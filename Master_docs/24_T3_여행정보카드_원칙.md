## §01 문서 목적 및 적용 범위

### 01.1 목적

본 문서는 SafeTrip 메인 화면 상단에 표시되는 **여행정보카드** 컴포넌트의 설계 원칙, 역할별 표시 규칙, 상태별 UI 변형, 여행 전환 메커니즘, 가디언 뷰 차이, DB 스키마 연동 방식을 정의한다.

여행정보카드는 사용자가 현재 참여 중인 여행의 컨텍스트를 인지하고 빠르게 진입할 수 있도록 하는 핵심 UI 진입점이다. 본 원칙은 비즈니스 원칙 v5.1을 기준으로 작성되었으며, 모든 하위 구현은 본 문서를 따른다.

### 01.2 적용 범위

| 대상 | 범위 |
|------|------|
| **Flutter 앱 (safetrip-mobile)** | 메인 화면 상단 여행정보카드 위젯 |
| **Node.js 서버 (safetrip-server-api)** | 여행 정보 조회 API, 상태 전환 로직 |
| **Firebase RTDB** | 실시간 위치 상태 반영 |
| **PostgreSQL** | TB_TRIP, TB_GROUP_MEMBER 스키마 |

### 01.3 이 문서에서 다루지 않는 것

- 일정 탭 상세 기능 → `18_T3_일정탭_원칙` 참조
- 멤버 탭 상세 기능 → `19_T3_멤버탭_원칙` 참조
- 온보딩 플로우 상세 → `14_T2_온보딩_UX_시나리오` 참조
- 가디언 연결/해제 상세 → 비즈니스 원칙 §03 참조

## §02 설계 철학 5원칙

여행정보카드는 다음 5가지 설계 철학(C1~C5)을 최우선으로 적용한다.

### C1 — 여행 컨텍스트 최우선

> "앱을 열면 현재 여행이 즉시 보인다."

- 진행 중(`active`) 여행이 있으면 자동으로 해당 여행 카드를 최상단에 표시한다.
- 복수의 `active` 여행이 있는 경우: **가장 최근에 시작한 여행**을 우선 표시한다.
- 여행이 없으면 탐색 모드 CTA 배너를 표시한다 (§06.3 참조).

### C2 — 최소 정보 노출

> "카드에는 핵심 정보만, 상세 정보는 탭 안에서."

- 여행정보카드는 여행명, 상태, 날짜, 멤버 수 등 핵심 요약 정보만 표시한다.
- 일정, 채팅, 지도 등 상세 기능은 탭 전환 후 확인한다.
- 카드 영역의 정보 밀도는 항상 최소화하여 인지 부하를 줄인다.

### C3 — 상태 기반 UI

> "여행 상태(planning/active/completed)에 따라 카드 디자인이 달라진다."

- `planning`, `active`, `completed` 각 상태별 카드 색상, 배지, 표시 정보가 다르다.
- 상태 전환은 실시간으로 반영되며, 별도 새로고침 없이 카드가 갱신된다.
- 15일 도래 시 자동 `completed` 전환 시나리오를 지원한다 (§04.5 참조).

### C4 — 빠른 접근성

> "1터치로 현재 여행 메인 화면에 진입한다."

- 카드 터치 시 해당 여행의 메인 화면(지도 탭)으로 즉시 진입한다.
- 여행 전환 버튼은 카드 오른쪽 상단에 항상 노출한다.
- 전환 후 현재 탭 컨텍스트(위치, 스크롤 상태)를 보존한다.

### C5 — 가디언 정보 분리

> "가디언 카드는 멤버 카드와 분리하여 표시한다."

- 여행 멤버(캡틴/크루장/크루) 카드와 가디언 카드는 별도 섹션으로 분리 표시한다.
- 가디언 카드에 표시되는 정보는 무료/유료 구분에 따라 다르다 (§05 참조).
- 가디언 역할에서는 연결된 멤버의 여행 정보만 제한적으로 표시한다.

## §03 여행정보카드 기본 레이아웃

### 03.1 카드 구조 (3행 레이아웃)

```
┌─────────────────────────────────────────────────────────────┐
│ [1행] 🌏 여행명                D-day / 여행 중   👥 N명   [전환] │
│ [2행] 📅 2026.07.01 ~ 2026.07.10 (10일)                      │
│ [3행] [🛡️ 안전 최우선] [강제 공유 · 24시간]                     │
└─────────────────────────────────────────────────────────────┘
```

| 행 | 표시 요소 | 설명 |
|----|----------|------|
| **1행** | 국가 플래그 + 여행명 + D-day + 멤버 수 + 전환 버튼 | 핵심 식별 정보 |
| **2행** | 여행 기간 (시작일 ~ 종료일, 총 N일) | 날짜 정보 |
| **3행** | 프라이버시 등급 배지 + 공유 유형 배지 | 공유 정책 현황 |

### 03.2 D-day 표시 규칙

- D-day 계산: `여행 시작일 - 현재 날짜`
- **D-15 ~ D-1**: "D-15", "D-14", ... "D-1" 형식으로 표시
- **D-0 (여행 당일)**: "여행 중" 텍스트로 전환
- **여행 종료 후**: "완료" 배지 표시
- **최대 표시 범위**: D-15까지만 카운트다운 표시 (§02.3 여행 기간 15일 제한 반영)
  - D-16 이상인 경우: 날짜 텍스트만 표시하고 D-day 카운트다운은 비표시

> 근거: 비즈니스 원칙 §02.3 — 1회 여행 최대 기간 15일. D-day 최대값이 곧 여행의 최대 기간과 일치한다.

### 03.3 프라이버시 등급 배지

| 등급 | DB 값 | 배지 표시 |
|------|--------|----------|
| 안전 최우선 | `safety_first` | `🛡️ 안전 최우선 · 강제 공유 · 24시간` |
| 표준 | `standard` | `📍 표준 · 24시간 공유` |
| 프라이버시 우선 | `privacy_first` | `🔒 프라이버시 우선 · 일정 연동` |

## §04 여행 상태별 카드 UI 변형

여행 상태(`tb_trip.status`)는 `planning`, `active`, `completed` 3단계이며, 각 상태에 따라 카드 UI가 달라진다.

### 04.1 planning (예정) 카드

```
┌─────────────────────────────────────────────────────────────┐
│ [예정] 🌏 도쿄 자유여행              D-7    👥 5명   [전환▼] │
│ 📅 2026.03.09 ~ 2026.03.15 (7일)                            │
│ [📍 표준 · 24시간 공유]                                       │
└─────────────────────────────────────────────────────────────┘
```

| 표시 요소 | 설명 |
|----------|------|
| **상태 배지** | `[예정]` — 회색 계열 |
| **날짜 + D-day** | 시작일까지 남은 일수 카운트다운 |
| **목적지** | 국가 플래그 + 여행명 |
| **멤버 수** | 현재 확정된 멤버 수 |
| **프라이버시 배지** | 설정된 등급 표시 |

**비활성 기능**: 실시간 위치 버튼 비활성화 (여행 미시작)

### 04.2 active (진행) 카드

```
┌─────────────────────────────────────────────────────────────┐
│ [진행 중] 🌏 도쿄 자유여행       여행 중   👥 5명   [전환▼] │
│ 📅 2026.03.09 ~ 2026.03.15 | 3일째 진행 중                  │
│ [오늘 일정: 시부야 쇼핑 → 아사쿠사 탐방]                      │
└─────────────────────────────────────────────────────────────┘
```

| 표시 요소 | 설명 |
|----------|------|
| **상태 배지** | `[진행 중]` — 초록 계열 |
| **D-day** | "여행 중"으로 전환 |
| **진행 일차** | "N일째 진행 중" |
| **오늘 일정** | 오늘 등록된 첫 번째 일정 요약 (최대 1줄) |
| **실시간 위치** | 위치 공유 활성 상태 인디케이터 표시 |

**추가 기능**: 오늘 일정 탭 바로가기 아이콘 활성화

### 04.3 completed (완료) 카드

```
┌─────────────────────────────────────────────────────────────┐
│ [완료] 🌏 도쿄 자유여행             완료    👥 5명   [열람▼] │
│ 📅 2026.03.09 ~ 2026.03.15 (7일)                            │
│ [이동 거리: 42.3km] [방문지: 8곳] [사진: 127장]              │
└─────────────────────────────────────────────────────────────┘
```

| 표시 요소 | 설명 |
|----------|------|
| **상태 배지** | `[완료]` — 회색/파란 계열 |
| **여행 기간** | 총 여행 일수 |
| **총 이동 거리** | 여행 중 누적 이동 거리 |
| **주요 방문지 수** | 등록된 장소 수 |
| **사진 수** | (향후 기능, Phase 3) |

**비활성 기능**: 위치 공유, 출석 체크, SOS 버튼 비활성화

### 04.4 탐색 모드 CTA (여행 없음)

참여 중인 여행이 없거나 모든 여행이 `completed` 상태인 경우 표시된다.

```
┌─────────────────────────────────────────────────────────────┐
│   여행이 없습니다                                             │
│   [+ 새 여행 만들기]   [초대코드 입력]                        │
└─────────────────────────────────────────────────────────────┘
```

- "새 여행 만들기" → 여행 생성 플로우 진입
- "초대코드 입력" → 초대코드 입력 화면 진입

### 04.5 여행 상태 전환 및 카드 연동

```
planning ──► active ──► completed
                           │
                    ◄──── (재활성화: 종료 후 24시간 이내 1회)
```

| 전환 | 트리거 | 카드 갱신 |
|------|--------|----------|
| `planning` → `active` | 캡틴 수동 시작 / 출발일 자동 전환 | 즉시 active 카드로 변형 |
| `active` → `completed` | 캡틴 수동 종료 / 귀국일 자동 종료 | 즉시 completed 카드로 변형 |
| **15일 도래 자동 종료** | 여행 end_date 도달 시 서버 자동 완료 | 자동으로 completed 카드 전환 |
| `completed` → `active` | 종료 후 24시간 이내 캡틴 재활성화 | completed 카드에 재활성화 버튼 표시 |

> 근거: 비즈니스 원칙 §02.3 — 15일 초과 불가. 비즈니스 원칙 §02.6 — 재활성화 횟수 최대 1회.

## §05 가디언 카드 뷰 차이

가디언 역할의 사용자는 여행 멤버 카드와 분리된 **가디언 전용 카드**를 통해 연결된 멤버의 여행 정보를 열람한다.

### 05.1 가디언 카드 위치

- 메인 화면에서 여행 멤버 카드 섹션 아래에 별도 `가디언으로 참여 중인 여행` 섹션으로 분리 표시한다.
- 하나의 사용자가 여행 멤버이면서 다른 여행의 가디언일 경우, 두 섹션 모두 표시한다.

### 05.2 무료 가디언 카드 (기본 2명 이내)

> 근거: 비즈니스 원칙 §03.1 — 멤버당 무료 가디언 최대 2명, §09.3 — 무료 범위

```
┌─────────────────────────────────────────────────────────────┐
│ [가디언] 홍길동의 여행             여행 중   [상세보기]        │
│ 📅 2026.03.09 ~ 2026.03.15                                  │
│ 현재 상태: 위치 공유 중                                       │
└─────────────────────────────────────────────────────────────┘
```

| 표시 요소 | 표시 여부 |
|----------|:--------:|
| 연결 멤버 이름 | ✅ |
| 여행 기간 | ✅ |
| 현재 여행 상태 (planning/active/completed) | ✅ |
| 현재 위치 공유 상태 | ✅ (ON/OFF 여부만) |
| 프라이버시 등급 배지 | ❌ |
| 오늘 일정 요약 | ❌ |
| 실시간 위치 정보 | ❌ |

**유료 전환 유도 메시지**: 카드 하단에 다음 문구 표시
```
"유료 가디언으로 전환하면 일정 요약, 프라이버시 등급 정보를 추가로 확인할 수 있습니다."
[유료로 전환 (1,900원/여행)] 버튼 표시
```

### 05.3 유료 가디언 카드 (3명째 이상, 1,900원/여행)

> 근거: 비즈니스 원칙 §09.3 — 3명째 개인 가디언부터 1,900원/여행 과금

```
┌─────────────────────────────────────────────────────────────┐
│ [유료 가디언] 홍길동의 여행    여행 중   [📍 표준]  [상세보기] │
│ 📅 2026.03.09 ~ 2026.03.15                                  │
│ 오늘: 시부야 쇼핑 → 아사쿠사 탐방 (현재 이동 중)              │
└─────────────────────────────────────────────────────────────┘
```

| 표시 요소 | 표시 여부 |
|----------|:--------:|
| 연결 멤버 이름 | ✅ |
| 여행 기간 | ✅ |
| 현재 여행 상태 | ✅ |
| 현재 위치 공유 상태 | ✅ (상세: 공유 중/일시 중지/비공유) |
| **프라이버시 등급 배지** | ✅ (유료 추가) |
| **오늘 일정 요약 (1줄)** | ✅ (유료 추가) |
| **실시간 위치 상태** | ✅ (이동 중/정지 중/오프라인) |

### 05.4 전체 가디언 카드 (여행당 최대 2명, 무료)

> 근거: 비즈니스 원칙 §03.1 — 전체 가디언 여행당 2명 무료

```
┌─────────────────────────────────────────────────────────────┐
│ [전체 가디언] 도쿄 자유여행         여행 중   5명    [상세보기]│
│ 📅 2026.03.09 ~ 2026.03.15                                  │
│ 전체 멤버 위치 공유 현황: 5/5명 공유 중                       │
└─────────────────────────────────────────────────────────────┘
```

전체 가디언은 여행 전체 멤버의 위치를 열람할 수 있으므로, 카드에 전체 공유 현황 요약을 표시한다.

## §07 프라이버시 등급별 동작 차이

여행정보카드에서 프라이버시 등급에 따라 카드 배지 및 가디언 정보 표시 방식이 달라진다.

### 07.1 안전 최우선 (`safety_first`)

```
카드 배지: [🛡️ 안전 최우선 · 강제 공유 · 24시간]
가디언 공유: 항상 실시간 공유 (ON/OFF 무관)
가디언 일시 중지: 불가 (버튼 비활성화)
대상: 미성년자 포함 여행 전용 (자동 강제)
```

- 미성년자 멤버가 1명이라도 포함된 여행은 이 등급 고정 (변경 불가 UI)
- 카드에 미성년자 포함 아이콘 표시 (Phase 2 이후)

### 07.2 표준 (`standard`)

```
카드 배지: [📍 표준 · 24시간 공유]
가디언 공유: ON 시간대 실시간, OFF 시간대 30분 스냅샷
가디언 일시 중지: 최대 12시간 가능
```

- 기본값 등급 (미설정 시 자동 적용)
- 일시 중지 중인 경우 카드에 "가디언 일시 중지 중" 표시

### 07.3 프라이버시 우선 (`privacy_first`)

```
카드 배지: [🔒 프라이버시 우선 · 일정 연동]
가디언 공유: 일정 ON 시간대만 공유, OFF 시간대 비공유
가디언 일시 중지: 최대 24시간 가능
```

- 일정 없는 날 "오늘 일정 없음 — 위치 비공유 상태" 안내 표시

### 07.4 등급 변경 UI 규칙

- 등급 변경은 캡틴만 가능 → 카드 내 설정 아이콘 → 등급 변경 모달
- 변경 전 확인 다이얼로그: "등급 변경 시 모든 멤버와 가디언에게 알림이 발송됩니다."
- 미성년자 포함 여행에서 `standard`/`privacy_first` 선택 시 경고: "미성년자 포함 여행은 안전 최우선 등급만 허용됩니다." → 자동으로 `safety_first`로 복원

## §08 여행 기간 15일 제한 UI (§02.3)

> 근거: 비즈니스 원칙 §02.3 — 1회 여행 최대 기간 15일 이내

### 08.1 신규 여행 생성 시 기간 선택 UI

- 여행 기간 선택은 **달력 피커(DateRangePicker)** 를 사용한다.
- 시작일 선택 후 **16일째 이후 날짜는 자동 비활성화** 처리한다.
  - 비활성화된 날짜: 회색 처리 + 터치 불가
  - 비활성화 이유 안내: "SafeTrip은 1회 여행 최대 15일까지 지원합니다."
- 기간 초과 시도 시 안내 메시지: "여행 기간은 최대 15일을 초과할 수 없습니다. 15일을 초과하는 여행은 분할하여 각각 별도 여행으로 생성하세요."

```dart
// DateRangePicker 비활성화 로직 (예시)
lastDate: selectedStartDate?.add(Duration(days: 15)) ?? DateTime.now().add(Duration(days: 365)),
```

### 08.2 기간 변경 시 검증

- `active` 상태 여행의 종료일 변경 시에도 15일 검증을 수행한다.
- 서버 검증: `end_date - start_date > 15` 조건 시 400 에러 + 사용자 안내 메시지 반환
- 클라이언트 사전 검증: 서버 요청 전 로컬에서 먼저 기간 계산 후 차단

### 08.3 서버 검증 오류 처리

| HTTP 상태 | 에러 코드 | 사용자 메시지 |
|-----------|----------|-------------|
| 400 | `TRIP_DURATION_EXCEEDED` | "여행 기간은 최대 15일까지 설정할 수 있습니다." |
| 400 | `TRIP_DATE_CONFLICT` | "기존 참여 여행과 날짜가 겹칩니다." |

### 08.4 15일 도래 알림 연동

| 시점 | 알림 내용 | 수신자 |
|------|----------|--------|
| D-1 (여행 종료 전날) | "내일 여행이 종료됩니다. 연장이 필요하면 새 여행을 생성하세요." | 캡틴 |
| D-0 (여행 종료일) | "오늘 여행이 종료됩니다." | 전체 멤버 |
| 종료일 자동 완료 | 서버에서 자동으로 `completed` 상태 전환 | 전체 멤버 + 가디언 |

## §09 여행 전환 메커니즘

### 09.1 복수 여행 참여 시 전환

- 사용자가 복수의 여행에 참여 중인 경우, 카드 우측 상단의 `[전환▼]` 버튼을 표시한다.
- `[전환▼]` 버튼 탭 → **바텀시트 모달**로 여행 목록 표시
- 목록 표시 순서:
  1. `active` 상태 여행 (시작일 최신 순)
  2. `planning` 상태 여행 (시작일 오름차순)
  3. 가디언으로 참여 중인 여행 (별도 섹션)
  4. `completed` 상태 여행 (최근 종료 순, 최대 5개)

### 09.2 복수 active 여행 처리

```
복수 active 여행 존재 시:
├── 자동 선택: 가장 최근 시작한 여행 (start_date 최신)
├── 동일 start_date: created_at 최신 여행 우선
└── 전환 모달에서 수동 전환 가능
```

- 진행 중 여행이 2개 이상인 경우 메인 카드에 경고 배지 표시:
  "진행 중인 여행 N개" + [관리] 버튼

### 09.3 여행 전환 시 컨텍스트 보존

- 현재 활성 탭(지도/일정/채팅 등) 위치 보존
- 바텀시트 스크롤 상태 보존
- 선택한 여행으로 전환 후 동일한 탭에서 해당 여행 데이터 갱신

> 근거: 설계 철학 C4 — 여행 전환 시 바텀시트 탭 위치·스크롤 상태 보존

### 09.4 신규 여행 생성 플로우 (카드에서 진입)

```
[+새 여행 만들기] 탭
      │
      ▼
1. 여행명 입력
2. 목적지 선택 (국가/도시)
3. 여행 기간 선택 (최대 15일 달력 피커)
4. 프라이버시 등급 선택 (3가지 옵션)
5. 위치 공유 모드 선택 (강제/자유)
6. 확인 → 여행 생성 + 캡틴 자동 배정
```

CREATE TABLE tb_trip (
    trip_id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    group_id                 UUID REFERENCES tb_group(group_id),
    trip_name                VARCHAR(200),
    destination              VARCHAR(200),

    -- ▼ 위치 공유 모드 (§04.2)
    sharing_mode             VARCHAR(20) DEFAULT 'voluntary',
        -- forced | voluntary

    schedule_type            VARCHAR(20) DEFAULT 'always',

    created_by               VARCHAR(128) REFERENCES tb_user(user_id),
    created_at               TIMESTAMPTZ DEFAULT NOW(),
    updated_at               TIMESTAMPTZ,
    deleted_at               TIMESTAMPTZ,
    -- ▼ §02.3: 여행 기간 최대 15일 CHECK 제약 (핵심)
    CONSTRAINT chk_trip_duration
        CHECK (end_date IS NULL OR start_date IS NULL OR end_date - start_date <= 15),
    -- ▼ §02.6: 재활성화 횟수 1회 제한
    CONSTRAINT chk_reactivation_count
        CHECK (reactivation_count <= 1)
);

CREATE INDEX idx_trips_group    ON tb_trip(group_id);
CREATE INDEX idx_trips_status   ON tb_trip(status);
CREATE INDEX idx_trips_dates    ON tb_trip(start_date, end_date);

    joined_at                TIMESTAMPTZ DEFAULT NOW(),
    left_at                  TIMESTAMPTZ,

### 11.3 TB_TRIP_CARD_VIEW (여행 카드 뷰 정의)

여행정보카드 렌더링에 필요한 데이터를 한 번의 쿼리로 조회하기 위한 뷰 정의.

```sql
CREATE OR REPLACE VIEW tb_trip_card_view AS
SELECT
    t.trip_id,
    t.trip_name,
    t.status,
    t.start_date,
    t.end_date,
    t.end_date - t.start_date                          AS trip_days,
    t.privacy_level,
    t.sharing_mode,
    t.schedule_type,
    t.country_code,
    t.country_name,
    t.destination_city,
    t.has_minor_members,
    t.reactivated_at,
    t.reactivation_count,
    -- ▼ D-day 계산
    CASE
        WHEN t.status = 'active'    THEN 0
        WHEN t.status = 'planning'  THEN (t.start_date - CURRENT_DATE)
        ELSE NULL
    END                                                 AS d_day,
    -- ▼ 현재 진행 일차
    CASE
        WHEN t.status = 'active'
            THEN (CURRENT_DATE - t.start_date + 1)
        ELSE NULL
    END                                                 AS current_day,
    -- ▼ 활성 멤버 수 (가디언 제외)
    (
        SELECT COUNT(*)
        FROM tb_group_member gm
        WHERE gm.trip_id = t.trip_id
          AND gm.status = 'active'
          AND gm.member_role IN ('captain', 'crew_chief', 'crew')
    )                                                   AS member_count,
    -- ▼ 재활성화 가능 여부 (종료 후 24시간 이내 + 재활성화 횟수 0)
    CASE
        WHEN t.status = 'completed'
         AND t.reactivation_count = 0
         AND t.updated_at > NOW() - INTERVAL '24 hours'
            THEN TRUE
        ELSE FALSE
    END                                                 AS can_reactivate
FROM tb_trip t
WHERE t.deleted_at IS NULL;
```

## §12 오프라인 대응

### 12.1 오프라인 상태 감지 및 표시

- 네트워크 연결이 끊어지면 여행정보카드 상단에 **"오프라인" 배지**를 표시한다.
- 오프라인 배지: 회색 배경 + "오프라인" 텍스트 + 마지막 동기화 시각 표시
  ```
  [오프라인 — 마지막 동기화: 14:32]
  ```

### 12.2 로컬 캐시 정책

| 캐시 항목 | 캐시 방식 | 유효 기간 |
|----------|----------|----------|
| 여행 기본 정보 (trip_name, status, dates) | SharedPreferences / Hive | 24시간 |
| 여행 상태 (planning/active/completed) | 앱 메모리 | 앱 세션 동안 |
| 멤버 수 | SharedPreferences | 24시간 |
| 프라이버시 등급 | SharedPreferences | 24시간 |
| 오늘 일정 요약 | 앱 메모리 | 앱 세션 동안 |

### 12.3 오프라인 시 허용/차단 기능

| 기능 | 오프라인 상태 |
|------|:------------:|
| 여행 카드 열람 (캐시 데이터) | ✅ 허용 |
| 여행 전환 (캐시 기반) | ✅ 허용 (마지막 캐시 기준) |
| 여행 상태 변경 | ❌ 차단 + 안내 |
| 신규 여행 생성 | ❌ 차단 + 안내 |
| 프라이버시 등급 변경 | ❌ 차단 + 안내 |
| D-day 계산 | ✅ 허용 (로컬 날짜 기준) |

### 12.4 재연결 후 동기화

- 네트워크 재연결 시 자동으로 최신 여행 정보를 갱신한다.
- 오프라인 중 변경 시도된 항목이 있다면 재연결 후 재시도 안내를 표시한다.

## §14 검증 체크리스트 (12항목)

여행정보카드 기능 구현 완료 후 다음 12개 항목을 모두 검증한다.

| # | 검증 항목 | 검증 방법 | 기준 문서 |
|---|----------|----------|---------|
| 1 | **15일 제한 달력 피커** | 시작일 선택 후 16일째 날짜 탭 시 비활성 확인 | §08.1, §02.3 |
| 2 | **서버 15일 검증** | `end_date - start_date > 15` API 요청 시 400 에러 반환 확인 | §08.2, DB CHECK 제약 |
| 3 | **D-day 계산 정확성** | D-1, D-0("여행 중"), D-15 각 케이스 UI 확인 | §03.2 |
| 4 | **상태 전환 카드 갱신** | planning→active→completed 전환 시 카드 UI 즉시 변형 확인 | §04.5 |
| 5 | **여행 전환 컨텍스트 보존** | 바텀시트 전환 후 탭 위치 및 스크롤 상태 유지 확인 | §09.3 |
| 6 | **가디언 카드 정보 분리** | 무료 가디언 카드에 프라이버시 배지·일정 요약 미표시 확인 | §05.2 |
| 7 | **유료 가디언 추가 정보** | 유료 가디언 카드에 프라이버시 배지 + 일정 요약 1줄 표시 확인 | §05.3 |
| 8 | **프라이버시 등급 배지** | 3등급 배지 텍스트·색상 정확 표시 확인 | §07 |
| 9 | **미성년자 등급 강제** | 미성년자 멤버 추가 시 `safety_first` 자동 전환 확인 | §07.4, §10.2 |
| 10 | **오프라인 배지 + 캐시** | 네트워크 차단 시 오프라인 배지 표시 + 캐시 데이터 렌더링 확인 | §12 |
| 11 | **재활성화 규칙** | completed 후 24시간 초과 및 2회 시도 시 버튼 비활성 확인 | §04.5, §10.5 |
| 12 | **복수 active 여행 처리** | 2개 이상 active 여행 시 최신 여행 자동 선택 + 경고 배지 표시 확인 | §09.2 |
