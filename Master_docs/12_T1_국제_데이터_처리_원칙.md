## 1. 문서 목적 및 적용 범위

### 1.1 목적

SafeTrip은 해외 여행을 지원하는 앱으로, 사용자의 데이터가 국경을 넘어 처리될 수 있다. 본 문서는 **국경을 초월한 데이터 처리**에 적용되는 GDPR, CCPA 등 국제 규정 대응 전략을 정의하고, AI 제공사와의 데이터 처리 계약(DPA) 요건을 명시한다.

### 1.2 적용 범위

- SafeTrip 앱을 통해 수집·처리되는 모든 개인정보
- 서버 외부로 전송되는 데이터 (AI 처리, 외부 API 연동)
- EU 시민(GDPR), 캘리포니아 거주자(CCPA) 해당 사용자
- 해외 여행 중 현지 데이터 처리

### 1.3 SafeTrip 국제 데이터 처리 핵심 원칙

| 원칙 | 설명 |
|------|------|
| **최소 이전** | 필요 최소한의 데이터만 국외로 이전 |
| **목적 제한** | 국외 이전 시 명시된 목적 외 처리 금지 |
| **동의 우선** | 국외 이전 전 명시적 동의 획득 |
| **안전성 확보** | 이전 시 암호화, DPA 체결 의무화 |
| **투명성** | 이전 국가, 목적, 처리 내용을 사용자에게 공개 |

## 2. 데이터 처리 서버 위치

### 2.1 주요 서버 위치

| 시스템 | 위치 | 데이터 종류 |
|--------|------|-----------|
| **주 서버 (PostgreSQL)** | 한국 (Seoul Region) | 모든 사용자 데이터, 여행 데이터, 위치 이력 |
| **Firebase RTDB** | 미국 (us-central1) | 실시간 위치, 채팅 메시지, SOS 상태 |
| **Firebase Auth** | 미국 (글로벌) | 인증 토큰, 전화번호 |
| **Firebase FCM** | 미국 (글로벌) | 푸시 알림 페이로드 |
| **AI LLM 서버** | 제공사별 상이 (§6 참조) | AI 처리용 컨텍스트 데이터 |

### 2.2 Firebase 국외 이전 법적 근거

Firebase(구글)는 한국 개인정보보호법 제28조의8 제1항에 따른 **표준 계약 조항(SCC)** 기반으로 국외 이전이 적용된다.

- **처리 목적**: 실시간 위치 공유, 채팅, 푸시 알림, 인증
- **법적 근거**: 정보통신서비스 제공 계약 이행 (개인정보보호법 §15 ①2)
- **보호 조치**: Google Cloud 표준 계약 조항(SCC), ISO 27001 인증

## 3. 국외 이전 시나리오

### 3.1 시나리오 분류

| 시나리오 | 이전 대상 | 이전 국가 | 법적 근거 | 비고 |
|--------|----------|----------|----------|------|
| **S1: 실시간 위치 공유** | GPS 좌표, 타임스탬프 | 미국 (Firebase) | 계약 이행 + SCC | 여행 active 상태 |
| **S2: 채팅 메시지** | 메시지 내용, 발신자 ID | 미국 (Firebase) | 계약 이행 + SCC | 텍스트/시스템 메시지 |
| **S3: 푸시 알림** | 알림 제목, 내용, user_id | 미국 (FCM) | 계약 이행 + SCC | SOS 포함 |
| **S4: AI 안전 어시스턴트** | 질문 텍스트, 여행 국가 | AI 제공사 위치 | 동의 + DPA | 개인식별 정보 비포함 |
| **S5: AI 일정 생성** | 여행 날짜, 국가, 선호도 | AI 제공사 위치 | 동의 + DPA | 최소화 전송 |
| **S6: AI Intelligence** | 이동 패턴, 행동 데이터 | AI 제공사 위치 | 명시적 동의 + DPA | 유료, 미성년자 제한 |

### 3.2 이전 금지 데이터

다음 데이터는 어떠한 경우에도 국외 이전하지 않는다:

- 전화번호 원문 (AI 서버 전송 시 마스킹 처리)
- 비식별화되지 않은 의료/건강 데이터
- 14세 미만 아동의 위치 상세 데이터 (응급 시 제외)

## 4. GDPR 대응 (EU 여행자)

### 4.1 적용 범위

EU/EEA 회원국 시민이 SafeTrip을 사용하는 경우 GDPR(EU 2016/679)이 적용된다. SafeTrip은 **Article 3(2)(a) 역외 적용** 기준(EU 거주자에게 서비스 제공)에 해당할 수 있으며, 아래 조치를 의무화한다.

### 4.2 GDPR 핵심 요건 대응

| GDPR 조항 | 요건 | SafeTrip 대응 방안 |
|----------|------|-------------------|
| Art. 5 | 처리 원칙 (목적 제한, 최소화 등) | 데이터 최소화, 목적 외 처리 금지 |
| Art. 6 | 적법 처리 근거 | 계약 이행(§6(1)(b)), 동의(§6(1)(a)), 정당한 이익(§6(1)(f)) |
| Art. 7 | 동의 요건 | 명확, 철회 가능, 기록 보존 |
| Art. 13-14 | 정보 제공 의무 | 온보딩 시 개인정보 처리 투명 고지 |
| Art. 17 | 삭제권 (잊혀질 권리) | 계정 삭제 30일 유예 → 완전 삭제 (§14) |
| Art. 20 | 데이터 이동권 | JSON 형식 내 정보 내보내기 기능 |
| Art. 22 | 자동화 의사결정 | AI Intelligence 기능 고지 및 동의 |
| Art. 25 | 프라이버시 바이 디자인 | 3단계 프라이버시 등급 기본 설계 |
| Art. 32 | 보안 조치 | AES-256-GCM 암호화, TLS 전송 |
| Art. 33-34 | 침해 통지 | 72시간 내 감독기관 통지 |
| Art. 37 | DPO (선택적) | 대규모 처리 시 DPO 지정 검토 |
| Art. 35 | DPIA | AI Intelligence 기능 도입 전 DPIA 수행 |

### 4.3 국제 데이터 이전 (GDPR Ch. 5)

EU → 제3국(한국, 미국 Firebase) 이전:

| 경로 | 법적 메커니즘 |
|------|-------------|
| EU → 한국 서버 | **적정성 결정** (한국은 2021년 EU 적정성 결정 획득) |
| EU → 미국 Firebase | **표준 계약 조항(SCC)** + Google Data Processing Agreement |
| EU → AI 제공사 | **SCC + DPA** (§6 참조) |

### 4.4 GDPR 동의 관리

- 온보딩 A-2 단계에서 **EU 사용자 식별 시** GDPR 전용 동의 화면 추가 표시
- 동의 기록은 `tb_user_consent` 테이블에 `consent_type = 'gdpr'`로 저장
- 동의 철회는 설정 메뉴 → 개인정보 관리 → 동의 관리에서 언제든 가능

## 5. CCPA 대응 (캘리포니아 여행자)

### 5.1 적용 범위

캘리포니아 거주자가 SafeTrip을 사용하는 경우 CCPA(Cal. Civ. Code §1798.100 et seq.)가 적용된다.

### 5.2 CCPA 소비자 권리 대응

| 권리 | 내용 | SafeTrip 대응 |
|------|------|--------------|
| 알권리 | 수집 항목, 목적, 공유 대상 | 개인정보처리방침 공개, 온보딩 고지 |
| 삭제권 | 개인정보 삭제 요청 | 계정 삭제 기능 (30일 유예 후 삭제) |
| 판매 거부권 | 개인정보 판매 거부 | SafeTrip은 개인정보를 제3자에게 판매하지 않음 |
| 비차별권 | 권리 행사에 따른 서비스 차별 금지 | 삭제 요청 후에도 동등한 서비스 (잔존 기간 내) |

### 5.3 CCPA "판매(Sale)" 해당 여부

SafeTrip은 사용자 개인정보를 **금전적 대가를 받고 제3자에게 제공하지 않는다.** CCPA 정의상 "판매"에 해당하지 않으므로 "판매 거부권(Right to Opt-Out of Sale)" 고지는 필수가 아니나, 명확성을 위해 개인정보처리방침에 명시한다.

## 6. AI 제공사 DPA 요건

### 6.1 AI 기능과 데이터 처리 계약 (DPA)

SafeTrip의 AI 기능(Safety AI, Convenience AI, Intelligence AI)은 외부 LLM 서비스를 사용하므로, 해당 제공사와 **데이터 처리 계약(Data Processing Agreement, DPA)**을 반드시 체결해야 한다.

### 6.2 DPA 필수 조항

| 조항 | 내용 |
|------|------|
| **처리 목적 한정** | SafeTrip 서비스 목적 외 AI 모델 학습에 데이터 사용 금지 명시 |
| **데이터 최소화** | API 전송 시 개인식별 정보(전화번호, 실명) 제외 의무화 |
| **처리 위치** | 데이터 처리 서버 위치 및 재이전 제한 |
| **보안 기준** | ISO 27001 또는 동등 수준 보안 인증 의무화 |
| **침해 통지** | 데이터 침해 발생 시 48시간 내 통지 의무 |
| **감사권** | SafeTrip의 연 1회 감사 또는 인증서 확인 권리 |
| **하위처리자** | 하위처리자 목록 공개 및 사전 승인 |
| **삭제** | 서비스 종료 시 데이터 완전 삭제 또는 반환 |
| **국제 이전** | 적법한 이전 메커니즘(SCC, 적정성 결정) 명시 |

### 6.3 AI 전송 데이터 익명화 규칙

AI 서버에 전송 시 다음 익명화 처리를 의무 적용한다:

| 원본 데이터 | AI 전송 형태 |
|----------|------------|
| 전화번호 | 삭제 (전송하지 않음) |
| 실명 | `user_{hash_id}` 형태로 치환 |
| 정확한 GPS 좌표 | ± 100m 반경 랜덤화 (Safety AI 제외) |
| SOS 위치 | Safety AI에만 원본 전송 허용 |
| 여행 국가 | 원본 전송 |
| 일정 내용 | 원본 전송 (사용자 동의 하) |

## 7. 여행 국가별 데이터 규제 매트릭스

### 7.1 주요 여행 국가 규제 대응

| 국가/지역 | 관련 규제 | SafeTrip 주요 대응 |
|----------|---------|-----------------|
| **EU/EEA** | GDPR | §4 전체 적용 |
| **미국 캘리포니아** | CCPA | §5 전체 적용 |
| **미국 (CA 외)** | 연방법 없음 (주별 상이) | 기본 개인정보 보호 방침 적용 |
| **중국** | PIPL (개인정보보호법) | 중국 서버 데이터 국외 이전 시 보안 평가 의무 (Phase 3) |
| **일본** | APPI | 적정성 결정 국가 간 이전 허용, 개인정보 취급방침 일본어 제공 |
| **태국** | PDPA | 동의 기반 처리, 권리 보장 |
| **베트남** | 사이버보안법 + PDPD | 로컬 데이터 보관 요건 검토 필요 |
| **인도네시아** | PDP법 (2022) | 동의 + 위반 시 행정 제재 |
| **싱가포르** | PDPA | 데이터 보호 표준 준수 |
| **필리핀** | DPA 2012 | 국가개인정보위원회 등록 |
| **호주** | Privacy Act 1988 | Australian Privacy Principles |
| **캐나다** | PIPEDA / QC Law 25 | 퀘벡 거주자는 강화된 동의 요건 |

### 7.2 여행 국가 감지 및 적용 로직

```
사용자 여행 목적지 국가(tb_trip.country_code) 감지
    ↓
해당 국가 규제 플래그 조회 (tb_country_regulation)
    ↓
규제 요건 자동 적용 (GDPR / CCPA / 기타)
```

### 7.3 tb_country_regulation 스키마 (참조용)

```sql
CREATE TABLE tb_country_regulation (
    country_code    CHAR(2)      PRIMARY KEY,   -- ISO 3166-1 alpha-2
    country_name    VARCHAR(100) NOT NULL,
    gdpr_applies    BOOLEAN      DEFAULT FALSE,
    ccpa_applies    BOOLEAN      DEFAULT FALSE,
    localization_required BOOLEAN DEFAULT FALSE, -- 현지 서버 보관 필요
    special_consent BOOLEAN      DEFAULT FALSE,  -- 추가 동의 필요
    notes           TEXT,
    updated_at      TIMESTAMPTZ  DEFAULT now()
);
```

## 8. 국외 이전 동의 절차

### 8.1 온보딩 동의 단계

온보딩 A-2 단계(약관 동의 화면)에서 국외 이전 동의를 포함한다:

| 동의 항목 | 필수/선택 | 내용 |
|----------|:-------:|------|
| Firebase 국외 이전 (미국) | **필수** | 실시간 위치, 채팅, 알림 처리를 위한 미국 Firebase 서버 이용 |
| AI 서비스 국외 이전 | **선택** | AI 기능 이용 시 질문 내용이 AI 서버(해외)로 전송됨 |
| GDPR 추가 동의 | EU 사용자 필수 | GDPR 기반 권리 및 처리 고지 |

### 8.2 설정 메뉴 동의 관리

설정 메뉴 → 개인정보 관리 → 국외 이전 동의에서:

- 현재 동의 상태 확인
- AI 국외 이전 동의 철회 (AI 기능 비활성화)
- Firebase 국외 이전은 서비스 이용 필수이므로 철회 불가 (서비스 탈퇴로만 가능)

### 8.3 동의 기록 저장

```sql
-- tb_user_consent 활용 (DB 설계 v3.4 참조)
INSERT INTO tb_user_consent (
    user_id,
    consent_type,     -- 'firebase_international', 'ai_international', 'gdpr'
    is_agreed,
    agreed_at,
    version
) VALUES (...);
```

## 11. 데이터 지역화(Localization) 전략

### 11.1 현재 지역화 전략

| 데이터 유형 | 저장 위치 | 지역화 필요 여부 |
|----------|---------|:-------------:|
| 사용자 프로필, 여행 데이터 | 한국 PostgreSQL | ✅ 한국 내 |
| 실시간 위치, 채팅 | 미국 Firebase | 법적 허용 범위 내 |
| 위치 이력 (90일 보존) | 한국 PostgreSQL | ✅ 한국 내 |

### 11.2 지역화가 필요한 국가 대응 (Phase 3)

중국, 러시아 등 데이터 로컬라이제이션 법이 엄격한 국가의 여행자를 지원할 경우:

- 해당 국가 내 보조 서버 운용 검토
- 또는 서비스 지원 국가 목록에서 제외 후 사전 안내

### 11.3 CDN 및 정적 자원

- 안전가이드 데이터(MOFA API 응답)는 **한국 내 CDN**에 캐싱
- 앱 바이너리 배포는 Google Play Store / Apple App Store 글로벌 CDN 사용 (개인정보 해당 없음)

## 12. 에러 및 엣지케이스 처리

| 케이스 | 처리 방법 |
|--------|----------|
| **AI 서버 장애** | 폴백: AI 기능 비활성화, 기본 정보 제공 (외부 API 연동 원칙 #33 참조) |
| **Firebase 서비스 점검** | 오프라인 모드 전환, 로컬 큐잉 (오프라인 원칙 #16 참조) |
| **GDPR 동의 미확인 EU 사용자** | AI 기능 비활성화, Firebase 이전은 SCC 기반 진행 |
| **국가 감지 실패** | 기본 정책(한국법) 적용, 사용자에게 국가 선택 안내 |
| **DPA 미체결 AI 서비스 이용 시도** | 서비스 이용 차단, 법무팀 알림 |
| **동의 철회 후 AI 기능 이용 시도** | "AI 기능을 위한 국외 이전 동의가 필요합니다" 안내 |
