## 1. 문서 목적 및 적용 범위

### 1.1 목적

본 문서는 SafeTrip 앱 최초 실행 시 스플래시 화면(#28) 이후에 신규 사용자에게 표시되는 **웰컴화면**의 설계 원칙을 정의한다.

웰컴화면은 SafeTrip의 첫인상을 결정하는 온보딩 핵심 화면으로, 3초 이내에 앱의 핵심 가치를 전달하고 사용자가 자신의 목적(캡틴/크루/가디언)을 자연스럽게 선택하도록 안내하는 역할을 한다.

비즈니스 원칙 v5.1 §06(회원가입 및 온보딩 원칙), §01(역할 체계 원칙), §03.1(가디언 연결 규칙), §09.3(가디언 추가 과금), §12.3(B2B 전용 기능)과 스플래시 화면 원칙(#28)의 분기 로직을 UI 레벨에서 구현하는 규칙을 명세한다.

### 1.2 적용 범위

## 2. 웰컴화면 설계 철학

웰컴화면은 아래 4가지 목표를 동시에 달성해야 한다.

### 2.1 감성 먼저, 기능 나중

사용자가 처음 마주치는 화면에서 기능·요금 정보를 앞세우지 않는다. 안전, 연결, 보호라는 감성적 메시지를 먼저 전달한 후 행동을 유도한다.

### 2.2 맥락 감지

딥링크 파라미터, 재방문 여부, 시간대를 감지하여 최적의 진입 경로를 자동 선택한다. 사용자가 의식하지 않아도 가장 적합한 화면 흐름으로 안내된다.

### 2.3 역할 용어 미노출

"캡틴", "크루", "가디언" 등 내부 역할 용어를 목적 선택 단계에서 직접 노출하지 않는다. 대신 "여행 만들기", "초대코드 입력", "먼저 둘러보기", "가디언으로 참여"와 같은 행동 중심 언어로 안내한다.

### 2.4 제로 좌절 원칙

어떤 선택지를 고르더라도 막힘 없이 다음 단계로 진행 가능해야 한다. 잘못된 선택이 없으며, 모든 경로는 결국 SafeTrip의 핵심 가치 체험으로 이어진다.

## 3. 본문 — 웰컴화면 핵심 원칙 및 화면 플로우

### §3.1 웰컴화면 5대 원칙 (W1~W5)

| 원칙 코드 | 원칙명 | 내용 |
|:--------:|--------|------|
| **W1** | 3초 가치 전달 | 첫 화면 3초 이내 SafeTrip의 핵심 가치(안전·연결·보호)를 전달한다. 슬라이드 1이 표시되는 즉시 가치 메시지가 인식 가능해야 한다. |
| **W2** | 감성 먼저 | 기능 설명 전 감성적 연결을 우선한다. 슬라이드 순서는 "안전 → 연결 → 보호 → CTA" 순으로 고정한다. |
| **W3** | 맥락 감지 | 딥링크 파라미터, 재방문 여부, 시간대를 감지하여 최적 진입 경로를 자동 분기한다. |
| **W4** | 역할 기반 목적 선택 | 내부 역할 용어(캡틴/크루/가디언) 노출 없이 자연스러운 행동 선택으로 안내한다. |
| **W5** | 제로 좌절 | 어떤 선택도 막힘 없이 진행 가능하다. 잘못된 선택이 없으며, 모든 경로는 SafeTrip 핵심 가치 체험으로 이어진다. |

---

### §3.2 화면 플로우 구조 (Phase 1~3)

#### Phase 1: 맥락 감지 분기 (스플래시 이후 진입)

스플래시 화면(#28)에서 전달받은 맥락 정보를 기반으로 즉시 분기한다.

```
스플래시 화면(#28) 종료
  │
  ├─ 신규 유저 (인증 토큰 없음)
  │     ├─ 딥링크 없음    → Phase 2 (가치 슬라이드)
  │     └─ 딥링크(초대코드) → Phase 3 직행 (초대코드 자동 입력)
  │
  └─ 재방문 유저 (인증 토큰 있음) → 메인화면 직행 (웰컴화면 표시 없음)
```

- 딥링크 파라미터는 스플래시 화면(#28) §6에서 파싱·보존된 값을 그대로 전달받는다.
- 재방문 유저의 메인화면 직행 처리는 스플래시 화면 원칙(#28) §4.2를 따른다.
- 딥링크 패턴 `safetrip://invite/{code}`: Phase 3에서 초대코드 필드에 자동 입력 (초대코드 원칙 #23 §04 참조).

#### Phase 2: 가치 제안 슬라이드 (3~4장)

신규 유저에게 SafeTrip의 핵심 가치를 감성적으로 전달하는 3~4장 슬라이드를 표시한다.

| 슬라이드 | 테마 | 카피 | 배경 | 일러스트 |
|:-------:|------|------|------|---------|
| 1 | 안전 | "여행, 더 안전하게" | 네이비 `#1A3A5C` | 방패 일러스트 |
| 2 | 연결 | "일정부터 위치까지, 함께" | 민트 `#00BFA5` | 그룹 일러스트 |
| 3 | 보호 | "누군가 지켜보고 있어요" | 코랄 `#FF6B6B` | 가디언 일러스트 |
| 4 | CTA | "지금 시작하세요" | 브랜드 컬러 | 전환 버튼 |

**슬라이드 동작 규칙**:

- 자동 전환: 5초마다 다음 슬라이드로 자동 전환 (사용자 조작 없을 때)
- 수동 전환: 좌우 스와이프 또는 인디케이터 도트 탭
- 스킵 가능: 우상단 "건너뛰기" 버튼 → Phase 3 목적 선택으로 즉시 이동
- 인디케이터 도트: 하단 중앙, 현재 슬라이드 강조 표시
- 슬라이드 전환 애니메이션: fade + slide, 전환 시간 ≤ 300ms

#### Phase 3: 목적 선택 화면

사용자가 SafeTrip을 어떤 목적으로 사용할지 선택한다. 역할 용어(캡틴/크루/가디언) 대신 행동 중심 언어를 사용한다.

| 버튼 | 표시 문구 | 진입 경로 | 비고 |
|------|---------|---------|------|
| 주요 버튼 1 | "여행 만들기" | 캡틴 온보딩 분기 | 비즈니스 원칙 §02.1 |
| 주요 버튼 2 | "초대코드 입력" | 크루 온보딩 분기 | 초대코드 원칙(#23) 참조 |
| 보조 버튼 | "먼저 둘러보기" | 데모 투어(#30) 분기 | 인증 불필요 |
| 추가 링크 | "가디언으로 참여" | 가디언 온보딩 분기 | §03.1 무료 가디언 직접 진입 |

**딥링크 자동 처리**:

- `safetrip://invite/{code}` 딥링크 진입 시: "초대코드 입력" 화면 자동 진입 + 코드 자동 입력
- 초대코드 검증은 초대코드 원칙(#23) §04의 8단계 서버 검증을 따른다.

**가디언 직접 진입 (비즈니스 원칙 §03.1 근거)**:

- "가디언으로 참여" 링크는 가디언 역할로의 직접 온보딩 경로를 제공한다.
- 비즈니스 원칙 §03.1에 따라 캡틴/크루장/크루는 멤버당 무료 개인 가디언 2명을 연결할 수 있다.
- 가디언 직접 진입 경로를 통해 가입한 사용자는 가디언 초대 링크 수락 후 연결이 완료된다.

### §3.2.1 시간대별 감성 색상 적용 (UI 속성)

시간대에 따라 웰컴화면의 배경 색조와 감성 톤을 자동으로 조정한다. 이는 순차적 진입 단계가 아닌 Phase 1~3 전체에 걸쳐 적용되는 UI 속성이다.

| 시간대 | 범위 | 색조 | 감성 톤 |
|--------|------|------|---------|
| 아침 | 07:00 ~ 12:00 | 밝은 하늘색 | 상쾌함, 설렘 |
| 낮 | 12:00 ~ 18:00 | 따뜻한 노랑 | 활기, 에너지 |
| 밤 | 18:00 ~ 07:00 | 딥 네이비 | 안정감, 신뢰 |

- 시간 기준: 단말 로컬 시간 (서버 호출 불필요)
- 색조 변화는 배경 오버레이에만 적용되며, 브랜드 팔레트 기본값을 덮어쓰지 않는다.

### §3.3 유저 페르소나별 웰컴 경험 목표

| 페르소나 | SafeTrip 역할 | 진입 목표 | 목표 완료 시간 |
|---------|:------------:|---------|:------------:|
| 교사 (인솔 책임자) | 캡틴 (captain) | 여행 생성 준비 완료 | ≤ 90초 |
| 학생 (여행 참여자) | 크루 (crew) | 초대코드 참여 완료 | ≤ 60초 |
| 학부모 (보호자) | 가디언 (guardian) | 가디언 참여 + 자녀 연결 | ≤ 90초 |
| 탐색 유저 (체험자) | 미결정 | 데모 체험 → 가치 인지 | ≤ 120초 |

**비즈니스 원칙 §01.4 B2B 역할 매핑 적용**:

- 학교 수학여행: 교사(캡틴) → 학생(크루) → 학부모(가디언) 진입 경로가 각각 존재
- 여행사 패키지: 여행 가이드(캡틴) → 참가자(크루) → 가족(가디언)
- 기업 출장: 팀장/관리자(캡틴) → 팀원(크루) → 안전담당자(가디언)

#### 타이포그래피 계층

| 레벨 | 사용 위치 | 예시 |
|------|---------|------|
| H1 | 슬라이드 메인 카피 | "여행, 더 안전하게" |
| H2 | 슬라이드 서브 카피 | 부연 설명 텍스트 |

#### 마이크로 인터랙션

- **패럴랙스 스크롤**: 슬라이드 전환 시 배경과 전경 요소의 이동 속도 차이로 깊이감 연출
- **카드 전환 애니메이션**: fade + slide 조합, 전환 시간 ≤ 300ms
- **버튼 리플 효과**: 탭 즉시 리플 애니메이션 발생 (사용자 입력 피드백)
- **슬라이드 인디케이터 도트**: 현재 슬라이드 도트 스케일 1.5× + 컬러 강조

|------|------|------|
| 색상 대비 | WCAG 2.1 AA | 텍스트와 배경 간 최소 대비비 4.5:1 이상 |
| 스크린 리더 | TalkBack / VoiceOver | 각 슬라이드에 alt-text 필수 (예: "슬라이드 1: 여행 안전을 상징하는 방패 일러스트") |
| 모션 감소 모드 | prefers-reduced-motion | 시스템 설정 감지 시 자동 전환 애니메이션 비활성화, 정적 전환으로 대체 |
| 폰트 크기 | 동적 크기 | 시스템 텍스트 크기 설정 반영 (최소 12sp ~ 최대 20sp) |
| 버튼 탭 영역 | 최소 크기 | 최소 44×44dp 보장 |

### §3.6 A/B 테스트 설계

| 변수 | 변형 A (기본) | 변형 B | 측정 지표 |
|------|------------|-------|---------|
| 슬라이드 수 | 4장 | 3장 (CTA 포함) | 스킵율, Phase 3 전환율 |

## 4. 역할별 접근 권한 매트릭스

**웰컴화면은 로그인 전 화면이므로 역할 판별 이전 단계이다. 모든 사용자가 동일하게 접근 가능하다.**

역할은 목적 선택(Phase 3) 이후 온보딩 과정에서 결정된다.

| 사용자 상태 | 가치 슬라이드 표시 | 목적 선택 화면 표시 | 특이사항 |
|-----------|:-----------------:|:-----------------:|---------|
| 미결정 (신규, 인증 없음) | ✅ 전체 표시 | ✅ 전체 버튼 표시 | 기본 온보딩 경로 |
| 딥링크(초대코드) 진입 | ⬛ 스킵 | ✅ 초대코드 자동 입력 | 슬라이드 건너뜀 |
| 재방문 유저 (인증 토큰 있음) | ⬛ 스킵 | ⬛ 스킵 | 메인화면 직행 (스플래시 #28 §4.2) |

**역할 결정 이후 분기 (참고)**:

| 선택 버튼 | 진입 역할 경로 | 비즈니스 원칙 참조 |
|---------|------------|----------------|
| "여행 만들기" | 캡틴 (captain) 온보딩 | §01.1, §02.1 |
| "초대코드 입력" | 크루 (crew) / 크루장 (crew_chief) 온보딩 | §01.1, §02.5 |
| "가디언으로 참여" | 가디언 (guardian) 온보딩 | §01.1, §03.1 |
| "먼저 둘러보기" | 미결정 (데모 투어) | 데모 투어 원칙(#30) |

## 5. 프라이버시 등급별 동작 차이

**N/A — 로그인 전 공개 화면**

웰컴화면은 로그인 전 공개 화면으로, 프라이버시 등급(safety_first / standard / privacy_first) 판별 이전 단계이다. 따라서 프라이버시 등급별 동작 차이가 존재하지 않는다.

| 프라이버시 등급 | 웰컴화면 동작 |
|--------------|:----------:|
| safety_first (안전 최우선) | N/A (등급 판별 전) |
| standard (표준) | N/A (등급 판별 전) |
| privacy_first (프라이버시 우선) | N/A (등급 판별 전) |

프라이버시 등급 기반 동작 차이는 메인 화면 진입 이후부터 적용된다. 위치 공유 모드, 가디언 공유 정책 등 등급별 차이는 비즈니스 원칙 §04.5 및 지도탭 원칙(#17)을 참조한다.

## 6. 에러 및 엣지케이스 처리

### 6.1 에러 유형별 처리 전략

| 에러 유형 | 감지 시점 | 처리 방법 | 사용자 노출 |
|---------|---------|---------|----------|
| **딥링크 파라미터 손실** | Phase 1 맥락 감지 시 | Phase 3 직행 후 초대코드 수동 입력 안내 (슬라이드 스킵, Phase 2로 되돌아가지 않음) | "초대코드를 직접 입력해 주세요." 안내 토스트 |
| **슬라이드 이미지 로딩 실패** | Phase 2 슬라이드 렌더링 시 | 색상 배경만으로 표시 (이미지 없이도 카피·버튼 동작) | 없음 (이미지 없이도 정상 동작 보장) |
| **기기 회전** | 슬라이드 표시 중 | 슬라이드 비율 자동 조정, 가로 모드 최적화 레이아웃 적용 | 없음 (자동 처리) |
| **렌더링 타임아웃 (500ms 초과)** | 웰컴화면 최초 렌더링 | 애니메이션 비활성화 후 정적 버전 즉시 표시 | 없음 (정적 버전으로 대체) |
| **딥링크 초대코드 만료** | Phase 3 초대코드 자동 입력 후 검증 | 초대코드 원칙(#23) §06 에러 처리 규칙 적용 | "초대코드가 만료되었습니다. 새 코드를 요청해 주세요." |
| **네트워크 없음** | 초대코드 검증 시도 시 | 오프라인 처리 규칙 적용 (§8 참조) | "네트워크 연결 후 재시도해 주세요." |

### 6.2 에러 분기 결정 트리

```
웰컴화면 진입
  │
  ├─ 딥링크 파라미터 존재?
  │     ├─ YES: 파라미터 파싱 성공? ─YES→ Phase 3 직행 (초대코드 자동 입력)
  │     │                           └─NO→ Phase 3 직행 + 수동 입력 안내 토스트
  │     │                                 (Phase 2로 되돌아가지 않음)
  │     └─ NO: Phase 2 가치 슬라이드 표시
  │
  ├─ Phase 2 슬라이드 렌더링
  │     ├─ 이미지 로딩 성공? ─YES→ 정상 슬라이드 표시
  │     │                    └─NO→ 색상 배경만으로 대체 표시
  │     └─ 타임아웃(500ms)? ─YES→ 정적 버전 즉시 표시
  │
  └─ Phase 3 목적 선택
        ├─ "초대코드 입력" 탭 → 초대코드 원칙(#23) §04 검증 흐름
        └─ 딥링크 자동 입력 코드 → 동일 검증 흐름
```

## 7. DB 스키마

### 7.1 웰컴화면 직접 DB 기록

**웰컴화면 자체는 DB 기록 없음**

웰컴화면은 로그인 전 공개 화면으로, 서버 DB에 직접 접근하지 않으며 사용자 데이터를 수집하지 않는다. 어떠한 사용자 행동 데이터도 이 단계에서 DB에 저장되지 않는다.

### 7.2 연계 DB 테이블 (간접)

| 연계 상황 | 관련 테이블 | 참조 문서 |
|---------|-----------|---------|
| 딥링크 초대코드 파라미터 → 코드 검증 | `tb_invite_code` | 초대코드 원칙(#23) |
| 가디언 직접 진입 → 가디언 링크 수락 | `tb_guardian_link` | 비즈니스 원칙 §03 |
| 목적 선택 후 온보딩 완료 → 유저 생성 | `tb_user` | DB 설계(#07) §3 |

### 7.3 Analytics 이벤트 (DB 외)

A/B 테스트 변형 트래킹 및 온보딩 퍼널 분석은 Analytics 이벤트로 처리한다.

| 이벤트명 | 발생 시점 | 주요 파라미터 |
|---------|---------|------------|
| `welcome_view` | 웰컴화면 최초 표시 | `ab_variant`, `time_of_day`, `deeplink_present` |
| `slide_viewed` | 각 슬라이드 표시 | `slide_index`, `auto_or_manual` |
| `slide_skipped` | 건너뛰기 탭 | `skipped_at_slide` |
| `purpose_selected` | 목적 선택 버튼 탭 | `purpose` (`create_trip` / `join_trip` / `demo` / `guardian`) |
