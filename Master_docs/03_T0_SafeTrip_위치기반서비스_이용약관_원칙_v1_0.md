## 1. 문서 목적 및 적용 범위

### 1.1 목적

본 문서는 SafeTrip 앱 서비스(이하 "서비스")의 위치기반서비스 이용약관에 관한 설계 원칙을 정의한다. 「위치정보의 보호 및 이용 등에 관한 법률」(이하 "위치정보법") 제12조 및 동법 시행령에 따라, 위치기반서비스 사업자는 이용약관을 인터넷 홈페이지 또는 서비스 제공 단말기의 첫 화면이나 연결화면을 통해 이용자가 언제든지 열람할 수 있도록 공개해야 한다.

본 문서는 두 가지 역할을 수행한다.

1. **내부 설계 원칙**: SafeTrip 개발·운영팀이 위치정보 수집·이용·제공 로직을 구현할 때 참조하는 기술적 기준
2. **외부 공개 약관의 원천**: 앱 내 및 웹사이트에 공개되는 위치기반서비스 이용약관의 원문 소스이자, 방송미디어통신위원회 신고 시 제출 자료의 기반

### 1.2 적용 범위

- SafeTrip 모바일 앱(iOS, Android)에서 제공하는 모든 위치기반서비스
- SafeTrip 웹 서비스(관리 콘솔)에서의 위치정보 열람·관리
- B2B 고객(학교, 여행사, 보험사)이 SafeTrip을 통해 이용하는 위치기반서비스
- 데모 모드에서의 위치 시뮬레이션(실제 위치정보 수집 제외)

### 1.3 법적 근거

## 3. 위치기반서비스의 내용

### 3.1 서비스 개요

SafeTrip은 여행 안전을 목적으로 개인위치정보를 수집·이용하여 다음의 위치기반서비스를 제공한다.

### 3.2 서비스 목록 및 이용 목적

| # | 서비스명 | 서비스 내용 | 위치정보 이용 목적 | 보유 기간 |
|:-:|---------|-----------|----------------|----------|
| 1 | **실시간 위치 공유** | 여행 그룹 내 멤버 간 실시간 위치를 지도 위에 표시 | 여행 그룹 멤버의 안전 확인 및 위치 파악 | 여행 종료 후 90일 |
| 2 | **SOS 긴급 알림** | SOS 발동 시 발동자의 현재 위치를 그룹 멤버 및 가디언(보호자)에게 전달 | 긴급 상황에서 구조 지원을 위한 위치 공유 | 여행 종료 후 3년 (법적 분쟁 대비) |
| 3 | **자동 SOS (Heartbeat 타임아웃)** | 서버와 통신이 설정된 시간 이상 두절될 때 마지막 알려진 위치를 자동 전송 | 비자발적 통신 두절 시 안전 경보 | 여행 종료 후 3년 |
| 4 | **지오펜스 알림** | 캡틴이 설정한 안전 구역의 출입 감지 시 관련 멤버에게 알림 전달 | 그룹 멤버가 안전 구역을 이탈·진입할 때 관리자 및 보호자에게 알림 제공 | 여행 종료 후 90일 |
| 5 | **이동기록 타임라인** | 여행 기간 중 멤버의 이동 경로를 타임라인 형태로 기록·표시 | 여행 기록 관리, 체류 지점 확인, 일정 매칭 | 여행 종료 후 30일 |
| 6 | **출석 체크** | 일정에 등록된 장소와 멤버의 현재 위치를 비교하여 출석 여부 판정 | 단체 일정 관리, 장소 도착 확인 | 여행 종료 후 90일 |
| 7 | **가디언(보호자) 위치 모니터링** | 가디언이 연결된 멤버의 위치를 프라이버시 등급에 따라 열람 | 보호자의 여행 안전 확인 | 여행 종료 후 90일 |
| 8 | **안전가이드 (위치 기반 정보)** | 현재 위치 또는 여행 목적지 기반으로 외교부 여행 경보, 대사관 위치 등 안전 정보 제공 | 이용자 위치에 적합한 안전 정보 제공 | 즉시 폐기 (위치 자체를 별도 저장하지 않음) |
| 9 | **AI 안전 추천** | 현재 위치와 일정 데이터를 분석하여 안전 관련 맞춤형 추천 제공 | 이용자 위치 기반 지능형 안전 분석 | AI 처리 후 즉시 폐기 (분석 결과만 저장) |

## 4. 위치정보의 수집 방법

### 4.1 수집 기술

회사는 다음의 기술을 사용하여 이용자의 위치정보를 수집한다.

| 수집 기술 | 설명 | 정확도 |
|----------|------|--------|
| GPS (위성항법장치) | 이용자 단말기의 GPS 수신기를 통해 위성 신호로 위치 측정 | 약 3~10m |
| Wi-Fi | 주변 Wi-Fi 액세스 포인트 정보를 활용한 위치 추정 | 약 15~40m |
| 이동통신 기지국 | 연결된 셀 타워 정보를 기반으로 위치 추정 | 약 100~1,000m |
| 복합 측위 | 상기 기술을 조합하여 최적의 위치 산출 | 환경에 따라 상이 |

### 4.2 수집 형태

수집되는 개인위치정보의 형태는 다음과 같다.

- 위도(latitude) 및 경도(longitude) 좌표
- 수집 시각(timestamp)
- 측위 정확도(accuracy, 미터 단위)
- 측위 방식(GPS / Wi-Fi / 셀룰러 / 복합)
- 이동 속도(speed, m/s — Heartbeat 판단용)
- 배터리 잔량 및 네트워크 상태(Heartbeat 데이터로서, 위치정보와 함께 전송)

## 5. 위치정보의 이용 및 제공

### 5.1 이용 범위

회사는 이용자의 동의를 얻어 수집한 개인위치정보를 본 약관 제3조(위치기반서비스의 내용)에 명시된 서비스 제공 목적 범위 내에서만 이용한다. 이용약관에 명시하거나 고지한 범위를 넘어 개인위치정보를 이용하지 않는다.

### 5.2 제3자 제공

회사는 이용자의 동의가 있거나 다음 각 호에 해당하는 경우를 제외하고 개인위치정보를 제3자에게 제공하지 않는다.

| # | 제공 대상 | 제공 목적 | 제공 항목 | 법적 근거 |
|:-:|----------|----------|----------|----------|
| 1 | 동일 여행 그룹 멤버 (캡틴, 크루장, 크루) | 그룹 내 안전 모니터링, 실시간 위치 공유 | 실시간 GPS 좌표, 최종 갱신 시각 | 이용자 동의 (위치정보법 제19조) |
| 2 | 가디언(보호자) | 연결된 멤버의 여행 안전 모니터링 | 프라이버시 등급에 따른 위치 정보 (§5.3 참조) | 이용자 동의 (위치정보법 제19조) |
| 3 | 긴급 구조기관 (119, 112, 해외 대사관 등) | SOS 발동 시 구조 지원 | SOS 발동자의 위치 좌표, 멤버 정보 | 위치정보법 제29조 (긴급구조) |
| 4 | 외교부 영사콜센터 (1335) | 해외 긴급 상황 시 영사 조력 | 이용자의 위치 좌표, 여행 목적지 | 이용자 동의 및 위치정보법 제29조 |
| 5 | AI 서비스 제공사 (LLM Provider) | AI 기반 안전 분석·추천 | 익명화 또는 최소화된 위치 데이터 | 이용자 동의 (AI 데이터 활용 선택 동의) |

> **제공 방식**: 그룹 멤버 및 가디언에 대한 위치 제공은 SafeTrip 앱 내에서만 이루어지며, 외부 반출(다운로드, 스크린 캡처 제외)되지 않는다.

### 5.3 프라이버시 등급별 위치정보 제공 범위

SafeTrip은 비즈니스 원칙 v5.1에 따라 3단계 프라이버시 등급을 운영하며, 등급에 따라 가디언에 대한 위치정보 제공 범위가 달라진다.

| 프라이버시 등급 | 그룹 내 멤버 간 | 가디언 (공유 ON 시간) | 가디언 (공유 OFF 시간) | 가디언 일시 중지 |
|:-------------:|:--------------:|:-------------------:|:--------------------:|:--------------:|
| 🛡️ 안전 최우선 | 항상 실시간 | 항상 실시간 | 실시간 공유 (항상) | 불가 |
| 📍 표준 | 설정에 따라 | 실시간 | 30분 스냅샷 | 최대 12시간 |
| 🔒 프라이버시 우선 | 설정에 따라 | 실시간 | 비공유 | 최대 24시간 |

> **SOS 예외**: 모든 프라이버시 등급에서 SOS 발동 시에는 위치 공유 설정·가디언 일시 중지와 무관하게 모든 그룹 멤버 및 가디언에게 위치가 제공된다 (SOS 원칙 §1.1 참조).

## 6. 위치정보 이용·제공사실 확인자료

### 6.1 기록·보존 의무

회사는 위치정보법 제16조 제2항에 따라 개인위치정보의 이용·제공사실 확인자료를 위치정보시스템에 자동으로 기록하며, 다음의 항목을 포함한다.

| 기록 항목 | 설명 |
|----------|------|
| 이용·제공 일시 | 개인위치정보가 이용되거나 제공된 시각 |
| 이용·제공 목적 | 어떤 서비스 기능을 위해 이용·제공되었는지 |
| 제공받은 자 | 위치정보를 열람·수신한 대상 (그룹 멤버, 가디언 등) |
| 이용·제공 내용 | 제공된 위치정보의 범위 (실시간 좌표, 스냅샷 등) |

### 6.2 보존 기간

위치정보 이용·제공사실 확인자료는 기록 시점으로부터 **6개월간** 보존한다. 이용자가 위치정보 수집·이용 동의를 철회한 경우, 철회 시점 이후의 확인자료만 보존하고 이전 자료는 지체 없이 파기한다.

### 6.3 DB 참조

```
TB_LOCATION_ACCESS_LOG
├── log_id (BIGINT, PK)
├── user_id (VARCHAR, 위치정보 주체)
├── accessed_by_user_id (VARCHAR, 열람자)
├── access_type (ENUM: 'realtime_view', 'history_view', 'sos_broadcast', 'geofence_alert', 'guardian_snapshot', 'guardian_request', 'attendance_check')
├── trip_id (VARCHAR)
├── latitude (DECIMAL — 암호화 저장)
├── longitude (DECIMAL — 암호화 저장)
├── access_purpose (VARCHAR, 서비스 목적 코드)
├── created_at (TIMESTAMPTZ)
└── expired_at (TIMESTAMPTZ, created_at + 6개월)
```

### 7.2 파기 절차

```
파기 프로세스
│
├── 1단계: 보존 기간 만료 감지 (일일 배치 — 매일 03:00 UTC)
│   · TB_LOCATION_HISTORY에서 expired_at < NOW() 레코드 스캔
│
├── 2단계: 소프트 삭제 (is_deleted = true)

│   · user_id → 해시 처리

│   · 통계 분석용으로만 보존 (선택)
│
└── 4단계: 물리적 삭제 (30일 후)
```

### 7.3 동의 철회 시 파기

위치정보법 제24조 제4항에 따라 이용자가 동의의 전부 또는 일부를 철회하면, 회사는 지체 없이 수집된 개인위치정보 및 위치정보 이용·제공사실 확인자료(동의의 일부를 철회하는 경우 해당 부분에 한함)를 파기한다. 단, 다른 법령에 따라 보존 의무가 있는 데이터(SOS 관련 기록 등)는 해당 법률에서 정한 기간까지 보관 후 파기한다.

## 8. 개인위치정보주체의 권리

### 8.1 권리 목록

이용자(개인위치정보주체)는 위치정보법 제24조에 따라 다음의 권리를 행사할 수 있다.

| # | 권리 | 설명 | 행사 방법 |
|:-:|------|------|----------|
| 1 | 동의 철회권 | 개인위치정보 수집·이용·제공에 대한 동의의 전부 또는 일부를 언제든 철회 | 앱 설정 > 개인정보 > 위치정보 동의 관리 |
| 2 | 일시 중지권 | 개인위치정보의 수집·이용·제공을 일시적으로 중지 요청 | 앱 설정 > 위치 공유 > 일시 중지 |
| 3 | 열람 청구권 | 본인의 개인위치정보 및 이용·제공사실 확인자료 열람 | 앱 설정 > 개인정보 > 내 위치정보 열람 |
| 4 | 고지 청구권 | 개인위치정보의 이용·제공 사실에 대한 통보 요청 | 앱 설정 > 알림 > 위치정보 이용 알림 |
| 5 | 오류 정정권 | 개인위치정보 및 확인자료에 오류가 있는 경우 정정 요청 | 앱 설정 또는 고객센터(support@safetrip.app) |

### 8.2 권리 행사 절차

```
권리 행사 요청
│
├── 앱 내 직접 행사 (즉시 처리)
│   ├── 동의 철회: 설정 > 위치정보 동의 관리 > 토글 OFF → 확인 다이얼로그
│   ├── 일시 중지: 설정 > 위치 공유 > 일시 중지 → 기간 선택
│   └── 열람: 설정 > 내 위치정보 열람 → 기간별·유형별 조회
│
├── 고객센터 접수 (3영업일 내 처리)
│   ├── 이메일: support@safetrip.app
│   └── 인앱 문의
│
└── 처리 완료 알림
    · 동의 철회: "위치정보 수집이 중지되었습니다" 푸시 + 인앱 알림
    · 열람 완료: 열람 가능 상태 안내 알림
    · 오류 정정: 정정 결과 안내 알림
```

### 8.3 동의 철회의 효과

| 항목 | 동의 철회 전 | 동의 철회 후 |
|------|-----------|-----------|
| 지도에서 내 위치 표시 | 다른 멤버에게 표시됨 | 표시 안 됨 ("위치 비공유" 상태) |
| 가디언에게 위치 제공 | 등급 정책에 따라 제공 | 제공 중단 |
| 이동기록 | 기록됨 | 기록 중단, 기존 데이터 파기 |
| SOS 발동 시 | 현재 위치 포함하여 전송 | ⚠️ **위치 미포함** 상태로 SOS 전송 (위치 동의 철회에도 SOS 기능 자체는 이용 가능) |
| 지오펜스 | 감지됨 | 감지 중단 |
| 출석 체크 | 자동 판정 | 수동 처리 필요 |

> **SOS와 동의 철회의 관계**: 위치정보 수집 동의를 철회한 경우에도 SOS 버튼은 사용 가능하나, 위치 좌표가 포함되지 않은 채로 알림이 발송된다. SOS 발동 시점에 일시적으로 위치 수집 동의를 재요청하는 긴급 동의 팝업은 별도 검토한다 (P2).

## 9. 법정대리인의 권리

### 9.1 14세 미만 아동

위치정보법 제25조 및 개인정보 보호법 제22조에 따라, 14세 미만 아동의 개인위치정보를 수집·이용·제공하려면 법정대리인의 동의를 받아야 한다.

| 항목 | 내용 |
|------|------|
| 동의 대상 | 14세 미만 이용자의 개인위치정보 수집·이용·제공 전체 |
| 동의 방법 | 법정대리인의 휴대전화 본인 인증 후 동의 (동의 내용을 인터넷 사이트에 게재하고 법정대리인의 휴대전화 문자메시지로 알리는 방법) |
| 법정대리인의 권리 | 제8조(개인위치정보주체의 권리)에 규정된 모든 권리를 대리 행사 가능 |
| 동의 철회 | 법정대리인은 언제든 동의의 전부 또는 일부를 철회 가능 |

### 9.2 8세 이하 아동 및 피성년후견인

위치정보법 제26조에 따라 다음에 해당하는 사람의 보호의무자가 개인위치정보의 수집·이용·제공에 동의할 수 있다.

- 8세 이하의 아동
- 피성년후견인
- 「장애인복지법」 제2조 제2항 제2호에 따른 정신적 장애를 가진 사람으로서 「장애인고용촉진 및 직업재활법」 제2조 제2호에 따른 중증장애인에 해당하는 사람

| 항목 | 내용 |
|------|------|
| 동의 주체 | 보호의무자 (친권자, 후견인 등) |
| 동의 방법 | 보호의무자의 본인 인증 + 보호 관계 증빙 후 동의 |
| 보호의무자의 권리 | 제8조의 모든 권리를 대리 행사 가능 |
| SafeTrip에서의 적용 | B2B 학교 시나리오에서 학생(8세 이하 해당 시) 참여 시 적용 |

### 9.3 연령별 동의 체계 요약

비즈니스 원칙 v5.1 §10에 정의된 연령별 3단계 동의 체계를 위치정보에도 적용한다.

| 연령 구간 | 위치정보 동의 주체 | 동의 방식 |
|----------|----------------|----------|
| 8세 이하 | 보호의무자 단독 | 보호의무자 본인 인증 + 관계 증빙 |
| 9~13세 (14세 미만) | 본인 + 법정대리인 | 본인 동의 + 법정대리인 휴대전화 인증 |
| 14~17세 | 본인 (법정대리인 동의 권장) | 본인 동의, 법정대리인에게 고지 |
| 18세 이상 | 본인 | 본인 동의 |

| 개인 가디언 (멤버당) | 2명까지 무료 / 3~5번째 각 1,900원/여행 | 멤버당 최대 2명 무료, 이후 유료 추가 (최대 5명) |
| 전체 가디언 (여행당) | 무료 | 멤버당 최대 2명 (무료, 추가 불가) |

가디언 과금은 서비스 이용약관에서 상세히 정의하며, 위치기반서비스 자체에 대한 별도 요금은 부과하지 않는다.

## 13. 사업의 양도·합병 시 조치

### 13.1 고지 의무

회사가 위치기반서비스사업의 전부 또는 일부를 양도하거나 합병·분할되는 경우, 회사는 양도·합병·분할 예정일의 30일 전까지 다음 사항을 이용자에게 통보한다.

- 위치기반서비스사업의 양도·합병·분할 사실
- 양수인 또는 합병·분할 후 존속하는 법인의 명칭, 주소, 연락처
- 이용자의 권리 및 그 행사 방법

### 13.2 통보 방법

- 앱 내 팝업 공지
- 푸시 알림
- 등록된 이메일(있는 경우)
- 인터넷 홈페이지 공지

## 16. 이용약관 변경

### 16.1 변경 절차

회사가 본 약관을 변경하려는 경우, 변경 사유 및 변경 내용을 지체 없이 공개하고 변경된 사항을 이용자가 쉽게 알아볼 수 있도록 조치한다.

| 변경 유형 | 사전 고지 기간 | 고지 방법 |
|----------|:------------:|----------|
| 이용자에게 불리한 변경 | 시행일 30일 전 | 앱 푸시 + 인앱 팝업 + 홈페이지 공지 |
| 단순 표현 수정, 법률 반영 등 | 시행일 7일 전 | 앱 내 공지 + 홈페이지 공지 |

### 16.2 변경 동의 처리

이용자에게 불리한 약관 변경 시, 이용자가 변경된 약관에 동의하지 않으면 서비스 이용을 중단하고 이용 계약을 해지할 수 있다. 고지된 기간 내에 별도의 거부 의사를 표시하지 않으면 변경된 약관에 동의한 것으로 간주한다.

### 17.1 위치 수집 관련

## 18. DB 스키마

### 18.1 TB_LBS_CONSENT (위치정보 동의 관리)

개인정보처리방침 원칙의 `TB_USER_CONSENT`와 연동하되, 위치정보 특화 동의 항목을 관리한다.

```sql
-- TB_USER_CONSENT의 consent_type에 위치정보 관련 유형 포함:
-- 'location_collection'     : 위치정보 수집·이용 동의 (필수)
-- 'lbs_terms'               : 위치기반서비스 이용약관 동의 (필수)
-- 'location_third_party'    : 위치정보 제3자 제공 동의 (그룹 공유)
-- 'guardian_location_share'  : 가디언 위치 공유 동의

-- 위치정보 이용·제공사실 확인자료 (위치정보법 제16조)
CREATE TABLE tb_location_access_log (
    log_id              BIGINT PRIMARY KEY GENERATED ALWAYS AS IDENTITY,
    user_id             VARCHAR(128) NOT NULL,      -- 위치정보 주체
    accessed_by_user_id VARCHAR(128),                -- 열람자 (NULL = 시스템)
    access_type         VARCHAR(30) NOT NULL,
        -- 'realtime_view'      : 실시간 위치 열람
        -- 'history_view'       : 이동기록 열람
        -- 'sos_broadcast'      : SOS 위치 전송
        -- 'geofence_alert'     : 지오펜스 알림 발동
        -- 'guardian_snapshot'   : 가디언 스냅샷 제공
        -- 'guardian_request'    : 가디언 긴급 위치 요청
        -- 'attendance_check'   : 출석 체크
        -- 'ai_analysis'        : AI 분석용 위치 이용
        -- 'safety_guide'       : 안전가이드 위치 조회
    trip_id             VARCHAR(128),
    location_data       JSONB,                       -- 암호화된 좌표 데이터
    access_purpose      VARCHAR(200),                -- 이용 목적 상세
    created_at          TIMESTAMPTZ DEFAULT NOW(),
    expired_at          TIMESTAMPTZ                  -- created_at + 6개월
);

CREATE INDEX idx_loc_access_user ON tb_location_access_log(user_id, created_at DESC);
CREATE INDEX idx_loc_access_type ON tb_location_access_log(access_type);
CREATE INDEX idx_loc_access_expired ON tb_location_access_log(expired_at);
```

### 18.2 TB_LOCATION_SHARING_PAUSE (가디언 일시 중지 이력)

```sql
-- 기존 TB_GUARDIAN_PAUSE와 연동하여 위치정보법 준수 이력 기록
CREATE TABLE tb_location_sharing_pause_log (
    pause_log_id        BIGINT PRIMARY KEY GENERATED ALWAYS AS IDENTITY,
    user_id             VARCHAR(128) NOT NULL,       -- 일시 중지 요청자
    guardian_user_id    VARCHAR(128) NOT NULL,        -- 대상 가디언
    trip_id             VARCHAR(128) NOT NULL,
    privacy_level       VARCHAR(20) NOT NULL,         -- 시점의 프라이버시 등급
    pause_duration_hours INTEGER NOT NULL,            -- 요청 중지 시간
    max_allowed_hours   INTEGER NOT NULL,             -- 등급별 허용 최대 시간
    paused_at           TIMESTAMPTZ NOT NULL,
    resumed_at          TIMESTAMPTZ,                  -- NULL = 아직 중지 중
    resume_reason       VARCHAR(30),
        -- 'auto_expire'    : 시간 만료 자동 해제
        -- 'user_manual'    : 사용자 수동 해제
        -- 'sos_override'   : SOS 발동으로 강제 해제
        -- 'admin_override' : 관리자 강제 해제
    created_at          TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_pause_log_user ON tb_location_sharing_pause_log(user_id, trip_id);
```

## 20. 오프라인 대응

### 20.1 오프라인 상태에서의 위치정보 처리

| 상황 | 위치 수집 | 위치 전송 | 동의 관리 |

| 네트워크 복구 | — | ✅ 큐잉 데이터 일괄 전송 | ✅ 큐잉 요청 전송 |

### 20.2 오프라인 위치 큐잉 규칙

- 오프라인 상태에서 수집된 위치 데이터는 단말기 로컬 저장소에 암호화하여 보관
- 최대 큐잉 용량: 72시간 분량 (약 8,640건 기준, 30초 주기)
- 네트워크 복구 시 전송 우선순위: SOS 관련 > 지오펜스 이벤트 > 일반 위치 데이터
- 동의 철회 요청이 큐잉된 경우, 네트워크 복구 시 철회 요청을 최우선 전송하고, 철회 시점 이후에 수집된 큐잉 데이터는 전송하지 않고 로컬에서 삭제
