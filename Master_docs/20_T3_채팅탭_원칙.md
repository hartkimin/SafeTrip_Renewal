## 1. 문서 목적 및 적용 범위

### 1.1 목적

본 문서는 SafeTrip 앱의 채팅탭(Chat Tab) 전체 설계 원칙을 정의한다. 채팅탭은 여행 그룹 내 커뮤니케이션 채널로, 텍스트 메시지·위치 공유 카드·일정 카드·투표 등의 메시지 유형과 SOS 시스템 메시지를 통합한다. 또한 가디언은 그룹 채팅에 접근할 수 없으며, 멤버 ↔ 가디언 전용 보호자 채널이 완전히 분리된다. 비즈니스 원칙 v5.1의 가디언 채널 분리(§03), 위치 공유(§04), SOS(§05), 오프라인(§11), 데이터 생명주기(§13)를 UI 레벨에서 구현하는 규칙을 명세한다.

### 1.2 적용 범위

| 범위 | 포함 여부 |
|------|:--------:|
| 그룹 채팅 (텍스트, 이미지, 위치 카드, 파일) | ✅ |
| 시스템 메시지 (멤버 합류/탈퇴, SOS, 출석 체크, 일정 변경) | ✅ |
| 리치 카드 (위치 카드, 일정 카드, 투표 카드) | ✅ |
| SOS 긴급 시스템 메시지 (자동 삽입) | ✅ |
| 가디언 채널 분리 (보호자 메시지 탭) | ✅ |
| 무료/유료 가디언 채널 기능 차이 | ✅ |
| 공지 고정 (최대 3건, 캡틴/크루장 전용) | ✅ |

| 조항 | 내용 |

## 2. 설계 철학 5원칙

| ID | 원칙명 | 내용 |
|----|--------|------|
| **C1** | **맥락 통합** | 일정·위치·SOS 정보를 채팅에 자연스럽게 통합한다. 별도 탭을 오가지 않아도 현재 상황을 채팅 맥락에서 파악할 수 있어야 한다. |
| **C2** | **빠른 공유** | 탭 2번으로 현재 위치 또는 일정을 채팅에 공유할 수 있다. 공유 흐름은 최소 탭 수를 원칙으로 설계한다. |
| **C3** | **안전 우선** | SOS 발신 시 채팅에 자동 시스템 메시지가 삽입된다. SOS 메시지는 CRITICAL 채널로 처리하여 읽지 않은 메시지보다 위에 고정된다. |
| **C4** | **오프라인 대응** | 오프라인 시 메시지 작성이 가능하며, 로컬에 큐잉(최대 100건)된 후 온라인 복귀 시 자동으로 순서대로 발송된다. |
| **C5** | **프라이버시 분리** | 그룹 채팅과 가디언 채널은 완전히 분리된다. 가디언은 그룹 채팅에 접근할 수 없으며, 가디언 채널에서 공유되는 위치 정보는 프라이버시 등급에 따라 제한된다. |

## 3. 화면 구성

### 3.1 그룹 채팅 화면 레이아웃

```
┌─────────────────────────────────────────────┐
│  ← 여행명  멤버 7명  🔍                      │  ← 헤더
├─────────────────────────────────────────────┤
│  📌 공지 1: "집합 장소: 광화문 광장 (3/15)" │  ← 고정 공지 (최대 3건)
│  📌 공지 2: "체크아웃 10시"                 │
├─────────────────────────────────────────────┤
│  ─────── 3월 15일 ────────────────────────  │  ← 날짜 구분선
│                                              │
│  [사진] 홍길동                               │  ← 상대방 메시지 (좌측)
│  안녕하세요! 모두 잘 이동 중이신가요?        │
│  오전 10:23                                  │
│                                              │
│  ─────── 오늘 ──────────────────────────── │
│                                              │
│  [🚨 SOS 시스템 메시지 카드]                │  ← CRITICAL, 자동 삽입
│                                              │
│               내 메시지 (우측) [✅ 읽음]    │
│                              오후 2:15      │
│                                              │
│  [📍 위치 카드 — 강남구 역삼동]              │  ← 리치 카드
│                                              │
├─────────────────────────────────────────────┤
│  [+]  메시지를 입력하세요…          [→전송]  │  ← 입력 영역
└─────────────────────────────────────────────┘
```

### 3.2 화면 구성 요소 상세

| 영역 | 설명 |
|------|------|
| **헤더** | 여행명 + 현재 온라인 멤버 수. 🔍 탭 시 메시지 검색 (P3). |
| **공지 영역** | 캡틴/크루장이 고정한 메시지 최대 3건. 탭 시 해당 메시지 위치로 스크롤. |
| **날짜 구분선** | "오늘", "어제", "MM월 DD일" 형식. 날짜 변경 시 자동 삽입. |
| **메시지 영역** | 스크롤 가능. 새 메시지 수신 시 자동 스크롤 다운 (현재 화면 최하단인 경우만). |
| **입력 영역** | 텍스트 입력 + [+] 첨부 메뉴 + 전송 버튼. 멀티라인 입력 지원 (최대 3줄 후 스크롤). |

### 3.3 탭 구조 (채팅탭 내부)

```
채팅탭
  ├── [그룹 채팅] (멤버 전용)
  └── [보호자 메시지] (멤버 ↔ 가디언 1:1 채널)
```

가디언은 채팅탭 진입 시 [보호자 메시지] 탭만 보인다.

## 4. 메시지 유형 3종

### 4.1 사용자 메시지

| 세부 유형 | 설명 | 제한 |
|----------|------|------|
| **텍스트** | 일반 텍스트 메시지 | 최대 2,000자 |
| **이미지/동영상** | 갤러리 또는 카메라 촬영 첨부 | 이미지 최대 20MB, 동영상 최대 50MB |
| **위치 공유 카드** | 현재 위치 공유 (미니 지도 임베드) | 실시간 위치 1회성 스냅샷 |
| **파일 첨부** | 일반 파일 | 최대 50MB |

### 4.2 시스템 메시지

시스템 메시지는 사용자가 전송하는 것이 아니라 서버에서 자동으로 삽입된다. 회색 배경에 중앙 정렬 텍스트로 표시한다.

| 시스템 메시지 유형 | 표시 텍스트 예시 |
|-------------------|----------------|
| 멤버 합류 | "[홍길동]님이 여행에 합류했습니다." |
| 멤버 탈퇴 | "[홍길동]님이 여행을 떠났습니다." |
| 멤버 강퇴 | "[홍길동]님이 여행에서 강퇴되었습니다." |
| 역할 변경 | "[홍길동]님이 크루장으로 변경되었습니다." |
| 리더 위임 | "[홍길동]님이 캡틴이 되었습니다." |
| 여행 시작 | "여행이 시작되었습니다." |
| 여행 종료 | "여행이 종료되었습니다." |
| SOS 발신 | *(§5 별도 정의 — CRITICAL 카드 형식)* |
| SOS 해제 | "[홍길동]님의 SOS가 해제되었습니다." |
| 출석 체크 시작 | "출석 체크가 시작되었습니다. 10분 내 응답해 주세요." |
| 출석 체크 완료 | "출석 체크 완료: ✅ 5명 / ❌ 1명 / ⏳ 1명" |
| 프라이버시 등급 변경 | "여행 프라이버시 등급이 [표준]으로 변경되었습니다." |
| 일정 변경 (연동 모드) | "[3월 16일] 일정이 변경되어 위치 공유 시간이 갱신되었습니다." |
| 가디언 추가 | "[홍길동]님의 가디언으로 [이모니]님이 연결되었습니다." |
| 가디언 해제 | "[홍길동]님의 가디언 [이모니]님이 해제되었습니다." |

### 4.3 리치 카드

리치 카드는 단순 텍스트가 아닌 인터랙티브한 카드 형태의 메시지이다.

| 카드 유형 | 표시 내용 | 탭 시 동작 |
|----------|----------|----------|
| **위치 카드** | 미니 지도 + 주소 + 공유 시각 | 지도탭에서 해당 위치 전체 화면 보기 |
| **일정 카드** | 일정 제목 + 날짜/시간 + 장소 | 일정탭에서 해당 일정 상세 보기 |
| **투표 카드** | 질문 + 선택지 + 현재 투표율 + 마감 시간 | 투표 선택 화면 |

## 5. SOS 시스템 메시지

### 5.1 SOS 자동 삽입 메시지 (CRITICAL)

SOS 발신 시 그룹 채팅에 다음 카드가 자동 삽입된다. 이 메시지는 읽지 않은 일반 메시지보다 위에 항상 고정된다.

```
┌──────────────────────────────────────────────┐
│  🚨 [홍길동]님이 SOS를 발신했습니다.          │
│  ─────────────────────────────────────────   │
│  📍 위치: 강남구 역삼동 (오후 3:32 기준)      │
│  🔋 배터리: 45%                               │
│  ─────────────────────────────────────────   │
│  [지도에서 확인]   [메시지]   [119 안내]      │
└──────────────────────────────────────────────┘
```

### 5.2 SOS 메시지 동작 규칙

- SOS 발신 시 FCM 최고 우선순위(`PRIORITY_HIGH`)로 채팅 알림 발송.
- 채팅 화면이 열려 있으면 SOS 카드가 최상단에 자동 스크롤.
- SOS가 해제되면 카드 배경이 회색으로 변경되고 "해제됨 (N시 N분)" 텍스트가 추가된다.
- SOS 카드는 삭제할 수 없다 (법적 근거: 위치정보법 제29조).
- SOS 발신자의 위치는 프라이버시 설정과 무관하게 카드에 포함된다 (§05.1).

### 5.3 지속 상태 배너 (안전최우선 등급)

`safety_first` 등급 여행에서 SOS가 활성 중인 동안, 채팅 화면 상단에 지속 배너가 고정된다:

```
[🚨 SOS 활성 중 — 홍길동님 · 강남구 역삼동] [지도]
```

## 6. 가디언 채널 분리

### 6.1 채널 분리 원칙 (§03.2)

- 가디언은 그룹 채팅에 접근할 수 없다.
- 멤버와 가디언은 1:1 보호자 채널을 통해서만 소통한다.
- 보호자 채널은 그룹 대화가 아닌 멤버 ↔ 특정 가디언 간의 독립 채널이다.
- RTDB 채널 키: `link_{linkId}` (tb_guardian_link.id 기반)

### 6.2 보호자 메시지 탭 구성

```
┌─────────────────────────────────────────────┐
│  ← 보호자 메시지                             │
├─────────────────────────────────────────────┤
│  [가디언 사진] 이모니 (무료 🆓)              │  ← 보호자 채널 목록
│  "잘 도착했나요?"  오전 10:15   [읽음]       │
├─────────────────────────────────────────────┤
│  [가디언 사진] 강감찬 (유료 💎)              │
│  📍 위치 카드  오후 2:30   [읽음]            │
└─────────────────────────────────────────────┘
```

### 6.3 무료/유료 가디언 채널 기능 차이 (§03.1, §09.3)

| 기능 | 무료 가디언 채널 | 유료 가디언 채널 |
|------|:--------------:|:--------------:|
| 텍스트 메시지 | ✅ | ✅ |
| 이미지 전송 | ✅ | ✅ |
| 위치 공유 카드 수신 | ❌ (위치 정보 제한) | ✅ |

### 6.4 가디언 → 멤버 메시지 발신 규칙

- 가디언이 메시지를 발신하면, 해당 멤버의 채팅탭 [보호자 메시지] 탭에 수신.
- 멤버는 가디언 채널에서 답장 가능.
- 캡틴/크루장은 멤버 ↔ 가디언 채널 내용을 열람하지 못한다 (1:1 프라이버시).

## 11. 프라이버시 등급별 동작 차이

### 11.1 채팅 본문 텍스트

채팅 텍스트 자체는 프라이버시 등급의 직접 영향을 받지 않는다. 멤버 간 텍스트 커뮤니케이션은 등급과 무관하게 항상 허용된다.

### 11.2 위치 공유 카드 전송 시 경고

| 프라이버시 등급 | 위치 비공유 시간대 중 위치 카드 전송 시 |
|--------------|:------------------------------------:|
| `safety_first` | 경고 없음 (항상 공유 중) |
| `standard` | 경고 없음 (저빈도 공유 중이나 카드 전송은 허용) |
| `privacy_first` | "현재 위치 비공유 시간대입니다. 위치 카드를 보내시겠습니까?" 확인 다이얼로그 |

- `privacy_first` 등급에서도 사용자가 명시적으로 동의하면 위치 카드 전송이 가능하다.
- 비공유 시간대에 전송된 위치 카드에는 "비공유 시간 발송" 배지가 표시된다.

### 11.3 SOS 시스템 메시지 지속 배너 (안전최우선)

| 프라이버시 등급 | SOS 발신 후 채팅 상단 배너 |
|--------------|:------------------------:|
| `safety_first` | 해제 시까지 지속 배너 유지 |
| `standard` | SOS 카드 삽입 후 배너 없음 |
| `privacy_first` | SOS 카드 삽입 후 배너 없음 |

### 11.4 가디언 채널 위치 카드 수신 제한

| 프라이버시 등급 | 가디언 채널 위치 카드 수신 조건 |
|--------------|:-----------------------------:|
| `safety_first` | 항상 수신 가능 (유료 가디언만) |
| `standard` + ON 시간대 | 수신 가능 (유료 가디언만) |
| `standard` + OFF 시간대 | "비공유 시간대" 배지 표시 후 카드 표시 |
| `privacy_first` + ON 시간대 | 수신 가능 (유료 가디언만) |
| `privacy_first` + OFF 시간대 | 위치 카드 내용 숨김 ("비공유 시간입니다") |

## 12. 에러 및 엣지케이스 처리

### 12.1 메시지 발송 실패

| 상황 | 처리 |
|------|------|
| 네트워크 오류로 발송 실패 | 메시지 옆 ⚠️ 아이콘 표시 + "재전송" 버튼 제공 |
| 오프라인 상태에서 발송 시도 | 로컬 큐잉 처리 + 시계 ⏳ 아이콘 표시 |
| 큐 100건 초과 | "오프라인 메시지 한도(100건)에 도달했습니다. 연결 후 발송됩니다." 안내 |
| 파일 크기 50MB 초과 | 첨부 시도 시점에 "파일 크기는 50MB 이하여야 합니다" 안내 + 차단 |

### 12.2 메시지 삭제 규칙

| 상황 | 처리 |
|------|------|
| 크루가 본인 메시지 삭제 | 메시지 내용 → "삭제된 메시지입니다" 표시 (메시지 자체는 유지) |
| 캡틴이 타인 메시지 삭제 | 동일하게 "삭제된 메시지입니다" 표시 |
| SOS 시스템 메시지 삭제 시도 | 삭제 불가 ("SOS 기록은 삭제할 수 없습니다") |
| 공지 고정 메시지 삭제 시도 | 공지 해제 후 삭제 가능 (캡틴/크루장만) |

### 12.3 투표 관련 엣지케이스

| 상황 | 처리 |
|------|------|
| 투표 마감 후 응답 시도 | "투표가 종료되었습니다" 안내 + 결과만 표시 |
| 투표 생성 중 네트워크 끊김 | 임시 저장 후 온라인 복귀 시 재전송 |
| 마감 시간이 여행 종료 후인 경우 | 여행 종료 시 자동 종료 처리 |

### 12.4 가디언 채널 엣지케이스

| 상황 | 처리 |
|------|------|
| 가디언 연결이 해제된 후 채널 접근 | "연결이 해제된 가디언입니다. 메시지를 더 이상 보낼 수 없습니다." 안내 |
| 무료 가디언이 위치 카드 수신 시도 | 카드 내용 숨김 + "유료 가디언으로 업그레이드하면 위치 정보를 받을 수 있습니다" 배너 |
| 가디언이 그룹 채팅 탭 직접 접근 시도 | "그룹 채팅은 여행 멤버만 접근할 수 있습니다" 안내 + 보호자 메시지 탭으로 리다이렉트 |

### 12.5 위치 카드 관련 엣지케이스

| 상황 | 처리 |
|------|------|
| 위치 권한 없이 위치 카드 발송 시도 | "위치 권한이 필요합니다. 설정에서 허용해 주세요." 안내 |
| GPS 신호 없는 상태에서 위치 카드 발송 | "위치를 가져오는 중 오류가 발생했습니다. 잠시 후 다시 시도해 주세요." 안내 |
| `privacy_first` 비공유 시간대 위치 카드 전송 | 확인 다이얼로그 표시 후 사용자 선택에 따라 처리 |

## 13. DB 스키마

### 13.1 핵심 테이블

```sql
-- 채팅 메시지 (그룹 채팅)
-- tb_chat_message
CREATE TABLE tb_chat_message (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    trip_id         UUID NOT NULL REFERENCES tb_trip(id),
    group_id        UUID NOT NULL REFERENCES tb_group(id),
    sender_id       UUID NOT NULL REFERENCES tb_user(id),
    message_type    VARCHAR(20) NOT NULL DEFAULT 'text'
                    CHECK (message_type IN ('text','system','rich_card')),
    content         TEXT,                                        -- 텍스트 메시지 본문
    media_url       TEXT,                                        -- 이미지/파일 URL
    card_data       JSONB,                                       -- 리치 카드 데이터
    is_pinned       BOOLEAN NOT NULL DEFAULT FALSE,              -- 공지 고정 여부
    pinned_by       UUID REFERENCES tb_user(id),                -- 고정한 캡틴/크루장
    pinned_at       TIMESTAMPTZ,
    deleted_at      TIMESTAMPTZ,                                 -- 소프트 삭제 (§14.1)
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_chat_message_trip_created
    ON tb_chat_message (trip_id, created_at DESC);

-- 투표
-- tb_chat_vote
CREATE TABLE tb_chat_vote (
    id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    message_id   UUID NOT NULL REFERENCES tb_chat_message(id),
    trip_id      UUID NOT NULL REFERENCES tb_trip(id),
    question     TEXT NOT NULL,
    options      JSONB NOT NULL,   -- [{"id": 1, "text": "한식"}, ...]
    expires_at   TIMESTAMPTZ NOT NULL,
    created_by   UUID NOT NULL REFERENCES tb_user(id),
    is_closed    BOOLEAN NOT NULL DEFAULT FALSE,
    closed_at    TIMESTAMPTZ,
    created_at   TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at   TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 투표 응답
-- tb_chat_vote_response
CREATE TABLE tb_chat_vote_response (
    id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    vote_id      UUID NOT NULL REFERENCES tb_chat_vote(id),
    voter_id     UUID NOT NULL REFERENCES tb_user(id),
    option_id    INTEGER NOT NULL,   -- options JSONB의 id
    voted_at     TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (vote_id, voter_id)       -- 1인 1투표
);

-- 보호자 메시지 채널 (멤버 ↔ 가디언 1:1)
-- tb_guardian_message
CREATE TABLE tb_guardian_message (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    trip_id       UUID NOT NULL REFERENCES tb_trip(id),
    link_id       UUID NOT NULL REFERENCES tb_guardian_link(id),  -- RTDB: link_{link_id}
    sender_type   VARCHAR(20) NOT NULL
                  CHECK (sender_type IN ('member','guardian')),
    sender_id     UUID NOT NULL REFERENCES tb_user(id),
    message_type  VARCHAR(20) NOT NULL DEFAULT 'text'
                  CHECK (message_type IN ('text','location_card','system')),
    content       TEXT,
    card_data     JSONB,                                           -- 위치 카드, 일정 카드
    is_read       BOOLEAN NOT NULL DEFAULT FALSE,
    sent_at       TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_guardian_message_link_sent
    ON tb_guardian_message (link_id, sent_at DESC);

-- 채팅 읽음 상태
-- tb_chat_read_status
CREATE TABLE tb_chat_read_status (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    trip_id         UUID NOT NULL REFERENCES tb_trip(id),
    user_id         UUID NOT NULL REFERENCES tb_user(id),
    last_read_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    last_message_id UUID REFERENCES tb_chat_message(id),
    UNIQUE (trip_id, user_id)
);
```

### 13.3 데이터 보존 (§13.1)

채팅 메시지 보존 기간: 여행 종료 후 90일.
90일 경과 후 `deleted_at` 마킹 → 추가 90일 후 물리 삭제.

```sql
-- 보존 기간 만료 메시지 소프트 삭제 (스케줄러)
UPDATE tb_chat_message
SET deleted_at = NOW()
WHERE trip_id IN (
    SELECT id FROM tb_trip WHERE completed_at < NOW() - INTERVAL '90 days'
)
AND deleted_at IS NULL;
```

## 14. 오프라인 대응

### 14.1 오프라인 시 채팅탭 동작

| 기능 | 오프라인 가용성 | 비고 |
|------|:--------------:|------|
| 메시지 목록 열람 | ✅ | 로컬 캐시 (최근 200건) |
| 텍스트 메시지 작성 | ✅ | 로컬 큐잉 (최대 100건) |
| 이미지 첨부 발송 | ⚠️ | 로컬 큐잉 (최대 100건 포함) |
| 파일 첨부 발송 | ❌ | 온라인 연결 필요 |
| 새 메시지 수신 | ❌ | 온라인 연결 필요 |
| 투표 응답 | ⚠️ | 로컬 큐잉 → 온라인 복귀 시 발송 |
| 위치 카드 발송 | ❌ | 위치 좌표 취득에 온라인 필요 |

### 14.2 오프라인 배너

채팅탭 진입 시 오프라인 상태이면 최상단에 표시:

```
[오프라인 모드] 메시지는 연결 후 자동 전송됩니다. 메시지 입력은 가능합니다.
```

### 14.3 전송 대기 메시지 표시

로컬 큐에 대기 중인 메시지는 다음과 같이 표시:

```
대기 중인 메시지 내용                    ⏳ 전송 대기
(회색 텍스트)                            오후 3:15
```

### 14.4 온라인 복귀 시 큐 처리

온라인 복귀 시 대기 메시지를 큐 순서대로 자동 발송. 발송 성공 시 ⏳ → ✅ 아이콘으로 변경.

동기화 우선순위 (비즈니스 원칙 §11.2):

1. SOS 이벤트 (최우선)
2. 가디언 긴급 알림/요청
3. 위치 데이터 업로드
4. 일정 변경 동기화
5. 채팅 메시지 발송 큐
6. 일반 이벤트 로그
