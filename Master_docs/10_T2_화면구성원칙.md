## 1. 문서 목적 및 적용 범위

### 1.1 목적

본 문서는 SafeTrip 앱의 **전체 화면 구조, 역할별 렌더링 규칙, 탭 구성, 화면 전환 로직**을 정의한다. 모든 탭별 원칙 문서(Tier 3)는 본 문서에서 정의한 공통 구조와 규칙을 준수해야 하며, 본 문서와 상충 시 본 문서가 우선한다.

### 1.2 적용 범위

- SafeTrip 모바일 앱(Flutter) 전 화면
- 여행 상태(`planning` / `active` / `completed` / `none`)에 관계없이 공통 적용
- 역할(캡틴 / 크루장 / 크루 / 가디언) 전체

### 1.3 핵심 설계 원칙

| 원칙 | 설명 |
|------|------|
| **안전 최우선** | SOS 버튼은 어떤 화면 상태에서도 항상 접근 가능해야 한다 |
| **역할 기반 UI** | 역할에 따라 표시되는 버튼·탭·정보가 달라진다 |
| **상태 기반 분기** | 여행 상태에 따라 화면 구성이 변경된다 |
| **프라이버시 존중** | 프라이버시 등급에 따라 표시 정보 범위가 달라진다 |

## 2. 메인 화면 전체 구조

### 2.1 레이아웃 계층

메인 화면은 **3개의 레이어**로 구성된다. 하위 레이어부터 상위 레이어 순으로 렌더링된다.

```
┌─────────────────────────────────────┐
│           [Layer 3] 상단 오버레이      │  ← 항상 최상단 (z-index: 최고)
│  여행정보카드 | 설정 아이콘 | 알림 아이콘 │
│                     [SOS 버튼]        │
├─────────────────────────────────────┤
│                                     │
│           [Layer 1] 지도 뷰           │  ← 기본 레이어
│        (Google Maps / Kakao Map)    │
│    멤버 위치 마커 | 지오펜스 오버레이    │
│                                     │
├─────────────────────────────────────┤
│      [Layer 2] 바텀시트 (탭 포함)      │  ← 지도 위에 슬라이딩
│  [일정] [멤버] [채팅] [안전가이드]       │
│              탭 컨텐츠               │
└─────────────────────────────────────┘
```

### 2.2 지도 뷰 (Layer 1)

| 요소 | 설명 | 표시 조건 |
|------|------|----------|
| **멤버 위치 마커** | 각 멤버의 현재 위치를 역할 색상으로 표시 | 위치 공유 ON 상태인 멤버만 |
| **가디언 연결 멤버 마커** | 가디언이 자신과 연결된 멤버 위치 확인 | 가디언 역할, 프라이버시 등급 조건 충족 시 |
| **지오펜스 영역** | 반투명 원형/다각형 오버레이 | 지오펜스 등록 시 (캡틴/크루장만 편집) |
| **내 위치 버튼** | 현재 기기 위치로 지도 중심 이동 | 항상 표시 |
| **전체 보기 버튼** | 모든 멤버가 보이도록 지도 축척 조정 | `active` 상태 + 멤버 2명 이상 |

### 2.3 바텀시트 높이 단계

바텀시트 상세 규칙은 **바텀시트 동작 규칙 v2.0** 문서를 준수한다.

| 단계 | 명칭 | 높이 기준 | 지도 노출 비율 |
|------|------|----------|:-------------:|
| 1단계 | collapsed | 탭 바 + 헤더만 노출 (약 72dp) | 90% |
| 2단계 | peek | 화면 하단 1/4 | 75% |
| 3단계 | half | 화면 절반 | 50% |
| 4단계 | expanded | 화면 하단 1/4만 지도 노출 | 25% |
| 5단계 | full | 전체 화면 | 0% |

## 3. 상단 오버레이 구성

### 3.1 오버레이 요소 배치

```
┌──────────────────────────────────────────────┐
│ [여행정보카드]                [알림🔔] [설정⚙️] │  ← 상단바 (항상 고정)
│  여행명 | 국가 | D+N | 멤버N명               │
├──────────────────────────────────────────────┤
│                                              │
│                  (지도 영역)                  │
│                                              │
│                              [SOS 버튼]       │  ← 우측 하단 고정 (바텀시트 위)
└──────────────────────────────────────────────┘
```

### 3.2 여행정보카드

| 표시 항목 | 내용 | 조건 |
|----------|------|------|
| 여행명 | `tb_trip.trip_name` | 항상 |
| 국가명 | `tb_trip.country_name` | 항상 |
| 경과일 (D+N) | 여행 시작일 기준 경과일 | `active` 상태 |
| 남은일 (D-N) | 여행 종료일까지 남은일 | `planning` 상태 |
| 참여 멤버 수 | 현재 참여 중인 멤버 수 | 항상 |
| 프라이버시 등급 아이콘 | 🛡️/📍/🔒 | 캡틴/크루장에게만 |

- 여행정보카드 탭 시 → 여행 상세 정보 바텀시트 열림 (여행 기간, 목적지, 멤버 현황)
- **여행이 없는 상태(`none`)**: 카드 대신 "+ 새 여행 만들기" CTA 표시

### 3.3 SOS 버튼

| 항목 | 규칙 |
|------|------|
| **표시 대상** | 캡틴, 크루장, 크루 (가디언은 표시하지 않음) |
| **표시 조건** | `active` 상태 여행에 소속된 경우 항상 표시 |
| **위치** | 화면 우측 하단, 바텀시트 탭 바 위에 고정 |
| **크기** | 최소 터치 영역 56dp × 56dp (접근성 기준) |
| **색상** | 빨간색 (`#D32F2F`) — 배경 + 흰색 텍스트 |
| **발동 방식** | 2초 길게 누르기 (오발송 방지) |
| **오버레이 우선** | SOS 발동 시 바텀시트 자동 `collapsed` 전환, 전체화면 SOS 모달 표시 |

> **근거**: 비즈니스 원칙 v5.1 §05.1 — SOS는 모든 프라이버시 설정을 무시하는 최상위 안전 기능이다.

### 3.4 설정 아이콘

- 탭 시 → 설정 메뉴 화면으로 이동 (별도 라우트, 바텀시트가 아닌 전체 화면)
- **설정 메뉴 원칙 문서** 참조

### 3.5 알림 아이콘

- 미읽은 알림 수 뱃지 표시 (최대 99+)
- 탭 시 → 알림 목록 화면으로 이동

## 4. 바텀시트 기반 탭 구성

### 4.1 탭 구성 (4개 탭)

| 탭 순서 | 탭명 | 아이콘 | 담당 원칙 문서 |
|:-------:|------|:------:|-------------|
| 1 | 일정 | 📅 | 일정탭 원칙 |
| 2 | 멤버 | 👥 | 멤버탭 원칙 |
| 3 | 채팅 | 💬 | 채팅탭 원칙 |
| 4 | 안전가이드 | 🛡️ | 안전가이드탭 원칙 |

### 4.2 탭별 역할 접근 규칙

| 탭 | 캡틴 | 크루장 | 크루 | 가디언 |
|----|:----:|:------:|:----:|:------:|
| 일정 | ✅ 편집 | ✅ 편집 | ✅ 열람 | ✅ 열람 |1
| 멤버 | ✅ 관리 | ✅ 제한적 관리 | ✅ 열람 | ✅ 연결 멤버만 |
| 채팅 | ✅ 전송 | ✅ 전송 | ✅ 전송 | ❌ 비표시 |
| 안전가이드 | ✅ | ✅ | ✅ | ✅ |

> **가디언의 채팅탭**: 가디언은 그룹 채팅에 참여하지 않는다. 가디언↔멤버 1:1 메시지는 멤버탭 내 별도 채널로 제공한다.

### 4.3 탭별 바텀시트 기본 높이

| 탭 | planning | active | completed |
|----|:--------:|:------:|:---------:|
| 일정 | half | half | half |
| 멤버 | peek | peek | peek |
| 채팅 | half | half | half |
| 안전가이드 | peek | peek | peek |

### 4.4 탭 전환 시 바텀시트 상태 처리

- 탭 전환 시 이전 탭의 스크롤 위치와 바텀시트 높이는 **유지**된다.
- 탭 전환 시 바텀시트가 `full` 상태이면 → 새 탭도 `full` 유지.
- 단, SOS 발동 시 → **모든 탭이 `collapsed`** 강제 전환.

## 5. 여행 상태별 화면 분기

### 5.1 4가지 여행 상태

```
none ──────────┐
               │
planning ──────┤──→ active ──→ completed
               │       │
               │       └──→ active (재활성화, 24시간 이내 1회)
               └── (여행 생성)
```

### 5.2 상태별 화면 구성 차이

| UI 요소 | none | planning | active | completed |
|---------|:----:|:--------:|:------:|:---------:|
| 지도 (멤버 위치 마커) | ❌ | ❌ | ✅ | ❌ (기록 열람 모드) |
| SOS 버튼 | ❌ | ❌ | ✅ | ❌ |
| D+N 경과일 표시 | ❌ | ❌ | ✅ | ❌ |
| D-N 남은일 표시 | ❌ | ✅ | ❌ | ❌ |
| 일정탭 | ❌ | ✅ 편집 가능 | ✅ 편집 가능 | ✅ 열람 전용 |
| 멤버탭 | ❌ | ✅ | ✅ | ✅ 열람 |
| 채팅탭 | ❌ | ✅ | ✅ | ✅ 열람 |
| 안전가이드탭 | ❌ | ✅ | ✅ | ✅ |
| 여행 시작 버튼 | ❌ | ✅ (캡틴만) | ❌ | ❌ |
| 여행 종료 버튼 | ❌ | ❌ | ✅ (캡틴만) | ❌ |
| 재활성화 버튼 | ❌ | ❌ | ❌ | ✅ (24h 이내, 캡틴) |

### 5.3 `none` 상태 (여행 없음)

- 홈 화면: 지도 배경 위에 "+ 새 여행 만들기" 버튼 중앙 표시
- 탭바 비활성화 (탭 4개 모두 회색 처리, 탭 전환 불가)
- 여행 정보 카드 대신 온보딩 안내 배너 표시

### 5.4 `completed` 상태

- 지도는 여행 종료 시점의 마지막 스냅샷 표시 (실시간 위치 아님)
- 모든 탭 열람 전용 (편집/전송 비활성화)
- 상단바에 "종료된 여행" 배지 표시
- 재활성화 버튼: 종료 시점으로부터 24시간 이내에만 캡틴에게 표시

| 설정 아이콘 | ✅ | ✅ | ✅ | ✅ |

### 6.2 지도 뷰 역할별 차이

| UI 요소 | 캡틴 | 크루장 | 크루 | 가디언 |
|---------|:----:|:------:|:----:|:------:|
| 전체 멤버 위치 마커 | ✅ | ✅ | 설정에 따라 | ❌ |

| 내 위치 버튼 | ✅ | ✅ | ✅ | ✅ |
| 전체 멤버 보기 버튼 | ✅ | ✅ | ✅ | ✅ |

> **크루의 타 멤버 위치 열람**: 공개 범위 설정(`visibility_type`)에 따라 달라진다. `all` → 전체 가능, `admin_only` → ❌, `specified` → 지정된 멤버만.

### 6.3 가디언 전용 화면 구성

가디언은 여행 멤버와 별도의 화면 구성을 가진다.

```
┌─────────────────────────────────────┐
│ [여행정보카드]       [알림🔔] [설정⚙️] │
├─────────────────────────────────────┤
│                                     │
│    지도 (연결된 멤버 위치만 표시)       │
│                                     │
├─────────────────────────────────────┤
│  [내 담당 멤버] [일정] [안전가이드]     │  ← 가디언 전용 탭 3개
└─────────────────────────────────────┘
```

| 가디언 탭 | 기능 |
|----------|------|

## 8. 가디언 과금 UI 컴포넌트

> **근거**: 비즈니스 원칙 v5.1 §09.3 가디언 추가 과금 체계

### 8.1 과금 진입 시나리오

가디언 추가 과금 UI는 멤버가 **3번째 개인 가디언을 추가**하려 할 때 표시된다.

```
무료 한도 (2명): 일반 가디언 추가 플로우
유료 한도 (3~5명): 과금 모달 → 결제 → 가디언 추가
```

### 8.2 과금 모달 (PaywallModal)

```
┌───────────────────────────────────┐
│  추가 가디언 연결                  │
│  ──────────────────────────────── │
│  현재 무료 가디언 2명을 모두 사용  │
│  중입니다.                         │
│                                   │
│  추가 가디언 (유료)                │
│  ┌─────────────────────────────┐  │
│  │  월 N,000원 / 1명           │  │
│  │  결제 즉시 적용, 환불 불가   │  │
│  └─────────────────────────────┘  │
│                                   │
│  [취소]          [결제하고 연결]   │
└───────────────────────────────────┘
```

| 항목 | 규칙 |
|------|------|
| 트리거 | 개인 가디언 3번째 이상 추가 시도 시 |
| 표시 위치 | 전체 화면 모달 (바텀시트 위) |
| "환불 불가" 안내 | 반드시 포함 (§09.3 기준) |
| 결제 실패 시 | 가디언 추가 취소, 실패 토스트 메시지 |
| 결제 성공 시 | 모달 닫힘, 가디언 추가 플로우 계속 진행 |

### 8.3 과금 상태 배너 (PaywallBanner)

유료 가디언이 이미 연결된 경우 멤버탭 상단에 배너 표시.

```
┌───────────────────────────────────────┐
│ 💳  유료 가디언 N명 연결 중           │
│     월 N,000원 청구 예정 | 관리 →    │
└───────────────────────────────────────┘
```

### 8.4 과금 한도 초과 안내

최대 허용 수(유료 3명)를 초과하려 할 때:

```
"가디언은 최대 5명(무료 2명 + 유료 3명)까지만 연결할 수 있습니다."
```

- 버튼 비활성화 처리 (그레이 아웃)
- 현재 연결 중인 가디언 수/한도를 항상 표시

## 9. 화면 전환 규칙

### 9.1 전환 애니메이션

| 전환 유형 | 애니메이션 | 지속 시간 |
|----------|----------|:--------:|
| 탭 전환 (바텀시트 내) | 슬라이드 (좌↔우) | 200ms |
| 바텀시트 ↔ 전체화면 | 슬라이드 업/다운 | 300ms |
| 설정/알림 화면 진입 | 우→좌 슬라이드 | 250ms |
| 설정/알림 화면 복귀 | 좌→우 슬라이드 | 250ms |
| SOS 모달 | 페이드 인 | 150ms |
| 과금 모달 | 슬라이드 업 | 300ms |

### 9.2 딥링크 규칙

| 딥링크 경로 | 이동 대상 | 조건 |
|-----------|----------|------|
| `safetrip://trip/{tripId}` | 해당 여행 메인 화면 | 해당 여행 멤버 또는 가디언인 경우 |
| `safetrip://trip/{tripId}/schedule` | 일정탭 | 동일 |
| `safetrip://trip/{tripId}/member` | 멤버탭 | 동일 |
| `safetrip://trip/{tripId}/chat` | 채팅탭 | 여행 멤버 (캡틴/크루장/크루)만 |
| `safetrip://trip/{tripId}/sos` | SOS 확인 화면 | 수신자 (멤버 + 가디언) |
| `safetrip://invite/{inviteCode}` | 초대 수락 화면 | 미로그인 시 → 로그인 후 리다이렉트 |

### 9.3 뒤로가기 규칙

- 바텀시트 `full` 상태에서 뒤로가기 → `half` 단계로 축소
- 바텀시트 `half` 이하에서 뒤로가기 → 앱 종료 확인 다이얼로그
- 설정/알림 화면에서 뒤로가기 → 이전 여행 메인 화면으로 복귀

## 10. 반응형 설계 원칙

### 10.1 지원 디바이스

## 12. 접근성 (a11y) 원칙

### 12.1 최소 터치 영역

| 컴포넌트 | 최소 크기 |
|---------|:--------:|

| 지도 마커 | 44dp × 44dp |
| 일반 버튼 | 44dp × 44dp |

### 12.2 스크린 리더 (TalkBack / VoiceOver)

- 모든 이미지/아이콘에 `contentDescription` 필수
- SOS 버튼: "긴급 SOS 발송 버튼, 2초간 누르세요"
- 위치 마커: "{멤버명}의 현재 위치"

- 역할별 마커 색상은 색상 + 모양(shape)을 함께 사용하여 색맹 사용자도 구분 가능

### 12.4 폰트 크기

- 시스템 폰트 크기 설정을 따른다.
- 최소 폰트: 12sp (법적 고지 등 소문자 영역)
- SOS 관련 텍스트: 최소 16sp

## 13. 에러 및 엣지케이스 처리

### 13.1 네트워크 에러

| 상황 | UI 처리 |
|------|---------|
| 위치 데이터 로딩 실패 | 마지막 캐시 위치 표시 + "위치 정보를 불러오는 중…" 오버레이 |
| 탭 데이터 로딩 실패 | 해당 탭 내 재시도 버튼 표시 |
| SOS 발송 실패 | 에러 다이얼로그 + 재시도 버튼 (자동 3회 재시도 후 표시) |
| 전체 오프라인 | 오프라인 배너 표시, 캐시 데이터로 열람만 허용 |

### 13.2 권한 에러

| 상황 | UI 처리 |
|------|---------|
| 위치 권한 없음 | 권한 요청 다이얼로그 → 거부 시 지도 비활성화 + 설정 이동 안내 |
| 알림 권한 없음 | SOS 수신 제한 경고 배너 (설정 이동 링크 포함) |
| 위치 서비스 OFF | "GPS를 켜주세요" 시스템 설정 이동 버튼 |

### 13.3 엣지케이스

| 상황 | 처리 규칙 |
|------|----------|
| 여행 기간 15일 초과 생성 시도 | 생성 화면에서 실시간 오류 표시, 완료 버튼 비활성화 |
| 멤버 0명 여행 | 멤버탭에 "아직 멤버가 없습니다 — 초대하기" CTA 표시 |
| 동시에 2개 여행 active | 서버 차단 + "현재 진행 중인 여행이 있습니다" 에러 메시지 |
| 가디언이 연결 멤버 0명 | "연결된 멤버가 없습니다. 멤버가 가디언으로 초대해야 합니다" 안내 |
| 재활성화 24시간 초과 | 재활성화 버튼 비표시, "재활성화 기간이 만료되었습니다" 안내 |
