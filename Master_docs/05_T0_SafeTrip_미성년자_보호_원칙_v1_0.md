## 1. 문서 목적 및 적용 범위

### 1.1 목적

본 문서는 SafeTrip 앱 서비스에서 미성년자(만 18세 미만)의 개인정보 보호, 위치정보 처리, 안전 기능 운영에 관한 통합 원칙을 정의한다. SafeTrip의 핵심 사용 시나리오인 "학생 단체 여행(수학여행)"은 미성년자의 실시간 위치를 성인(교사, 보호자)이 추적하는 구조이므로, 미성년자의 권익 보호와 안전 확보를 동시에 달성하기 위한 명확한 기준이 필요하다.

본 문서는 다음 두 가지 역할을 수행한다.

1. **내부 설계 원칙**: SafeTrip 개발·운영팀이 미성년자 관련 기능을 설계·구현할 때 참조하는 기술적·운영적 기준
2. **외부 신뢰 기반**: B2B 학교 고객, 학부모, 규제 기관에 미성년자 보호 체계를 입증하는 근거 문서

### 1.2 적용 범위

- SafeTrip 모바일 앱(iOS, Android)에서 미성년자가 이용하는 모든 기능
- 미성년자의 개인정보 및 위치정보를 처리하는 모든 서버 로직
- B2B 학교 시나리오에서 학생 일괄 등록 및 관리 기능
- 가디언-미성년자 연결 관계의 생성·유지·해제
- 미성년자가 포함된 여행에서의 AI 기능 동작 범위
- 데모 모드에서의 미성년자 시나리오 체험 (실제 데이터 미수집)

### 1.3 미성년자 정의

| 구분 | 연령 범위 | SafeTrip 내 분류 |
|------|----------|-----------------|
| 8세 이하 아동 | 만 0~8세 | `minor_child` |
| 14세 미만 아동 | 만 9~13세 | `minor_under14` |
| 14세 이상 미성년자 | 만 14~17세 | `minor_over14` |
| 성인 | 만 18세 이상 | `adult` |

> **판별 기준**: TB_USER.date_of_birth 기반으로 서버에서 실시간 계산한다. 생년월일 미입력 시 성인으로 간주하되, B2B 학교 시나리오에서는 학교 관리자가 CSV 등록 시 연령 정보를 필수로 포함해야 한다.

### 1.4 법적 근거

| 법령 | 관련 조항 | 적용 사항 |
|------|----------|----------|
| 개인정보 보호법 | 제22조 | 14세 미만 아동의 개인정보 수집 시 법정대리인 동의 필수 |
| 위치정보법 | 제25조 | 14세 미만 아동의 개인위치정보 수집 시 법정대리인 동의 필수 |
| 위치정보법 | 제26조 | 8세 이하 아동·피성년후견인의 보호의무자 동의 |
| 아동복지법 | 제3조, 제15조 | 아동(18세 미만) 안전 보장 의무 |
| 전자상거래법 | 제17조 | 미성년자의 법률행위(유료 서비스 결제) 제한 |

### 1.5 미작성 시 리스크

- 🔴 **치명적**: 14세 미만 아동의 법정대리인 동의 없는 개인정보·위치정보 수집 시 형사 제재 (개인정보 보호법 제75조, 위치정보법 제39조)
- 🔴 **치명적**: B2B 학교 고객의 계약 전제 조건 미충족으로 핵심 시장 진입 불가
- 🟠 **높음**: 미성년자 위치 추적에 대한 사회적·브랜드 리스크
- 🟠 **높음**: 앱 스토어 심사 시 아동 보호 정책 미비로 거부 가능 (COPPA, 한국 법령)

## 2. 연령별 동의 체계

### 2.1 3단계 동의 체계

비즈니스 원칙 v5.1 §10에 정의된 연령별 동의 체계를 구현한다.

| 연령 구간 | 동의 주체 | 동의 방식 | 법적 근거 |
|----------|----------|----------|----------|
| 8세 이하 | 보호의무자 단독 | 보호의무자 본인 인증 + 관계 증빙 | 위치정보법 제26조 |
| 9~13세 (14세 미만) | 본인 + 법정대리인 | 본인 동의 + 법정대리인 휴대전화 인증 | 개인정보 보호법 제22조, 위치정보법 제25조 |
| 14~17세 | 본인 + 보호자 이중 동의 | 본인 동의 + 보호자 확인 SMS/인앱 승인 | 비즈니스 원칙 v5.1 §10.1 |

### 2.2 연령 판별 프로세스

```
회원가입 / 여행 참여 시
│
├── 1단계: 생년월일 입력 (필수 — B2B 시나리오)
│   · B2C: 선택 항목이나, 미입력 시 미성년자 보호 기능 미적용 고지
│   · B2B: CSV 일괄 등록 시 필수 컬럼
│
├── 2단계: 서버 측 연령 계산
│   · age = (현재 날짜 - date_of_birth) 기반 만 나이 계산
│   · 계산 시점: 가입 시 + 여행 참여 시 + 매일 배치 (생일 도래 시 등급 자동 변경)
│
├── 3단계: 연령 구간 분류
│   ├── age ≤ 8  → minor_child    → 보호의무자 동의 플로우
│   ├── age ≤ 13 → minor_under14  → 법정대리인 동의 플로우
│   ├── age ≤ 17 → minor_over14   → 본인 동의 + 보호자 고지 플로우
│   └── age ≥ 18 → adult          → 일반 동의 플로우
│
└── 4단계: TB_USER.minor_status 업데이트
    · minor_status ENUM: 'adult', 'minor_over14', 'minor_under14', 'minor_child'
```

### 2.3 법정대리인 동의 확인 방법

| 방법 | 적용 시나리오 | 구현 우선순위 |
|------|-------------|:----------:|
| 법정대리인 휴대전화 본인 인증 | B2C 개인 가입 | P0 |
| 법정대리인 이메일 인증 | 보조 수단 (SMS 불가 시) | P1 |
| B2B 학교 관리자 일괄 확인 | 학교 CSV 등록 시 동의서 일괄 취득 확인 | P0 |
| 오프라인 서면 동의서 | B2B 학교 시나리오 (보관 의무) | P1 |

### 2.4 법정대리인 동의 플로우 (B2C)

```
미성년자 회원가입 진입
│
├── [화면] 생년월일 입력
│   → age ≤ 13 판별
│
├── [화면] 법정대리인 동의 안내
│   "14세 미만 이용자는 법정대리인(보호자)의 동의가 필요합니다."
│   "보호자의 휴대전화번호를 입력해주세요."
│
├── [입력] 법정대리인 휴대전화번호
│   → SMS 인증번호 발송
│
├── [화면] 법정대리인 동의 화면 (인증 완료 후)
│   · 개인정보 수집·이용 동의 (필수)
│   · 위치정보 수집·이용 동의 (필수)
│   · SafeTrip 서비스 이용 동의 (필수)
│   · "법정대리인으로서 위 내용에 동의합니다" 체크
│
├── [저장] TB_MINOR_CONSENT 레코드 생성
│   · consent_type: 'legal_guardian'
│   · guardian_phone: 인증된 번호
│   · consented_at: 현재 시각
│
└── [완료] 회원가입 계속 진행
```

### 2.5 B2B 학교 시나리오 일괄 동의

```
학교 관리자 (캡틴) CSV 업로드
│
├── CSV 필수 컬럼:
│   · student_name, date_of_birth, phone_number, guardian_phone
│   · guardian_consent_obtained (Y/N) — "N"이면 해당 학생 등록 거부
│
├── 서버 검증:
│   ├── 전체 학생 date_of_birth 확인 → 연령 구간 분류
│   ├── guardian_consent_obtained = 'Y' 확인
│   └── 14세 미만 학생 중 동의 미취득 건 → 오류 리포트 생성
│
├── 오프라인 동의서 보관 의무:
│   · 학교는 법정대리인 서면 동의서를 여행 종료 후 3년간 보관
│   · SafeTrip은 동의 취득 여부(Y/N)만 기록, 동의서 원본은 학교가 보관
│
└── 등록 완료:
    · 동의 취득 확인된 학생만 여행에 참여 가능
    · 미동의 학생 → "보호자 동의 대기 중" 상태로 표시
```

## 3. 미성년자 데이터 보호 강화

### 3.1 데이터 보유 기간 단축

개인정보처리방침 원칙 v1.0 §10.3에 정의된 미성년자 데이터 보호 강화 정책을 상세화한다.

| 데이터 유형 | 일반 보유 기간 | 미성년자 보유 기간 | 근거 |
|-----------|:----------:|:------------:|------|
| 실시간 위치 데이터 | 여행 종료 후 90일 | 여행 종료 후 30일 | 아동 보호 강화 |
| 체류 지점 데이터 | 여행 종료 후 30일 | 여행 종료 후 15일 | 최소 보유 원칙 |
| 이동기록 타임라인 | 여행 종료 후 30일 | 여행 종료 후 15일 | 최소 보유 원칙 |
| SOS 관련 위치 기록 | 여행 종료 후 3년 | 여행 종료 후 3년 (동일) | 법적 분쟁 대비 — 미성년자도 동일 |
| 채팅 메시지 | 영구 보존 | 여행 종료 후 1년 | 미성년자 대화 내용 보호 |
| 프로필 정보 | 탈퇴 시까지 | 탈퇴 시까지 (동일) | 서비스 이용 |

### 3.2 자동 파기 배치 스케줄 (미성년자 전용)

| 대상 데이터 | 파기 조건 | 실행 주기 | 구현 우선순위 |
|-----------|----------|----------|:----------:|
| 미성년자 위치 데이터 | trip.end_date + 30일 | 매일 02:00 UTC | P0 |
| 미성년자 체류 지점 | trip.end_date + 15일 | 매일 02:00 UTC | P0 |
| 미성년자 채팅 이미지 | trip.end_date + 30일 | 매일 02:30 UTC | P1 |
| 미성년자 채팅 텍스트 | trip.end_date + 365일 | 매주 일요일 04:00 UTC | P1 |
| 법정대리인 동의 기록 | 동의 후 3년 | 매월 1일 05:00 UTC | P1 |

### 3.3 데이터 접근 제한

미성년자의 위치 데이터에 접근할 수 있는 역할을 엄격히 제한한다.

| 접근 주체 | 실시간 위치 | 이동기록 | 체류 지점 | SOS 위치 |
|----------|:--------:|:------:|:-------:|:-------:|
| 캡틴 (교사) | ✅ | ✅ | ✅ | ✅ |
| 크루장 (부인솔) | ✅ | ✅ | ✅ | ✅ |
| 크루 (다른 학생) | ⚠️ 등급별 | ❌ | ❌ | ✅ (SOS 시) |
| 가디언 (보호자) | ✅ 등급별 | ⚠️ 제한적 | ❌ | ✅ |
| SafeTrip 운영팀 | ❌ (사고 조사 시 제외) | ❌ | ❌ | ⚠️ 사고 조사 |

> **크루 간 위치 공유 제한**: 안전 최우선 등급에서 크루(학생) 간 실시간 위치 공유는 캡틴이 설정으로 활성화/비활성화할 수 있다. 기본값은 "활성화" (학생 간 서로 위치 확인 가능)이나, 학교 요청 시 "비활성화"로 변경 가능하다.

### 3.4 미성년자 데이터 암호화 강화

| 보호 조치 | 일반 사용자 | 미성년자 |
|----------|:---------:|:------:|
| 전송 암호화 | TLS 1.3 | TLS 1.3 (동일) |
| 저장 암호화 (위치 좌표) | AES-256 | AES-256 (동일) |
| 접근 로그 기록 | 기록 | 기록 + 접근 사유 필수 입력 |
| 데이터 내보내기 제한 | 가능 | 법정대리인 인증 후 가능 |
| 익명화 처리 시점 | 보유 기간 만료 7일 후 | 보유 기간 만료 즉시 |

## 6. 가디언-미성년자 연결 관리

### 6.1 가디언 연결 유형

| 유형 | 설명 | 연결 방식 | 과금 |
|------|------|----------|------|
| 개인 가디언 | 특정 미성년자 1~3명과 연결 | 초대코드 + 멤버 선택 | 여행당 무료 2명, 유료 추가 3명 |
| 전체 가디언 | 여행 전체 미성년자 모니터링 | 캡틴이 지정 | 여행당 최대 2명 무료 (추가 불가) |
| 학교 가디언 (B2B) | 학교 관리자 / 학부모 일괄 연결 | CSV 일괄 또는 캡틴 초대 | B2B 계약에 따름 |

### 6.2 가디언 연결 생성 조건

| 조건 | 필수 여부 | 설명 |
|------|:--------:|------|
| 가디언이 성인(18세 이상)일 것 | 필수 | 미성년자가 가디언이 될 수 없다 |
| 미성년자 본인(14세 이상) 또는 법정대리인(14세 미만)의 동의 | 필수 | 동의 없이 가디언 연결 불가 |
| 캡틴의 승인 | 필수 | 캡틴이 가디언 초대코드를 생성 |
| 미성년자와의 관계 증빙 (B2B) | 권장 | 학교 시나리오에서 학부모 확인 |

### 6.3 가디언 연결 해제 조건

| 해제 트리거 | 해제 주체 | 즉시/유예 | 미성년자 알림 |
|-----------|----------|:--------:|:-----------:|
| 여행 종료 | 시스템 자동 | 즉시 | ✅ |
| 캡틴이 해제 | 캡틴 | 즉시 | ✅ |
| 가디언 본인 탈퇴 | 가디언 | 즉시 | ✅ + 캡틴 알림 |
| 법정대리인 동의 철회 | 법정대리인 | 즉시 | ✅ + 캡틴 알림 |
| 미성년자 본인 요청 (14세 이상) | 미성년자 | ⚠️ 캡틴 승인 후 | ✅ |
| 미성년자 본인 요청 (14세 미만) | 법정대리인 대리 | ⚠️ 캡틴 승인 후 | ✅ |
| 가디언 부정 사용 감지 | 운영팀 | 즉시 | ✅ + 전체 알림 |

> **안전 최우선 등급 특칙**: 안전 최우선 등급에서는 여행 진행 중 가디언 연결 해제가 제한된다. 캡틴만 해제할 수 있으며, 미성년자 본인의 해제 요청은 여행 종료 후 처리된다.

### 6.4 가디언 부재 시 안전망

미성년자에게 가디언이 연결되지 않은 경우의 대응 체계이다.

```
미성년자 여행 참여
│
├── 가디언 연결 여부 확인
│   ├── 연결됨 → 정상 진행
│   └── 미연결 → 안전망 동작
│
├── 안전망 단계:
│   ├── 1단계: 캡틴에게 "미성년자 [이름]에게 가디언이 연결되지 않았습니다" 알림
│   ├── 2단계: 미성년자에게 "보호자를 초대하시겠습니까?" 안내
│   ├── 3단계: B2B 학교 시나리오 → 학교 관리자(전체 가디언)가 자동 커버
│   └── 4단계: SOS 발동 시 → 가디언 미연결이라도 캡틴·크루장에게 전달
│
└── 가디언 미연결 상태 표시:
    · 멤버탭에서 해당 미성년자에 "⚠️ 보호자 미연결" 배지 표시
    · 캡틴 대시보드에 경고 카드 표시
```

## 7. AI 기능 제한

### 7.1 미성년자 AI 기능 접근 매트릭스

AI 기능 원칙 v1.0에 정의된 3가지 AI 분류(Safety AI, Convenience AI, Intelligence AI)에 따라 미성년자 접근을 제한한다.

| AI 분류 | AI 기능 | 미성년자 접근 | 제한 사유 |
|---------|---------|:----------:|----------|
| Safety AI | AI 안전 어시스턴트 | ✅ 무제한 | 안전 기능은 항상 허용 |
| Safety AI | AI 안전 브리핑 | ✅ 열람 | 안전 정보 접근 보장 |
| Safety AI | AI 긴급 현지어 카드 | ✅ 무제한 | 안전 기능은 항상 허용 |
| Safety AI | AI 비상 키워드 감지 | ✅ 자동 동작 | 안전 기능 비활성화 불가 |
| Convenience AI | AI 빠른 번역 | ✅ 무료 한도 | 편의 기능 허용 |
| Convenience AI | AI 일정 요약 봇 | ✅ 수신 | 편의 기능 허용 |
| Convenience AI | AI 환율 변환 | ✅ | 편의 기능 허용 |
| Intelligence AI | AI 그룹 상태 요약 (개인 분석) | ❌ 차단 | 미성년자 프로파일링 방지 |
| Intelligence AI | AI 일정 최적화 | ❌ 차단 | 개인 행동 패턴 분석 방지 |
| Intelligence AI | AI 주변 추천 (위치 기반) | ⚠️ 제한적 | 위치 기반 추천은 캡틴/크루장에게만 |

### 7.2 Intelligence AI 차단 세부 규칙

| 규칙 | 상세 |
|------|------|
| 개인 분석 차단 | 미성년자의 이동 패턴, 체류 시간, 행동 데이터를 AI에 전달하지 않음 |
| 그룹 분석 허용 (비식별) | 그룹 전체의 분산 상태는 분석 가능하되, 미성년자 개인을 식별하는 분석 결과는 표시하지 않음 |
| AI 데이터 전달 시 연령 필터 | AI Gateway에서 minor_status != 'adult'인 사용자의 개인 데이터를 자동 제외 |
| 캐시 분리 | 미성년자 관련 AI 응답은 별도 캐시 키로 분리하여 개인 데이터 유출 방지 |

### 7.3 AI 기능 연령 전환 처리

미성년자가 여행 기간 중 만 18세가 되는 경우의 처리이다.

```
매일 배치 (01:00 UTC)
│
├── TB_USER.date_of_birth 스캔
│   → 오늘 기준 만 18세 도래 사용자 식별
│
├── minor_status 업데이트
│   · 'minor_over14' → 'adult'
│
├── AI 기능 잠금 해제
│   · Intelligence AI 접근 허용
│   · "🎉 성인 전환 — AI 기능이 확장되었습니다" 알림 (여행 종료 후)
│
└── 주의: 여행 진행 중에는 변경사항을 적용하지 않음
    · 여행 시작 시점의 minor_status를 여행 종료까지 유지
    · 여행 종료 후 다음 여행부터 성인 기능 적용
```

## 9. B2B 학교 시나리오 특칙

### 9.1 학교 여행 기본 설정

B2B 학교 고객이 학생 단체 여행을 생성할 때 적용되는 기본값이다.

| 설정 항목 | 기본값 | 변경 가능 여부 |
|----------|--------|:----------:|
| 프라이버시 등급 | 안전 최우선 | ✅ (경고 표시와 함께) |
| 위치 공유 모드 | 강제 공유 | ✅ (캡틴만) |
| Auto-SOS | 활성화 | ❌ (비활성화 불가) |
| 출석 체크 | 활성화 | ✅ |
| 가디언 지오펜스 알림 | 항상 전달 | ❌ (고정) |
| 크루 간 위치 공유 | 활성화 | ✅ (캡틴 설정) |
| 미성년자 위치 공유 일시 중지 | 불가 | ❌ (고정) |

### 9.2 학교 관리자 역할 (캡틴/전체 가디언 겸직)

```
학교 관리자
├── 역할 1: 캡틴 (인솔 교사)
│   · 여행 생성, 학생 일괄 등록, 프라이버시 설정
│   · 출석 체크, 지오펜스 관리, SOS 대응
│
├── 역할 2: 전체 가디언 (학교 담당자)
│   · 원격에서 전체 학생 위치 모니터링
│   · 학부모 가디언의 보조 역할
│
└── B2B 특칙:
    · 학교 관리자 계정은 개인 계정과 별도 관리
    · 학교 관리자는 여행 종료 후 데이터 내보내기 가능 (익명화 버전)
    · 학교 관리자 변경 시 캡틴 위임 + 가디언 재연결
```

### 9.3 학부모 가디언 연결 프로세스 (B2B)

```
학교 관리자 CSV 업로드 (학생 + 보호자 정보)

├── 시스템이 학생별 개인 가디언 초대코드 자동 생성

│   "앱 다운로드: [딥링크]"

├── 학부모 앱 설치 → 초대코드 입력 → 가디언 참여 완료

└── 미참여 학부모 관리:
    · 여행 시작 3일 전: 미참여 학부모에게 리마인더 발송
```

## 13. 에러 및 엣지케이스

### 13.1 연령 판별 관련

| 상황 | 처리 |
|------|------|
| 생년월일 미입력 회원이 미성년자 포함 여행에 참여 | "생년월일을 입력해주세요. 미성년자 보호를 위해 필요합니다." 안내 → 미입력 시 성인으로 간주하되 캡틴에게 고지 |
| 생년월일 입력 오류 (미래 날짜, 비현실적 날짜) | 입력 검증 실패 → "올바른 생년월일을 입력해주세요" |
| 여행 중 만 18세 도래 | 여행 시작 시점의 minor_status 유지 → 여행 종료 후 성인 전환 |
| 여행 중 만 14세 도래 | 여행 시작 시점의 minor_status 유지 → 다음 여행부터 동의 체계 변경 |
| B2B CSV에 생년월일 형식 오류 | 해당 행 스킵 + 오류 리포트에 포함 → "생년월일 형식을 확인해주세요 (YYYY-MM-DD)" |

### 13.2 동의 관련

| 상황 | 처리 |
|------|------|
| 법정대리인 인증 중 네트워크 단절 | "인증이 완료되지 않았습니다. 네트워크 연결 후 다시 시도해주세요." → 인증 상태 임시 저장 |
| 법정대리인 동의 후 철회 | 즉시 서비스 이용 중단 → 활성 여행에서 자동 탈퇴 → 가디언 연결 해제 |
| 법정대리인 전화번호 변경 | 설정 > 보호자 정보 > 전화번호 변경 → 재인증 필수 |
| B2B 동의서 일괄 취득 후 개별 철회 | 해당 학생만 여행에서 탈퇴 처리 → 학교 관리자에게 알림 |
| 동의 유효 기간(3년) 만료 | 만료 30일 전 법정대리인에게 재동의 요청 알림 → 미갱신 시 서비스 이용 제한 |

### 13.3 가디언 연결 관련

| 상황 | 처리 |
|------|------|
| 미성년자에게 가디언이 0명인 상태로 여행 시작 | 캡틴에게 경고 → B2B: 전체 가디언이 자동 커버 / B2C: 여행 시작 가능하나 지속적 안내 |
| 가디언이 앱을 삭제한 경우 | FCM 토큰 무효화 감지 → 캡틴에게 "가디언 [이름]의 앱 연결이 끊어졌습니다" 알림 |
| 미성년자가 가디언을 차단하려 시도 | 차단 불가 → "보호자 연결은 여행 중 해제할 수 없습니다" 안내 |
| 동일 미성년자에게 가디언 5명 초과 연결 시도 | "최대 5명까지 가디언을 연결할 수 있습니다" (무료 2 + 유료 3) |
| 미성년자가 가디언 역할로 참여 시도 | 연령 확인 → "18세 이상만 보호자로 참여할 수 있습니다" 안내 |

### 13.4 AI 기능 관련

| 상황 | 처리 |
|------|------|
| 미성년자가 Intelligence AI 기능 접근 시도 | "이 기능은 18세 이상 이용자만 사용할 수 있습니다" 안내 + 대체 기능 제안 |
| 미성년자 데이터가 AI에 의도치 않게 전달될 경우 | AI Gateway에서 minor_status 필터 작동 → 전달 차단 + 오류 로그 기록 |
| B2B 학교에서 AI 분석 리포트 요청 | 미성년자 개인 식별 불가한 집계 데이터만 제공 → 개인별 분석 차단 |
