## 1. 문서 목적 및 적용 범위

### 1.1 목적

본 문서는 SafeTrip 앱의 **안전가이드 탭(Safety Guide Tab)** 전체 기능을 정의한다. 안전가이드는 여행자가 목적지 국가의 외교부(MOFA) 공식 안전 정보, 긴급연락처, 의료 정보, 입국 요건, 현지 생활 정보를 한곳에서 확인할 수 있는 정보 허브다.

SafeTrip 내 유일하게 **모든 역할이 동등하게 접근**하는 탭으로, 프라이버시 등급이나 여행 참여 여부와 무관하게 전체 기능이 활성화된다.

### 1.2 적용 범위

| 범위 | 포함 여부 |
|------|:--------:|
| 안전가이드 탭 6개 서브 탭 전체 기능 | ✅ |
| MOFA API 연동 및 캐시 전략 | ✅ |
| 원터치 긴급전화 버튼 | ✅ |
| 컨텍스트 기반 국가 자동 선택 | ✅ |
| 여행 유형별 강조 서브탭 로직 | ✅ |
| 오프라인 캐시 및 폴백 처리 | ✅ |
| 역할별 접근 권한 (전 역할 동등) | ✅ |
| AI 안전 어시스턴트 (P2) | 부분 포함 |

### 1.3 관련 비즈니스 원칙 조항

| 조항 | 내용 |
|------|------|
| §02.3 | 여행 기간 최대 15일 제한 (컨텍스트 여행 결정 기준) |
| §03.1 | 가디언 무료 2명 / 유료 3명 (가디언 컨텍스트 자동 선택 적용) |
| §04 | 프라이버시 등급 3단계 정의 (안전가이드는 등급 무관 동등 접근) |
| §05 | 알림 및 SOS 원칙 (긴급연락처 원터치 전화 연계) |

## 2. 안전가이드 설계 철학 6원칙

| ID | 원칙명 | 내용 |
|----|--------|------|
| **S1** | 컨텍스트 우선 | 여행 목적지 기반 자동 필터링. 현재 active 여행의 목적지 국가를 자동으로 선택하여 불필요한 국가 검색 없이 즉시 관련 정보를 제공한다. |
| **S2** | 외교부(MOFA) 신뢰성 | 공식 데이터 최우선. 안전등급, 여행경보, 입국 요건 등 모든 공식 정보는 외교부 API를 단일 소스로 사용하며, 자의적 데이터 편집을 금지한다. |
| **S3** | 오프라인 안정성 | 핵심 정보 사전 캐시 필수. 여행 중 인터넷이 불안정한 상황에서도 긴급연락처 및 기본 안전 정보는 반드시 로컬 캐시에서 제공 가능해야 한다. |
| **S4** | 여행 유형 적응 | 수학여행/출장/패키지/자유여행별 강조 서브탭을 자동 적용하여 여행 목적에 맞는 정보를 우선 노출한다. |
| **S5** | 역할 무관 동등 접근 | 안전가이드는 SafeTrip 내 유일하게 캡틴/크루장/크루/가디언/미참여 유저 모두 동등하게 전체 기능에 접근한다. 안전 정보는 어떤 권한 체계보다 우선한다. |
| **S6** | 즉시 행동 | 긴급연락처를 원터치 전화 버튼으로 제공하여 위기 상황에서 지체 없이 구조기관에 연락할 수 있도록 한다. 탭 진입 후 1초 이내에 긴급연락처가 노출되어야 한다. |

## 3. 안전가이드 탭 구성 — 서브 탭 6개

### 3.1 서브 탭 구조

```
[안전가이드 탭]
├── 개요 (Overview)
├── 안전 (Safety)
├── 의료 (Medical)
├── 입국 (Entry)
├── 긴급연락 (Emergency)
└── 현지생활 (Local Life)
```

### 3.2 탭별 상세 콘텐츠

#### 3.2.1 개요 (Overview)

| 항목 | 내용 |
|------|------|
| 국가 플래그 이미지 | ISO 3166-1 alpha-2 코드 기반 |
| 국가명 | 한국어 공식 명칭 |
| 외교부 안전등급 배지 | 1~4단계 색상 구분 (초록/노랑/주황/빨강) |
| 수도 | 현지어 + 한국어 병기 |
| 통화 | 통화 코드 + 기호 + 현재 환율 (참고용) |
| 언어 | 공식 언어 목록 |
| 시차 | UTC 기준 표시, 한국 시간 대비 시차 |

#### 3.2.2 안전 (Safety)

| 항목 | 내용 |
|------|------|
| **여행경보 단계** | 1단계: 여행유의 (초록), 2단계: 여행자제 (노랑), 3단계: 출국권고 (주황), 4단계: 여행금지 (빨강) |
| 치안 현황 | 외교부 공식 치안 평가 요약 |
| 최근 안전 공지 | 최대 5건, 발행일 기준 최신순 |
| 지역별 경보 상세 | 국가 내 특정 지역의 별도 경보 표시 |

> **[4단계 여행금지 경고 배너]** 4단계 적용 국가 진입 시 화면 최상단에 빨간 배너 표시:
> "외교부에서 여행금지를 권고하는 국가입니다. 방문이 불가피한 경우 영사 확인서를 반드시 취득하세요."

#### 3.2.3 의료 (Medical)

| 항목 | 내용 |
|------|------|
| 주요 병원 목록 | 지도 핀 포함, 한국어 통역 가능 여부 표시 |
| 해외여행자보험 안내 | 여행자보험 가입 체크리스트, 보험사 링크 |
| 약국 찾기 | 현지 약국 검색 (지도 연동) |
| 현지 응급 대처 가이드 | 단계별 응급 처치 안내 (오프라인 캐시 필수) |

#### 3.2.4 입국 (Entry)

| 항목 | 내용 |
|------|------|
| 비자 요건 | 한국 여권 기준 무비자/비자 필요 구분 |
| 입국 서류 | 필수 서류 체크리스트 |
| 통관 규정 | 주요 금지 품목, 신고 기준 금액 |
| 여권 유효기간 최소 요건 | 입국일 기준 최소 유효기간 (예: 6개월) |

#### 3.2.5 긴급연락 (Emergency)

> **★ 핵심 탭 — 가장 빠른 접근이 보장되어야 한다.**

| 연락처 유형 | 내용 |
|------------|------|
| 현지 경찰 | 현지 112 등가 번호, **원터치 전화 버튼** |
| 소방/구급 | 현지 119 등가 번호, **원터치 전화 버튼** |
| 한국 대사관/영사관 | 전화번호, 주소, 운영 시간, **원터치 전화 버튼** |
| 영사콜센터 | +82-2-3210-0404 (24시간), **원터치 전화 버튼** |

**원터치 전화 버튼 UX 규칙:**
- 버튼 탭 → 즉시 네이티브 전화 앱 연결 (다이얼 팝업 없이)
- 버튼 최소 높이: 56dp (손가락 터치 오류 방지)
- 색상: 빨간 배경 + 흰 전화기 아이콘
- 오프라인 상태에서도 캐시된 번호로 전화 발신 가능

#### 3.2.6 현지생활 (Local Life)

| 항목 | 내용 |
|------|------|
| 교통 안내 | 주요 대중교통 정보 (공항 → 시내) |
| SIM 카드/통신 | 현지 SIM 구매 안내, 데이터 로밍 팁 |
| 팁 문화 | 팁 권장 여부, 일반적인 팁 금액 |
| 전압/콘센트 | 전압 규격, 콘센트 타입 |
| 물가 참고 | 식사/교통/숙박 대략 비용 (참고용) |
| 문화 주의사항 | 현지 금기 행동, 복장 규정 등 |

### 3.3 컨텍스트 기반 국가 자동 선택

```
앱 실행
    ↓
현재 사용자 상태 확인
    ├── [active 여행 참여 중]
    │       └─ 현재 여행의 목적지 국가 자동 선택
    │
    ├── [가디언 역할]
    │       └─ 연결된 멤버의 여행 목적지 자동 선택
    │           (멤버 복수 연결 시 → 최근 active 여행 기준)
    │
    ├── [여행 없음 / 미참여]
    │       └─ 자유 탐색 모드
    │           ├── 국가 검색 (검색창)
    │           └── 현재 위치 기반 자동 인식 (GPS 허용 시)
    │
    └── [수동 변경]
            └─ 우상단 [국가 변경] 버튼 → 국가 목록 바텀시트
```

**수동 변경 후 컨텍스트 복원:**
- 수동 변경 후 여행 상태가 변경되면(예: 새 여행 active) 컨텍스트 자동 선택으로 복원
- 수동 변경 상태는 세션 종료 시 초기화

### 3.4 MOFA API 연동

#### 3.4.1 API 엔드포인트 6개

| 엔드포인트 | 용도 | 데이터 |
|-----------|------|--------|
| `/travel-alert` | 여행경보 단계 및 지역별 상세 | 국가별 경보 단계, 지역 목록 |
| `/safety-notice` | 최근 안전 공지 | 최대 20건, 발행일/제목/내용 |
| `/country-info` | 국가 기본 정보 | 수도, 통화, 언어, 시차, 공휴일 |
| `/entry-info` | 입국 요건 및 비자 정보 | 비자 종류, 유효기간, 필요 서류 |
| `/embassy` | 대사관/영사관 목록 | 위치, 전화번호, 운영 시간 |
| `/incident-report` | 최근 사건사고 (최근 30일) | 발생일, 종류, 지역, 요약 |

#### 3.4.2 API 캐시 전략

| 항목 | 내용 |
|------|------|
| **캐시 TTL** | 24시간 (국가별) |
| **경보 변동 감지** | WebSocket 구독 또는 10분 주기 polling |
| **경보 변동 시 처리** | 즉시 캐시 무효화 → 재조회 → 인앱 알림 발송 |
| **API 장애 시 폴백** | 마지막 캐시 데이터 표시 + "데이터가 최신이 아닐 수 있습니다 (최종 갱신: N시간 전)" 배너 |
| **캐시 저장소** | SQLite (tb_safety_guide_cache) |
| **국가별 최대 용량** | 5MB |

### 3.5 여행 유형별 강조 서브탭

| 여행 유형 | 강조 서브탭 | 추가 정보 표시 |
|----------|-----------|----------------|
| **수학여행** | 안전, 의료, 긴급연락 | 학생 단체 이용 가능 병원, 학교 긴급연락처 |
| **기업 출장** | 안전, 입국, 현지생활 | 비즈니스 문화 주의사항, 주요 비즈니스 지구 |
| **패키지 투어** | 입국, 의료, 현지생활 | 가이드 긴급연락처, 패키지 여행사 번호 |
| **자유여행** | 전체 균등 표시 | 개인 여행자보험 안내, 현지 커뮤니티 링크 |

**강조 방식:**
- 강조 서브탭은 탭 목록 상단에 핀 고정 (배지 또는 색상 구분)
- 나머지 탭은 동일 순서 유지
- 강조 탭 변경: 여행 생성/편집 → `trip_type` 필드 기준 자동 적용

**유일한 프라이버시 관련 예외:**

## 6. 에러 및 엣지케이스 처리

### 6.1 API 오류 처리

| 오류 상황 | 처리 방법 |
|----------|----------|
| MOFA API 응답 없음 (timeout) | 캐시 데이터 표시 + 황색 배너 "외교부 서버에 연결할 수 없습니다. 캐시 데이터를 표시합니다." |
| MOFA API 500 에러 | 동일 (캐시 폴백) + 서버 로그 기록 |
| 캐시 없음 + API 오류 | 기본 정보만 표시 (하드코딩된 대사관 번호, 영사콜센터) + 빨간 배너 경고 |
| 국가 코드 미지원 | "해당 국가의 정보는 현재 준비 중입니다." + 영사콜센터 번호 표시 |
| 응답 데이터 파싱 오류 | 빈 섹션 표시 금지 → 섹션별 "정보를 불러오지 못했습니다." 표시 |

### 6.2 컨텍스트 선택 엣지케이스

| 상황 | 처리 방법 |
|------|----------|
| 복수 active 여행 동시 참여 | 시작일이 가장 최근인 여행의 목적지 자동 선택 |
| 가디언이 복수 멤버 연결 | 가장 최근 active 여행의 멤버 기준 자동 선택 |
| 여행 목적지 국가 코드 없음 | 자유 탐색 모드로 폴백 + 사용자에게 국가 선택 안내 토스트 |
| 여행 종료 시점 | 직전 여행 목적지 3시간 유지 후 자유 탐색 모드 전환 |
| 국가 변경 중 여행 시작 | 수동 선택 즉시 무효화 → 컨텍스트 자동 선택 우선 적용 |

### 6.3 긴급전화 엣지케이스

| 상황 | 처리 방법 |
|------|----------|
| 전화 권한 미허용 | 버튼 탭 시 "전화 권한이 필요합니다." 시스템 다이얼로그 표시 |
| 오프라인 상태 | 캐시된 긴급번호로 전화 발신 (전화는 오프라인 무관) |
| 국가 긴급번호 미등록 | 영사콜센터 (+82-2-3210-0404) 대체 표시 |

### 6.4 여행 기간 관련 엣지케이스 (§02.3)

여행 기간 최대 15일 제한에 따라 컨텍스트 자동 선택은 active 상태인 여행만 적용한다. 여행이 15일을 초과하는 경우는 서버 단에서 차단되므로 안전가이드 탭에서 별도 처리는 불필요하다.

## 7. DB 스키마

### 7.1 tb_safety_guide_cache

안전가이드 MOFA API 응답 캐시 테이블

```sql
CREATE TABLE tb_safety_guide_cache (
    id              BIGSERIAL PRIMARY KEY,
    country_code    VARCHAR(3)   NOT NULL,          -- ISO 3166-1 alpha-3
    data_type       VARCHAR(30)  NOT NULL,           -- 'travel_alert'|'safety_notice'|'country_info'
                                                    -- |'entry_info'|'embassy'|'incident_report'
    content         JSONB        NOT NULL,           -- MOFA API 응답 원본
    fetched_at      TIMESTAMPTZ  NOT NULL DEFAULT now(),
    expires_at      TIMESTAMPTZ  NOT NULL,           -- fetched_at + 24h
    created_at      TIMESTAMPTZ  NOT NULL DEFAULT now(),
    updated_at      TIMESTAMPTZ  NOT NULL DEFAULT now(),

    CONSTRAINT uq_cache_country_type UNIQUE (country_code, data_type)
);

CREATE INDEX idx_safety_cache_country ON tb_safety_guide_cache (country_code);
CREATE INDEX idx_safety_cache_expires ON tb_safety_guide_cache (expires_at);
```

**컬럼 설명:**
- `country_code`: ISO 3166-1 alpha-3 코드 (예: `KOR`, `JPN`, `USA`)
- `data_type`: MOFA API 엔드포인트 구분자 (6종)
- `content`: 원본 JSON 전체 저장 (파싱 없이 보존)
- `expires_at`: TTL 만료 시각 (24시간 기준)

### 7.2 tb_emergency_contact

긴급연락처 로컬 저장 테이블 (캐시 만료 시에도 유지되는 핵심 정보)

```sql
CREATE TABLE tb_emergency_contact (
    id              BIGSERIAL PRIMARY KEY,
    country_code    VARCHAR(3)   NOT NULL,
    contact_type    VARCHAR(20)  NOT NULL,           -- 'police'|'fire'|'ambulance'
                                                    -- |'embassy'|'consulate_call_center'
    phone_number    VARCHAR(30)  NOT NULL,
    description_ko  VARCHAR(100),                   -- 한국어 설명
    is_24h          BOOLEAN      NOT NULL DEFAULT TRUE,
    created_at      TIMESTAMPTZ  NOT NULL DEFAULT now(),
    updated_at      TIMESTAMPTZ  NOT NULL DEFAULT now()
);

CREATE INDEX idx_emergency_contact_country ON tb_emergency_contact (country_code);
```

**고정 레코드 (하드코딩 시드):**
- 영사콜센터: `country_code = 'ALL'`, `contact_type = 'consulate_call_center'`, `phone_number = '+82-2-3210-0404'`, `is_24h = TRUE`

## 8. 오프라인 대응

### 8.1 오프라인 동작 원칙

안전가이드 탭은 여행 중 인터넷 불안정 환경에서도 핵심 정보를 제공해야 한다. 특히 긴급 상황에서 연락처 정보는 반드시 오프라인 상태에서도 접근 가능해야 한다.

### 8.2 캐시 구조

| 항목 | 내용 |
|------|------|
| **캐시 저장소** | SQLite (로컬 디바이스) |
| **캐시 대상** | 마지막 열람 국가의 전체 6개 서브탭 데이터 |
| **국가당 최대 용량** | 5MB |
| **갱신 트리거** | 1) 앱 실행 시, 2) Wi-Fi 연결 시, 3) 사용자 수동 새로고침 |
| **갱신 우선 국가** | 현재 컨텍스트 선택 국가 (여행 목적지) |

### 8.3 오프라인 상태 UX

```
[오프라인 감지]
    ↓
탭 상단에 황색 배너 표시:
┌──────────────────────────────────────────────────┐
│ [오프라인] 저장된 데이터를 표시합니다.              │
│ 최종 갱신: 2시간 전 (2026-03-01 14:30 KST)       │
└──────────────────────────────────────────────────┘
    ↓
캐시 데이터로 전체 서브탭 표시
    ↓
긴급연락 탭: 캐시된 번호로 원터치 전화 발신 가능
```

### 8.4 오프라인 우선순위

| 우선순위 | 항목 | 저장 방식 |
|---------|------|----------|
| 1순위 (필수) | 긴급연락처 전화번호 | SQLite 영구 저장 |
| 2순위 (필수) | 여행경보 단계 | SQLite 24시간 캐시 |
| 3순위 (권장) | 대사관 위치/주소 | SQLite 24시간 캐시 |
| 4순위 (선택) | 현지 응급 가이드 | SQLite 24시간 캐시 |
| 5순위 (선택) | 국가 기본 정보 | SQLite 24시간 캐시 |
