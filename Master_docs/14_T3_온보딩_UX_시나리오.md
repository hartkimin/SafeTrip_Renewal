## 1. 문서 목적 및 적용 범위

### 1.1 목적

본 문서는 SafeTrip 앱의 사용자 유형별(신규/복귀/초대/B2B) 온보딩 여정 전체를 정의한다. 특히 비즈니스 원칙 v5.1의 약관 동의 체계(§06.5), 여행 기간 제한(§02.3), 가디언 연결 플로우(§03.1, §09.3)를 온보딩 흐름에 완전히 반영한다.

### 1.2 적용 범위

- SafeTrip 앱 최초 설치 후 ~ 메인 화면 진입 전체 흐름
- 스플래시 → 웰컴 → 약관 동의 → 전화번호 인증 → 역할 배정 → 메인 화면
- B2B CSV 일괄 초대 플로우 포함

## 3. 시나리오 A — 여행장 주도 신규 가입

```
[A-1] 스플래시
    ↓
[A-2] 웰컴화면 (3~4장 가치 슬라이드)
    ↓
[A-3] 목적 선택: "여행 만들기" / "여행 참여" / "체험하기"
    → "여행 만들기" 선택
    ↓
[A-4] 전화번호 입력
    ↓
[A-5] OTP 인증 (6자리, 3분 유효)
    ↓
[A-6] 약관 동의 화면 ← ★ §06.5 필수 단계
    ↓
[A-7] 생년월일 입력 (14세 미만 여부 판단 → §12 분기)
    ↓
[A-8] 프로필 설정 (이름, 사진)
    ↓
[A-9] 여행 생성 가이드 (선택적)
    - 여행명, 국가, 날짜 설정
    - ★ 15일 초과 시 분할 안내 (§11)
    ↓
[A-10] 메인 화면 (캡틴으로 배정)
```

## 4. 시나리오 B — 초대코드 참여 (크루/크루장)

```
[B-1] 딥링크 또는 초대코드 입력
    ↓
[B-2] 스플래시 (딥링크 파라미터 보존)
    ↓
[B-3] 전화번호 입력 (이미 가입된 경우 → 시나리오 D)
    ↓
[B-4] OTP 인증
    ↓
[B-5] 약관 동의 화면 (§8)
    ↓
[B-6] 생년월일 입력 (미성년자 분기 → §12)
    ↓
[B-7] 프로필 설정
    ↓
[B-8] 초대코드 확인 화면
    - 여행명, 캡틴 이름, 역할(크루장/크루) 표시
    - [참여 확인] / [거절]
    ↓
[B-9] 메인 화면 (코드에 지정된 역할로 배정)
```

### 4.1 초대코드 인식 플로우

| 진입 방법 | 처리 |
|----------|------|
| 앱 미설치 + 딥링크 | 스토어 유도 → 설치 후 딥링크 파라미터 복원 |
| 앱 설치 + 딥링크 | 스플래시에서 파라미터 캡처 → B-8 단계 자동 진행 |
| 앱 내 초대코드 입력 | 메인 화면 상단 CTA "초대코드 입력" → B-8 단계 |

## 5. 시나리오 C — 가디언 초대 참여

```
[C-1] 멤버가 가디언 초대 발송 (전화번호 입력)
    ↓
[C-2] 가디언 기기에 SMS 수신
    "SafeTrip: {멤버명}님이 가디언으로 초대했습니다. [링크]"
    ↓
[C-3] 링크 탭 → 스플래시 (딥링크: guardian_invite)
    ↓
[C-4] 전화번호 입력 → OTP 인증
    ↓
[C-5] 약관 동의 (§8)
    ↓
[C-6] 가디언 초대 확인 화면
    - 멤버 이름, 여행 정보 표시
    - [수락] / [거절]
    ↓
[C-7] 수락 시 → 가디언 역할로 배정
    가디언 특성 안내:
    - "멤버의 위치를 확인하고 긴급 알림을 받을 수 있습니다"
    - "별도 비용이 발생할 수 있습니다" (3번째 가디언 이상)
    ↓
[C-8] 메인 화면 (가디언 전용 UI)
```

## 6. 시나리오 D — 복귀 사용자 (재인증)

비즈니스 원칙 §06.2: 재인증 시 불필요한 단계를 건너뛴다.

```
[D-1] 스플래시
    ↓
저장된 인증 토큰 유효 여부 확인
    ├── 유효: → 메인 화면 직행 (온보딩 생략)
    └── 만료/없음:
            ↓
[D-2] 전화번호 입력
    ↓
[D-3] OTP 인증
    ↓ (약관/프로필 설정 건너뜀 — 이미 완료)
[D-4] 메인 화면 → 가장 가까운 여행 자동 배정 (§06.2)
```

### 6.1 자동 여행 배정 우선순위

| 우선순위 | 조건 |
|:-------:|------|
| 1 | `active` 상태 여행 |
| 2 | `planning` 상태 여행 (시작일이 현재에 가장 가까운 순) |
| 3 | 동일 우선순위: 시작일이 빠른 여행 선택 |

## 7. 시나리오 E — B2B CSV 일괄 등록

비즈니스 원칙 §12.3, §01.4 기반.

```
[E-1] B2B 관리자 포털에서 CSV 업로드
    CSV 컬럼: 이름, 전화번호, 역할(captain/crew_chief/crew/guardian), 이메일(선택)
    ↓
[E-2] 서버: CSV 파싱 → 역할별 초대코드 일괄 생성
    ↓
[E-3] 각 전화번호로 SMS 발송:
    "SafeTrip: {조직명} {여행명} 참여 초대입니다. [링크]"
    ↓
[E-4] 사용자 링크 탭 → 시나리오 B-3 이후와 동일
    단, B2B 역할 매핑 자동 적용 (§01.4)
    ├── 학교: 캡틴=인솔교사, 크루=학생, 가디언=학부모
    ├── 여행사: 캡틴=가이드, 크루=참가자
    └── 기업: 캡틴=팀장, 크루=팀원, 가디언=안전담당자
    ↓
[E-5] 미성년자(학교 시나리오): 학부모 동의 프로세스 자동 트리거 (§12)
    ↓
[E-6] 메인 화면 (CSV에서 지정된 역할로 자동 배정)
```

## 8. A-2 약관 동의 화면 상세

비즈니스 원칙 §06.5 기반. 모든 신규 가입 시 반드시 거쳐야 하는 단계.

### 8.1 동의 항목 구성

```
┌──────────────────────────────────────┐
│          서비스 이용을 위한           │
│           약관 동의가 필요합니다       │
├──────────────────────────────────────┤
│ ☑ 전체 동의                          │
├──────────────────────────────────────┤
│ [필수] ☑ 서비스 이용약관              │  → [전문 보기] ↗
│ [필수] ☑ 개인정보처리방침             │  → [전문 보기] ↗
│ [필수] ☑ 위치기반서비스 이용약관      │  → [전문 보기] ↗
│ [선택] ☐ 마케팅 정보 수신 동의        │
├──────────────────────────────────────┤
│ [EU 사용자만 표시]                    │
│ [필수] ☑ GDPR 개인정보 처리 동의      │
│ [필수] ☑ Firebase 국외 이전 동의      │
├──────────────────────────────────────┤
│           [다음] (필수 모두 체크 시)   │
└──────────────────────────────────────┘
```

### 8.2 동의 기록 저장

```sql
-- 항목별 동의 시간·버전 기록 (DB 설계 v3.4 참조)
INSERT INTO tb_user_consent (user_id, consent_type, is_agreed, agreed_at, version)
VALUES
  (:user_id, 'terms_of_service',      TRUE, now(), '2026-03-01'),
  (:user_id, 'privacy_policy',        TRUE, now(), '2026-03-01'),
  (:user_id, 'lbs_terms',             TRUE, now(), '2026-03-01'),
  (:user_id, 'marketing',             :is_agreed, now(), '2026-03-01'),
  -- EU 사용자 추가
  (:user_id, 'gdpr',                  TRUE, now(), '2026-03-01'),
  (:user_id, 'firebase_international', TRUE, now(), '2026-03-01');
```

### 8.3 필수 동의 미체크 시

- 필수 동의 미체크 상태에서 [다음] 탭 → 진행 차단
- 미체크 항목 하이라이트 + "필수 항목에 동의해야 계속할 수 있습니다" 안내

## 9. 프라이버시 등급 선택 안내

여행 생성 시 또는 온보딩 완료 직후 프라이버시 등급을 선택한다.

```
┌──────────────────────────────────────┐
│     위치 공유 방식을 선택해주세요      │
├──────────────────────────────────────┤
│ 🛡️ 안전 최우선                        │  ← 기본 추천
│    항상 정확한 위치를 공유합니다      │
├──────────────────────────────────────┤
│ 📍 표준                               │
│    설정한 시간에만 위치를 공유합니다  │
├──────────────────────────────────────┤
│ 🔒 프라이버시 우선                    │
│    최소한의 위치 정보만 공유합니다    │
└──────────────────────────────────────┘
```

- **미성년자 포함 여행**: `안전 최우선` 강제 → 선택 UI 표시 안 함 (§10.2)
- 등급 선택 후 설정 메뉴에서 언제든 변경 가능

## 10. 가디언 연결 플로우

온보딩 완료 후 (또는 설정 메뉴에서) 가디언을 연결하는 플로우. §03.1, §09.3 기반.

```
[멤버 앱] 설정 → 가디언 관리
    ↓
가디언 전화번호 입력
    ↓
무료 2명 한도 확인:
    ├── 미초과 (1~2번째): 바로 초대 발송
    └── 초과 (3번째+): 과금 안내 모달 표시 ★

[과금 안내 모달 (§09.3)]
┌──────────────────────────────────────┐
│  가디언 추가 과금 안내               │
│  3번째 가디언부터 1,900원/여행        │
│  결제 후 최대 5명까지 연결 가능합니다  │
│  [결제하기] [취소]                   │
└──────────────────────────────────────┘
    ↓ (결제 완료)
초대 SMS 발송 → 시나리오 C 진입
```

```

    └── 15일 초과 시도:
            ↓
    [15일 초과 안내 모달]
    ┌──────────────────────────────────────┐
    │  여행 기간은 최대 15일입니다          │
    │                                      │
    │  30일 여행을 계획 중이신가요?         │
    │  두 개의 여행으로 나누어 생성하세요.  │
    │  예: 1차 여행 (1~15일) + 2차 (16~30일) │
    │                                      │
    │  [1차 여행 생성] [취소]               │
    └──────────────────────────────────────┘
```

- 서버 API에서도 CHECK 제약으로 15일 초과를 차단한다 (DB 설계 v3.4 `tb_trip` 참조)

## 12. 미성년자 동의 분기

비즈니스 원칙 §10.1 기반. 생년월일 입력 후 나이 계산.

```
생년월일 입력
    ↓
나이 계산
    ├── 18세 이상: 일반 성인 동의 절차
    ├── 14세~17세:
    │       ↓
    │   보호자 이중 동의 화면:
    │   "보호자 전화번호를 입력해주세요"
    │       ↓
    │   보호자 기기에 SMS → 보호자 인앱 승인
    └── 14세 미만:
            ↓
        법정대리인(부모/보호자) 동의 화면:
        "부모님 또는 보호자의 동의가 필요합니다"
            ↓
        법정대리인 전화번호 입력 → OTP 인증
            ↓
        법정대리인 동의 완료 후 가입 허용
        + `tb_user.is_minor = TRUE` 저장
        + 여행 등급 safety_first 강제 적용
```

## 16. 에러 및 엣지케이스 처리

| 케이스 | 처리 방법 |
|--------|----------|
| **OTP 만료 (3분)** | 재발송 버튼 표시, 1분 후 재활성화 |
| **OTP 3회 오입력** | 1분 대기 후 재시도 허용 |
| **딥링크 만료 (초대코드)** | "초대코드가 만료되었습니다. 캡틴에게 재발급 요청하세요" |
| **초대코드 역할 불일치** | 코드의 역할 표시 후 확인 요구 |
| **보호자 동의 미완료 (미성년자)** | 동의 대기 중 상태 표시, 48시간 내 미완료 시 가입 취소 |
| **이미 가입된 전화번호** | 시나리오 D (복귀 사용자) 자동 전환 |
| **약관 동의 중단** | 앱 재진입 시 동의 화면부터 재시작 |
| **여행 기간 15일 초과 입력** | 모달 안내, 직접 차단 (날짜 선택 불가) |
