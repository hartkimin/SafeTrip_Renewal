## §2. 목적 및 적용 범위

### 2.1 목적

본 문서는 SafeTrip 앱에서 제공하는 **AI 기능 전체**에 대한 설계 원칙을 정의한다. AI 기능은 Safety AI / Convenience AI / Intelligence AI 3가지로 분류되며, 각 분류별 무료/유료 경계, 탭별 배치, 역할별 접근 권한, 프라이버시 등급별 동작 차이, 미성년자 제한, AI 데이터 처리 원칙, 모델 폴백 전략을 규정한다.

비즈니스 원칙 v5.1의 다음 항목과 직접 연계된다:
- §04 위치 공유 원칙 (프라이버시 등급별 AI 동작)
- §09.4 부가 상품 (AI Plus / AI Pro 과금 체계)
- §10.3 미성년자 AI 기능 제한
- §11 오프라인 기본 정책 (Safety AI 오프라인 폴백)

### 2.2 적용 범위

| 대상 | 범위 |
|------|------|
| 적용 기능 | 앱 내 모든 AI 기반 기능 (Safety AI / Convenience AI / Intelligence AI) |
| 적용 역할 | 캡틴, 크루장, 크루, 가디언 (역할별 접근 범위 차등) |
| 적용 등급 | 안전최우선(safety_first) / 표준(standard) / 프라이버시우선(privacy_first) |
| 적용 Phase | Phase 1 (Safety AI, 기본 Convenience) / Phase 2 (AI 챗봇, 고급 Convenience) / Phase 3 (Intelligence AI) |
| 비적용 범위 | 서버 인프라 ML 파이프라인 내부 구현 상세 (별도 기술 아키텍처 문서 정의) |

### 2.3 용어 정의

| 용어 | 정의 |
|------|------|
| Safety AI | 안전 목적의 AI 기능. 위험 감지, SOS 자동 판단, 이탈 감지, 집합 지연 감지 포함. 전원 무료 |
| Convenience AI | 편의 목적의 AI 기능. 일정 추천, 장소 추천, 짐 리스트 생성, 번역, AI 챗봇 포함. 기본 기능 무료 / 고급 기능 유료 |
| Intelligence AI | 인사이트·분석·예측 목적의 AI 기능. 여행 인사이트, 패턴 분석, 이동 예측 포함. 전원 유료(AI 구독) |
| AI Plus | Convenience AI 고급 기능 + Intelligence AI 전체를 포함한 유료 구독 (4,900원/월 또는 2,900원/여행) |
| AI Pro | AI Plus + 프리미엄 브리핑 + 일정 최적화를 포함한 최상위 유료 구독 (9,900원/월 또는 5,900원/여행) |
| 할루시네이션 | LLM이 잘못된 정보를 사실인 것처럼 생성하는 현상. 여행 안전 정보에서는 특히 위험 |
| 폴백(Fallback) | 기본 AI 모델 장애 시 차선 모델·방식으로 자동 전환하는 전략 |
| DPA | Data Processing Agreement. AI 모델 공급자와 체결하는 데이터 처리 계약서 |
| 온디바이스 모델 | 서버 통신 없이 기기 내부에서 실행되는 경량 AI 모델 |

## §3. AI 3분류 체계

### 3.1 분류 정의 및 무료/유료 경계

SafeTrip의 AI 기능은 **목적**에 따라 3가지로 분류한다. 안전과 관련된 기능은 과금 상태와 무관하게 항상 무료로 제공한다(§09.5 무료 구간 보장 원칙).

```
[Safety AI] ─────────────── 전원 무료 ──────────────────────────
  ├── 위험 지역 자동 감지 (국가 안전 등급, 재난 정보 연계)
  ├── SOS 자동 판단 (움직임 패턴 + 비활성 감지)
  ├── 이탈 감지 (지오펜스 이탈 + 미귀환 판단)
  └── 집합 지연 감지 (집합 장소 미도착 자동 알림)

[Convenience AI] ─────── 기본: 무료 / 고급: AI Plus 이상 ────────
  ├── [무료] 일정 자동 완성 (여행 날짜·국가 기반)
  ├── [무료] 이동 시간 추정 (교통수단 선택)
  ├── [무료] 짐 리스트 기본 생성 (국가·기간·인원 기반)
  ├── [AI Plus] 장소 추천 (위치 기반 맞춤 POI 추천)
  ├── [AI Plus] AI 챗봇 어시스턴트 (여행 관련 Q&A)
  ├── [AI Plus] 실시간 번역 (채팅·안내판 텍스트)
  └── [AI Plus] 채팅 요약 (장문 채팅 핵심 요약)

[Intelligence AI] ─────── 전원 유료 (AI Plus 이상) ────────────
  ├── 여행 인사이트 (일별 이동 거리, 체류 패턴 분석)
  ├── 패턴 분석 (멤버 행동 패턴, 이탈 위험 점수)
  ├── 이동 예측 (다음 행선지 예측, 도착 예상 시간)
  └── 맞춤 안전 브리핑 [AI Pro] (위험 지역·날씨·행사 종합 브리핑)
```

### 3.3 AI 구독 결제 흐름

```
[사용자 AI 유료 기능 터치]
       │
       ▼
[현재 구독 상태 확인]
       │
  구독 중? ─── YES ──→ [기능 즉시 제공]
       │
      NO
       │
       ▼
[AI Plus / AI Pro 구독 안내 모달 표시]
  ├── AI Plus: 4,900원/월 또는 2,900원/여행
  └── AI Pro: 9,900원/월 또는 5,900원/여행
       │
       ▼
[인앱 결제 (App Store / Play Store IAP)]
       │
  결제 성공? ─── YES ──→ [tb_subscription 구독 레코드 생성]
       │                     → [tb_payment 결제 레코드 저장]
      NO                      → [AI 기능 잠금 해제]
       │
       ▼
[결제 실패 토스트 + 재시도 또는 취소]
```

- **결제 유형(payment_type)**: `addon_ai_plus` 또는 `addon_ai_pro` (§09.6)
- **B2B 기업 고객**: AI 구독을 개인 결제 없이 `b2b_contract` 계약으로 일괄 제공 가능. 계약 범위 초과 시 별도 계약 필요.
- **환불 규칙**: AI Plus/Pro 월 구독은 결제 후 7일 이내 미사용 시 전액 환불. 여행 단위 구독은 여행 `planning` 상태에서만 전액 환불.

## §5. AI 데이터 처리 원칙

### 5.1 LLM 호출 시 개인정보 마스킹

LLM(외부 AI 모델) 호출 페이로드에는 개인정보를 포함하지 않는다. 서버에서 전처리 후 마스킹된 데이터만 전달한다.

| 원본 데이터 | LLM 전달 형태 | 비고 |
|-----------|-------------|------|
| 사용자 이름 | "멤버A", "멤버B" (순번 익명화) | 식별 불가 |
| 전화번호 | 전송 금지 | 절대 포함 불가 |
| 이메일 | 전송 금지 | 절대 포함 불가 |
| 정확 위치 좌표 | 구(區) 수준 그리드로 뭉툭하게 변환 | 100m → 1km 단위 |
| 여행명 | 내부 ID로 치환 ("trip_a1b2") | 식별 불가 |

### 5.2 위치 데이터 AI 학습 미사용 원칙

- SafeTrip 사용자의 **실시간 위치 데이터는 AI 모델 학습에 사용하지 않는다**.
- 외부 LLM 공급자와 체결하는 DPA(Data Processing Agreement)에 학습 미사용 조항을 명시한다.
- 국제 데이터 처리 원칙(#12)과 정합: 위치 데이터의 국경 간 이전 시 DPA + 표준 계약 조항(SCC) 병행 적용.
- AI 사용 로그는 **90일 보존** 후 자동 삭제한다 (`tb_ai_usage_log.expires_at` 기준).

### 5.3 AI 응답 캐싱 정책

| 대상 요청 | 캐시 유효 시간 | 캐시 키 구성 |
|---------|-------------|------------|
| 국가 위험 정보 조회 | 6시간 | country_code + threat_type |
| 장소 추천 (동일 지역) | 24시간 | lat_grid + lon_grid + category |
| 일정 자동 완성 | 1시간 | trip_id + schedule_hash |
| 채팅 요약 | 메시지 변경 전까지 유효 | chat_room_id + message_count |
| 안전 브리핑 | 4시간 | trip_id + destination + date |

- 캐시 미적중 시 LLM 호출 → 응답 저장 → 반환.
- 캐시 히트 시 저장된 응답 즉시 반환 (LLM 호출 생략 → 비용 절감 + 응답 속도 향상).

### 5.4 AI 사용 로그 기록 항목

`tb_ai_usage_log`에 기록되는 항목:

| 컬럼 | 내용 |
|------|------|
| `ai_type` | `safety` / `convenience` / `intelligence` |
| `feature_name` | 구체 기능명 (예: `sos_auto_detect`, `schedule_autocomplete`) |
| `user_id` | 요청 사용자 ID |
| `trip_id` | 연관 여행 ID (nullable) |
| `model_used` | 실제 사용된 모델명 (예: `gpt-4o`, `claude-3`, `on_device_v2`, `rule_based`) |
| `latency_ms` | 응답 소요 시간 (밀리초) |
| `is_cached` | 캐시 응답 여부 |
| `is_fallback` | 폴백 모델 사용 여부 |
| `expires_at` | 로그 보존 만료일 (90일 후) |

## §6. AI 모델 선택 및 폴백 전략

### 6.1 모델 3단계 폴백 구조

```
[1차] 클라우드 LLM (외부 API)
  - OpenAI GPT-4o (Convenience / Intelligence AI)
  - Anthropic Claude 3 (Safety AI 문맥 분석)
  - 응답 시간 초과(Safety: 2초, Convenience: 5초) 또는 API 장애 시 → 2차로 전환
  │
  ▼
[2차] 경량 온디바이스 모델
  - 기기 내 실행 (서버 불필요)
  - Convenience AI 기본 기능 한정 (번역, 일정 자동완성 경량 버전)
  - Safety AI 단순 패턴 판단 지원
  - 기기 저장공간 부족 또는 지원 기기 미충족 시 → 3차로 전환
  │
  ▼
[3차] 규칙 기반 대응 (Rule-Based)
  - AI 모델 없이 사전 정의된 규칙으로 동작
  - Safety AI 전 기능 유지 (이탈 감지: GPS 반경 초과 판단, SOS: 비활성 시간 기준)
  - Convenience AI: 기능 비활성화 + "현재 AI 서비스 점검 중" 안내
  - Intelligence AI: 전체 비활성화
```

### 6.2 폴백 발동 조건

| 조건 | 폴백 단계 |
|------|----------|
| 클라우드 LLM API 응답 없음 (타임아웃) | 1차 → 2차 |
| 클라우드 LLM API 오류 (5xx) | 1차 → 2차 |
| 네트워크 완전 단절 (오프라인) | 1차·2차 동시 → 3차 |
| 온디바이스 모델 메모리 부족 | 2차 → 3차 |
| 오프라인 + 온디바이스 미설치 | 직접 → 3차 |

### 6.3 모델별 다국어 지원

| 언어 | 클라우드 LLM | 온디바이스 모델 | 규칙 기반 |
|------|------------|--------------|---------|
| 한국어 | 완전 지원 | 완전 지원 | 한국어 고정 문자열 |
| 영어 | 완전 지원 | 완전 지원 | 영어 고정 문자열 |
| 일본어 | 완전 지원 | 부분 지원 | 일본어 고정 문자열 |
| 기타 언어 | 부분 지원 | 미지원 (영어 대체) | 영어 고정 문자열 |

## §7. AI 응답 품질 기준

### 7.1 응답 시간 SLA

| AI 분류 | 응답 시간 기준 | 초과 시 처리 |
|---------|-------------|-----------|
| Safety AI | ≤ 2초 | 즉시 규칙 기반 폴백 |
| Convenience AI | ≤ 5초 | 로딩 스피너 표시 + 5초 이후 타임아웃 안내 |
| Intelligence AI | ≤ 10초 | 로딩 애니메이션 + 백그라운드 처리 후 푸시 알림 |

### 7.2 할루시네이션 방지 원칙

여행 안전 정보(위험 지역, 재난, 의료 정보 등)에서의 할루시네이션은 인명 피해와 직결될 수 있으므로 다음 원칙을 적용한다:

- **Safety AI 응답**: LLM 생성 텍스트를 직접 사용하지 않는다. 사전 정의된 안전 등급 DB(외교부 여행경보, 재난 API)와 구조화된 규칙으로만 판단한다.
- **Convenience AI 응답**: LLM 생성 응답에 "AI가 생성한 정보로, 실제와 다를 수 있습니다" 면책 문구를 첨부한다.
- **Intelligence AI 응답**: 분석 근거 데이터(로그 기간, 샘플 수)를 함께 표시하여 신뢰성을 명시한다.
- **정확도 기준**: 안전가이드 탭 AI 정보는 최소 2개 이상의 공식 데이터 소스와 교차 검증 후 제공한다.

### 7.3 AI 응답 품질 모니터링

- 사용자 피드백 버튼(응답 하단 엄지 업/다운)을 제공한다.
- 부정 피드백 비율이 15%를 초과하면 해당 기능의 AI 모델 재검토 트리거를 발동한다.
- `tb_ai_usage_log`의 `latency_ms`를 7일 단위로 집계하여 SLA 준수 여부를 모니터링한다.

## §9. 프라이버시 등급별 AI 동작 차이

비즈니스 원칙 §04의 3단계 프라이버시 등급은 AI 기능의 데이터 접근 범위와 활성화 여부에 직접 영향을 미친다.

### 9.1 등급별 AI 기능 활성화 요약

| AI 기능 | 안전최우선(safety_first) | 표준(standard) | 프라이버시우선(privacy_first) |
|---------|:---------------------:|:------------:|:------------------------:|
| Safety AI 전체 | 완전 활성 | 완전 활성 | 완전 활성 |
| Convenience AI — 일정 기반 추천 | 활성 | 일정 연동 시간대만 | 일정 연동 시간대만 |
| Convenience AI — 위치 기반 추천 | 활성 | 활성 (위치 공유 ON 시간대) | **비활성** |
| Convenience AI — 번역/챗봇 | 활성 | 활성 | 활성 |
| Intelligence AI — 패턴 분석 | 활성 | 위치 공유 ON 시간대 데이터만 | **비활성** |
| Intelligence AI — 여행 인사이트 | 활성 | 공유 데이터 범위 내 | 공유 데이터 범위 내 |

### 9.2 프라이버시 등급별 상세 동작

#### 안전최우선 (`safety_first`)

- 모든 AI 기능 활성화.
- 위치 데이터를 24시간 AI 처리에 사용 가능.
- Safety AI가 가장 높은 정확도로 동작 (최다 데이터 기반).
- 미성년자 포함 여행에서 강제 적용되는 등급 (§10.2).

#### 표준 (`standard`)

- Safety AI: 완전 활성 (등급 무관 원칙).
- Convenience AI: 위치 기반 추천은 위치 공유 ON 시간대에만 활성. 번역·챗봇은 시간대 무관 활성.
- Intelligence AI: 위치 공유 ON 시간대에 수집된 데이터만 분석에 사용. 비공유 시간대 데이터는 분석에서 제외.

#### 프라이버시우선 (`privacy_first`)

- Safety AI: 완전 활성 (SOS·이탈 감지는 등급과 무관하게 최상위 작동).
- Convenience AI: 위치 기반 장소 추천 비활성. 일정 기반 추천·번역·챗봇은 활성.
- Intelligence AI: 위치 기반 패턴 분석 비활성. 비위치 기반 인사이트(체크리스트 완성률, 일정 준수율)만 제공.

## §10. 미성년자 AI 기능 제한

비즈니스 원칙 §10.3의 미성년자 AI 기능 제한을 구체적으로 정의한다.

### 10.1 미성년자 판별 기준

| 판별 방법 | 내용 |
|----------|------|
| 프로필 생년월일 | `tb_user.birth_date` 기준으로 만 18세 미만이면 미성년자로 판단 |
| 캡틴 미성년자 플래그 | `tb_group_member.is_minor = TRUE` — 캡틴이 수동으로 지정하거나 생년월일 자동 판단 |
| 복합 판단 | 생년월일 미입력 시 캡틴 플래그를 우선 적용 |

### 10.2 미성년자 역할 제약

미성년자(크루)가 포함된 여행은 `safety_first` 등급이 강제 적용되므로(§10.2), Safety AI는 가장 높은 수준으로 동작한다.

| AI 기능 | 미성년자 크루 적용 규칙 |
|---------|---------------------|
| Safety AI 전체 | **제한 없이 사용 가능** (보호 목적 최우선) |
| Convenience AI — 일정·짐 리스트·번역 | **제한 없이 사용 가능** |
| Convenience AI — 위치 기반 장소 추천 | **제한** (위치 정보 과다 노출 방지) |
| Convenience AI — AI 챗봇 직접 대화 | **제한** (14세 미만: 완전 차단 / 14~17세: 여행 관련 주제만 허용) |
| Intelligence AI — 그룹 단위 인사이트 | 사용 가능 (개인 식별 불가 집계 데이터) |
| Intelligence AI — 개인 데이터 기반 분석 | **제한** (미성년자 개인 행동 패턴 분석 금지) |

### 10.3 미성년자 AI 챗봇 제한 UX

- 14세 미만 미성년자: AI 챗봇 진입 자체를 차단. "보호자 동의가 필요한 기능입니다" 안내 표시.
- 14~17세 미성년자: AI 챗봇 사용 가능하나 여행 관련 주제 외 대화 시 "여행과 관련된 질문을 해주세요" 리다이렉트.
- 주제 판별은 서버 사이드 분류 모델로 처리 (클라이언트 단독 처리 금지).

### 10.4 미성년자 위치 데이터 보존 기간

- 미성년자 크루의 위치 데이터는 여행 종료 후 **30일** 보존 후 자동 삭제 (성인 90일보다 단축, §10.2).
- AI 사용 로그(`tb_ai_usage_log`)에서 미성년자 관련 레코드도 동일하게 30일 적용.

## §11. 에러 및 엣지케이스 처리

### 11.1 AI 응답 지연·장애 UX

| 상황 | 클라이언트 처리 | 서버 처리 |
|------|--------------|---------|
| AI 응답 지연 (2~5초) | 로딩 스피너 표시 | 폴백 준비 시작 |
| AI 응답 타임아웃 초과 | "응답 지연 중입니다. 잠시 후 다시 시도해주세요" 토스트 + 수동 재시도 버튼 | 폴백 모델로 전환 |
| LLM API 전체 장애 | "AI 서비스 일시 점검 중입니다" 배너 + 규칙 기반 Safety AI 유지 | 규칙 기반 폴백 전환 + 장애 로그 기록 |
| 캐시 만료 + 네트워크 단절 | 마지막 캐시 응답 표시 + "최신 정보가 아닐 수 있습니다" 경고 | — |
| AI 응답 내용 부적절 (신고) | 신고 버튼 → AI 팀 검토 큐 전달 | 응답 로그 플래그 처리 |

### 11.2 과금 관련 엣지케이스

| 상황 | 처리 규칙 |
|------|----------|
| AI Plus 구독 중 만료 | 유료 기능 즉시 잠금 + 갱신 안내 배너 (만료 7일 전 선제 알림) |
| AI Plus 결제 실패 | 3일 유예 기간 부여 → 이후 잠금 |
| 여행 중 구독 만료 | 여행 종료일까지 유지 → 다음 여행부터 잠금 (사용자 경험 보호) |
| B2B 계약 범위 초과 | 초과 사용 기능 잠금 + 관리자에게 계약 초과 알림 |
| 미성년자가 유료 AI 결제 시도 | 차단 + "보호자 동의 후 결제 가능합니다" 안내 |

### 11.3 데이터 처리 엣지케이스

| 상황 | 처리 규칙 |
|------|----------|
| 위치 데이터 없음 (GPS 없음) | 위치 기반 AI 기능 비활성화 + "위치 정보가 없어 일부 AI 기능이 제한됩니다" 안내 |
| 여행 종료 후 AI 기능 요청 | `completed` 상태 여행의 Intelligence AI 인사이트는 보존 기간 내 열람 가능 |
| 동일 요청 연속 발송 (스팸) | 사용자당 동일 기능 분당 10회 요청 제한 (rate limiting) |
| DPA 미체결 LLM 공급자 | 해당 공급자 API 호출 차단. 다음 폴백 단계로 전환 |
| 개인정보 마스킹 실패 | LLM API 호출 전 마스킹 실패 시 해당 요청 전체 중단. 로그 기록 후 사용자에게 "요청을 처리할 수 없습니다" 반환 |

## §12. DB 스키마

### 12.1 tb_ai_usage_log (AI 사용 이력)

```sql
CREATE TABLE tb_ai_usage_log (
    log_id          UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    user_id         UUID REFERENCES tb_user(user_id) ON DELETE SET NULL,
    trip_id         UUID REFERENCES tb_trip(trip_id) ON DELETE SET NULL,
    ai_type         VARCHAR(20) NOT NULL
                    CHECK (ai_type IN ('safety', 'convenience', 'intelligence')),
    feature_name    VARCHAR(50) NOT NULL,
                    -- 예: 'sos_auto_detect', 'schedule_autocomplete',
                    --     'place_recommend', 'chat_summary', 'travel_insight'
    model_used      VARCHAR(50),
                    -- 예: 'gpt-4o', 'claude-3-opus', 'on_device_v2', 'rule_based'
    is_cached       BOOLEAN DEFAULT FALSE,
    is_fallback     BOOLEAN DEFAULT FALSE,
    fallback_reason VARCHAR(100),
                    -- 예: 'api_timeout', 'network_offline', 'oom'
    latency_ms      INTEGER,
    is_minor_user   BOOLEAN DEFAULT FALSE,   -- 미성년자 사용 여부
    privacy_level   VARCHAR(20),             -- 사용 시점 프라이버시 등급
    feedback        SMALLINT CHECK (feedback IN (-1, 0, 1)),
                    -- -1: 부정, 0: 무응답, 1: 긍정
    created_at      TIMESTAMPTZ DEFAULT NOW(),
    expires_at      TIMESTAMPTZ              -- created_at + 90일 (미성년자: +30일)
);

-- 인덱스
CREATE INDEX idx_ai_usage_log_user    ON tb_ai_usage_log (user_id);
CREATE INDEX idx_ai_usage_log_trip    ON tb_ai_usage_log (trip_id);
CREATE INDEX idx_ai_usage_log_type    ON tb_ai_usage_log (ai_type, feature_name);
CREATE INDEX idx_ai_usage_log_expires ON tb_ai_usage_log (expires_at);
```

### 12.2 tb_ai_subscription (AI 구독 정보)

```sql
CREATE TABLE tb_ai_subscription (
    subscription_id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    user_id         UUID NOT NULL REFERENCES tb_user(user_id) ON DELETE CASCADE,
    plan_type       VARCHAR(20) NOT NULL
                    CHECK (plan_type IN ('ai_plus', 'ai_pro')),
    billing_cycle   VARCHAR(10) NOT NULL
                    CHECK (billing_cycle IN ('monthly', 'per_trip')),
    trip_id         UUID REFERENCES tb_trip(trip_id) ON DELETE SET NULL,
                    -- per_trip 구독 시 연결 여행, monthly 구독 시 NULL
    status          VARCHAR(20) DEFAULT 'active'
                    CHECK (status IN ('active', 'cancelled', 'expired', 'grace_period')),
    started_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    expires_at      TIMESTAMPTZ NOT NULL,
    grace_until     TIMESTAMPTZ,             -- 결제 실패 시 3일 유예 종료일
    payment_id      UUID REFERENCES tb_payment(payment_id) ON DELETE SET NULL,
    created_at      TIMESTAMPTZ DEFAULT NOW(),
    updated_at      TIMESTAMPTZ DEFAULT NOW()
);

-- 인덱스
CREATE INDEX idx_ai_subscription_user    ON tb_ai_subscription (user_id);
CREATE INDEX idx_ai_subscription_status  ON tb_ai_subscription (status, expires_at);
CREATE INDEX idx_ai_subscription_trip    ON tb_ai_subscription (trip_id)
    WHERE trip_id IS NOT NULL;

-- 활성 월 구독 1건 유지를 위한 부분 유니크 인덱스
CREATE UNIQUE INDEX idx_ai_subscription_active_monthly
    ON tb_ai_subscription (user_id, plan_type)
    WHERE billing_cycle = 'monthly' AND status IN ('active', 'grace_period');
```

## §13. 오프라인 대응

### 13.1 오프라인 시 AI 기능 동작 원칙

오프라인 상태에서도 Safety AI는 규칙 기반 폴백으로 **핵심 안전 기능을 유지**한다. Convenience AI와 Intelligence AI는 기능이 제한되며 사용자에게 명확히 안내한다.

| AI 분류 | 오프라인 동작 |
|---------|------------|
| Safety AI | 규칙 기반 폴백으로 완전 유지 — 이탈 감지(GPS 반경 기준), SOS(비활성 시간 기준) |
| Convenience AI | 마지막 캐시 응답 사용 (캐시 만료 시 기능 비활성) + "오프라인 상태" 배너 |
| Intelligence AI | 전체 비활성 + "인터넷 연결 시 사용 가능합니다" 안내 |

### 13.2 오프라인 Safety AI 규칙 기반 동작 상세

- **이탈 감지**: 집합 장소로부터 GPS 반경 300m 초과 + 10분 이상 지속 → 로컬 알림 발생.
- **SOS 자동 판단**: 가속도 센서 기반 낙상 감지 + 15분 비활성 → 로컬 SOS 발동 큐잉 → 온라인 복귀 시 즉시 전송.
- **지오펜스 이탈**: 기기 OS 지오펜스 API 사용 (서버 연결 불필요).

### 13.3 오프라인 → 온라인 복귀 시 동기화

```
[오프라인 중]
  └── Safety AI 이벤트 큐잉 (로컬 SQLite)
  └── 미전송 SOS 로컬 저장

[온라인 복귀 시]
  ├── 큐잉된 Safety AI 이벤트 즉시 서버 전송
  ├── AI 캐시 갱신 요청
  └── Intelligence AI 분석 데이터 동기화 (백그라운드)
```
