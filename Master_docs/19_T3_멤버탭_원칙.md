## 1. 문서 목적 및 적용 범위

### 1.1 목적

본 문서는 SafeTrip 앱의 멤버탭(Members Tab) 전체 설계 원칙을 정의한다. 멤버탭은 여행 참여자의 현재 상태·역할·위치 정보를 한눈에 파악하고, 캡틴/크루장이 여행 멤버와 가디언을 관리하는 중심 화면이다. 비즈니스 원칙 v5.1의 역할 체계(§01), 가디언 시스템(§03), 위치 공유(§04), 과금 모델(§09), 미성년자 보호(§10)를 UI 레벨에서 구현하는 규칙을 명세한다.

### 1.2 적용 범위

| 범위 | 포함 여부 |
|------|:--------:|
| 멤버 목록 표시 (캡틴/크루장/크루/가디언 섹션) | ✅ |
| 멤버 카드 디자인 (역할 배지, 상태 인디케이터, 위치 텍스트) | ✅ |
| 가디언 무료/유료 배지 및 슬롯 표시 (§03.1, §09.3) | ✅ |
| 가디언 관리 하프시트 (캡틴 전용 과금 관리 UI) | ✅ |
| B2B 커스텀 역할명 표시 (§01.4) | ✅ |
| 멤버 정렬 규칙 (SOS 우선, 역할 순, 온/오프라인 순) | ✅ |
| 출석 체크 시스템 (6인 이상 유료, TB_ATTENDANCE_CHECK 연동) | ✅ |
| 가디언 전용 상태 탭 (연결 멤버 카드, 긴급 위치 요청) | ✅ |
| 미성년자 멤버 가디언 해제 캡틴 승인 (§10) | ✅ |
| 지도탭 마커 연동 (멤버 카드 탭 → 지도 이동) | 지도탭 원칙(#17) 참조 |

### 1.3 관련 비즈니스 원칙 조항

| 조항 | 내용 |
|------|------|
| §01.1 | 역할 4종 정의 (captain/crew_chief/crew/guardian) |
| §01.4 | B2B 역할 매핑 (학교/여행사/기업) |
| §03.1 | 가디언 무료 2명 / 유료 최대 3명 (멤버당) |
| §03.2 | 가디언은 연결된 멤버의 위치만 공유 받음 |
| §03.3 | 가디언 해제 규칙 (미성년자 = 캡틴 승인 필요) |
| §03.4 | 가디언 긴급 위치 요청 (1시간 최대 3회) |
| §04 | 프라이버시 등급별 위치 공유 동작 |
| §05.5 | 출석 체크 (캡틴/크루장 실행, 크루 응답) |
| §09.3 | 가디언 3번째부터 1,900원/여행, 환불 불가 |
| §10.2 | 미성년자 포함 시 safety_first 강제, 가디언 해제 캡틴 승인 |

## 2. 설계 철학 5원칙

| ID | 원칙명 | 내용 |
|----|--------|------|
| **M1** | **상태 중심** | 멤버의 현재 상태(온라인/오프라인/SOS/배터리)를 어떤 정보보다 우선하여 표시한다. 역할보다 상태가 먼저 눈에 들어와야 한다. |
| **M2** | **역할 구분** | 캡틴/크루장/크루/가디언을 시각적으로 명확히 구분한다. 배지 색상, 아이콘, 섹션 분리로 역할을 즉각 인식할 수 있게 한다. |
| **M3** | **위험 강조** | SOS 중인 멤버와 오프라인 멤버를 최상단에 배치하고 빨간색으로 강조한다. 안전 위협 상황은 절대 리스트 중간에 묻히지 않는다. |
| **M4** | **원터치 액션** | 멤버 카드 탭 1회로 위치 확인 / 메시지 전송 / SOS 지원 중 적합한 액션을 즉시 수행할 수 있다. 3단계 이상의 탐색이 필요한 액션은 설계하지 않는다. |
| **M5** | **프라이버시 준수** | 위치 비공유 멤버의 위치 텍스트는 숨긴다. 프라이버시 등급별 비공유 시간대에는 마지막 갱신 시각만 표시하며 실시간 위치는 표시하지 않는다. |

## 3. 화면 구성

### 3.1 전체 레이아웃

```
┌─────────────────────────────────────┐
│  [경고 배너] SOS 활성 / 오프라인 멤버  │  ← 조건부 표시 (최상단 고정)
├─────────────────────────────────────┤
│  ─── 관리자 섹션 ───                 │
│  [캡틴 카드]                          │
│  [크루장 카드 1]  [크루장 카드 2] …   │
├─────────────────────────────────────┤
│  ─── 멤버 섹션 ───                   │
│  [크루 카드 1]                        │
│  [크루 카드 2] …                      │
├─────────────────────────────────────┤
│  ─── 보호자 섹션 ───                  │
│  무료 가디언 (2/2 사용)               │
│  [가디언 카드 🆓]  [가디언 카드 🆓]   │
│  유료 가디언 (1/3 사용) — 💎          │
│  [가디언 카드 💎]                     │
│  [+ 멤버 초대] [+ 가디언 추가]        │  ← 캡틴/크루장만 표시
└─────────────────────────────────────┘
```

### 3.2 섹션 정의

| 섹션 | 포함 역할 | 표시 조건 |
|------|----------|----------|
| **관리자 섹션** | 캡틴, 크루장 | 항상 표시 |
| **멤버 섹션** | 크루 | 항상 표시 |
| **보호자 섹션** | 가디언 (무료/유료 구분) | 가디언이 1명 이상일 때 표시 |
| **경고 배너** | — | SOS 활성 OR 오프라인 멤버 존재 시 상단 고정 |

### 3.3 경고 배너 상세

```
─ SOS 활성 배너 (빨간 배경, 최우선) ─────────────
🚨 [홍길동]님이 SOS를 발신 중입니다.
   📍 강남구 역삼동 · 3분 전   [지도에서 보기]
──────────────────────────────────────────────

─ 오프라인 멤버 배너 (주황 배경) ──────────────────
⚠️  [이순신]님이 오프라인입니다. (32분 전 마지막 접속)
──────────────────────────────────────────────
```

- SOS 배너와 오프라인 배너가 동시에 존재하면 SOS 배너가 위에 표시된다.
- 오프라인 멤버가 2명 이상이면 "N명이 오프라인입니다" 요약 표시 후 탭 시 목록 펼침.

## 4. 멤버 카드 디자인

### 4.1 카드 레이아웃

```
┌───────────────────────────────────────────┐
│ [●] 프로필사진  역할배지  이름  (b2b역할명) │
│  40dp×40dp    [👑 캡틴]  홍길동           │
│                                            │
│  📍 강남구 역삼동 · 3분 전   🔋 85%        │
│     (위치 텍스트)            (배터리)       │
└───────────────────────────────────────────┘
```

### 4.2 구성 요소 명세

| 요소 | 규격 / 규칙 |
|------|------------|
| **프로필 사진** | 40×40dp, 원형 클리핑. 사진 없을 시 이니셜 아바타 표시. |
| **상태 인디케이터 (점)** | 사진 우하단에 8dp 원형 점. 온라인: 녹색 `#4CAF50`, 오프라인: 회색 `#9E9E9E`, SOS: 빨간 펄싱 애니메이션 `#F44336` |
| **역할 배지** | 캡틴: 👑 금색, 크루장: 🔷 파란색, 크루: ⚪ 회색, 가디언: 🛡️ 녹색 |
| **이름** | 실명 표시. B2B 여행에서는 `(커스텀 역할명)` 병기 |
| **위치 텍스트** | `"[행정동] · [N분/시간] 전"` 형식. 프라이버시 비공유 시 숨김. |
| **배터리 인디케이터** | 배터리 잔량 % 텍스트 + 아이콘. 20% 이하 시 빨간색 표시. |

### 4.3 SOS 활성 카드 처리

SOS 발신 중인 멤버의 카드는 다음과 같이 강조 처리한다:

```
┌─ 빨간 테두리 + 배경 (빨간 오버레이) ─────────────┐
│ [🔴펄싱] 프로필사진   🚨 SOS 발신 중              │
│                       홍길동                      │
│  📍 강남구 역삼동 · 방금                           │
│  [지도에서 보기]  [메시지]  [119 안내]             │
└───────────────────────────────────────────────────┘
```

## 5. 가디언 무료/유료 배지 및 과금 관리 UI

### 5.1 가디언 카드 배지 표시 (§03.1, §09.3)

| 가디언 유형 | 배지 | 레이블 |
|------------|------|--------|
| 무료 가디언 (1~2번째) | 🆓 | "무료 가디언" |
| 유료 가디언 (3~5번째) | 💎 | "프리미엄 가디언" |

**슬롯 상태 표시 (보호자 섹션 헤더)**

```
보호자 섹션
  무료: 2/2 사용 (슬롯 가득 참 → 회색)
  유료: 1/3 사용 (추가 가능 → 파란색)
```

### 5.2 가디언 관리 하프시트 (캡틴 전용)

캡틴이 가디언 카드 또는 "가디언 관리" 버튼을 탭하면 하프시트가 열린다.

```
══════════════════════════════════════
    가디언 관리
══════════════════════════════════════

  무료 슬롯 (2/2 사용)
  ┌─────────────────────────────────┐
  │ [사진] 홍길동   🆓             [해제] │
  │ [사진] 이순신   🆓             [해제] │
  └─────────────────────────────────┘

  유료 슬롯 (1/3 사용) — 1,900원/여행
  ┌─────────────────────────────────┐
  │ [사진] 강감찬   💎  결제완료   [해제] │
  │ [+ 유료 가디언 추가] → 결제 모달  │
  │ [+ 유료 가디언 추가] → 결제 모달  │
  └─────────────────────────────────┘

  ※ 가디언 해제 시 이미 결제된 요금은 환불되지 않습니다.
══════════════════════════════════════
```

### 5.3 가디언 추가 과금 결제 모달 (§09.3)

무료 슬롯 초과 시 자동 표시:

```
─────────────────────────────────────
  💎 프리미엄 가디언 추가
─────────────────────────────────────
  이 여행에 가디언을 추가하려면
  1,900원이 결제됩니다.

  · 여행 단위 과금 (이 여행에만 적용)
  · 가디언 해제 후에도 환불 불가

  [취소]    [1,900원 결제 후 추가]
─────────────────────────────────────
```

### 5.4 해제 확인 다이얼로그

유료 가디언 해제 시:

```
"[강감찬]님과의 가디언 연결을 해제하시겠습니까?
이미 결제된 1,900원은 환불되지 않습니다."
[취소]  [해제]
```

미성년자 크루의 가디언 해제 요청 시 (캡틴에게 승인 요청 알림 발송):

```
"미성년자 멤버의 가디언 해제는 캡틴 승인이 필요합니다.
캡틴에게 해제 요청을 보내시겠습니까?"
[취소]  [요청 전송]
```

## 6. B2B 역할 표시

### 6.1 B2B 여행에서의 역할명 표시 (§01.4)

B2B 여행(`tb_trip.is_b2b = TRUE`)에서 멤버 카드에 커스텀 역할명을 병기한다.

| B2B 고객 유형 | 캡틴 표시 | 크루장 표시 | 크루 표시 | 가디언 표시 |
|--------------|----------|------------|----------|------------|
| 학교 (수학여행) | 👑 인솔교사 | 🔷 부담임교사 | ⚪ 학생 | 🛡️ 학부모 |
| 여행사 (패키지) | 👑 가이드 | 🔷 보조가이드 | ⚪ 참가자 | 🛡️ 참가자가족 |
| 기업 (출장) | 👑 팀장 | 🔷 부팀장 | ⚪ 팀원 | 🛡️ 안전담당자 |

### 6.2 커스텀 역할명 표시 규칙

- `tb_group_member.b2b_role_name`이 NULL이 아닌 경우, 역할 배지 옆에 커스텀 역할명을 우선 표시한다.
- B2B 커스텀 역할명이 없으면 기본 역할명("캡틴"/"크루장"/"크루"/"가디언")을 표시한다.
- 역할 배지(아이콘/색상)는 B2B 여부와 무관하게 동일한 시각 규칙을 유지한다.

## 8. 출석 체크 시스템

### 8.1 개요 (§05.5)

출석 체크는 6인 이상 유료 여행에서만 제공한다. 캡틴 또는 크루장이 출석 체크를 시작하면 크루에게 응답 요청 알림이 발송된다.

### 8.2 출석 체크 흐름

```
캡틴/크루장: [출석 체크 시작] 버튼 탭
    ↓
출석 체크 세션 생성 (TB_ATTENDANCE_CHECK)
응답 마감 시간 설정 (기본 10분)
    ↓
전체 크루에게 FCM 알림 발송:
"출석 체크가 시작되었습니다. 10분 내 응답해 주세요."
    ↓
크루: 앱 상단 배너 탭 → "위치 확인" 버튼 탭 → 응답 (TB_ATTENDANCE_RESPONSE)
    ↓
10분 경과 시:
  · 미응답 크루 → 자동 'absent' 처리
  · 캡틴에게 결과 요약 알림 발송
```

### 8.3 출석 현황 실시간 표시 (캡틴/크루장 전용)

출석 체크 진행 중 멤버탭 상단에 실시간 카운트 배너가 표시된다:

```
┌──────────────────────────────────────────┐
│  출석 체크 진행 중  |  남은 시간: 7:23   │
│  ✅ 확인 5명   ⏳ 미응답 3명   ❌ 부재 1명 │
└──────────────────────────────────────────┘
```

### 8.4 출석 응답 유형

| 응답 유형 (`response_type`) | 표시 | 조건 |
|:---------------------------:|------|------|
| `present` | ✅ 확인 | 크루가 직접 응답 |
| `absent` | ❌ 미확인 | 캡틴/크루장이 수동 처리 |
| `unknown` | ⏳ 미응답 | 마감 시간 이전 응답 없음 |

마감 경과 후 `unknown` 상태는 자동으로 `absent`로 전환된다.

## 9. 가디언 전용 상태 탭

가디언은 그룹 멤버탭에 접근할 수 없다. 대신 별도의 **가디언 상태 탭**이 제공된다.

### 9.1 가디언 상태 탭 구성

```
┌─────────────────────────────────────────┐
│  내 연결 멤버 — [여행명]                  │
├─────────────────────────────────────────┤
│  [사진] 홍길동 (크루)                    │
│  📍 강남구 역삼동 · 5분 전               │
│  🔋 72%   ⚪ 온라인                      │
│  [메시지]   [긴급 위치 요청]             │
├─────────────────────────────────────────┤
│  연결 멤버 일정 요약 (유료 가디언만 표시) │
│  ─────────────────────────────────────  │
│  오늘 15:00  광화문 광장 방문             │
│  오늘 18:00  인사동 저녁 식사             │
└─────────────────────────────────────────┘
```

### 9.2 긴급 위치 요청 (§03.4)

- 버튼 탭 시 연결 멤버에게 FCM 높은 우선순위 알림 발송.
- 이벤트 타입: `GUARDIAN_LOCATION_REQUEST`
- 요청 빈도 제한: 1시간 최대 3회. 초과 시 버튼 비활성화 + 남은 시간 카운트다운 표시.
- 프라이버시 등급별 동작:
  - `safety_first`: 이미 항상 공유 중 → 자동 응답 (안부 확인 용도)
  - `standard`: 멤버에게 알림 → 1회성 위치 자동 전달
  - `privacy_first`: 멤버에게 알림 → 멤버 승인 후 전달

### 9.3 유료/무료 가디언 기능 차이

| 기능 | 무료 가디언 | 유료 가디언 |
|------|:-----------:|:-----------:|
| 연결 멤버 위치 확인 | ✅ (등급별 제한) | ✅ (등급별 제한) |
| 연결 멤버 배터리 확인 | ✅ | ✅ |
| 연결 멤버 SOS 알림 수신 | ✅ | ✅ |
| 멤버 메시지 발송 | ✅ (텍스트만) | ✅ (텍스트 + 위치 카드) |
| 연결 멤버 일정 요약 | ❌ | ✅ |
| 긴급 위치 요청 | ✅ (1시간 3회) | ✅ (1시간 3회) |

## 11. 프라이버시 등급별 동작 차이

### 11.1 위치 텍스트 표시 기준

| 프라이버시 등급 | 스케줄 ON 시간대 | 스케줄 OFF 시간대 |
|--------------|:--------------:|:---------------:|
| `safety_first` (안전 최우선) | 실시간 위치 표시 | 실시간 위치 표시 (항상) |
| `standard` (표준) | 실시간 위치 표시 | "마지막 갱신: N분 전" (30분 스냅샷) |
| `privacy_first` (프라이버시 우선) | 실시간 위치 표시 | 위치 텍스트 숨김 ("위치 비공유 중") |

### 11.2 가디언 상태 탭에서의 위치 표시

| 프라이버시 등급 | 가디언 위치 수신 |
|--------------|:--------------:|
| `safety_first` | 항상 실시간 위치 표시 |
| `standard` + ON 시간대 | 실시간 위치 표시 |
| `standard` + OFF 시간대 | "마지막 갱신: N분 전" (30분 스냅샷) |
| `privacy_first` + ON 시간대 | 실시간 위치 표시 |
| `privacy_first` + OFF 시간대 | "비공유 시간입니다" + 마지막 갱신 시각 |

### 11.3 SOS 시 프라이버시 무시

SOS 활성 멤버의 위치는 프라이버시 등급·스케줄 설정과 무관하게 모든 멤버와 연결 가디언에게 실시간으로 표시된다. (비즈니스 원칙 §05.1)

### 11.4 가디언 일시 중지 UI 표시

멤버가 가디언 공유를 일시 중지한 경우, 가디언 카드에 "일시 중지 중" 배지가 표시된다.

| 등급 | 일시 중지 가능 여부 | 최대 시간 |
|------|:-----------------:|:--------:|
| `safety_first` | ❌ 불가 | — |
| `standard` | ✅ | 12시간 |
| `privacy_first` | ✅ | 24시간 |

## 13. DB 스키마

### 13.1 핵심 테이블

```sql
-- 그룹 멤버 (역할, B2B 커스텀 역할명 포함)
-- tb_group_member
CREATE TABLE tb_group_member (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    group_id        UUID NOT NULL REFERENCES tb_group(id),
    trip_id         UUID NOT NULL REFERENCES tb_trip(id),   -- 절대 NULL 불가 (§08.1)
    user_id         UUID NOT NULL REFERENCES tb_user(id),
    member_role     VARCHAR(20) NOT NULL                    -- 'captain'|'crew_chief'|'crew'
                    CHECK (member_role IN ('captain','crew_chief','crew')),
    b2b_role_name   VARCHAR(30),                            -- B2B 커스텀 역할명 (§01.4)
    status          VARCHAR(20) NOT NULL DEFAULT 'active'
                    CHECK (status IN ('active','left','kicked')),
    is_admin        BOOLEAN NOT NULL DEFAULT FALSE,
    invited_by      UUID REFERENCES tb_user(id),
    joined_at       TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    left_at         TIMESTAMPTZ,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (group_id, user_id)
);

-- 캡틴 유일성 보장 (부분 유니크 인덱스)
CREATE UNIQUE INDEX idx_group_member_captain
    ON tb_group_member (group_id)
    WHERE member_role = 'captain' AND status = 'active';

-- 가디언 링크 (무료/유료 구분 포함)
-- tb_guardian_link
CREATE TABLE tb_guardian_link (
    id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    trip_id           UUID NOT NULL REFERENCES tb_trip(id),
    member_user_id    UUID NOT NULL REFERENCES tb_user(id),   -- 연결된 멤버
    guardian_user_id  UUID REFERENCES tb_user(id),            -- 가입된 가디언 (NULL = 미가입)
    guardian_phone    VARCHAR(20),                             -- 미가입 가디언 초대용
    link_type         VARCHAR(20) NOT NULL DEFAULT 'personal'
                      CHECK (link_type IN ('personal','group')),
    status            VARCHAR(20) NOT NULL DEFAULT 'pending'
                      CHECK (status IN ('pending','accepted','rejected')),
    is_paid           BOOLEAN NOT NULL DEFAULT FALSE,          -- ★ v5.1 추가 (§09.3)
    payment_id        UUID REFERENCES tb_payment(id),          -- 유료 시 결제 레코드
    paused_until      TIMESTAMPTZ,                             -- 일시 중지 만료 시각
    created_at        TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at        TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 가입된 가디언 중복 방지 (guardian_user_id NOT NULL인 경우)
CREATE UNIQUE INDEX idx_guardian_link_unique_member
    ON tb_guardian_link (trip_id, member_user_id, guardian_user_id)
    WHERE guardian_user_id IS NOT NULL;

-- 미가입 초대 중복 방지 (guardian_user_id NULL인 경우)
CREATE UNIQUE INDEX idx_guardian_link_unique_phone
    ON tb_guardian_link (trip_id, member_user_id, guardian_phone)
    WHERE guardian_user_id IS NULL;

-- 출석 체크 세션
-- tb_attendance_check
CREATE TABLE tb_attendance_check (
    id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    trip_id      UUID NOT NULL REFERENCES tb_trip(id),
    group_id     UUID NOT NULL REFERENCES tb_group(id),
    started_by   UUID NOT NULL REFERENCES tb_user(id),      -- 캡틴 또는 크루장
    deadline_at  TIMESTAMPTZ NOT NULL,                       -- 응답 마감 시각
    status       VARCHAR(20) NOT NULL DEFAULT 'active'
                 CHECK (status IN ('active','completed')),
    created_at   TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at   TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 출석 응답
-- tb_attendance_response
CREATE TABLE tb_attendance_response (
    id             UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    check_id       UUID NOT NULL REFERENCES tb_attendance_check(id),
    member_user_id UUID NOT NULL REFERENCES tb_user(id),
    response_type  VARCHAR(20) NOT NULL DEFAULT 'unknown'
                   CHECK (response_type IN ('present','absent','unknown')),
    responded_at   TIMESTAMPTZ,
    created_at     TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at     TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (check_id, member_user_id)
);
```

### 13.2 가디언 슬롯 집계 쿼리 (멤버당 무료/유료 카운트)

```sql
-- 특정 멤버의 가디언 슬롯 현황 조회
SELECT
    COUNT(*) FILTER (WHERE is_paid = FALSE AND status = 'accepted') AS free_count,
    COUNT(*) FILTER (WHERE is_paid = TRUE  AND status = 'accepted') AS paid_count
FROM tb_guardian_link
WHERE trip_id = $1
  AND member_user_id = $2
  AND status != 'rejected';
```

## 14. 오프라인 대응

### 14.1 오프라인 시 멤버탭 동작

| 데이터 | 오프라인 가용성 | 비고 |
|--------|:--------------:|------|
| 멤버 목록 (역할, 이름) | ✅ | 마지막 동기화 데이터 캐시 표시 |
| 멤버 상태 (온/오프라인) | ⚠️ | "오프라인 모드 — N분 전 기준" 배너 표시 |
| 위치 텍스트 | ⚠️ | "마지막 갱신: N분 전"으로 표시 |
| 배터리 정보 | ⚠️ | 캐시된 마지막 값 표시 (오래됨 표시) |
| 가디언 목록 | ✅ | 캐시 표시 |
| 출석 체크 응답 | ⚠️ | 로컬 큐잉 → 온라인 복귀 시 자동 전송 (마감 시간 이내인 경우) |

### 14.2 오프라인 배너

멤버탭 진입 시 오프라인 상태이면 상단에 표시:

```
[오프라인 모드] 마지막 동기화: 5분 전. 연결 후 자동 갱신됩니다.
```

### 14.3 온라인 복귀 시 동기화

1. SOS 이벤트 동기화 (최우선)
2. 가디언 긴급 알림/요청 동기화
3. 멤버 위치/상태 갱신
4. 출석 체크 응답 큐 전송
