## 1. 문서 목적 및 적용 범위

### 1.1 목적

본 문서는 SafeTrip 앱의 **알림 시스템 전체**를 정의한다. SOS 긴급 알림부터 일반 정보 알림까지 5계층 우선순위 체계, FCM 채널 구조, 역할별 수신 필터링, 딥링크 매핑, 배지 카운트 규칙을 포함한다.

알림버튼은 SafeTrip의 안전 기능과 직결되며, 특히 **Layer 5 SOS 알림은 어떤 상황에서도 수신자에게 전달**되어야 한다.

### 1.2 적용 범위

| 범위 | 포함 여부 |
|------|:--------:|
| FCM 푸시 알림 전체 (Android/iOS) | ✅ |
| 인앱 알림 (앱 내 알림 센터) | ✅ |
| 알림 우선순위 5계층 체계 | ✅ |
| 역할별 알림 수신 필터링 | ✅ |
| 배지 카운트 및 읽음 처리 | ✅ |
| 딥링크 매핑 | ✅ |
| 가디언 과금 알림 (§09.3) | ✅ |
| 여행 기간 D-1/D-day 알림 (§02.3) | ✅ |
| SMS 폴백 (Layer 5 전용) | ✅ |
| 이메일 알림 | ❌ (별도 정책) |

### 1.3 관련 비즈니스 원칙 조항

| 조항 | 내용 |
|------|------|
| §02.3 | 여행 기간 최대 15일 제한 → D-1/D-day 알림 발송 기준 |
| §03.1 | 가디언 무료 2명 / 유료 3명 → 알림 수신 권한 차등 |
| §05 | 알림 및 SOS 원칙 → SOS 알림 최상위 보장 |
| §09.3 | 가디언 추가 과금 (3번째부터 1,900원/여행) → 과금 알림 트리거 |

## 2. 알림버튼 설계 철학 6원칙

| ID | 원칙명 | 내용 |
|----|--------|------|
| **N1** | 안전 최우선 전달 | SOS 알림은 DND(방해금지) 설정, 무음 모드, 배터리 절약 모드 등 모든 기기 설정을 무시하고 반드시 수신자에게 전달된다. |
| **N2** | 5계층 우선순위 | 알림은 Layer 1(정보)~Layer 5(긴급) 5계층으로 분류하며, 계층별 FCM 채널, 사운드, 진동, 화면 표시 방식을 명확히 차등화한다. |
| **N3** | 역할 기반 필터링 | 역할별로 필요한 알림만 수신한다. 불필요한 알림 과부하를 방지하여 실제 중요 알림의 주의를 높인다. |
| **N4** | 인앱 알림 불멸 | 사용자가 알림을 삭제하거나 읽음 처리해도 서버 이력(tb_notification)은 30일간 보존된다. |
| **N5** | 원터치 네비게이션 | 알림 탭 → 관련 화면으로 즉시 이동. 사용자가 알림을 받은 후 관련 화면을 직접 찾는 수고를 없앤다. |
| **N6** | 투명한 알림 이력 | 모든 알림 히스토리는 30일간 보존되며, 사용자는 앱 내 알림 센터에서 전체 이력을 확인할 수 있다. |

### 3.2 Layer 5: CRITICAL (SOS 긴급)

| 항목 | 내용 |
|------|------|
| **알림 유형** | SOS 발신/수신, Watchdog 자동 SOS |
| **FCM 우선순위** | `PRIORITY_HIGH` (Android) / `critical-alert` (iOS) |
| **DND 무시** | ✅ 강제 통과 |
| **사운드** | 커스텀 `sos_alert` 사운드 (기본 사운드와 명확히 구별) |
| **진동** | 반복 강진동 (패턴: 500ms on, 200ms off, 3회 반복) |
| **화면 표시** | 풀스크린 알림 (잠금화면 포함) |
| **배지** | 빨간 배지, 앱 종료 후에도 유지 |
| **SMS 폴백** | ✅ FCM 미수신 시 SMS로 자동 재전송 |
| **해제 방법** | SOS 발신자 본인 또는 캡틴만 해제 가능 |
| **발신 이벤트 예시** | SOS 버튼 3초 장누르기, Watchdog 자동 SOS, 지오펜스 이탈+무응답 |

### 3.3 Layer 4: HIGH (긴급 안전)

| 항목 | 내용 |
|------|------|
| **알림 유형** | 가디언 긴급 위치 요청, 여행경보 3~4단계 변동 |
| **FCM 우선순위** | `PRIORITY_HIGH` |
| **DND 무시** | ❌ (기본 DND 적용, 단 Layer 4 전용 DND 예외 설정 권장) |
| **사운드** | 기본 알림음 (볼륨 높음) |
| **진동** | 강한 진동 (패턴: 300ms on, 100ms off, 2회) |
| **화면 표시** | 잠금화면 알림 표시 |
| **배지** | 파란 배지 |
| **발신 이벤트 예시** | 가디언의 멤버 위치 긴급 요청, 외교부 여행경보 3단계 이상 변동 |

### 3.4 Layer 3: NORMAL (안전 모니터링)

| 항목 | 내용 |
|------|------|
| **알림 유형** | 지오펜스 이탈, 출석 체크 요청, Heartbeat 미수신 경고 |
| **FCM 우선순위** | `PRIORITY_NORMAL` |
| **DND 무시** | ❌ |
| **사운드** | 기본 알림음 |
| **진동** | 기본 진동 (1회) |
| **화면 표시** | 상단 배너 (앱 사용 중) / 알림 바 (배경) |
| **배지** | 파란 배지 (카운트 증가) |

### 3.5 Layer 2: LOW (일반 여행 이벤트)

| 항목 | 내용 |
|------|------|
| **알림 유형** | 일정 변경, 멤버 합류/탈퇴, 채팅 메시지 |
| **FCM 우선순위** | `PRIORITY_LOW` |
| **DND 무시** | ❌ |
| **사운드** | 조용한 알림음 |
| **진동** | ❌ (소리만) |
| **화면 표시** | 알림 바만 (배너 없음) |
| **배지** | 파란 배지 (카운트 증가) |

### 3.6 Layer 1: INFO (정보)

| 항목 | 내용 |
|------|------|
| **알림 유형** | 설정 변경 확인, 가디언 과금 알림, 서비스 공지, 여행 자동 완료 처리 |
| **FCM 우선순위** | `PRIORITY_LOW` (silent) 또는 미사용 |
| **DND 무시** | ❌ |
| **사운드** | ❌ |
| **진동** | ❌ |
| **화면 표시** | 인앱 알림 센터에만 표시 (OS 알림 바 미표시) |
| **배지** | 배지 카운트 미증가 |

### 3.7 비즈니스 원칙 연동 알림

#### 3.7.1 가디언 과금 관련 알림 (§09.3)

| 이벤트 | 알림 Layer | 수신 대상 | 알림 내용 |
|--------|-----------|----------|----------|
| 유료 가디언(3번째~) 결제 완료 | Layer 1 | 캡틴 + 해당 가디언 | "[가디언명]님이 유료 가디언으로 추가되었습니다. 결제금액: 1,900원" |
| 유료 가디언 만료 임박 (D-3) | Layer 2 | 캡틴 | "유료 가디언 [가디언명]님의 여행 연결이 3일 후 만료됩니다." |
| 무료 슬롯 부족 (2명 초과 시도) | Layer 1 | 캡틴 | "가디언을 더 추가하려면 유료 결제가 필요합니다. (1,900원/여행)" |
| 결제 실패 | Layer 2 | 캡틴 | "가디언 추가 결제에 실패했습니다. 결제 수단을 확인해 주세요." |

> **과금 알림 원칙**: 가디언 과금은 §09.3 기준 3번째 가디언부터 1,900원/여행이 부과된다. 캡틴에게는 결제 전 반드시 과금 안내 알림이 선행되어야 하며, 결제 확인 없이 자동 결제되지 않는다.

#### 3.7.2 여행 기간 관련 알림 (§02.3)

여행 기간 최대 15일 제한에 따라 아래 알림이 자동 발송된다.

| 이벤트 | 알림 Layer | 수신 대상 | 발송 시각 |
|--------|-----------|----------|----------|
| D-1 (여행 종료 1일 전) | Layer 2 | 캡틴 | 여행 종료일 전날 오전 9시 |
| D-day (여행 종료 당일) | Layer 2 | 전체 여행 멤버 | 여행 종료일 오전 9시 |
| 자동 종료 처리 완료 | Layer 1 | 전체 여행 멤버 | 종료일 자정 처리 후 즉시 |

| 이벤트 | 알림 내용 예시 |
|--------|-------------|
| D-1 캡틴 | "내일 여행 '[여행명]'이 종료됩니다. 일정을 확인해 주세요." |
| D-day 전체 | "오늘 여행 '[여행명]'이 종료됩니다. 안전하게 귀국하세요." |
| 자동 종료 전체 | "여행 '[여행명]'이 완료 처리되었습니다. 이용해 주셔서 감사합니다." |

## 4. FCM 채널 구조

### 4.1 Android 채널 정의

| 채널 ID | 중요도 | DND 무시 | 사운드 | 진동 | 대상 Layer |
|---------|--------|---------|--------|------|-----------|
| `safetrip_critical` | IMPORTANCE_HIGH (5) | ✅ | sos_alert | 강한 반복 | Layer 5 |
| `safetrip_high` | IMPORTANCE_HIGH (4) | ❌ | 기본 알림음 | 강 | Layer 4 |
| `safetrip_normal` | IMPORTANCE_DEFAULT (3) | ❌ | 기본 알림음 | 기본 | Layer 3 |
| `safetrip_low` | IMPORTANCE_LOW (2) | ❌ | 조용한 알림음 | 없음 | Layer 2 |

Layer 1 INFO 알림은 FCM silent push 또는 인앱 폴링으로 처리하며 별도 채널 없음.

### 4.2 iOS APNs 설정

| Layer | priority | interruption-level | sound |
|-------|---------|-------------------|-------|
| Layer 5 | 10 | `critical` | `sos_alert.caf` |
| Layer 4 | 10 | `time-sensitive` | 기본 |
| Layer 3 | 5 | `active` | 기본 |
| Layer 2 | 5 | `passive` | 조용한 알림음 |

### 4.3 배지 카운트 규칙

| 상황 | 배지 유형 | 표시 |
|------|----------|------|
| Layer 3~4 미읽음 | 파란 배지 | 숫자 (1~99) |
| Layer 5 미처리 SOS | 빨간 배지 | 숫자 (항상 최우선 표시) |
| 99개 초과 | 배지 | "99+" |
| 전체 읽음 처리 | — | 배지 제거 |
| Layer 1 INFO | — | 배지 미증가 |

**읽음 처리 방식:**
- 개별 읽음: 알림 탭 즉시
- 전체 읽음: 알림 센터 상단 "전체 읽음" 버튼
- Layer 5 SOS: 개별 읽음만 허용 (전체 읽음 대상 제외)

## 6. 프라이버시 등급별 동작 차이

> 비즈니스 원칙 §04 기준 3단계 프라이버시 등급이 알림 수신에 영향을 미친다.
> **단, Layer 5 SOS 알림은 프라이버시 등급 무관 항상 수신된다.**

| 프라이버시 등급 | 위치 관련 알림 수신 | Layer 5 SOS | 비고 |

## 7. 에러 및 엣지케이스 처리

### 7.1 FCM 전달 실패 처리

| 상황 | 처리 방법 |
|------|----------|
| FCM 토큰 만료 | 토큰 갱신 요청 후 재발송 (최대 3회 재시도) |
| FCM 서버 장애 | 지수 백오프(1s→2s→4s) 재시도 후 SMS 폴백 (Layer 4~5만) |
| 디바이스 오프라인 | FCM 메시지 큐 저장 (최대 4주), 온라인 복귀 시 일괄 전달 |
| Layer 5 전달 실패 | 즉시 SMS 재전송 (SOS 원칙 §06 참조) |
| 알림 권한 미허용 | 인앱 알림 센터에만 저장, 알림 권한 요청 모달 표시 |

### 7.2 배지 카운트 엣지케이스

| 상황 | 처리 방법 |
|------|----------|
| 복수 디바이스 로그인 | 한 디바이스에서 읽음 처리 시 모든 디바이스 동기화 |
| 앱 삭제 후 재설치 | 서버 미읽음 이력 기준으로 배지 카운트 복원 |
| Layer 5 SOS 미처리 중 전체 읽음 | SOS 배지는 유지, 일반 배지만 초기화 |
| 알림 30일 만료 | 만료된 알림은 배지 카운트에서 자동 제외 |

### 7.3 딥링크 오류 처리

| 상황 | 처리 방법 |
|------|----------|
| 연결된 여행이 삭제됨 | 홈 화면으로 이동 + "해당 여행이 삭제되었습니다." 토스트 |
| 대상 메시지 삭제됨 | 채팅 탭 상단으로 이동 |
| 권한 없는 화면 딥링크 | 홈 화면으로 이동 + 권한 없음 안내 |

### 7.4 알림 과부하 방지

| 규칙 | 내용 |
|------|------|
| 동일 유형 알림 묶음 처리 | 1분 내 동일 유형 알림 3건 이상 발생 시 → "N건의 알림" 묶음 표시 |
| Heartbeat 경고 중복 방지 | 미처리 Heartbeat 경고가 있는 경우 중복 발송 금지 |
| 채팅 알림 묶음 | 5분 내 동일 채팅방 알림 → 최신 1건으로 묶음 ("외 N건") |
| Layer 5 묶음 금지 | SOS 알림은 묶음 처리 절대 금지 (각 SOS 독립 표시) |

### 7.5 여행 기간 엣지케이스 (§02.3)

| 상황 | 처리 방법 |
|------|----------|
| 여행 기간 15일 초과 불가 | 서버 단에서 차단 → 알림 탭에 별도 처리 없음 |
| 여행 종료 후 SOS 수신 | 여행 종료 직전 미처리 SOS는 72시간 유예 후 아카이브 |
| D-1 알림 발송 실패 | 재시도 1회 후 미발송 시 서버 로그 기록 (Layer 2이므로 SMS 폴백 없음) |

## 8. DB 스키마

### 8.1 tb_notification

알림 이력 저장 테이블 (30일 보존)

```sql
CREATE TABLE tb_notification (
    id                  BIGSERIAL PRIMARY KEY,
    user_id             BIGINT       NOT NULL REFERENCES tb_user(id),
    trip_id             BIGINT       REFERENCES tb_trip(id),     -- NULL 가능 (서비스 공지 등)
    notification_type   VARCHAR(50)  NOT NULL,                   -- 알림 유형 코드
    priority_layer      INTEGER      NOT NULL CHECK (priority_layer BETWEEN 1 AND 5),
    title               VARCHAR(100) NOT NULL,
    body                TEXT         NOT NULL,
    deep_link           VARCHAR(200),                            -- 딥링크 경로
    fcm_channel         VARCHAR(30),                             -- FCM 채널 ID
    is_read             BOOLEAN      NOT NULL DEFAULT FALSE,
    sent_at             TIMESTAMPTZ  NOT NULL DEFAULT now(),
    read_at             TIMESTAMPTZ,                             -- NULL이면 미읽음
    expires_at          TIMESTAMPTZ  NOT NULL                    -- sent_at + 30일
                        GENERATED ALWAYS AS (sent_at + INTERVAL '30 days') STORED,
    metadata            JSONB,                                   -- 알림 유형별 추가 데이터
    created_at          TIMESTAMPTZ  NOT NULL DEFAULT now()
);

CREATE INDEX idx_notification_user_id     ON tb_notification (user_id, sent_at DESC);
CREATE INDEX idx_notification_trip_id     ON tb_notification (trip_id);
CREATE INDEX idx_notification_layer       ON tb_notification (priority_layer);
CREATE INDEX idx_notification_unread      ON tb_notification (user_id, is_read) WHERE is_read = FALSE;
CREATE INDEX idx_notification_expires     ON tb_notification (expires_at);
```

**notification_type 코드 목록 (주요):**

| 코드 | 설명 | Layer |
|------|------|-------|
| `SOS_SENT` | SOS 발신 알림 | 5 |
| `SOS_RECEIVED` | SOS 수신 알림 | 5 |
| `SOS_AUTO` | Watchdog 자동 SOS | 5 |
| `GUARDIAN_LOCATION_REQUEST` | 가디언 긴급 위치 요청 | 4 |

| `GEOFENCE_EXIT` | 지오펜스 이탈 | 3 |
| `ATTENDANCE_CHECK` | 출석 체크 요청 | 3 |
| `HEARTBEAT_MISSED` | Heartbeat 미수신 경고 | 3 |
| `SCHEDULE_CHANGED` | 일정 변경 | 2 |
| `MEMBER_JOINED` | 멤버 합류 | 2 |
| `MEMBER_LEFT` | 멤버 탈퇴 | 2 |
| `CHAT_MESSAGE` | 채팅 메시지 | 2 |

## 9. 오프라인 대응

### 9.1 오프라인 알림 처리 원칙

| Layer | 오프라인 시 처리 | 온라인 복귀 시 처리 |
|-------|----------------|-------------------|
| Layer 5 (SOS) | SMS 자동 발송 (SOS 원칙 §06) + 로컬 알림 표시 | FCM 재발송 불필요 (SMS로 전달됨) |
| Layer 4~2 | FCM 큐에 저장 (최대 4주) | 온라인 복귀 시 FCM 일괄 전달 |
| Layer 1 | 서버 저장 | 앱 실행 시 인앱 알림 센터에서 조회 |

### 9.2 온라인 복귀 시 동기화

```
온라인 복귀 감지
    ↓
서버에서 미수신 알림 조회 (최대 100건)
    ↓
Layer 기준 내림차순 정렬 (5→1)
    ↓
Layer 5: 즉시 표시 (풀스크린)
Layer 4~3: 묶음 알림으로 표시
Layer 2~1: 인앱 알림 센터에만 추가
    ↓
배지 카운트 서버 기준으로 동기화
```

### 9.3 오프라인 로컬 알림

| 유형 | 처리 방법 |
|------|----------|
| 출석 체크 알림 | 기기 내 스케줄러로 로컬 알림 예약 (서버 동기화 기준 시간) |
| 여행 기간 D-day 알림 | 앱 설치 시 30일치 로컬 알림 예약 |
| SOS Watchdog | 백그라운드 서비스로 오프라인 상태에서도 Heartbeat 감시 (배터리 허용 범위) |
