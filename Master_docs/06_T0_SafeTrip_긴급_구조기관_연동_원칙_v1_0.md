## 1. 설계 철학 및 핵심 원칙

### 1.1 핵심 가치

> **"SOS는 앱 내 알림으로 끝나지 않는다. 실제 구조기관 연결까지가 SafeTrip의 책임 범위이다."**

SafeTrip SOS 원칙 v1.0은 앱 내부의 긴급 알림 시스템(그룹 멤버·가디언 알림)을 정의하지만, 실제 긴급 상황에서는 공식 구조기관에 연락하는 것이 최우선이다. 본 문서는 SOS 발동 이후 "마지막 1마일"—실제 구조기관 전화 연결, 위치 정보 전달, 대사관·영사콜센터 연계—을 정의한다.

### 1.2 설계 원칙

| 원칙 | 설명 |
|------|------|
| **공식 구조기관 최우선** | SafeTrip SOS는 보조 수단이며, 공식 긴급 전화(119/112/911 등)를 대체하지 않는다. SOS 발동 시 공식 구조기관 연결을 항상 최우선으로 안내한다 |
| **원터치 연결** | 긴급 상황에서 최소 탭 수(최대 2탭)로 현지 긴급 전화에 연결되어야 한다 |
| **컨텍스트 자동 판별** | 현재 위치 기반으로 해당 국가의 긴급 전화번호를 자동 선택한다. 유저가 번호를 검색할 필요가 없다 |
| **오프라인 복원력** | 인터넷 연결 없이도 긴급 전화번호 확인 및 발신이 가능해야 한다 (로컬 캐시) |
| **투명한 한계 고지** | SafeTrip이 직접 구조기관과 통신하는 것이 아니라, 유저의 전화 발신을 지원하는 것임을 명확히 고지한다 |
| **이중 트리거 방지** | SOS 발동과 구조기관 연결은 별개 액션이며, SOS 발송이 자동으로 긴급 전화를 발신하지 않는다 |

### 1.3 법적 근거

| 법령 | 조항 | 적용 |
|------|------|------|
| 위치정보법 제29조 | 긴급구조 시 위치정보 제공 | SOS 발동 시 구조기관 요청에 따른 위치 제공 근거 |
| 개인정보보호법 제15조 제1항 제3호 | 급박한 생명·신체 위험 방지 | SOS 시 위치·개인정보 동의 없이 제공 가능 근거 |
| 재외국민보호법 | 재외국민 안전 및 보호 | 영사콜센터·대사관 연계의 법적 기반 |

## 2. 긴급 구조기관 연결 체계 개요

### 2.1 연결 대상 기관 분류

```
긴급 구조기관 체계
─────────────────────────────────────────────────
Layer 1 (최우선)  │ 현지 긴급 전화 (경찰/소방/응급)
                  │ 112, 911, 110, 119 등 국가별 번호
─────────────────────────────────────────────────
Layer 2           │ 한국 영사콜센터 (1335 / +82-2-3210-0404)
                  │ 24시간 운영, 해외 한국인 긴급 지원
─────────────────────────────────────────────────
Layer 3           │ 주재국 한국 대사관·영사관
                  │ 근무시간 한정, 대면 지원 가능
─────────────────────────────────────────────────
Layer 4 (보조)    │ SafeTrip 그룹 내부 알림
                  │ 캡틴·크루장·가디언 알림 (SOS 원칙 관할)
─────────────────────────────────────────────────
```

### 2.2 연결 시나리오 매트릭스

| 시나리오 | 1차 연결 대상 | 2차 연결 대상 | 3차 연결 대상 |
|---------|-------------|-------------|-------------|
| 신체 위험 (폭행, 사고) | 현지 경찰/응급 | 영사콜센터 | 대사관 |
| 의료 응급 (질병, 부상) | 현지 응급/소방 | 가까운 병원 | 영사콜센터 |
| 자연재해 (지진, 태풍) | 현지 긴급 전화 | 영사콜센터 | 대사관 |
| 도난·분실 (여권, 소지품) | 현지 경찰 | 대사관 | 영사콜센터 |
| 납치·강제 억류 | 현지 경찰 | 영사콜센터 | 대사관 |
| 일반 안전 불안 | 영사콜센터 | 캡틴/크루장 | — |
| 통신 두절 (Heartbeat 타임아웃) | 캡틴 확인 | 영사콜센터 | 현지 경찰 |

### 2.3 SafeTrip 역할의 한계

SafeTrip은 **긴급 전화 발신을 지원**하는 앱이지, 구조기관에 **직접 연락하는** 시스템이 아니다.

```
SafeTrip이 하는 것:
  ✅ 현재 국가의 긴급 전화번호 자동 표시
  ✅ 원터치 전화 발신 연결 (전화 앱으로 전달)
  ✅ 유저의 위치·상태 정보를 구조기관 제공용으로 정리
  ✅ 대사관·영사콜센터 연락처 제공
  ✅ 오프라인에서도 긴급 번호 확인 가능

SafeTrip이 하지 않는 것:
  ❌ 구조기관에 자동 전화 발신 (유저 의사 확인 없이)
  ❌ 구조기관과의 실시간 데이터 연동 (API 미존재)
  ❌ 구조 완료 보장
  ❌ 긴급 전화 통화 중계
```

## 3. 국가별 긴급 전화번호 DB 관리

### 3.1 관리 대상 데이터

| 데이터 항목 | 설명 | 소스 | 갱신 주기 |
|-----------|------|------|----------|
| 국가별 종합 긴급 번호 | 112(EU), 911(미국) 등 단일 긴급 번호 | 자체 DB + MOFA API | 분기 |
| 경찰 전화번호 | 국가별 경찰 신고 번호 | 자체 DB | 분기 |
| 소방/응급 전화번호 | 국가별 소방·구급 번호 | 자체 DB | 분기 |
| 한국 대사관·영사관 | 주소, 전화번호, 근무시간, 좌표 | MOFA API | 실시간 API + 일 1회 캐시 |
| 영사콜센터 | 국내 1335 / 해외 +82-2-3210-0404 | 고정 | — |
| 여행자 보험 긴급 전화 | 주요 보험사 해외 긴급 번호 | 자체 DB | 반기 |

### 3.2 주요 국가 긴급 전화번호 샘플

| 국가 | 종합 긴급 | 경찰 | 소방/응급 | 비고 |
|------|:--------:|:----:|:--------:|------|
| 한국 | 112 | 112 | 119 | — |
| 일본 | — | 110 | 119 | 종합 번호 없음 |
| 미국/캐나다 | 911 | 911 | 911 | 단일 번호 |
| EU 대부분 | 112 | 112 | 112 | 유럽 공통 |
| 영국 | 999 | 999 | 999 | 112도 가능 |
| 중국 | — | 110 | 120 (응급) / 119 (소방) | 분리 운영 |
| 태국 | — | 191 | 1669 (응급) / 199 (소방) | 관광 경찰: 1155 |
| 호주 | 000 | 000 | 000 | 단일 번호 |
| 베트남 | 113 | 113 | 115 (응급) / 114 (소방) | — |
| 필리핀 | — | 117 | 911 | 911은 마닐라 중심 |

### 3.3 데이터 갱신 프로세스

```
데이터 소스별 갱신 플로우
─────────────────────────────────────────

[MOFA API] ──(일 1회 자동 동기화)──→ [SafeTrip 서버 캐시]
                                          │
                                          ▼
[관리자 수동 입력] ──(분기 갱신)──→ [TB_EMERGENCY_NUMBER]
                                          │
                                          ▼
                                    [클라이언트 캐시]
                                    (앱 설치 시 + 여행 생성 시 다운로드)
                                          │
                                          ▼
                                    [오프라인 로컬 DB]
                                    (SQLite, 전화번호 전용)
```

### 3.4 데이터 검증 규칙

| 검증 항목 | 규칙 | 실패 시 처리 |
|----------|------|------------|
| 국가 코드 유효성 | ISO 3166-1 alpha-2 코드 필수 | 입력 거부 |
| 전화번호 형식 | E.164 또는 현지 번호 형식 | 경고 + 수동 검토 |
| 대사관 좌표 유효성 | 해당 국가 영토 내 좌표 | 경고 + 수동 검토 |
| 중복 데이터 | 동일 국가·동일 유형 중복 방지 | 업데이트로 처리 |
| 갱신 지연 감지 | 90일 이상 미갱신 국가 | 관리자 알림 + 수동 확인 |

## 4. 자동 다이얼 프로토콜

### 4.1 긴급 전화 발신 플로우

SOS 발동 후 또는 안전가이드 긴급연락 탭에서 긴급 전화를 발신하는 전체 플로우이다.

```
유저: 긴급 전화 버튼 탭
    │
    ▼
현재 위치 국가 자동 판별
    ├── GPS 기반 판별 성공 → 해당 국가 긴급 번호 표시
    ├── GPS 실패 → 여행 목적지 국가 긴급 번호 표시
    └── 둘 다 실패 → 국가 선택 드롭다운 표시
    │
    ▼
긴급 전화 유형 선택 화면
    ┌──────────────────────────────────────┐
    │  🆘 긴급 전화 — {국가명}              │
    │                                      │
    │  🚔 경찰     {경찰번호}   [📞 전화]   │
    │  🚑 응급     {응급번호}   [📞 전화]   │
    │  🚒 소방     {소방번호}   [📞 전화]   │
    │  ──────────────────────────────────  │
    │  🇰🇷 영사콜센터 (24시간)              │
    │  +82-2-3210-0404          [📞 전화]   │
    │  🏛️ 가까운 대사관          [📞 전화]   │
    └──────────────────────────────────────┘
    │
    ▼
[📞 전화] 탭
    │
    ├── 현지 번호 (110, 119 등)
    │   → 즉시 전화 앱으로 전달 (확인 없이)
    │
    └── 국제 번호 (+82-2-3210-0404)
         │
         ▼
        확인 다이얼로그
        "국제 전화 요금이 발생할 수 있습니다.
         발신하시겠습니까?"
        [발신]  [취소]
    │
    ▼
전화 앱 실행 (Intent/URL Scheme)
    ├── Android: Intent.ACTION_CALL + tel:{번호}
    └── iOS: UIApplication.open(tel:{번호})
    │
    ▼
통화 종료 감지 (가능한 경우)
    │
    ▼
SafeTrip 복귀 시 후속 조치 안내 표시
    "📋 통화가 종료되었습니다.
     [위치 정보 복사] — 구조기관에 알려주세요
     [대사관 연락]
     [그룹에 상황 알리기]"
```

### 4.2 현지 번호 vs 국제 번호 처리

| 번호 유형 | 확인 다이얼로그 | 발신 형식 | 비용 안내 |
|----------|:-------------:|----------|:--------:|
| 현지 긴급 번호 (110, 119 등) | ❌ 즉시 발신 | 그대로 발신 | ❌ |
| 현지 일반 번호 (대사관 현지번호) | ❌ 즉시 발신 | 그대로 발신 | ❌ |
| 국제 번호 (+82-xxx) | ✅ 확인 필요 | E.164 형식 | ✅ 요금 경고 |
| 영사콜센터 (국내 1335) | ❌ 즉시 발신 | 해외에서는 +82-2-3210-0404 자동 변환 | ✅ 요금 경고 |

### 4.3 위치 정보 클립보드 복사 기능

긴급 전화 발신 시, 유저가 구조기관에 현재 위치를 구두로 전달할 수 있도록 위치 정보를 클립보드에 자동 복사한다.

```
클립보드 복사 데이터 (현지어 + 한국어):

"[SafeTrip 긴급 위치 정보]
위치: 東京都渋谷区道玄坂2-1 / 도쿄도 시부야구 도겐자카 2-1
GPS: 35.6580, 139.6994
시각: 2026-03-15 14:23 (현지시간)
배터리: 42%
요청자: 홍길동"
```

| 항목 | 포함 여부 | 형식 |
|------|:--------:|------|

## 6. SOS 이후 구조기관 연결 플로우

### 6.1 SOS 유형별 구조기관 연결 분기

SOS 원칙 v1.0에서 정의한 6가지 시나리오에 대해, 각각의 구조기관 연결 방식을 정의한다.

| SOS 시나리오 | 구조기관 연결 방식 | 자동/수동 | 설명 |
|-------------|------------------|:--------:|------|
| A. 정상 SOS | SOS 완료 후 구조기관 안내 화면 표시 | 수동 | 유저가 직접 전화 발신 선택 |
| B. 오프라인 SOS | SMS 폴백 + 로컬 캐시 긴급번호 표시 | 수동 | 오프라인에서도 전화 발신 가능 |
| C. Heartbeat 타임아웃 | 캡틴에게 구조기관 연락 CTA 제공 | 수동 (캡틴) | 캡틴이 상황 판단 후 연락 |
| D. 전원 꺼짐 | 캡틴/가디언에게 구조기관 연락 CTA 제공 | 수동 (캡틴/가디언) | 캡틴/가디언이 대리 연락 |
| E. 납치·강제 전원 차단 | Auto-SOS 알림에 구조기관 연락처 포함 | 수동 (수신자) | 수신자가 대리 연락 |
| F. 전원 복구 | 해당 없음 | — | 상황 해소 |

### 6.2 수신자 측 구조기관 연결 (대리 연락)

SOS를 수신한 캡틴·가디언이 발송자를 대신하여 구조기관에 연락하는 시나리오이다.

```
SOS 수신 알림 (캡틴/가디언)
┌──────────────────────────────────────────────┐
│  🚨 SOS 긴급 알림                              │

│                                              │
│  📍 위치: {주소}                               │
│  🔋 배터리: {N}%                               │
│  ⏰ 발송 시각: {시각}                           │
│                                              │
│  [위치 확인하기]                                │
│                                              │
│  ⚠️ 긴급 상황이 의심되면 구조기관에 연락하세요     │
│  🚔 현지 경찰: {번호}        [📞 전화]          │
│  🚑 현지 응급: {번호}        [📞 전화]          │
│  🇰🇷 영사콜센터              [📞 전화]          │

### 6.3 Auto-SOS 수신 시 구조기관 연결 강화

Auto-SOS(SOS 원칙 §6.4)는 시스템이 자동 발송하는 것으로, 해당 멤버가 직접 연락할 수 없는 상태일 가능성이 높다. 따라서 수신자에게 더 적극적인 구조기관 연결 안내를 제공한다.

```
Auto-SOS 수신 알림 (캡틴/가디언)
┌──────────────────────────────────────────────┐
│  🚨 자동 SOS — 연결 끊김 감지                    │

│                                              │
│  📍 마지막 위치: {주소}                          │
│  ⏰ 마지막 접속: {시각} ({N}분 전)                │
│  🔋 배터리(마지막): {N}%                         │
│                                              │
│  ═══════════════════════════════════════════  │
│  🔴 즉시 확인이 필요합니다                        │
│                                              │
│  [📞 멤버에게 전화 걸기]                         │
│  [📞 현지 경찰: {번호}]                          │
│  [📞 영사콜센터: +82-2-3210-0404]               │
│  [📞 가까운 대사관: {대사관명}]                    │
│                                              │
│  [📋 멤버 마지막 위치 정보 복사]                   │
│                                              │
│  ──────────────────────────────────────────  │
│  💡 대응 가이드                                  │
│  1. 먼저 멤버에게 직접 전화를 시도하세요            │
│  2. 연락이 안 되면 현지 경찰에 신고하세요           │
│  3. 영사콜센터에 상황을 알려주세요                  │
└──────────────────────────────────────────────┘
```

### 7.2 SMS 기반 긴급 연락 대체

SOS 원칙 v1.0 §3.3에서 정의한 SMS 폴백과 연계하여, 구조기관 연락을 지원한다.

```
오프라인 + 긴급 상황
    │
    ├── 전화 가능 (음성 통화)
    │   → 로컬 캐시에서 긴급 번호 조회 → 직접 전화 발신
    │
    ├── 전화 불가 + SMS 가능
    │   → 캡틴에게 SMS 발송 (SOS 원칙 §3.3)
    │   → SMS 내용에 위치 좌표 포함
    │
    └── 전화 + SMS 모두 불가
        → "통신 불가 상태입니다" 안내
        → "가까운 사람에게 도움을 요청하세요" 안내
        → 오프라인 SOS 큐에 저장 (네트워크 복구 시 자동 발송)

## 8. 오발송(가짜 SOS) 처리 절차

### 8.1 오발송 유형 분류

| 유형 | 설명 | 빈도 예상 | 위험도 |
|------|------|:--------:|:-----:|
| 실수 터치 | SOS 버튼 의도치 않게 탭 | 높음 | 🟡 |
| 테스트 목적 | "진짜 작동하나?" 호기심 | 보통 | 🟡 |
| 장난·악의적 | 그룹 혼란 유도 목적 | 낮음 | 🔴 |
| Auto-SOS 오탐 | Heartbeat 타임아웃 오판 | 보통 | 🟠 |

### 8.2 오발송 방지 메커니즘

```
SOS 버튼 탭
    │
    ▼
[방지 1] 확인 다이얼로그 (SOS 원칙 §2.2)
    "SOS를 발송하시겠습니까?"
    "⚠️ 허위 SOS 반복 발송 시 이용이 제한될 수 있습니다."
    [취소]  [SOS 발송 🚨]
    │
    ▼ (발송 완료)
    │
[방지 2] SOS 즉시 취소 기능 (5초 유예)
    "SOS가 발송되었습니다.
     실수라면 5초 내에 취소할 수 있습니다.
     [SOS 취소] ■■■■■ (5초 카운트다운)"
    │
    ├── 5초 내 [취소] 탭 → SOS 취소, 그룹에 "SOS 취소됨" 알림
    │
    └── 5초 경과 → SOS 확정, 취소 불가
```

### 8.3 SOS 취소 후 처리

```
SOS 취소 시:
    │
    ├── 서버: SOS 이벤트 상태 → "cancelled"
    │
    ├── 그룹 멤버: "⚠️ {멤버명}의 SOS가 취소되었습니다 (실수 발송)"
    │
    ├── 가디언: "ℹ️ {멤버명}의 SOS가 취소되었습니다. 실수 발송입니다."
    │
    └── 로그: TB_EVENT_LOG에 cancel 이벤트 기록
         {event_type: "SOS_CANCEL", cancel_reason: "user_cancelled", ...}
```

### 8.4 반복 오발송 제재

| 기간 | 오발송 횟수 | 조치 |
|------|:----------:|------|
| 24시간 내 | 3회 이상 취소 | 경고 메시지 + SOS 발송 시 추가 확인 단계 (슬라이드 투 센드) |
| 7일 내 | 5회 이상 취소 | SOS 발송 시 10초 대기 + 관리자 알림 |
| 30일 내 | 10회 이상 취소 | 계정 경고 + 고객지원 검토 |
| 악의적 판단 | — | 서비스 이용약관 §9.2 금지행위에 따른 이용 제한 |

### 8.5 Auto-SOS 오탐 처리

Auto-SOS 오탐 시(예: 기기 재부팅 후 Heartbeat 재개), 상황 해소 알림을 발송한다.

```
Auto-SOS 오탐 해소:
    │
    ├── 멤버 기기 복귀 (Power Recovery 수신)
    │   → "✅ {멤버명}의 기기가 다시 연결되었습니다. 자동 SOS 해제."
    │
    ├── 캡틴이 "상황 확인 완료" 선택
    │   → "✅ 캡틴이 {멤버명}의 상황을 확인했습니다. 자동 SOS 해제."
    │
    └── 구조기관 연락 이미 진행된 경우
        → "⚠️ 자동 SOS가 해소되었지만, 이미 구조기관에 연락하셨다면
            상황 종료를 알려주세요."
```

## 10. 프라이버시 등급별 동작 차이

### 10.1 핵심 원칙

**긴급 구조기관 연결 기능은 프라이버시 등급의 영향을 받지 않는다.** (SOS 원칙 §11.1과 동일)

| 기능 | 🛡️ 안전 최우선 | 📍 표준 | 🔒 프라이버시 우선 |
|------|:------------:|:------:|:----------------:|
| 긴급 전화 발신 | ✅ 동일 | ✅ 동일 | ✅ 동일 |
| 영사콜센터 연결 | ✅ 동일 | ✅ 동일 | ✅ 동일 |
| 대사관 연결 | ✅ 동일 | ✅ 동일 | ✅ 동일 |
| SOS 시 위치 클립보드 복사 | ✅ | ✅ | ✅ |
| SOS 수신자 구조기관 CTA | ✅ | ✅ | ✅ |
| 오프라인 긴급번호 캐시 | ✅ | ✅ | ✅ |

### 10.2 프라이버시 등급별 차이점

유일하게 차이가 발생하는 부분은 **대리 신고 시 위치 정보 상세도**이다.

| 상황 | 안전 최우선 | 표준 | 프라이버시 우선 |
|------|:---------:|:----:|:-------------:|
| 수신자가 복사하는 멤버 위치 (SOS 발동 시) | 실시간 GPS 좌표 + 주소 | 실시간 GPS 좌표 + 주소 | 실시간 GPS 좌표 + 주소 |
| 수신자가 복사하는 멤버 위치 (Auto-SOS 시) | 마지막 위치 + 이동 경로 요약 | 마지막 위치 | 마지막 위치 |

> **SOS 예외 원칙 적용**: SOS/Auto-SOS 발동 시 모든 등급에서 위치가 강제 포함되므로, 구조기관 연결에서의 실질적 차이는 Auto-SOS 시 이동 경로 포함 여부뿐이다.

## 13. DB 스키마

### 13.1 TB_EMERGENCY_NUMBER (국가별 긴급 전화번호)

```sql
CREATE TABLE tb_emergency_number (
    number_id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    country_code       VARCHAR(2) NOT NULL,          -- ISO 3166-1 alpha-2
    number_type        VARCHAR(20) NOT NULL,          -- general | police | fire | ambulance | coast_guard
    phone_number       VARCHAR(30) NOT NULL,          -- 현지 번호 (110, 119 등)
    phone_number_intl  VARCHAR(30),                   -- 국제 번호 (해당 시)
    display_name_ko    VARCHAR(100) NOT NULL,          -- 한국어 표시명
    display_name_en    VARCHAR(100),                   -- 영어 표시명
    display_name_local VARCHAR(100),                   -- 현지어 표시명
    description        TEXT,                           -- 추가 설명
    is_primary         BOOLEAN DEFAULT FALSE,          -- 해당 유형의 대표 번호 여부
    is_free_call       BOOLEAN DEFAULT TRUE,           -- 무료 통화 여부
    available_24h      BOOLEAN DEFAULT TRUE,           -- 24시간 운영 여부
    notes              TEXT,                           -- 특이사항 (예: "영어 가능", "관광경찰")
    source             VARCHAR(50) DEFAULT 'manual',   -- manual | mofa_api | external
    verified_at        TIMESTAMPTZ,                    -- 마지막 검증 일시
    created_at         TIMESTAMPTZ DEFAULT NOW(),
    updated_at         TIMESTAMPTZ,
    
    UNIQUE(country_code, number_type, phone_number)
);

CREATE INDEX idx_emergency_number_country ON tb_emergency_number(country_code);
CREATE INDEX idx_emergency_number_type ON tb_emergency_number(country_code, number_type);
```

### 15.1 오프라인 가용 기능

| 기능 | 오프라인 가용 | 대응 방식 |
|------|:----------:|----------|

| 역지오코딩 (좌표→주소) | ❌ | GPS 좌표만 표시 |
| 가까운 대사관 자동 선택 | ❌ (제한적) | 캐시된 좌표 기반 거리 계산 (오프라인 가능) |

### 15.2 로컬 캐시 갱신 정책

| 데이터 | 초기 다운로드 시점 | 갱신 트리거 | 최대 캐시 유효기간 |
|--------|:---------------:|-----------|:----------------:|
| 전 세계 긴급 전화번호 | 앱 설치 시 | 앱 업데이트 시 | 무제한 (앱 내장) |
| 여행 목적지 국가 상세 데이터 | 여행 생성 시 | 여행 시작 전날 | 30일 |

| 최근 MOFA 안전 공지 | 여행 시작 시 | 4시간마다 | 24시간 |

### 15.3 캐시 무결성 확인

```
앱 실행 시 캐시 검증
    │
    ├── 긴급 전화번호 DB 존재 확인 → 없으면 내장 DB에서 복원

    └── 갱신 실패 시 → 기존 캐시 사용 + "데이터가 최신이 아닐 수 있습니다" 배지
```

## 16. 에러 및 엣지케이스

### 16.1 전화 발신 실패

| 상황 | 처리 |
|------|------|
| 전화 앱 권한 미부여 | "전화 발신 권한이 필요합니다. 설정에서 허용해주세요" + 설정 이동 CTA |
| SIM 카드 미삽입 | "SIM 카드가 없어 전화가 불가능합니다. Wi-Fi 전화 또는 주변에 도움을 요청하세요" |
| 로밍 미설정 | "해외 로밍이 설정되지 않았습니다. 현지 SIM 또는 Wi-Fi 전화를 이용해주세요" |
| 통화 권한 거부 (iOS) | 번호를 화면에 표시 + "위 번호를 직접 입력해주세요" 안내 |
| 전화 앱 강제 종료 | SafeTrip 복귀 시 "통화가 완료되었나요?" + 후속 조치 안내 |

### 16.2 위치 관련 에러

| 상황 | 처리 |
|------|------|
| GPS 비활성 | SOS 시 "위치 정보 없이 SOS가 발송됩니다. GPS를 켜면 더 정확한 위치가 전달됩니다" |
| 국가 판별 실패 | 여행 목적지 국가 번호를 기본 표시 + 국가 변경 드롭다운 제공 |
| 역지오코딩 실패 | GPS 좌표만 표시 (주소 없이) |
| 캐시 데이터 손상 | 내장 기본 DB에서 복원 시도 → 실패 시 "영사콜센터 +82-2-3210-0404" 고정 표시 |

### 16.3 네트워크 관련 엣지케이스

| 상황 | 처리 |
|------|------|
| Wi-Fi만 연결 (전화 불가) | VoIP 전화 가능 여부 안내 + "Wi-Fi 전화가 가능한 경우 시도해보세요" |
| 비행기 모드 | "비행기 모드를 해제하면 긴급 전화를 걸 수 있습니다" |
| 긴급 전화 전용 모드 (SIM 없이) | "SIM 카드 없이도 일부 국가에서 긴급 전화가 가능합니다. {번호}로 발신을 시도하세요" |

### 16.4 SOS 취소 관련

| 상황 | 처리 |
|------|------|
| 5초 카운트다운 중 앱 크래시 | SOS 확정 처리 (안전 우선) |
| 5초 카운트다운 중 네트워크 끊김 | 로컬에서 취소 처리 → 네트워크 복구 시 서버에 취소 전송 |
| 취소 요청이 서버에 도달하지 않음 | SOS 확정 유지 + "취소가 처리되지 않았습니다. 그룹에 직접 안내해주세요" |
| 캡틴의 Auto-SOS 취소와 실제 SOS 발송이 동시 | 실제 SOS 우선 처리 (안전 우선 원칙) |

## 17. 면책 및 투명성 고지

### 17.1 면책 범위

서비스 이용약관 원칙 v1.0 §8과 일관되게, 다음을 명확히 고지한다.

| 고지 항목 | 내용 |
|----------|------|
| 서비스 성격 | SafeTrip의 긴급 전화 연결 기능은 공식 구조 서비스를 대체하지 않는 보조 수단이다 |
| 통화 보장 | SafeTrip은 긴급 전화의 실제 연결·응답을 보장하지 않는다. 연결은 이동통신 환경에 의존한다 |
| 데이터 정확성 | 긴급 전화번호·대사관 정보는 주기적으로 갱신하나, 실시간 정확성을 보장하지 않는다 |
| 구조 책임 | SafeTrip은 구조기관의 출동·대응·구조 결과에 대해 책임을 지지 않는다 |
| Auto-SOS 한계 | 자동 SOS는 Heartbeat 타임아웃 기반 추정이며, 실제 위험 여부를 보장하지 않는다 |

### 17.2 고지 시점

| 시점 | 고지 내용 | 방식 |
|------|----------|------|
| 회원가입 시 | "SOS는 보조 수단이며 공식 구조 서비스를 대체하지 않습니다" | 약관 동의 화면 |
| 최초 여행 생성 시 | 긴급 전화 기능 안내 + 한계 고지 | 인앱 카드 |
| SOS 발송 확인 시 | "공식 긴급 전화도 함께 이용해주세요" | SOS 확인 다이얼로그 |
| 오프라인 긴급 화면 | "캐시 데이터 기준이며 최신이 아닐 수 있습니다" | 화면 상단 배지 |
| SOS 수신 시 | "긴급 상황이 의심되면 공식 구조기관에 연락하세요" | 수신 알림 하단 |
