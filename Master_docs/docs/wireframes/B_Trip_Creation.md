## 1. 개요

- **화면 수:** 8개 (B-01 ~ B-08)
- **Phase:** P0 8개 (전 화면 런칭 필수)
- **핵심 역할:** 캡틴 (여행 생성 B-01~B-06), 크루장/크루 (여행 참여 B-07~B-08)
- **여행 상태:** 생성 완료 시 `planning` 상태로 진입
- **핵심 스펙:** 여행 기간 최대 15일 (§02.3), 프라이버시 3등급 (§02.7), 초대코드 7자리 (#23), 강제/자유 위치공유 모드 (§04.2)

### 여행 생성 규칙 요약 (비즈니스 원칙 §02 기반)

> - 여행 생성 시 생성자는 자동으로 **캡틴**이 된다 (§02.1).
> - 여행 1개 = 그룹 1개 (1:1 매핑) (§02.1).
> - 1회 여행의 최대 기간은 **15일** 이내로 제한된다 (§02.3).
> - 캡틴은 여행 생성 시 **프라이버시 등급**(§02.7)과 **위치 공유 모드**(§04.2)를 선택한다.
> - 미설정 시 프라이버시 등급은 `standard`, 위치 공유 모드는 `mandatory`가 기본값이다.

### User Journey Flow

```
[프로필 설정 완료 / 재로그인 (여행 없음)]
 → B-01 여행없음 홈
    ├─ "여행 만들기" → B-02 여행 정보 → B-03 국가 선택 → B-04 프라이버시 등급
    │                  → B-05 위치공유 모드 → B-06 여행 확인 → C-01 메인맵 (planning)
    └─ "코드 입력"  → B-07 초대코드 입력 → B-08 여행 미리보기 → C-01 메인맵 (planning)
```

### 외부 진입/이탈 참조

| 출발 | 조건 | 도착 | 카테고리 |
|------|------|------|---------|
| A-06 프로필 설정 완료 | 여행 없음 | B-01 | A → B |
| 앱 재실행 | 인증 유효 + 여행 없음 | B-01 | 시스템 → B |
| 딥링크 `safetrip://invite/{code}` | 앱 설치 + 인증 완료 + 여행 없음 | B-07 (코드 사전 입력) | 외부 → B |
| B-06 | 여행 생성 완료 | C-01 메인맵 | B → C |
| B-08 | 여행 참여 완료 | C-01 메인맵 | B → C |
