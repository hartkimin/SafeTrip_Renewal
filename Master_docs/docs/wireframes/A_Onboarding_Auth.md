## 1. 개요

- **화면 수:** 6개 (A-01, A-02, A-04, A-05, A-06, A-07)
- **Phase:** P0 6개 (전 화면 런칭 필수)
- **핵심 역할:** 신규 사용자 (역할 미지정 상태), 복귀 사용자
- **여행 상태:** `none` (온보딩 완료 전)
- **법적 근거:** 개인정보보호법 제30조 (개인정보처리방침), 위치정보법 제15조 (LBS 이용약관), 전자상거래법 (서비스이용약관), 개인정보보호법 제22조 (14세 미만 동의)

### 온보딩 5대 시나리오 연결 관계

| 시나리오 | 진입 조건 | 거치는 화면 | 비즈니스 원칙 |
|---------|---------|-----------|------------|
| **A: 여행장 주도 신규 가입** | 최초 가입 + "여행 만들기" 선택 | A-01 → A-02 → A-07 → A-04 → A-05 → A-06 | §06.1, §02.1 |
| **B: 초대코드 참여** | 초대코드/딥링크 보유 | A-01 → A-02(Phase 3 직행) → A-07 → A-04 → A-05 → A-06 | §06.1, §02.5 |
| **C: 가디언 초대 참여** | 가디언 초대 링크 수신 | A-01 → A-07 → A-04 → A-05 → A-06 | §03.1 |
| **D: 복귀 사용자 (재인증)** | 토큰 만료/없음 | A-01 → A-04 → A-05 → 메인화면 | §06.2, §06.4 |
| **E: B2B CSV 일괄 등록** | SMS/이메일 링크 수신 | A-01 → A-07 → A-04 → A-05 → A-06 | §12.3, §01.4 |

### User Journey Flow

```
A-01 Splash
 ├── [토큰 유효] ────────────────────────────→ C-01 메인맵
 ├── [딥링크(초대코드)] ─→ A-02 Welcome (Phase 3 직행, 초대코드 자동입력)
 ├── [딥링크(가디언초대)] → A-07 Consent → A-04 Phone Auth
 └── [첫 실행 / 토큰 만료]
      ↓
A-02 Welcome (Phase 2: 가치 슬라이드 3~4장 → Phase 3: 목적 선택)
      ├── [여행 만들기] ────→ A-07 Consent
      ├── [초대코드 입력] ──→ A-07 Consent
      ├── [먼저 둘러보기] ──→ 데모 투어 (#30)
      └── [가디언으로 참여] → A-07 Consent
           ↓
A-07 Consent (약관 4종 필수 동의 §06.5)
      ↓
A-04 Phone Auth (전화번호 입력)
      ↓
A-05 OTP (6자리, 3분 유효)
      ↓
A-06 Profile Setup (이름, 생년월일, 프로필 사진)
      ├── [14세 미만] → 법정대리인 동의 분기 (§10.1)
      ├── [14~17세] → 보호자 이중 동의 분기 (§10.1)
      └── [설정 완료] → C-01 메인맵 (탐색 모드 §06.3 / 초대코드 시 해당 여행 배정)
```

**배경 초기화 작업 (스플래시 원칙 #28 §5):**

| 작업 | 종류 | 실패 시 |
|------|------|--------|
| Firebase 인증 토큰 갱신 (`silentRefresh`) | 필수 | 토큰 삭제 → 신규 유저 경로(A-02) |
| FCM 토큰 유효성 확인 및 갱신 | 선택 | 실패 무시, 이후 재시도 |
| 앱 버전 확인 (서버 min/recommended) | 필수 | 캐시 사용 / 없으면 스킵 |
| 로컬 캐시 무결성 체크 | 선택 | 손상 캐시 초기화 후 진행 |
| 딥링크 파라미터 파싱 | 조건부 | 딥링크 무시, 정상 분기 |

**분기 로직 (스플래시 원칙 #28 §4):**

```
스플래시 초기화 완료
  ├─ 경로 A (신규): 로컬 토큰 없음 → A-02 웰컴화면
  ├─ 경로 B (복귀): 토큰 유효 → C-01 메인화면
  │    └─ active 여행 > planning 여행 > 탐색모드 (§06.2)
  └─ 경로 C (딥링크): 딥링크 파라미터 존재
       ├─ 로그인 상태 → 딥링크 대상 화면 직접 이동
       └─ 비로그인 → 인증 흐름 → 완료 후 딥링크 대상 이동
```
