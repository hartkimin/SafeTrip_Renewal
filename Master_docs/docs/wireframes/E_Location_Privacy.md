## 1. 개요

- **화면 수:** 4개 (E-01, E-04, E-05, E-06)
- **Phase:** P0 1개 (E-01), P1 3개 (E-04, E-05, E-06)
- **핵심 역할:** 캡틴 (프라이버시 등급/모드 변경), 크루장/크루 (개인 위치 설정), 가디언 (미관여 — 수신 전용)
- **여행 상태:** `active` 상태에서만 위치 공유 기능 동작
- **법적 근거:** 위치정보법 제15조 (위치정보 수집 동의), 위치정보법 제18조 (위치정보 이용 동의), 개인정보보호법 제15조 (개인정보 수집 동의)

### 프라이버시 3등급 핵심 원칙

> **프라이버시 등급은 여행 단위로 캡틴이 설정하며, 가디언 공유 정책/스케줄 기본값/공개 범위를 결정한다.** (비즈니스 원칙 §02.7)
> SOS는 모든 프라이버시 설정을 무시하는 최상위 안전 기능이다. (비즈니스 원칙 §05.1)

### 프라이버시 등급별 동작 매트릭스

| 동작 | 안전최우선 (`#DA4C51`) | 표준 (`#00A2BD`) | 프라이버시우선 (`#A7A7A7`) |
|------|:---------------------:|:----------------:|:-------------------------:|
| 스케줄 ON 시 마커 | 실시간 | 실시간 | 실시간 |

| 가디언 일시정지 | 불가 | 최대 12h | 최대 24h |

| SOS 시 위치 강제 포함 | 동일 | 동일 | 동일 |

### User Journey Flow

```
C-01 메인맵 → 멤버탭/설정
 │
 └──→ E-01 위치공유 설정 (모달 바텀시트)
       ├── [가시범위] ────→ E-04 가시범위 설정 (자유 모드만)
       └── [가디언 일시정지] → E-05 가디언 접근 일시정지

K-01 설정 (Layer 2 여행별) → 캡틴 전용
 └──→ E-06 프라이버시 등급 변경 확인 (다이얼로그)
```

> **v2.0 구조 변경 사항:**
> - E-02 (공유 스케줄 설정), E-03 (이벤트 연동 설정)을 E-01에 통합 — 스케줄/이벤트 설정을 위치공유 설정 내 인라인 섹션으로 배치
> - E-07 (공유모드 전환 확인)을 E-06에 통합 — 프라이버시 등급 변경 다이얼로그 내에서 모드 전환도 함께 처리
> - E-08 (지오펜스 관리)은 Master_docs 미정의로 제거 (향후 별도 와이어프레임으로 분리 예정)

## 3. 에러 & 엣지케이스

| # | 케이스 | 처리 방법 |
|---|--------|----------|
| 1 | **위치 권한 미허용** | E-01 진입 시 권한 요청 다이얼로그 → 거부 시 "위치 권한이 필요합니다" Card_Alert + 설정 이동 버튼, 토글 비활성 |
| 2 | **스케줄 저장 실패 (네트워크)** | 로컬에 변경사항 캐시 → 네트워크 복구 시 자동 재전송, Toast "오프라인 상태입니다. 네트워크 복구 시 자동 저장됩니다" |
| 3 | **프라이버시 등급 변경 중 네트워크 단절** | 로컬 캐시에 변경 저장 → 복구 시 재시도, 실패 시 이전 등급 유지 |
| 4 | **일시정지 중 SOS 발동** | 일시정지 강제 해제, 가디언에게 즉시 위치 공유 재개, Toast "SOS 발동으로 일시정지가 해제되었습니다" |
| 5 | **일시정지 중 프라이버시 등급 변경** | 새 등급의 최대 일시정지 시간 적용 — 초과 시 자동 단축 (표준→안전최우선: 즉시 해제) |
| 6 | **가디언 미연결 상태에서 일시정지 시도** | "연결된 가디언이 없습니다" 안내 + 시작 버튼 비활성 |
| 7 | **강제 모드에서 개인 설정 시도** | 해당 UI 비활성 (opacity 0.4) + Toast "강제 모드에서는 개인 설정을 변경할 수 없습니다" |
| 8 | **미성년자 포함 여행에서 등급 변경 시도** | Card_Alert: "미성년자 포함 여행은 안전최우선 등급만 허용됩니다", 변경 버튼 비활성 |
| 9 | **미성년자 크루의 가디언 일시정지 시도** | E-05 진입 차단, Card_Alert: "미성년자는 가디언 일시정지를 사용할 수 없습니다" |
| 10 | **동시 다중 여행 소속** | 각 여행별 독립 설정, E-01 진입 시 현재 선택된 여행의 설정만 표시 |
| 11 | **일시정지 만료 시 앱 미실행** | 서버에서 자동 만료 처리, 앱 재진입 시 비활성 상태로 표시, Push 알림 발송 |
| 12 | **가시범위 "지정 멤버" 후 멤버 탈퇴** | 탈퇴 멤버 자동 제거, 남은 멤버 0명이면 "전체 멤버"로 자동 변경 |

## 4. 오프라인 대응

| 상황 | 동작 |
|------|------|
| **오프라인 시 E-01 스케줄 변경** | 로컬 SQLite에 변경사항 저장, 상단 배너 "오프라인 — 네트워크 복구 시 자동 반영", 네트워크 복구 시 `PUT` 자동 재전송 |
| **오프라인 시 가디언 일시정지 시작** | 로컬에서 타이머 시작, 서버 동기화 대기, 네트워크 복구 시 `POST` 전송, 가디언에게 "일시정지됨" 알림 지연 발송 |
| **오프라인 시 등급 변경** | 변경 불가 — "네트워크 연결이 필요합니다" Toast, 변경 버튼 비활성 (등급 변경은 전 멤버에게 영향이므로 오프라인 변경 차단) |
| **오프라인 시 가시범위 변경** | 로컬에 저장, 네트워크 복구 시 자동 반영 |
| **위치 공유 ON/OFF 토글 (오프라인)** | 로컬 상태 즉시 변경 + 서버 동기화 큐잉, 네트워크 복구 시 `PATCH` 전송 |
| **일시정지 카운트다운 중 오프라인** | 로컬 타이머 지속, 만료 시 로컬에서 해제 처리 → 복구 시 서버와 상태 동기화 |
