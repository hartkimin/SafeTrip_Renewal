## 1. 개요

- **화면 수:** 8개 (F-01 ~ F-08)
- **Phase:** P0 6개 (F-01 ~ F-06), P1 2개 (F-07, F-08)
- **핵심 역할:** 캡틴 (전체 가디언 관리, 과금 결제, 미성년자 해제 승인), 크루장/크루 (개인 가디언 관리), 가디언 (위치 열람, 긴급 알림, 메시지)
- **여행 상태:** `planning`/`active` 상태에서 가디언 관리 가능, `completed` 이후 읽기 전용
- **법적 근거:** 위치정보법 제15조 (위치정보 수집 동의), 아동복지법 제17조 (미성년자 보호)

### 가디언 과금 핵심 원칙

> **개인 가디언:** 멤버당 무료 2명 + 유료 3명 (1,900원/여행/명), 환불 불가 (비즈니스 원칙 §03.1, §09.3)
> **전체 가디언:** 여행당 무료 2명, 추가 불가 (캡틴만 관리)
> **미성년자:** 가디언 해제 시 캡틴 승인 필수 (비즈니스 원칙 §10.2)

### 가디언 초대코드

> 가디언 연결은 **전화번호 기반 초대** 또는 **7자리 초대코드**로 이루어진다.

### 프라이버시 등급별 가디언 동작 매트릭스

| 동작 | 안전최우선 | 표준 | 프라이버시우선 |
|------|:---------:|:----:|:-------------:|
| 스케줄 ON 시 가디언 위치 수신 | 실시간 | 실시간 | 실시간 |
| 스케줄 OFF 시 가디언 위치 수신 | 실시간 (항상) | 30분 스냅샷 | 비공유 |
| 가디언 일시정지 | 불가 | 최대 12시간 | 최대 24시간 |
| SOS 시 가디언 위치 수신 | 항상 | 항상 | 항상 |
| 지오펜스 이벤트 가디언 전달 | 항상 | 스케줄 ON 시만 | 전달 안 함 |
| 긴급 위치 요청 응답 | 자동 (항상 공유 중) | 1회성 자동 전달 | 멤버 승인 후 전달 |

### User Journey Flow

```
[여행 멤버 측 플로우]

F-02 가디언 관리 (개인 + 전체 가디언 목록)
 ├── [+ 무료 추가] → F-01 가디언 추가 (전화번호/코드 입력)
 │                    └── [요청 전송] → F-02 (대기 중 상태 반영)
 ├── [+ 유료 추가] → 과금 모달 (1,900원) → 결제 후 F-01
 ├── [가디언 카드 탭] → 상태별 분기
 │    ├── 연결됨 → 해제 확인 Dialog
 │    ├── 대기 중 → 요청 취소 확인 Dialog
 │    └── 거절됨 → 재요청 또는 삭제
 └── [전체 가디언 관리] (캡틴 전용 탭) → F-02 캡틴뷰 확장

[가디언 측 플로우]

F-03 가디언 링크 승인 (푸시/인앱 알림 → 승인 화면)
 ├── [수락] → F-04 가디언 홈
 └── [거절] → 알림 목록으로 복귀

F-04 가디언 홈 (연결 멤버 카드 목록)
 ├── [멤버 카드 탭] → F-05 가디언 멤버 상세
 │                      ├── [메시지 보내기] → F-06 가디언 메시지
 │                      └── [긴급 알림] → F-07 긴급 알림 발송
 └── [긴급 알림 버튼] → F-07 긴급 알림 발송

F-08 위치 요청 (F-05에서 진입, 프라이버시등급별 상태분기)
 └── [요청 전송] → 프라이버시 등급별 분기
      ├── 안전최우선 → 자동 승인 (위치 즉시 수신)
      ├── 표준 → 1회성 위치 자동 전달
      └── 프라이버시우선 → 멤버 승인 대기 (5분 만료)
```

> **v2.0 구조 변경 사항:**
> - F-09 (위치 요청 승인)을 F-08에 통합 — 프라이버시등급별 상태분기로 처리 (프라이버시우선 등급의 승인 다이얼로그를 F-08 내부 상태로 관리)
> - F-10 (전체여행 가디언 관리)을 F-02에 통합 — 캡틴뷰 확장 탭으로 구현 (개인 가디언 / 전체 가디언 탭 전환)
> - 최종 화면 수: 10개 → 8개

## 3. 에러 & 엣지케이스

### 3.1 가디언 연결 관련

| 상황 | 처리 |
|------|------|
| 무료 슬롯 초과 가디언 추가 시도 | 과금 안내 모달 자동 표시 (§09.3) |
| 유료 가디언 결제 실패 | "결제에 실패했습니다. 카드 정보를 확인해 주세요." 안내 후 결제 재시도 |

| 전체 가디언 슬롯 소진 (2/2) 후 추가 시도 | "전체여행 가디언은 최대 2명까지 지정 가능합니다" 안내 |
| 가디언 연결 요청 후 상대방 미수락 | 카드에 "수락 대기 중" 배지 표시, 7일 후 자동 만료 |
| 이미 연결된 번호로 재요청 | SnackBar "이미 연결된 가디언입니다" |
| 동일 여행 멤버를 가디언으로 추가 시도 | SnackBar "같은 여행의 멤버는 가디언으로 추가할 수 없습니다" (비즈니스원칙 §01.2) |
| 가디언 초대코드 만료 | "초대코드가 만료되었습니다. 새 초대를 요청해주세요" 안내 |

| 1시간 3회 한도 초과 시 요청 시도 | 버튼 비활성화 + "N분 후 다시 요청 가능합니다" 표시 |
| 연결 멤버가 오프라인 상태에서 요청 | 요청 발송 성공 + "연결 멤버가 오프라인입니다. 온라인 전환 시 알림이 전달됩니다." 안내 |
| 프라이버시우선 승인 대기 중 멤버 앱 종료 | 푸시 알림으로 재안내, 5분 만료 시 자동 거절 |

| 네트워크 끊김 중 메시지 전송 | 로컬 큐잉 → 복귀 시 자동 전송, 버블에 "전송 대기 중" 표시 |
