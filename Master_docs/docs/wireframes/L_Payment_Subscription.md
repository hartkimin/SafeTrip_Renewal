## 1. 개요

- **화면 수:** 8개 (L-01 ~ L-05, L-07 ~ L-09)
- **Phase:** P2 전체
- **핵심 역할:** 캡틴 (여행 기본 요금 결제), 캡틴/크루장/크루 (가디언 슬롯 구매, AI 구독), 전체 (결제 내역 조회)
- **여행 상태:** `planning` 또는 `active` 상태에서 결제 가능 (§09.7)

### 과금 모델 요약 (비즈니스 원칙 §09)

| 항목 | 가격 | 과금 단위 | 환불 규칙 |
|------|:----:|----------|----------|
| 여행 기본 요금 (1~5명) | 무료 | — | — |
| 여행 기본 요금 (6~15명) | 9,900원 | 여행당 | §09.7 적용 |
| 여행 기본 요금 (16~30명) | 14,900원 | 여행당 | §09.7 적용 |
| 여행 기본 요금 (31명+) | 19,900원 | 여행당 | §09.7 적용 |
| 추가 가디언 (3~5번째) | 1,900원 | 인/여행 | **환불 불가** |
| AI Plus | 4,900원/월 or 2,900원/여행 | 월/여행 | §09.7 적용 |
| AI Pro | 9,900원/월 or 5,900원/여행 | 월/여행 | §09.7 적용 |

> **환불 규칙 (§09.7):** planning=전액, active+24h이내=50%, active+24h이후=불가, completed=불가.
> **가디언 슬롯은 환불 불가** (§09.3).
> **무료 보장**: 안전 기능(SOS, 가디언 긴급알림, 안전가이드, 기본 위치 공유)은 과금 무관 항상 사용 가능 (§09.5).

### 결제 수단

| 구분 | 수단 | 비고 |
|------|------|------|
| 인앱결제 | Google Play / Apple IAP | AI 구독(월간)은 인앱결제 전용 |
| 간편결제 | 카카오페이, 네이버페이, 토스 | 여행 요금, 가디언 슬롯, AI 여행단위 |

### User Journey Flow

```
[여행 기본 요금]
L-01 여행 요금 안내
 └── [결제하기]
      ↓
L-02 결제 수단 선택
      ↓
L-03 결제 확인
      ├── [결제 성공] → L-04 결제 완료
      └── [결제 실패] → L-03 에러 상태 (인라인 표시)
                         ├── [다시 시도] → L-03 재시도
                         └── [다른 수단] → L-02

[가디언 슬롯]
F-xx 가디언 관리 > 슬롯 추가 → L-05 가디언 슬롯 구매 → L-02 → L-03 → L-04

[AI 구독]
K-01 설정 / AI 기능 업그레이드 유도 → L-07 AI 요금제 → L-02 → L-03 → L-04

[환불/내역]
K-01 설정 > 결제 내역 → L-09 결제 내역 → (환불 가능 항목) → L-08 환불 요청
```

## 4. 오프라인 대응

| 상황 | 동작 |
|------|------|
| **오프라인에서 L-01 진입** | 마지막 캐시된 요금 정보 표시 + "오프라인 상태 - 결제 불가" 배너, 결제 버튼 비활성 |
| **오프라인에서 결제 시도** | 결제 버튼 비활성, "인터넷 연결 후 결제해주세요" Toast |
| **결제 진행 중 네트워크 단절** | PG 응답 대기 (10초 타임아웃) → 타임아웃 시 "결제가 완료되지 않았습니다. 네트워크 확인 후 다시 시도해주세요." 안내 + 서버에서 결제 상태 확인 API 제공 |
| **오프라인에서 L-09 진입** | 마지막 캐시된 결제 내역 표시 + "오프라인 상태" 배너, 환불 요청 불가 |
| **오프라인에서 환불 요청** | "인터넷 연결 후 요청해주세요" Toast, 환불 요청 버튼 비활성 |

> 결제는 네트워크 필수 기능이다. 오프라인에서는 열람만 가능하고, 결제/환불 트랜잭션은 불가하다.
