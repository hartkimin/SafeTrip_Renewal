## 1. 개요

채팅탭은 여행 그룹 내 실시간 커뮤니케이션 채널로, 바텀시트 탭 컨텐츠(C-04의 채팅 탭)로 제공된다. 텍스트 메시지, 위치 공유 카드, 일정 카드, 투표 등의 메시지 유형과 SOS 시스템 메시지를 통합한다. 가디언은 그룹 채팅에 접근할 수 없으며, 멤버와 가디언 간 소통은 RTDB `link_{linkId}` 기반의 1:1 보호자 채널로 완전히 분리된다.

| 항목 | 내용 |
|------|------|
| 화면 수 | 3개 (I-01, I-03, I-04) |
| Phase | P0 1개 (I-01), P1 1개 (I-03), P3 1개 (I-04) |
| 핵심 역할 | 캡틴/크루장/크루 (그룹 채팅), 가디언 (보호자 채널만) |
| RTDB 채널 | 그룹 채팅: `trip_{tripId}_chat`, 보호자 채널: `link_{linkId}` |
| 메시지 보존 | 여행 종료 후 90일 (§13.1) |
| 오프라인 큐 | 최대 100건 (§11.1) |

### 설계 철학 5원칙 (채팅탭 원칙 §2)

| ID | 원칙명 | 내용 |
|----|--------|------|
| C1 | 맥락 통합 | 일정, 위치, SOS 정보를 채팅에 자연스럽게 통합한다. |
| C2 | 빠른 공유 | 탭 2번으로 현재 위치 또는 일정을 채팅에 공유할 수 있다. |
| C3 | 안전 우선 | SOS 발신 시 채팅에 자동 시스템 메시지가 삽입된다. SOS 메시지는 삭제 불가. |
| C4 | 오프라인 대응 | 오프라인 시 메시지 작성이 가능하며, 최대 100건 로컬 큐잉 후 자동 발송. |
| C5 | 프라이버시 분리 | 그룹 채팅과 가디언 채널은 완전히 분리된다. |

### 2.1 [I-01] 그룹 채팅 탭

> I-01은 기존 I-02(채팅 입력)와 I-05(오프라인 큐)를 통합한 화면이다.
> 입력 영역은 I-01 하단에 인라인으로 포함되며, 오프라인 큐는 I-01 내 상태 분기로 처리한다.

#### 메타데이터

| 항목 | 값 |
|------|-----|
| ID | I-01 |
| 화면명 | 그룹 채팅 탭 (Group Chat Tab) |
| Phase | P0 |
| 역할 | 캡틴, 크루장, 크루 (그룹 채팅) / 가디언 (보호자 메시지 탭만) |
| 진입 경로 | C-04 바텀시트 --> 채팅 탭 선택 --> I-01 |
| 이탈 경로 | I-01 --> I-03 (미디어 탭) / I-01 --> I-04 (검색) / I-01 --> C-01 (위치 카드) |
| 표시 형태 | 바텀시트 탭 컨텐츠 (독립 화면 아님) |

#### 역할별 분기

| 역할 | 접근 | 동작 |
|------|:----:|------|
| 캡틴 | O | 그룹 채팅 전체 기능. 공지 고정/해제 가능. 투표 생성 가능 (P2). 타인 메시지 삭제 가능. 보호자 메시지 탭에서 본인 가디언 채널 접근. |
| 크루장 | O | 캡틴과 동일. 다만 타인 메시지 삭제는 본인 메시지만 가능. |
| 크루 | O | 그룹 채팅 메시지 발송/수신. 본인 메시지만 삭제 가능. 공지 고정/해제 불가. 투표 생성 불가, 참여만 가능. 보호자 메시지 탭에서 본인 가디언 채널 접근. |
| 가디언 | 제한 | 그룹 채팅 접근 불가. 보호자 메시지 탭만 표시. 연결된 멤버와 1:1 채널에서 텍스트/이미지 발송. 위치 카드 수신은 유료 가디언만 가능. |

#### 여행상태별 분기

| 여행 상태 | 동작 |
|----------|------|
| none | 여행 미참여 상태. 채팅탭 미표시. |
| planning | 채팅 가능. 여행 시작 전에도 그룹 대화 허용. 모든 메시지 유형 전송 가능. |
| active | 정상 동작. SOS 시스템 메시지, 출석 체크 시스템 메시지 활성. |
| completed | 입력 영역 비활성화. placeholder: "종료된 여행입니다". 전송 버튼 disabled. 메시지 열람만 가능. 메시지 보존 기간: 여행 종료 후 90일 (§13.1). |

#### 프라이버시등급별 분기

| 프라이버시 등급 | 동작 |
|--------------|------|
| 안전최우선 (safety_first) | 위치 카드 전송 시 경고 없음 (항상 공유 중). SOS 활성 시 채팅 상단에 지속 배너 고정 ("SOS 활성 중 -- {이름}님 [지도]"). |
| 표준 (standard) | 위치 카드 전송 시 경고 없음. 비공유 시간대에 전송된 위치 카드에 "비공유 시간 발송" 배지 표시. SOS 카드 삽입 후 배너 없음. |
| 프라이버시우선 (privacy_first) | 비공유 시간대 위치 카드 전송 시 "현재 위치 비공유 시간대입니다. 위치 카드를 보내시겠습니까?" 확인 다이얼로그 (P2). 가디언 채널에서 비공유 시간대 위치 카드 내용 숨김 ("비공유 시간입니다"). SOS 카드 삽입 후 배너 없음. |

## 4. 오프라인 대응

### 오프라인 시 채팅탭 동작

| 데이터 | 오프라인 가용성 | 비고 |
|--------|:--------------:|------|
| 메시지 목록 열람 | O | 로컬 캐시 (최근 200건) |
| 텍스트 메시지 작성 | O | 로컬 큐잉 (최대 100건) |
| 이미지 첨부 발송 | 부분 | 로컬 큐잉 (최대 100건 포함) |
| 파일 첨부 발송 | X | 온라인 연결 필요 |
| 새 메시지 수신 | X | 온라인 연결 필요 |
| 투표 응답 | 부분 | 로컬 큐잉 --> 온라인 복귀 시 발송 |
| 위치 카드 발송 | X | 위치 좌표 취득에 온라인 필요 |

### 오프라인 배너

채팅탭 진입 시 오프라인 상태이면 입력 영역 상단에 표시:

```
[오프라인 모드] 메시지는 연결 후 자동 전송됩니다. 메시지 입력은 가능합니다.
```

- 배경: `secondaryAmber` (`#FFB800`)
- 텍스트: `#FFFFFF`, bodySmall (12sp)

### 전송 대기 메시지 표시

로컬 큐에 대기 중인 메시지는 채팅 목록 내에서 인라인 표시:

```
대기 중인 메시지 내용                    [시계] 전송 대기
(회색 텍스트)                            오후 3:15
```

- 전송 대기: `onSurfaceVariant` (`#8E8E93`) 시계 아이콘
- 전송 실패: `semanticError` (`#DA4C51`) 경고 아이콘 + "재전송" 버튼

### 온라인 복귀 시 큐 처리

온라인 복귀 시 대기 메시지를 큐 순서대로 자동 발송. 발송 성공 시 시계 아이콘 --> 체크 아이콘 변경.

동기화 우선순위 (비즈니스 원칙 §11.2):

1. SOS 이벤트 (최우선)
2. 가디언 긴급 알림/요청
3. 위치 데이터 업로드
4. 일정 변경 동기화
5. 채팅 메시지 발송 큐
6. 일반 이벤트 로그
