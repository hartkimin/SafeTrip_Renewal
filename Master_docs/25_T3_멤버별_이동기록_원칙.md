## §2. 목적 및 적용 범위

### 2.1 목적

본 문서는 SafeTrip 앱의 멤버탭 내 **개별 멤버 이동기록 화면**에 대한 설계 원칙을 정의한다. 이동기록 화면은 타임라인 뷰와 지도 뷰를 이중으로 제공하며, 체류 지점 분석, 프라이버시 마스킹, 가디언 과금별 접근 범위 차등 및 이동기록 인사이트 기능을 포함한다.

비즈니스 원칙 v5.1의 다음 항목과 직접 연계된다:
- §04 위치 공유 원칙 (프라이버시 등급별 동작)
- §09.3 가디언 추가 과금 (무료/유료 가디언 접근 범위 차등)
- §13 데이터 생명주기 기본 정책 (30일/90일 보존 이원화)

### 2.2 적용 범위

| 대상 | 범위 |
|------|------|
| 적용 화면 | 멤버탭 > 멤버 선택 > 이동기록 |
| 적용 역할 | 캡틴, 크루장, 크루, 가디언 (역할별 조회 범위 차등) |
| 적용 등급 | 안전최우선(safety_first) / 표준(standard) / 프라이버시우선(privacy_first) |
| 적용 Phase | Phase 1 (기본 기능) / Phase 2 (인사이트, 히트맵) |
| 비적용 범위 | 그룹 발자취(Group Footprint) — 별도 문서 정의 |

### 2.3 용어 정의

| 용어 | 정의 |
|------|------|
| 이동기록 | 특정 멤버의 시간 순서별 위치 포인트 집합 및 이동 구간 분석 결과 |
| 이동 세션 | `movement_session_id`로 묶인 단일 이동 구간 (출발~도착) |
| 체류 지점 | 반경 100m 내에서 5분 이상 머문 정지 구간 (TB_STAY_POINT) |
| 타임라인 뷰 | 시간 축 기반으로 이동·체류 이벤트를 수직 나열한 뷰 |
| 지도 뷰 | 경로 폴리라인과 체류 지점 마커를 지도 위에 표시하는 뷰 |
| 무료 가디언 | 멤버당 2명까지 무료 연결 가능한 가디언 (`is_paid = FALSE`) |
| 유료 가디언 | 3명째 이상 연결 시 1,900원/여행 과금된 가디언 (`is_paid = TRUE`) |
| 마스킹 | 프라이버시우선 등급에서 일정 미연동 시간대의 위치 정보를 반경 500m 원으로 흐림 처리하는 기능 |

## §3. 이동기록 설계 철학 — 5대 원칙 (M1~M5)

### M1. 투명성 (Transparency)

> **본인 이동기록은 항상 확인 가능하다.**

- 크루 포함 모든 역할의 멤버는 **자신의 이동기록에 대한 접근이 제한되지 않는다.**
- 프라이버시우선 등급이더라도 본인 이동기록 조회 시 마스킹이 적용되지 않는다.
- 이동기록이 타인에게 어떻게 공유되는지 설정 화면에서 확인할 수 있어야 한다.

### M2. 이중 뷰 (Dual View)

> **타임라인(시간순) + 지도(공간) 이중 뷰를 병행 제공한다.**

- 두 뷰는 동일한 데이터를 서로 다른 차원으로 표현한다.
- 타임라인에서 이벤트 선택 시 지도 뷰가 해당 위치로 자동 이동한다.
- 지도에서 경로 구간 선택 시 타임라인이 해당 시간대로 스크롤된다.
- 양방향 연동(Two-Way Binding)이 필수이다.

### M3. 시간순 (Chronological Order)

> **이동기록의 시간 방향은 과거 → 현재로 고정한다.**

- 타임라인은 최상단 = 가장 오래된 기록, 최하단 = 가장 최근 기록이다.
- 시간대 변경(해외 여행) 시 **로컬 시간**으로 표시하되, 시간대 표기(예: `UTC+9`, `CET`)를 명시한다.
- 날짜 변경선(날짜 구분선) UI는 로컬 자정 기준으로 표시한다.

### M4. 프라이버시 마스킹 (Privacy Masking)

> **프라이버시 등급에 따라 이동기록에 마스킹을 적용한다.**

- 마스킹 적용 대상: 프라이버시우선 등급의 **일정 미연동 시간대**
- 마스킹 범위: 반경 500m 원으로 흐림 처리 (위치 좌표를 500m 격자 중심으로 스냅)
- 마스킹 해제 조건: **SOS 발송 시 자동 해제** (비즈니스 원칙 §04.8 참조)
- 마스킹 로그: 해제 이력을 `tb_event_log`에 기록 (감사 추적)
- 위치 데이터 정책(#09)과의 정합 필수 — 마스킹 규칙은 두 문서가 동일해야 한다.

### M5. 인사이트 확장 (Insight Expansion)

> **Phase 2에서 이동기록 분석 인사이트를 제공한다.**

- 일별 이동 거리, 체류 핫스팟, 그룹 내 가장 먼 멤버 등
- Phase 1에서는 기반 데이터(TB_LOCATION, TB_STAY_POINT)만 수집, 분석 UI는 Phase 2에서 활성화

## §4. 이동기록 데이터 구조

### 4.1 데이터 계층

```
이동기록 데이터
├── [1] 위치 포인트 (TB_LOCATION)
│       └── movement_session_id로 구간 그룹화
├── [2] 이동 세션 (TB_LOCATION.movement_session_id 기반 집계)
│       └── 세션 지도 이미지 캐시 (TB_SESSION_MAP_IMAGE)
├── [3] 체류 지점 (TB_STAY_POINT)
│       └── 연속 위치 포인트 클러스터 분석 결과
└── [4] 경로 세그먼트 (이동 세션 간 연결선, 앱 클라이언트 렌더링)
```

### 4.2 위치 수집 주기

| 상태 | 수집 빈도 | 근거 |
|------|:--------:|------|
| SOS 활성 상태 | 10초 간격 | 비즈니스 원칙 §13.3 |
| 스케줄 ON + 이동 중 | 30초 간격 | 비즈니스 원칙 §13.3 |
| 스케줄 ON + 정지 | 120초 간격 | 비즈니스 원칙 §13.3 |
| 스케줄 OFF | 수집 안 함 | 비즈니스 원칙 §13.3 |

### 4.3 배터리 최적화

- `tracking_mode` 컬럼으로 수집 모드 제어:
  - `normal`: 표준 빈도 수집 (스케줄 ON + 이동 중)
  - `power_saving`: 빈도 절반 (배터리 20% 이하 시 자동 전환)
  - `minimal`: 최소 빈도 (배터리 10% 이하 시 자동 전환, 오프라인 저장)
  - `sos`: 최대 빈도 (SOS 활성 시 강제 적용, 배터리 무시)
- 배터리 절약 모드 전환 시 멤버에게 인앱 알림 표시

### 4.4 위치 정확도 기준

| 정확도 등급 | accuracy 값 | 지도 표시 |
|-----------|:-----------:|---------|
| 높음 | ≤ 15m | 정확한 위치 포인트 |
| 보통 | 15~50m | 불확실 원 표시 |
| 낮음 | > 50m | 흐림 처리 + 경고 아이콘 |
| 미확인 | NULL | "GPS 약함" 배지 표시 |

## §5. 타임라인 뷰 구성 원칙

### 5.1 기본 레이아웃

```
┌─────────────────────────────────────────┐
│  [멤버 이름] 이동기록   [날짜 선택]      │
│─────────────────────────────────────────│
│  [타임라인 뷰] ↔ [지도 뷰]  탭 전환     │
│─────────────────────────────────────────│
│  ◀──────[시간 슬라이더]──────────▶      │
│  00:00                           23:59  │
│─────────────────────────────────────────│
│  ● 08:30  출발 (숙소)                   │
│  │                                      │
│  ▼ 이동 중 (도보 15분)                  │
│  │                                      │
│  ■ 08:45  체류 (카페 A, 45분)           │
│  │                                      │
│  ▼ 이동 중 (차량 20분)                  │
│  │                                      │
│  ■ 09:50  체류 (박물관, 2시간 10분)     │
│  ...                                    │
└─────────────────────────────────────────┘
```

### 5.2 시간 슬라이더

- 분 단위 이동 가능 (00:00 ~ 23:59)
- 슬라이더 드래그 시 해당 시점의 위치를 지도 뷰에 실시간 반영
- 배율 조절: 핀치 줌으로 시간 범위 확대/축소
  - 기본 배율: 1시간 = 화면 1/4 높이
  - 최대 배율(확대): 10분 = 화면 1/4 높이 (이동 세부 경로 확인)
  - 최소 배율(축소): 12시간 = 화면 1/4 높이 (하루 전체 조망)

### 5.3 이벤트 마커 종류

| 마커 유형 | 아이콘 | 설명 |
|---------|:------:|------|
| 이동 출발 | ● (초록) | 이동 세션 시작 지점 |
| 이동 종료 | ● (빨강) | 이동 세션 종료 지점 |
| 체류 지점 | ■ (파랑) | 반경 100m 내 5분 이상 정지 |
| SOS 이벤트 | ⚠ (빨강) | SOS 발송 시각 |
| 알림 이벤트 | 🔔 (주황) | 지오펜스 진입/이탈, 안전 알림 |
| 일정 시간대 | 📅 (보라) | 여행 일정(TB_SCHEDULE) 연동 시간대 |
| GPS 단절 | ✕ (회색) | 위치 데이터 없음 구간 |
| 마스킹 구간 | ░ (연회색) | 프라이버시우선 등급 비연동 시간대 |

### 5.4 데이터 밀도 조절

- 확대 시: 개별 위치 포인트 표시 (accuracy 낮은 포인트 회색 처리)
- 기본 배율: 이동 세션 단위 표시 (세션 시작/종료 마커)
- 축소 시: 체류 지점 + 주요 이벤트만 표시 (클러스터링)
- 클러스터링 기준: 시간 범위 기준 ±5분 이내 이벤트를 하나의 클러스터로 묶음

### 5.5 시간대 변경 처리 (해외 여행)

- 위치 포인트의 `recorded_at`은 항상 UTC로 저장
- 타임라인 표시는 **해당 위치의 로컬 시간** 사용 (역지오코딩 + 시간대 DB)
- 시간대 전환 지점에는 구분선과 함께 "(예: UTC+9 → CET)" 표시
- 날짜 변경 시 날짜 구분선 자동 삽입

## §6. 지도 뷰 구성 원칙

### 6.1 기본 레이아웃

- 경로 폴리라인: 이동 세션별 좌표를 순서대로 연결한 선
- 체류 지점 마커: 클러스터링 반경 100m 이내 체류 지점 표시
- 현재 위치 / 선택된 시점 위치: 별도 강조 마커
- 지도 기본화면(#17) 레이어 시스템과의 연동 규칙 준수 (§6.4 참조)

### 6.2 폴리라인 스타일

| 항목 | 값 |
|------|-----|
| 선 색상 | 이동 세션별 색상 (최대 8색 순환) |
| 선 두께 | 3px (기본) / 5px (선택된 세션) |
| 선 불투명도 | 과거 세션: 60% / 오늘 세션: 100% |
| 화살표 | 이동 방향 표시 (10초 간격마다 화살표 아이콘) |
| 마스킹 구간 | 점선 + 회색 처리 |

### 6.3 체류 지점 클러스터링

- 화면 줌 레벨 12 이상: 개별 체류 지점 마커 표시
- 화면 줌 레벨 11 이하: 인접 체류 지점 클러스터 마커로 병합
  - 클러스터 마커: 원 + 숫자(포함된 체류 지점 수)
- 체류 지점 탭 시: 팝업으로 주소 + 체류 시간 표시

### 6.4 지도 기본화면(#17)과의 레이어 연동 규칙

- 이동기록 레이어는 지도 기본화면(#17)의 **레이어 스택 최상단**에 추가된다.
- 지도 기본화면의 그룹 멤버 마커와 이동기록 마커가 겹치는 경우: **이동기록 마커 우선** 표시
- 이동기록 화면 진입 시 지도 기본화면의 다른 멤버 마커는 **반투명(30%) 처리**
- 이동기록 화면 종료 시 지도 기본화면 레이어 원상 복귀

### 6.5 히트맵 모드 (Phase 2)

- 이동 경로 위에 체류 시간 기반 히트맵 오버레이
- 색상 스케일: 짧은 체류(연파랑) → 장시간 체류(진빨강)
- 히트맵 전환 버튼: 지도 우측 상단 토글

| 이동기록 내보내기 (본인) | ✅ | ✅ | ✅ | - | - |
| 이동기록 내보내기 (연결 멤버) | - | - | - | ❌ | ✅ |
| 체류 지점 상세 주소 조회 | ✅ | 조 내 ✅ | 본인 ✅ | 등급별 | 등급별 |
| 이동기록 인사이트 조회 | ✅ | 조 내 ✅ | 본인 ✅ | ❌ | ✅ |

### 7.2 역할별 상세 규칙

**캡틴**
- 여행 내 **전체 멤버**의 이동기록을 조회할 수 있다.
- 프라이버시우선 등급 멤버의 마스킹 처리된 이동기록도 조회 가능하나 마스킹 해제 권한은 없다.
- SOS 발송 시에는 해당 멤버의 마스킹이 자동 해제된다.

**크루장**
- **소속 조 멤버**의 이동기록만 조회할 수 있다.
- 다른 조 멤버 또는 전체 멤버 이동기록 조회 불가.
- 조 편성 변경 시 변경 전 조원 이동기록은 더 이상 조회 불가.

**크루**
- **본인 이동기록만** 조회할 수 있다.
- 다른 멤버(캡틴/크루장 포함) 이동기록 조회 불가.
- 본인 이동기록 내보내기(PDF/CSV) 가능.

**가디언 (무료 / 유료 차등)**
- 연결된 멤버의 이동기록만 조회 가능 (연결되지 않은 멤버 조회 불가)
- 무료 가디언(`is_paid = FALSE`): **당일(24시간 이내)** 이동기록만 조회 가능
- 유료 가디언(`is_paid = TRUE`): **전체 보존 기간(30일 또는 90일)** 이동기록 조회 가능
- 접근 범위 제어: `tb_guardian_link.is_paid` 필드로 서버 측 검증

## §8. 프라이버시 등급별 동작 차이

### 8.1 등급별 이동기록 표시 규칙

| 기능 | 안전최우선 | 표준 | 프라이버시우선 |
|------|:--------:|:----:|:------------:|
| 전체 경로 표시 | ✅ | ✅ | 일정 연동 시간대만 ✅ |
| 체류 지점 표시 | ✅ | ✅ (반경 표시) | 일정 연동 시간대만 ✅ |
| 정확한 주소 표시 | ✅ | ❌ (도로명만) | ❌ (구/동 수준) |
| 비연동 시간대 위치 | ✅ | ✅ | ░ 마스킹 처리 |
| SOS 발송 시 마스킹 | 해당 없음 | 해당 없음 | 자동 해제 |

### 8.2 안전최우선 등급 (safety_first)

- **전체 경로 + 체류 지점 + 정확 주소** 표시
- 타임라인 모든 시간대 데이터 표시
- 지도 뷰에서 위치 정확도 표시 원 표시
- 마스킹 적용 없음

### 8.3 표준 등급 (standard)

- **경로 + 체류 지점(반경 표시)** 표시, 정확 주소 비공개
- 체류 지점 팝업: 정확 주소 대신 "도로명 수준" 표시 (예: "강남대로 인근")
- 타임라인 모든 시간대 데이터 표시
- 위치 정확도 표시 원 미표시

### 8.4 프라이버시우선 등급 (privacy_first)

- **일정 연동 시간대(TB_LOCATION_SCHEDULE ON 구간)만** 경로·체류 표시
- 비연동 시간대: 마스킹 처리 (반경 500m 원으로 흐림)
  - 타임라인: 해당 구간 회색 배경 + "비공개 구간" 텍스트
  - 지도: 해당 시간대 폴리라인 점선 + 회색 처리
- 정확 주소 비공개 (구/동 수준만 표시)
- 마스킹 로그: `tb_event_log`에 `event_type = 'privacy_mask_applied'` 기록

### 8.5 프라이버시 마스킹 상세 규칙

| 항목 | 규칙 |
|------|------|
| 마스킹 적용 대상 | 프라이버시우선 등급 + 비연동 시간대 |
| 마스킹 범위 | 실제 좌표를 반경 500m 격자 중심으로 스냅 |
| 마스킹 해제 조건 | SOS 발송 시 해당 위치 자동 해제 |
| 마스킹 해제 범위 | SOS 발송 시점 기준 ±30분 구간 해제 |
| 마스킹 해제 로그 | `tb_event_log`에 `event_type = 'privacy_mask_lifted'` + 해제 사유 기록 |
| 마스킹 재적용 | SOS 종료 후 자동 재적용 |
| 본인 조회 시 마스킹 | 적용 안 함 (본인은 마스킹 없이 전체 경로 확인 가능) |

## §9. 가디언 과금별 이동기록 접근 범위 차등 (§09.3 연계)

### 9.1 접근 범위 정의

| 가디언 유형 | is_paid | 조회 가능 범위 | 과금 |
|-----------|:-------:|:------------:|:----:|
| 무료 가디언 | FALSE | 당일(24시간 이내) 이동기록 | 무료 |
| 유료 가디언 | TRUE | 전체 보존 기간(30일 또는 90일) 이동기록 | 1,900원/여행 |

### 9.2 서버 측 접근 범위 제어 로직

```
가디언 이동기록 조회 요청
    │
    ▼
tb_guardian_link.status = 'accepted' 확인
    │
    ├─ NO → 403 Forbidden
    │
    ▼
tb_guardian_link.is_paid 확인
    │
    ├─ FALSE (무료) → 조회 범위: recorded_at >= NOW() - INTERVAL '24 hours'
    │
    └─ TRUE (유료) → 조회 범위: recorded_at >= trip.started_at (전체 보존 기간)
```

### 9.3 과금 전환 유도 UI

- 무료 가디언이 당일 범위를 초과하는 이동기록 조회 시도 시:
  - "2026-03-01 이동기록을 보려면 유료 가디언으로 전환하세요" 안내 모달 표시
  - 모달 내 "1,900원으로 전체 기간 조회하기" CTA 버튼
  - 취소 시 당일 이동기록만 표시

### 9.4 tb_guardian_link.is_paid 연동

- 결제 완료 시: `TB_PAYMENT → TB_GUARDIAN_LINK.is_paid = TRUE` 업데이트
- 가디언 해제 후 재연결 시: 동일 여행 내 재연결은 추가 과금 없음 (비즈니스 원칙 §09.3)
- `is_paid = TRUE`인 가디언이 연결 해제되어도 이미 결제된 요금은 환불 불가

## §10. 이동기록 보존 기간 정책 (§13 연계)

### 10.1 보존 기간 이원화

| 조건 | 보존 기간 | 근거 |
|------|:--------:|------|
| 일반 멤버 (유료 가디언 미연결) | **30일** | 비즈니스 원칙 §13.1 |
| 유료 가디언 연결 멤버 | **90일** | 비즈니스 원칙 §13.1 |
| 미성년자 멤버 위치 데이터 | **30일** | 비즈니스 원칙 §13.1, §10.2 |
| SOS 관련 위치 데이터 | **3년** | 비즈니스 원칙 §13.1 |

> 보존 기간 기산점: 여행 종료(`tb_trip.ended_at`) 시점

### 10.2 보존 만료 전 알림

- 보존 만료 **7일 전**: 해당 멤버 및 유료 가디언에게 알림 발송
  - 알림 메시지: "N일 후 이동기록이 삭제됩니다. 내보내기로 보관하세요."
- 무료 가디언에게는 만료 알림 미발송 (당일 이동기록만 조회 가능하므로)

### 10.3 수동 백업 (내보내기)

- 멤버 본인: 본인 이동기록 내보내기 가능 (PDF / CSV)
- 유료 가디언: 연결 멤버 이동기록 내보내기 가능 (PDF / CSV)
- 내보내기 형식:
  - **PDF**: 지도 이미지(경로 캡처) + 타임라인 텍스트 요약
  - **CSV**: 위치 포인트(위도, 경도, 시각, 체류 여부) 원시 데이터
- 내보내기 이력: `tb_location_access_log`에 기록

### 10.4 삭제 방식 (비즈니스 원칙 §13.2 적용)

1. **소프트 삭제**: 여행 종료 즉시 `deleted_at` 마킹
2. **익명화**: 보존 기간 만료 후 `user_id` 제거, 통계 데이터만 보관
3. **물리 삭제**: 익명화 후 추가 90일 경과 시 완전 삭제

## §12. 체류 지점 분석 알고리즘

### 12.1 체류 판정 기준

| 조건 | 값 |
|------|-----|
| 반경 | 100m 이내 |
| 최소 체류 시간 | 5분 이상 |
| 위치 포인트 최소 수 | 3개 이상 |
| 이동 속도 기준 | activity_type = 'still' 또는 speed < 1.0 m/s |

**판정 알고리즘 흐름:**
```
위치 포인트 스트림 입력
    │
    ▼
이동 세션 종료 감지 (is_movement_end = TRUE)
    │
    ▼
이전 포인트들 역방향 탐색
    │
    ├─ 반경 100m 내, 5분 이상, 3포인트 이상 → 체류 지점 생성 (TB_STAY_POINT INSERT)
    │
    └─ 조건 미충족 → 체류 지점 미생성
```

### 12.2 클러스터링 알고리즘 (인접 체류 지점 병합)

- **병합 조건**: 두 체류 지점의 중심 간 거리 ≤ 200m AND 체류 시간 간격 ≤ 30분
- 병합 시: `center_lat/center_lng`은 두 중심의 가중 평균 (체류 시간 기준 가중)
- 병합 결과 `duration_minutes`는 두 체류 시간의 합산
- 지도 뷰 줌 레벨에 따라 클러스터링 동적 조정 (§6.3 참조)

### 12.3 체류 지점 이상 감지

- **위치 점프**: 두 포인트 간 이동 거리 > 10km 이며 시간 차이 < 60초 → 이상 포인트로 마킹, 체류 판정 제외
- **정확도 불량**: accuracy > 100m → 체류 판정에서 제외, 타임라인에 "GPS 약함" 표시

## §14. 에러 및 엣지케이스 처리

### 14.1 GPS 관련 에러

| 상황 | 처리 방법 |
|------|----------|
| GPS 신호 없음 (터널, 실내) | "마지막 갱신: N분 전" 표시, 타임라인에 회색 간격 표시 |
| GPS 정확도 불량 (> 50m) | 위치 불확실 원 표시 + "GPS 약함" 배지 |
| GPS 완전 꺼짐 | "위치 서비스를 켜주세요" 배너, 마지막 알려진 위치 표시 |
| 위치 점프 이상 감지 | 이상 포인트 필터링 (§12.3), 타임라인에 "⚠ 비정상 위치" 마크 |

### 14.2 기기 전원 꺼짐 처리

- 전원 꺼지기 직전 `tb_power_event`에 이벤트 기록 (기기 펌웨어 레벨)
- 재부팅 후 재접속 시 간격 구간을 타임라인에 "기기 꺼짐" 이벤트로 표시
- 간격 발생 처리:
  - 간격 < 10분: 앞뒤 포인트를 직선으로 연결 (추정 경로 표시)
  - 간격 10분~1시간: 타임라인에 "--- 데이터 없음 (기기 꺼짐) ---" 구분자
  - 간격 > 1시간: 이동 세션 분리 처리

### 14.3 시간대 변경 (해외 여행)

- 비행 중 시간대 변경: 두 포인트 사이 시간대 전환이 감지된 경우 타임라인에 시간대 변경 이벤트 삽입
- 역방향 시간 발생 (서머타임 종료 등): UTC 기반으로 시간 정렬, 로컬 표시는 중복 표시 허용
- 날짜 변경선(IDL) 통과: UTC 기준으로 날짜 계산, 로컬 날짜 구분선 양쪽에 표시

### 14.4 오프라인 및 동기화 에러

- 오프라인 저장 실패: 로컬 DB에 저장 후 `is_offline = TRUE` 마킹, 복귀 시 재시도
- 동기화 충돌: 서버 타임스탬프(`recorded_at`) 기준 Last Write Wins
- 중복 포인트: `(user_id, recorded_at)` 기준 중복 제거 (UPSERT)

### 14.5 보존 기간 만료 에러

- 만료된 이동기록 조회 요청: `404 Not Found` + "이동기록 보존 기간(N일)이 만료되었습니다." 메시지
- 부분 만료 (일부 날짜만 삭제): 삭제된 날짜 구간을 타임라인에 "--- 기간 만료 ---" 표시

### 14.6 가디언 접근 권한 에러

| 상황 | 에러 처리 |
|------|----------|
| 무료 가디언이 당일 초과 조회 시도 | 과금 전환 안내 모달 표시 (§9.3) |
| 연결 해제된 가디언의 조회 시도 | `403 Forbidden` + "가디언 연결이 해제되었습니다." |
| 연결 대기중(pending) 가디언의 조회 시도 | `403 Forbidden` + "가디언 초대를 수락하면 이동기록을 확인할 수 있습니다." |

```sql
CREATE TABLE tb_location (
    location_id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id             VARCHAR(128) NOT NULL REFERENCES tb_user(user_id) ON DELETE CASCADE,

    -- 좌표
    latitude            DECIMAL(10, 8) NOT NULL,
    longitude           DECIMAL(11, 8) NOT NULL,
    accuracy            DECIMAL(6, 2),
    altitude            DECIMAL(8, 2),

    -- PostGIS 지원

    -- 주소 정보 (역지오코딩)

    battery_level       INTEGER,
    network_type        VARCHAR(20),

    -- 추적 모드
    tracking_mode       VARCHAR(20) DEFAULT 'normal',
                        -- normal | power_saving | minimal | sos

    -- 이동 세션
    movement_session_id UUID,
    is_movement_start   BOOLEAN DEFAULT FALSE,
    is_movement_end     BOOLEAN DEFAULT FALSE,

    -- Activity Recognition
    activity_type       VARCHAR(20),
                        -- still | walking | running | on_bicycle | in_vehicle | unknown
    activity_confidence INTEGER,

    -- 순차 인덱스
    i_idx               BIGINT,

    -- 메타데이터

    synced_at           TIMESTAMPTZ,

    CONSTRAINT chk_latitude  CHECK (latitude  BETWEEN -90  AND 90),
```

### 15.4 이동기록 조회 쿼리 예시

**무료 가디언 당일 이동기록 조회:**
```sql
SELECT l.*
FROM tb_location l
JOIN tb_guardian_link gl
    ON gl.member_id = l.user_id
    AND gl.guardian_id = :guardian_user_id
    AND gl.trip_id = :trip_id
    AND gl.status = 'accepted'
    AND gl.is_paid = FALSE
WHERE l.user_id = :member_id
  AND l.recorded_at >= NOW() - INTERVAL '24 hours'
ORDER BY l.recorded_at ASC;
```

**유료 가디언 전체 기간 이동기록 조회:**
```sql
SELECT l.*
FROM tb_location l
JOIN tb_guardian_link gl
    ON gl.member_id = l.user_id
    AND gl.guardian_id = :guardian_user_id
    AND gl.trip_id = :trip_id
    AND gl.status = 'accepted'
    AND gl.is_paid = TRUE
JOIN tb_trip t ON t.trip_id = gl.trip_id
WHERE l.user_id = :member_id
  AND l.recorded_at >= t.started_at
ORDER BY l.recorded_at ASC;
```

## §16. 오프라인 대응

### 16.1 오프라인 시 이동기록 로컬 저장

- 네트워크 단절 시 위치 포인트를 **기기 로컬 DB**(SQLite)에 저장
- 로컬 저장 최대 용량: 10,000 포인트 (약 48시간 분량, 30초 간격 기준)
- 용량 초과 시 **FIFO(가장 오래된 포인트 삭제)** 방식으로 순환 저장
- `is_offline = TRUE` 마킹으로 서버 동기화 미완료 포인트 식별

### 16.2 네트워크 복귀 시 동기화

```
네트워크 복귀 감지
    │
    ▼
로컬 DB에서 is_offline = TRUE 포인트 일괄 조회
    │
    ▼
서버 POST /api/v1/locations/batch (배치 업로드)
    │
    ├─ 성공 → 서버 저장 완료, synced_at = NOW(), is_offline = FALSE
    │
    └─ 충돌 감지 (동일 user_id + recorded_at 중복)
           │
           └─ UPSERT (ON CONFLICT DO NOTHING) 처리
```

### 16.3 충돌 해결 규칙

| 충돌 유형 | 해결 방법 |
|----------|----------|
| 중복 포인트 (`user_id + recorded_at` 동일) | 기존 서버 데이터 유지 (무시) |
| 타임스탬프 역전 | UTC 기준 `recorded_at` 오름차순 정렬 후 삽입 |
| 오프라인 체류 지점 | 동기화 완료 후 서버에서 TB_STAY_POINT 재계산 트리거 |

### 16.4 오프라인 동작 통합 원칙(#16)과의 연동

- 이동기록 오프라인 정책의 세부 기술 사항은 **오프라인 동작 통합 원칙(#16)**에 위임
- 본 문서의 §16은 이동기록 도메인에 특화된 규칙만 정의
