## 1. 문서 목적 및 적용 범위

### 1.1 목적

본 문서는 SafeTrip 앱의 **바텀시트(Bottom Sheet)** 컴포넌트의 5단계 상태 머신, 제스처 규칙, 탭 전환 시 상태 보존 방식, SOS 발동 시 동작을 정의한다. 모든 탭별 원칙 문서(Tier 3)는 본 문서의 바텀시트 규칙을 준수해야 한다.

### 1.2 적용 범위

- SafeTrip 모바일 앱(Flutter) 메인 화면의 바텀시트
- 4개 탭(일정 / 멤버 / 채팅 / 안전가이드) 전체
- 역할(캡틴 / 크루장 / 크루 / 가디언) 전체
- 여행 상태(`planning` / `active` / `completed` / `none`) 전체

### 1.3 구현 기술 기반

Flutter `DraggableScrollableSheet` 위젯을 기반으로 구현하며, 각 상태(snapSize)는 미디어 쿼리 기반 비율로 계산한다.

## 2. 바텀시트 5단계 상태 정의

바텀시트는 아래 5가지 상태를 가진다. 각 상태는 `BottomSheetState` enum으로 관리한다.

| 단계 | 상태명 | 높이 기준 | 지도 노출 비율 | 설명 |
|:----:|--------|----------|:-------------:|------|
| **1** | `collapsed` | 탭 바 + 헤더만 노출 (약 72dp) | 90% | 탭 제목과 바만 보이는 최소 상태 |
| **2** | `peek` | 화면 하단 1/4 | 75% | 탭 컨텐츠 일부 미리보기 |
| **3** | `half` | 화면 절반 | 50% | 기본 콘텐츠 탐색 상태 |
| **4** | `expanded` | 화면 하단 1/4만 지도 노출 | 25% | 상세 콘텐츠 열람 상태 |
| **5** | `full` | 전체 화면 | 0% | 지도가 완전히 숨겨지는 최대 상태 |

### 2.1 상태 변수 정의

```dart
enum BottomSheetState {
  collapsed, // snapSize: 0.10
  peek,      // snapSize: 0.25
  half,      // snapSize: 0.50
  expanded,  // snapSize: 0.75
  full,      // snapSize: 1.00
}
```

### 2.2 각 상태에서의 UI

| 상태 | 지도 조작 | 탭 제목 표시 | 탭 컨텐츠 표시 | SOS 버튼 |
|------|:--------:|:-----------:|:-------------:|:--------:|
| collapsed | ✅ 가능 | ✅ | ❌ | ✅ 표시 |
| peek | ✅ 부분 | ✅ | 일부 | ✅ 표시 |
| half | △ 제한 | ✅ | ✅ | ✅ 표시 |
| expanded | ❌ 불가 | ✅ | ✅ | ✅ 표시 |
| full | ❌ 불가 | ✅ | ✅ | ✅ 표시 (탭 바 위 고정) |

> **SOS 버튼은 모든 상태에서 항상 접근 가능해야 한다.** (화면 구성 원칙 §3.3 안전 최우선 원칙)

## 3. 상태 전환 트리거 및 규칙

### 3.1 전환 트리거 매트릭스

| 트리거 | 이전 상태 | 이후 상태 | 애니메이션 |
|--------|----------|----------|-----------|
| 위로 드래그 (긴 제스처) | any | expanded / full | spring (300ms) |
| 위로 드래그 (짧은 제스처) | collapsed | peek | spring (200ms) |
| 아래로 드래그 | any | 이전 단계 / collapsed | spring (200ms) |
| 탭 탭(tap) 클릭 | collapsed | half | easeInOut (250ms) |
| 탭 콘텐츠 스크롤 최상단 → 추가 아래 드래그 | expanded/full | half | spring (300ms) |
| 키보드 출현 | any | full (자동) | 시스템 키보드 애니메이션 동기화 |
| 키보드 닫힘 | full (키보드) | 이전 기억 상태 | spring (250ms) |
| SOS 발동 | any | collapsed (강제) | fade (150ms) |
| SOS 해제 | collapsed (SOS) | peek | spring (250ms) |
| 탭 전환 | any | 탭별 기본 높이 (§5 참조) | easeInOut (200ms) |

### 3.2 상태 전환 다이어그램

```
collapsed ⇌ peek ⇌ half ⇌ expanded ⇌ full
    ↑                                    |
    └────────────────────────────────────┘
           (SOS 발동: 모든 상태 → collapsed)
```

### 3.3 전환 제약

- **full → collapsed** 직접 전환: 아래로 빠른 플릭(velocity > 2000 dp/s)일 때만 허용
- **collapsed → full** 직접 전환: 두 손가락 위 스와이프 제스처로만 허용
- **SOS 상태에서** 바텀시트는 `collapsed` 이하로 잠금 (사용자가 위로 드래그해도 collapsed 유지)

## 4. 제스처 규칙

### 4.1 드래그 제스처

| 제스처 | 설명 | 임계값 |
|--------|------|--------|
| **위 드래그 (Up Drag)** | 바텀시트 높이 증가 | 시작 Y 기준 delta > 20dp |
| **아래 드래그 (Down Drag)** | 바텀시트 높이 감소 | 시작 Y 기준 delta < -20dp |
| **빠른 플릭 위 (Up Flick)** | 다음 snap 포인트로 이동 | velocity > 1000 dp/s |
| **빠른 플릭 아래 (Down Flick)** | 이전 snap 포인트로 이동 | velocity > 1000 dp/s |

### 4.2 Snap 동작

드래그 종료 시 **가장 가까운 snap 포인트**로 자동 이동한다.

- snap 포인트: `[0.10, 0.25, 0.50, 0.75, 1.00]`
- 방향성 고려: 드래그 방향이 위라면 현재 위치 이상의 snap, 아래라면 이하의 snap 우선 선택

### 4.3 콘텐츠 스크롤과 드래그 충돌 해소

| 조건 | 동작 |
|------|------|
| 바텀시트가 `full` 미만 + 콘텐츠 최상단 | 위 드래그 시 바텀시트 높이 증가 |
| 바텀시트가 `full` + 콘텐츠 최상단 + 아래 드래그 | 바텀시트 높이 감소 |
| 바텀시트가 `full` + 콘텐츠 스크롤 중 | 내부 콘텐츠 스크롤 (바텀시트 높이 변경 없음) |

### 4.4 탭 바 탭(Tap) 동작

| 현재 상태 | 탭 탭(tap) 결과 |
|----------|----------------|
| collapsed | → half로 전환 |
| peek / half | 동일 탭 재탭 시 → collapsed로 전환 |
| expanded / full | 동일 탭 재탭 시 → half로 전환 |
| 다른 탭 탭(tap) | → 탭 전환 + 탭별 기본 높이로 이동 |

## 5. 탭별 기본 높이 및 최대 높이

### 5.1 탭 전환 시 적용 높이

| 탭 | 기본 높이 (진입 시) | 최대 허용 높이 | 비고 |
|----|:------------------:|:--------------:|------|
| **일정탭** | `half` | `full` | 일정 목록 스크롤 고려 |
| **멤버탭** | `peek` | `full` | 멤버 수 적을 경우 peek 충분 |
| **채팅탭** | `expanded` | `full` | 채팅 입력창 고려 → 키보드 출현 시 full |
| **안전가이드탭** | `half` | `full` | 외교부 정보 탐색 고려 |

### 5.2 여행 상태별 초기 높이

| 여행 상태 | 앱 시작 시 초기 높이 | 비고 |
|----------|:-----------------:|------|
| `none` (여행 없음) | `collapsed` | 지도 노출 최대화 |
| `planning` | `collapsed` | — |
| `active` | `collapsed` | 위치 확인이 주 목적 |
| `completed` | `half` | 기록 탐색 |

## 6. 키보드 출현 시 동작

### 6.1 키보드 감지 및 반응

| 단계 | 동작 |
|------|------|
| **키보드 출현** | 바텀시트를 `full`로 자동 전환 (입력 영역 확보) |
| **바텀시트 상태 기억** | 키보드 출현 직전 상태를 `_preKeyboardState`에 저장 |
| **키보드 닫힘** | `_preKeyboardState`로 복원 |
| **뒤로 가기 제스처** | 키보드 닫힘 → 상태 복원 |

### 6.2 채팅탭 특수 처리

채팅탭은 메시지 입력창이 상시 표시되므로 다음 규칙을 추가 적용한다:

- 입력창 탭 시 → 바텀시트 `full` + 키보드 출현
- 메시지 목록이 아래로 자동 스크롤 (최신 메시지 표시)
- 키보드 닫힘 시 → `expanded` 상태로 복원 (채팅 입력 컨텍스트 유지)

## 7. 탭 전환 시 상태 보존 규칙

### 7.1 기본 정책

**탭 전환 시 현재 바텀시트 상태를 유지하는 것을 원칙**으로 한다. 단, 일부 탭은 최소 높이 요구사항이 있어 예외를 적용한다.

### 7.2 탭 전환 시 높이 결정 로직

```
if (현재 높이 >= 탭의 최소 요구 높이):
    현재 높이 유지
else:
    탭의 기본 높이로 전환 (easeInOut 200ms)
```

### 7.3 탭별 최소 요구 높이

| 탭 | 최소 요구 높이 | 이유 |
|----|:-----------:|------|
| 일정탭 | `peek` | 일정 항목 최소 1개 이상 표시 |
| 멤버탭 | `peek` | 멤버 카드 최소 1개 표시 |
| 채팅탭 | `expanded` | 입력창 + 메시지 영역 확보 |
| 안전가이드탭 | `peek` | 서브탭 선택 영역 표시 |

### 7.4 탭 내 세부 화면(상세뷰) 전환 시

탭 내부에서 세부 화면으로 전환 시(예: 일정 상세, 멤버 프로필) 바텀시트를 `full`로 전환한다. 뒤로 가기 시 이전 탭 상태로 복원한다.

## 8. 역할별 접근 권한 매트릭스

| 기능 | 캡틴 | 크루장 | 크루 | 가디언 |
|------|:----:|:------:|:----:|:------:|
| 바텀시트 수동 조작 | ✅ | ✅ | ✅ | ✅ |
| 탭 전환 (일정탭) | ✅ | ✅ | ✅ | ✅ |
| 탭 전환 (멤버탭) | ✅ | ✅ | ✅ | ✅ |
| 탭 전환 (채팅탭) | ✅ | ✅ | ✅ | ✅ |
| 탭 전환 (안전가이드탭) | ✅ | ✅ | ✅ | ✅ |
| 탭 내 편집 기능 접근 | 역할별 상이 | 역할별 상이 | 제한 | 읽기 전용 |

### 8.1 가디언 특수 규칙

- 가디언은 **자신과 연결된 멤버의 여행 데이터만** 열람 가능 (비즈니스 원칙 §01.1)
- 가디언은 SOS 버튼이 표시되지 않음 (비즈니스 원칙 §05)
- 채팅탭: 가디언은 멤버→가디언 채널만 접근 가능 (그룹 채팅 접근 불가)

### 8.2 여행 없는 상태(`none`)에서의 바텀시트

| 역할 | 표시 내용 |
|------|----------|
| 모든 역할 | collapsed 상태, 탭 바 비활성화, "+ 새 여행 만들기" CTA만 표시 |
| 가디언 | 연결된 여행 없는 경우 연결 요청 안내 화면 |

## 9. 프라이버시 등급별 동작 차이

프라이버시 등급(비즈니스 원칙 §04)에 따라 바텀시트 내 표시 정보가 달라진다.

| 등급 | 명칭 | 바텀시트 내 위치 정보 표시 |
|:----:|------|--------------------------|
| **1등급** | 안전 최우선 | 멤버탭: 정확한 위치 주소 표시 |
| **2등급** | 표준 | 멤버탭: 대략적 위치(동 단위) 표시 |
| **3등급** | 프라이버시 우선 | 멤버탭: 위치 숨김, 마지막 업데이트 시간만 표시 |

> 프라이버시 등급 변경 시 바텀시트 내 표시 정보는 **즉시(real-time)** 반영된다.

## 10. SOS 발동 시 바텀시트 동작

### 10.1 SOS 발동 흐름

```
SOS 버튼 3초 장누르기 (또는 Watchdog 자동 발동)
    ↓
바텀시트 → collapsed 강제 전환 (fade 150ms)
    ↓
SOS 오버레이 화면 표시 (전체 화면 최상단 z-index)
    ↓
지도 자동 센터링 (SOS 발신자 위치)
    ↓
바텀시트 잠금 (사용자 드래그 비활성화)
```

### 10.2 SOS 상태 중 바텀시트 규칙

| 항목 | 규칙 |
|------|------|
| 바텀시트 상태 | `collapsed` 강제 고정 |
| 사용자 드래그 | 비활성화 |
| 탭 전환 | 비활성화 |
| SOS 버튼 | 항상 표시 (SOS 해제 버튼으로 전환) |
| 오버레이 | SOS 알림 배너 상단 고정 표시 |

### 10.3 SOS 해제 시 복원

- SOS 해제 후 바텀시트 잠금 해제
- `peek` 상태로 복원 (spring 250ms)
- 지도 센터링 유지 → 사용자가 지도 조작 시 자동 해제

### 10.4 SOS 수신자 (가디언/다른 멤버) 동작

| 역할 | SOS 수신 시 바텀시트 동작 |
|------|--------------------------|
| 멤버 (캡틴/크루장/크루) | 푸시 알림 → 앱 진입 시 멤버탭 자동 열림 (SOS 발신자 강조) |
| 가디언 | 푸시 알림 → 앱 진입 시 SOS 알림 상세 화면 |

## 11. 에러 및 엣지케이스 처리

| 케이스 | 처리 방법 |
|--------|----------|
| **화면 회전 (가로/세로)** | 상태 유지, 절대 높이(px) 재계산하여 비율 보정 |
| **분할 화면(Split Screen)** | collapsed 상태로 리셋, 분할 화면 안내 메시지 표시 |
| **바텀시트 컨텐츠 로딩 중** | 스켈레톤 UI 표시, 높이는 peek 유지 |
| **데이터 없음(Empty State)** | 탭별 Empty State UI 표시, 높이는 half 유지 |
| **네트워크 에러** | 에러 배너 표시, 바텀시트 상태 유지 |
| **앱 백그라운드 → 포그라운드** | 이전 상태 복원 |
| **딥링크 진입** | 딥링크 대상 탭 자동 전환 + 해당 탭 기본 높이 적용 |
| **접근성 모드(TalkBack/VoiceOver)** | full 상태 기본 유지 (화면 판독기 접근성 고려) |

## 12. 오프라인 대응

바텀시트 동작 자체는 로컬 UI 로직이므로 오프라인 영향을 최소화한다.

| 상황 | 동작 |
|------|------|
| 오프라인 상태 진입 | "오프라인 모드" 배너를 바텀시트 상단에 표시 |
| 탭 컨텐츠 로딩 불가 | 로컬 캐시 데이터 표시, "마지막 업데이트 시간" 표시 |
| SOS 오프라인 | SOS 동작은 SMS 폴백으로 처리 (오프라인 동작 원칙 §16 참조) |
| 탭 전환 제한 없음 | 오프라인 여부에 관계없이 탭 전환 가능 |
