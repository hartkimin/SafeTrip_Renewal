## 1. 문서 목적 및 적용 범위

### 1.1 목적

본 문서는 SafeTrip 메인 화면의 **지도 영역**이 제공하는 고유 기능 원칙을 정의한다. 7단계 레이어 시스템, 마커 클러스터링 규칙, 지도 상태 자동 전환, 터치 인터랙션 우선순위, 역할별 접근 범위, 추가 기능의 Phase 배치 등을 포함한다.

### 1.2 적용 범위

- **적용 대상**: SafeTrip Flutter 앱의 메인 지도 화면 (MapScreen)
- **적용 여행 상태**: `active` (실시간 위치 표시), `planning` (정적 장소 마커만), 미참여 (데모 투어 모드)
- **기준 문서**: SafeTrip_비즈니스_원칙_v5.1
- **제외 범위**: 멤버별 이동기록 상세 화면, 안전가이드 지도, 오프라인 안전가이드 지도 (각 별도 문서 참조)

## 2. 지도 설계 철학 5대 원칙

### G1: 지도는 항상 갱신된다 (정적 UI 배제)

지도 위의 모든 마커, 레이어, 상태 표시는 실시간으로 갱신된다. 고정된 스크린샷이나 캐시된 정적 UI를 사용하지 않는다. 단, 오프라인 상태에서는 마지막으로 알려진 위치(last known location)를 표시하며, 이 경우 "마지막 업데이트 시각"을 마커에 명시한다.

### G2: 정보 레이어 분리 (물리 지도 / 멤버 / 일정 / 안전 / 긴급)

지도 위의 정보는 논리적으로 분리된 레이어로 관리된다. 각 레이어는 독립적으로 ON/OFF 가능하며, 레이어 간 우선순위에 따라 렌더링 순서가 결정된다. 레이어 분리를 통해 다양한 역할과 상황에서 최적의 정보 밀도를 제공한다.

### G3: 시간축 정보 통합 (현재 위치 + 일정 경로 + 과거 발자취)

지도는 단순한 현재 위치 표시를 넘어, 과거(발자취), 현재(실시간 위치), 미래(일정 경로)의 시간축 정보를 통합하여 표시한다. 이를 통해 여행의 전체 흐름을 공간적으로 이해할 수 있게 한다.

### G4: 역할별 정보 밀도 차등 (캡틴 > 크루장 > 크루 > 가디언)

지도에 표시되는 정보의 양과 종류는 역할에 따라 다르다. 캡틴은 전체 멤버, 지오펜스, 일정, 안전시설 등 모든 정보를 열람한다. 가디언은 연결된 특정 멤버의 위치만 확인한다. 역할별 정보 밀도 차등은 정보 과부하를 방지하고 각 역할에 필요한 정보에 집중할 수 있도록 한다.

### G5: SOS 0초 진입 (지도가 어느 상태에서도 SOS 버튼 항상 표시)

지도 화면의 어떤 상태, 어떤 줌 레벨, 어떤 레이어 조합에서도 SOS 버튼은 항상 최상단에 표시된다. 지도가 로딩 중이거나 오프라인 상태에서도 SOS 버튼은 반드시 사용 가능한 상태를 유지한다. 이는 안전 최우선 원칙(G5)을 UI 레벨에서 보장하는 핵심 규칙이다.

## 3. 7단계 레이어 시스템

지도의 정보는 Layer 0(최하단)부터 Layer 6(최상단)까지 7단계 레이어로 구성된다. 레이어 번호가 높을수록 더 높은 우선순위로 렌더링된다.

### 레이어 정의 테이블

| 레이어 | 이름 | 내용 | 토글 가능 여부 | 우선순위 |
|--------|------|------|:--------------:|:--------:|
| **Layer 6** | 긴급 오버레이 | SOS 발동 시 전체 화면 빨간 오버레이, 발신자 정보 표시 | ❌ (SOS 시 강제 표시) | 최우선 |
| **Layer 5** | UI 컨트롤 | SOS 버튼, 여행정보카드, 레이어 설정 버튼, 알림 아이콘 | ❌ (항상 표시) | — |
| **Layer 4** | 이벤트/알림 마커 | 지오펜스 경보 마커, 출석 체크 위치 마커, 이벤트 팝업 | ✅ (캡틴/크루장 전용) | P1 |
| **Layer 3** | 일정/장소 마커 | 여행 일정 장소 핀, 일정 경로 라인, 장소 라벨 | ✅ | P1 |
| **Layer 2** | 멤버 위치 마커 | 역할별 색상 마커, 프라이버시 등급별 표시, 클러스터링 | ✅ | P0 |
| **Layer 1** | 안전시설 오버레이 | 병원, 경찰서, 대사관 핀 마커 (Phase 1 추가) | ✅ | P1 |
| **Layer 0** | 지도 타일 | Google Maps SDK / 오프라인 캐시 타일 | ❌ (항상 표시) | — |

### 레이어 토글 UI

- 레이어 설정 버튼 탭 시 레이어 토글 패널이 바텀시트로 표시된다.
- Layer 0, Layer 5는 토글 불가 (항상 표시).
- Layer 6은 SOS 발동 시 자동 활성화, 해제 시 자동 비활성화.
- 각 레이어의 ON/OFF 상태는 사용자 기기에 로컬 저장된다 (앱 재시작 시 유지).

## 4. 지도 상태 자동 전환 규칙

지도 카메라와 레이어 상태는 특정 이벤트 발생 시 아래 규칙에 따라 자동으로 전환된다.

### 전환 규칙 5종

| 우선순위 | 트리거 이벤트 | 지도 카메라 동작 | 레이어 동작 | 구현 단계 |
|:--------:|--------------|----------------|------------|:---------:|
| **P0** | SOS 발동 | SOS 발신자 위치로 즉시 이동 (줌 레벨 16 고정) | Layer 6 긴급 오버레이 강제 활성화 | Phase 1 |
| **P1** | 지오펜스 이탈 | 이탈 멤버 마커 위치로 카메라 이동 | Layer 4 이탈 경보 마커 표시, 경보음 | Phase 1 |
| **P1** | 앱 복귀 (백그라운드→포어그라운드) | 현재 내 위치로 카메라 복귀 | 직전 레이어 상태 유지 | Phase 1 |
| **P2** | 멤버 오프라인 | 카메라 이동 없음 (현재 뷰 유지) | 해당 멤버 마커에 "마지막 알려진 위치" 표시 | Phase 1 |
| **P2** | 일정 시작 | 일정 장소로 카메라 이동 (선택 알림, 강제 이동 아님) | Layer 3 해당 일정 마커 강조 표시 | Phase 1 |

### 자동 전환 충돌 해결 규칙

여러 전환 이벤트가 동시에 발생하면 우선순위 순서로 처리한다.

```
P0 (SOS) > P1 (지오펜스 이탈 / 앱 복귀) > P2 (멤버 오프라인 / 일정 시작)
```

P0 상태 (SOS 발동) 중에는 P1/P2 자동 전환을 큐에 보관하고, SOS 해제 후 순서대로 처리한다.

### 지도 카메라 기본 위치 규칙

| 여행 상태 | 기본 카메라 위치 | 줌 레벨 |
|----------|----------------|:-------:|
| `active` | 내 현재 위치 | 15 |
| `planning` | 여행 목적지 (첫 번째 장소 마커) | 12 |
| 여행 미참여 (데모) | 데모 투어 기본 위치 | 12 |

### 가디언 카메라 규칙

가디언 사용자의 지도 카메라 기본 위치는 **연결된 멤버만 포함하는 바운딩 박스**를 자동으로 피트(fit)한다.

- 연결 멤버가 1명: 해당 멤버 위치 중심, 줌 레벨 15
- 연결 멤버가 2명 이상: 전체 멤버 위치를 포함하는 최소 바운딩 박스 자동 피트
- 연결 멤버 위치 미확인 시: 여행 목적지 기준 중심 이동

## 5. 역할별 접근 권한 매트릭스

### 5.1 지도 기능 7종 × 4역할 매트릭스

| 기능 | 캡틴 | 크루장 | 크루 | 가디언 |
|------|:----:|:------:|:----:|:------:|
| 전체 멤버 위치 마커 열람 | ✅ | ✅ | ✅ | ❌ |
| 연결 멤버 위치 마커 열람 | ✅ | ✅ | ✅ | ✅ (연결 멤버만) |
| 일정 장소 마커 열람 | ✅ | ✅ | ✅ | 유료만 ✅ |
| 지오펜스 영역 표시 열람 | ✅ | ✅ | ✅ | ❌ |
| 안전시설 오버레이 열람 | ✅ | ✅ | ✅ | ✅ |
| 오프라인 지도 다운로드 | ✅ | ✅ | ✅ | ❌ |
| SOS 버튼 사용 | ✅ | ✅ | ✅ | ❌ |

> 가디언의 SOS 발송은 지도가 아닌 가디언 전용 안전 화면에서 처리한다.

### 5.2 마커 클러스터링 (공통)

| 줌 레벨 | 표시 방식 |
|:--------:|----------|
| 15 이상 | 개별 마커 (이름 라벨 포함) |
| 12 ~ 14 | 3명 이하 개별 표시, 4명 이상 클러스터 (숫자 배지) |
| 11 이하 | 클러스터만 표시, 여행 바운딩 박스 자동 피트 |

### 5.3 멤버 마커 역할별 색상

| 역할 | 마커 색상 | 비고 |
|------|:--------:|------|
| 캡틴 | 황금색 (#FFD700) | 별 아이콘 |
| 크루장 | 주황색 (#FF8C00) | 다이아몬드 아이콘 |
| 크루 | 파란색 (#2196F3) | 원형 아이콘 |
| 내 위치 | 초록색 (#4CAF50) | 현재 위치 펄스 애니메이션 |
| 가디언 (연결 멤버) | 보라색 (#9C27B0) | 방패 아이콘 |

### 5.4 터치 인터랙션 우선순위

지도에서 터치(탭) 이벤트가 발생하면 아래 순서로 처리된다.

```
SOS 버튼 > 멤버 마커 > 이벤트 마커 > 일정 마커 > 지오펜스 영역 > 빈 영역
```

- **멤버 마커 탭**: 멤버 프로필 미니 카드 표시 (이름, 역할, 마지막 업데이트 시각, 배터리)
- **이벤트 마커 탭**: 이벤트 상세 바텀시트 표시 (지오펜스 이탈, 출석 체크 등)
- **일정 마커 탭**: 일정 상세 미니 카드 표시 + [지도▶] 연동 정보
- **지오펜스 영역 탭**: 지오펜스 정보 팝업 (캡틴/크루장에게는 편집 버튼 포함)
- **빈 영역 탭**: 현재 열려있는 모든 미니 카드 및 팝업 닫기

## 6. 프라이버시 등급별 마커 표시 동작

비즈니스 원칙 §04 (위치 공유 원칙) 및 §02.7 (여행 프라이버시 등급)에 따라 멤버 마커 표시 방식이 결정된다.

### 6.1 안전 최우선 등급 (`safety_first`)

- **전체 멤버 마커를 항상 실시간으로 표시한다.**
- 비공유 시간 없음. 가디언에게도 항상 위치 표시.
- 마커 숨김 기능 비활성화.
- 미성년자 포함 여행에 강제 적용 (§10).

```
[지도 표시] 모든 멤버 마커 → 실시간 갱신 (항상)
[가디언 뷰] 연결 멤버 마커 → 실시간 갱신 (항상)
```

### 6.2 표준 등급 (`standard`)

- **위치 공유 ON 상태인 멤버만 마커를 표시한다.**
- 기본 24시간 공유 설정이므로, 특별히 OFF 설정하지 않은 한 전체 표시.
- 스케줄 OFF 시간대: 가디언에게 30분 간격 스냅샷 위치 (마커 업데이트 빈도 감소).

```
[지도 표시] 공유 ON 멤버 마커 → 실시간 갱신
            공유 OFF 멤버 → 마커 숨김 (비공유 상태 표시 없음, 프라이버시 보호)
[가디언 뷰] 스케줄 ON → 실시간 / 스케줄 OFF → 30분 스냅샷 (타임스탬프 표시)
```

### 6.3 프라이버시 우선 등급 (`privacy_first`)

- **일정 연동 시간대에만 멤버 마커를 표시한다.**
- 일정이 없는 시간 / 일정 종료 후: 마커를 숨기고 회색 "비공유" 상태 라벨 표시.
- 버퍼 구간(일정 시작 전 15분 ~ 종료 후 15분) 포함하여 마커 표시.
- 가디언에게는 일정 연동 시간대에만 위치 공유 (OFF 시간대 비공유).

```
[지도 표시] 일정 연동 ON 시간 → 실시간 마커 표시
            일정 연동 OFF 시간 → 마커 숨김 + "비공유" 회색 라벨 표시
[가디언 뷰] ON 시간대 → 실시간 / OFF 시간대 → 마커 없음 (비공유)
```

### 6.4 프라이버시 등급별 비교 요약

| 구분 | 안전 최우선 | 표준 | 프라이버시 우선 |
|------|:-----------:|:----:|:--------------:|
| 멤버 마커 기본 표시 | 항상 | 공유 ON 시 | 일정 연동 시간만 |
| 비공유 시 마커 표시 | — | 숨김 | "비공유" 회색 표시 |
| 가디언 마커 업데이트 | 실시간 | 실시간 / 30분 스냅샷 | 연동 시간만 |
| 마커 강제 숨김 가능 | ❌ | ✅ | ✅ |

## 7. 에러 및 엣지케이스 처리

### 7.1 위치 정보 관련 오류

| 상황 | 처리 방법 |
|------|----------|
| 멤버 GPS 신호 없음 | "마지막 알려진 위치" 마커 표시 + 업데이트 시각 표시 (회색 처리) |
| 멤버 위치 업데이트 5분 이상 없음 | 마커에 "오프라인" 배지 표시, 캡틴/크루장에게 알림 발송 |
| 모든 멤버 오프라인 | 지도에 "현재 위치 정보를 불러올 수 없습니다" 배너 표시 |
| 위치 권한 없음 (기기) | 위치 권한 안내 다이얼로그 표시, 내 위치 마커만 미표시 (타 멤버 마커는 정상) |
| 가디언 연결 멤버 오프라인 | 가디언 화면에 "마지막 알려진 위치 — HH:MM" 표시 |

### 7.2 지도 렌더링 오류

| 상황 | 처리 방법 |
|------|----------|
| Google Maps 로딩 실패 | 오프라인 캐시 타일로 대체 표시, 상태바에 "오프라인 지도" 안내 |
| 오프라인 캐시 없음 | 회색 배경 + 멤버 마커만 좌표 기반 점 표시 (타일 없이) |
| 마커 렌더링 실패 | 기본 핀 아이콘으로 폴백 (역할 색상은 유지) |
| 지오펜스 영역 로딩 실패 | Layer 4 비활성화, 재시도 버튼 표시 |

### 7.3 SOS 관련 엣지케이스

| 상황 | 처리 방법 |
|------|----------|
| SOS 발동 중 앱 재시작 | 앱 재시작 시 SOS 상태 서버에서 복원, Layer 6 즉시 재활성화 |
| SOS 발동 중 오프라인 | Layer 6 유지, SOS 로컬 큐잉, 네트워크 복구 시 자동 전송 |
| 동시 다수 SOS | 가장 최근 SOS 발신자 위치로 카메라 이동, SOS 목록 Layer 6에 표시 |
| SOS 발신자 위치 미확인 | "SOS 발신 — 위치 확인 중" 오버레이 표시 (마지막 알려진 위치로 이동) |

### 7.4 가디언 관련 엣지케이스

| 상황 | 처리 방법 |

## 8. DB 스키마

본 섹션은 지도 화면에서 직접 참조하는 테이블 구조를 명세한다. 전체 스키마는 `07_T2_DB_설계_및_관계_v3_4.md`를 참조한다.

### 8.1 `tb_location` — 멤버 위치 데이터

```sql
-- 멤버 실시간 위치 (지도 마커 소스)
CREATE TABLE tb_location (
  id                   UUID         PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id              UUID         NOT NULL REFERENCES tb_user(id),
  trip_id              UUID         NOT NULL REFERENCES tb_trip(id) ON DELETE CASCADE,
  latitude             DECIMAL(10,8) NOT NULL,
  longitude            DECIMAL(11,8) NOT NULL,
  geom                 GEOMETRY(Point, 4326),  -- PostGIS 공간 인덱스용
  accuracy_meters      DECIMAL(6,2),
  altitude_meters      DECIMAL(8,2),
  movement_session_id  UUID         REFERENCES tb_movement_session(id),
  recorded_at          TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
  created_at           TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);

-- 공간 인덱스
CREATE INDEX idx_location_geom ON tb_location USING GIST(geom);
-- 여행별 최신 위치 조회 최적화
CREATE INDEX idx_location_trip_user_recorded ON tb_location(trip_id, user_id, recorded_at DESC);
```

### 8.2 `tb_geofence` — 지오펜스 영역 (Layer 4)

```sql
-- 지오펜스 영역 정의 (지도 Layer 4 소스)
CREATE TABLE tb_geofence (
  id              UUID         PRIMARY KEY DEFAULT gen_random_uuid(),
  trip_id         UUID         NOT NULL REFERENCES tb_trip(id) ON DELETE CASCADE,
  center_lat      DECIMAL(10,8) NOT NULL,
  center_lng      DECIMAL(11,8) NOT NULL,
  radius_meters   INTEGER      NOT NULL CHECK (radius_meters > 0),
  label           VARCHAR(100),
  is_active       BOOLEAN      NOT NULL DEFAULT TRUE,
  created_by      UUID         REFERENCES tb_user(id),
  created_at      TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
  updated_at      TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);
```

CREATE TABLE tb_guardian_link (

## 9. 오프라인 대응

비즈니스 원칙 §11 (오프라인 기본 정책) 및 오프라인 동작 통합 원칙 (#16)에 따른 지도 화면 오프라인 대응을 정의한다.

### 9.1 오프라인 지도 타일 (사전 다운로드)

| 항목 | 규칙 |
|------|------|
| 다운로드 범위 | 여행 목적지 중심 반경 50km |
| 줌 레벨 | 10~16 |
| 최대 용량 | 500MB/국가 |
| 자동 다운로드 시점 | 여행 상태 `planning` → `active` 전환 시 Wi-Fi 연결 상태에서 자동 시작 |
| 수동 다운로드 | 설정 메뉴 → "오프라인 지도 다운로드" (Wi-Fi 권장 안내 포함) |
| 다운로드 만료 | 여행 종료(`completed`) 후 30일, 또는 용량 부족 시 가장 오래된 여행 순 삭제 |

### 9.2 오프라인 상태별 지도 동작

| 기능 | 오프라인 상태 | 대응 |
|------|:-----------:|------|
| 지도 타일 표시 | ⚠️ | 사전 다운로드된 오프라인 캐시 타일 표시 |
| 내 위치 표시 | ✅ | GPS(위성 신호)로 내 위치 계속 표시 가능 |
| 타 멤버 위치 | ❌ | 마지막 알려진 위치 + "오프라인 — HH:MM 기준" 표시 |
| SOS 발송 | ✅ | 로컬 큐잉 → 네트워크 복구 시 자동 재전송 (§11.1) |
| 지오펜스 감지 | ✅ | 기기 로컬에서 판정 (결과는 복구 후 서버 전송) |
| 안전시설 마커 | ⚠️ | 로컬 캐시된 안전시설 정보만 표시 |
| 마커 실시간 업데이트 | ❌ | 네트워크 필수 |

### 9.3 오프라인 UI 표시

- 화면 상단에 오렌지색 배너: "오프라인 상태 — 실시간 위치가 업데이트되지 않습니다"

- SOS 버튼 및 기능은 오프라인에서도 항상 활성 상태 유지 (G5 원칙)
