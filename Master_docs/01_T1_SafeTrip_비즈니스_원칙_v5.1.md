## §01 역할 체계 원칙

### 01.1 역할 정의

| 역할 | DB 값 | 설명 | 분류 |
|------|--------|------|------|
| 캡틴 | `captain` | 여행 생성자, 최상위 관리 권한 | 여행 멤버 |
| 크루장 | `crew_chief` | 부인솔자, 일정/지오펜스 편집 가능 | 여행 멤버 |
| 크루 | `crew` | 일반 여행 멤버 | 여행 멤버 |
| 가디언 | `guardian` | 보호자, 열람 + 긴급 알림 전용 | 보호자 |

### 01.2 역할 겸직 규칙

- 한 유저는 **서로 다른 여행**에서 다른 역할을 가질 수 있다.
- **캡틴 + 가디언 동시 보유 가능** (서로 다른 여행 기준)
- **동일 여행 내에서 캡틴/크루장/크루는 하나만 가능하다.**
- 동일 여행 내에서 여행 멤버(캡틴/크루장/크루)이면서 동시에 가디언일 수는 없다.
- 가디언은 독립적으로 동작하며, **다른 여행**의 멤버 역할과 별개로 부여 가능하다.

### 01.3 역할 변경

- 유저는 가입 후 여행 내에서 역할을 변경할 수 있다.
- 역할 변경은 여행 단위로 적용된다 (전역 변경이 아닌 여행별 변경).
- 캡틴 → 다른 역할 변경은 리더 위임을 통해서만 가능하다 (§07 참조).

### 01.4 B2B 역할 매핑

B2B 고객 유형별로 SafeTrip 역할이 다음과 같이 매핑된다:

| B2B 고객 | 캡틴 | 크루장 | 크루 | 가디언 |
|---------|------|-------|------|--------|
| 학교 (수학여행) | 인솔 교사 | 부담임/보조 교사 | 학생 | 학부모/학년 부장 |
| 여행사 (패키지) | 여행 가이드 | 보조 가이드 | 참가자 | 참가자 가족 |
| 기업 (출장) | 팀장/관리자 | 부팀장 | 팀원 | 팀원 가족/회사 안전담당자 |
| 보험사 (제휴) | 보험 계약자 | — | 피보험자 | 보험사 모니터링 |

> B2B 계약에서 역할 매핑이 사전에 정의되며, 초대코드 일괄 생성 시 해당 매핑이 자동 적용된다 (§12 참조).

## §02 여행 관리 원칙

### 02.1 여행 생성

- 여행 생성 시 생성자는 자동으로 해당 여행의 **캡틴**이 된다.
- 여행 1개 = 그룹 1개 (1:1 매핑).
- 여행 생성 시 캡틴은 다음을 함께 지정한다:
  - 여행 기본 정보 (여행명, 국가, 도시, 날짜)
  - **여행 프라이버시 등급** (02.6절 참조)
  - 위치 공유 모드 (§04 참조)
  - 미설정 시 기본값이 적용된다.

### 02.2 여행 참여 제약

- **캡틴/크루장/크루는 동일한 일정(날짜 범위가 겹치는)에 1개의 여행에만 참여할 수 있다.**
  - 일정 충돌 검증: 새 여행의 `start_date ~ end_date`가 기존 참여 여행의 기간과 1일이라도 겹치면 참여를 차단한다.
  - 검증 대상: `captain`, `crew_chief`, `crew` 역할이며, `active` 또는 `planning` 상태의 여행만 검사한다.
  - 검증 시점: 여행 생성, 초대코드 사용, 직접 멤버 추가, 기존 여행 일정 변경 시 모두 적용.
- **가디언은 일정 중복 제한이 없다.**

### 02.3 여행 기간 정책

여행 기간은 캡틴이 자유롭게 설정하되, 다음 규칙을 따른다.

- **1회 여행의 최대 기간**: 15일 이내로 자유롭게 설정 가능하다.
- **15일을 초과하는 여행**: 여행을 분할하여 각각 별도의 여행으로 생성해야 한다.
- **여행 분할 시 과금 기준**: 각 분할된 여행은 독립적인 여행으로 간주하여 개별 과금한다.
  - 예시: 30일 여행을 15일 + 15일로 분할 시 → 2건의 여행으로 각각 과금
  - 예시: 7일간의 일정을 3일 + 4일로 분할 시 → 2건의 개별 여행으로 각각 과금
- **과금 산정 시점**: 여행 생성 시 설정된 기간과 인원을 기준으로 산정한다. 기간 내에서 일자별 분리를 하더라도 같은 여행 내라면 추가 과금 없이 자유롭게 일정을 구성할 수 있다.

### 02.4 여행 삭제 및 탈퇴

| 행위 | 캡틴 | 크루장 | 크루 | 가디언 |
|------|:----:|:------:|:----:|:------:|
| 여행 삭제 | ✅ | ❌ | ❌ | ❌ |
| 여행 탈퇴 | 조건부 | ✅ | ✅ | ✅ |

- **여행 삭제는 캡틴만 가능하다.**
- 캡틴의 탈퇴 규칙:
  - 다른 멤버가 있는 경우 (`active` 또는 `planning` 상태): **리더 권한을 위임한 후에만** 탈퇴 가능.
  - **캡틴만 남은 경우**: 여행을 종료(`completed`)하거나 삭제한 후 탈퇴한다.
  - 여행 기간 종료 후(`completed` 상태): 위임 없이 탈퇴 가능.
- 탈퇴한 멤버의 위치 공유는 즉시 중단되며, 해당 멤버에 연결된 가디언 관계도 자동 해제된다.

### 02.5 여행 초대

- **캡틴과 크루장만 다른 유저를 여행에 초대할 수 있다.**
- 초대 방법: 초대코드, QR코드, 링크, 직접 검색 추가.
- 초대코드에는 대상 역할이 지정되어 있으며, 코드 사용 시 해당 역할로 자동 배정된다.

### 02.6 여행 상태 전환

```
planning → active → completed
                        │
                        ▼ (캡틴 확인 후, 24시간 이내 1회)
                     active (재활성화)
```

- `planning` → `active`: 캡틴이 여행 시작 처리 (수동 또는 출발일 기준 자동)
- `active` → `completed`: 캡틴이 여행 종료 처리 (수동 또는 귀국일 기준 자동)
- `completed` → `active` **재활성화**: 종료 후 24시간 이내에 캡틴이 1회 재활성화할 수 있다.
  - DB 컬럼: `tb_trip.reactivated_at`(마지막 재활성화 시각), `reactivation_count`(횟수, 최대 1회)
  - CHECK 제약: `reactivation_count <= 1`로 DB 레벨에서 1회 제한을 강제한다.

### 02.7 여행 프라이버시 등급

캡틴은 여행 생성 시 **프라이버시 등급**을 선택한다. 이 등급은 가디언 공유 정책, 기본 공유 스케줄, 기본 공개 범위의 기본값을 결정한다.

#### 등급 정의

| 등급 | DB 값 | 대상 여행 유형 | 가디언 공유 정책 | 기본 스케줄 |
|------|--------|---------------|:---------------:|------------|
| **안전 최우선** | `safety_first` | 미성년자 단체, 어학연수, 가족여행(미성년) | 항상 공유 | 24시간 |
| **표준** | `standard` | 일반 그룹 여행, 가족 여행(성인), 개인 여행 | 스케줄 연동 | 24시간 |
| **프라이버시 우선** | `privacy_first` | 업무 출장, 성인 패키지 투어, 컨퍼런스 | 스케줄 연동 | 일정 연동 |

- **미설정 시 기본값**: `standard` (표준)
- 프라이버시 등급은 여행 도중에도 캡틴이 변경할 수 있다. 변경 시 전 멤버 + 가디언에게 알림을 발송한다.
- 미성년자 멤버가 포함된 여행은 `safety_first` 등급만 허용된다 (§10 참조). 캡틴이 다른 등급을 선택하면 경고 후 자동 전환.

#### 등급별 가디언 공유 정책 상세

| 동작 | 안전 최우선 | 표준 | 프라이버시 우선 |
|------|:----------:|:----:|:--------------:|
| 스케줄 ON 시간대 가디언 공유 | ✅ | ✅ | ✅ |
| 스케줄 OFF 시간대 가디언 공유 | ✅ 유지 | ✅ 유지 (저빈도)* | ❌ 비공유 |
| 가디언 일시 중지 | ❌ 불가 | ✅ 최대 12시간 | ✅ 최대 24시간 |
| SOS 시 가디언 위치 수신 | ✅ 항상 | ✅ 항상 | ✅ 항상 |
| 지오펜스 이벤트 가디언 전달 | ✅ 항상 | ✅ 스케줄 ON 시만 | ❌ |

> *표준 등급의 "저빈도": 스케줄 OFF 시간대에 가디언에게 실시간 위치 스트리밍 대신 **30분 간격 스냅샷**을 제공한다. 실시간 추적이 아닌 안부 확인 수준의 정보를 제공하여, 안전 확보와 프라이버시의 균형을 맞춘다.

#### 등급별 기본 설정 프리셋 (캡틴이 수정 가능)

**안전 최우선 (`safety_first`)**
```
위치 공유 모드: 강제 공유
기본 스케줄: 24시간 공유
가디언 공유: 항상 (스케줄 무관)
가디언 일시 중지: 불가
권장 대상: 수학여행, 미성년자 어학연수
```

**표준 (`standard`)**
```
위치 공유 모드: 자유 설정
기본 스케줄: 24시간 공유
가디언 공유: 항상 (OFF 시간대 저빈도)
가디언 일시 중지: 최대 12시간
권장 대상: 일반 그룹 여행, 성인 가족 여행
```

**프라이버시 우선 (`privacy_first`)**
```
위치 공유 모드: 자유 설정
기본 스케줄: 일정 연동 (등록된 일정 시간에만 공유)
가디언 공유: 스케줄 연동 (OFF 시간대 비공유)
가디언 일시 중지: 최대 24시간
권장 대상: 업무 출장, 성인 패키지 투어
```

#### UI에서의 등급 선택 가이드

여행 생성 화면에서 캡틴이 등급을 선택할 때, 각 등급의 의미를 직관적으로 안내한다:

```

   "보호자가 항상 위치를 확인할 수 있습니다"
   미성년자 단체 여행, 어학연수에 적합

📍 표준 (기본값)
   "보호자가 위치를 확인하되, 프라이버시 시간을 존중합니다"
   일반 그룹 여행, 가족 여행에 적합

🔒 프라이버시 우선
   "일정 시간에만 위치를 공유합니다"
   업무 출장, 성인 패키지 투어에 적합
```

## §03 가디언(보호자) 시스템 원칙

### 03.1 가디언 연결 규칙

- **캡틴/크루장/크루는 각각 기본 최대 2명의 개인 가디언을 무료로 연결할 수 있다.**
- **3명째 개인 가디언부터는 추가 가디언 요금이 부과된다** (§09 과금 모델 참조).
  - 추가 가디언은 멤버당 최대 3명까지 유료로 연결 가능하다.
  - 따라서 1명의 멤버에 연결 가능한 개인 가디언 최대 수: 무료 2명 + 유료 3명 = 5명.
- **전체 가디언은 개인 가디언 상한과 별도로 관리한다.**
  - 전체 가디언은 여행당 최대 2명까지 등록할 수 있다 (무료).
  - 1명의 멤버에 연결될 수 있는 가디언의 최대 수: 개인 최대 5명 + 전체 2명 = 7명.
- 가디언은 동시에 여러 멤버의 가디언이 될 수 있다.
- 전체 가디언은 이후 새로 합류하는 멤버에게도 자동으로 관계가 적용된다.

### 03.2 가디언 위치 공유 범위

- 가디언은 **자신에게 연결된 멤버의 위치만** 공유받을 수 있다.
- 연결되지 않은 멤버의 위치는 조회할 수 없다.
- **가디언이 위치를 수신하는 조건은 여행 프라이버시 등급(§02.7)에 따라 달라진다.**

### 03.3 가디언 해제

- 여행 멤버가 자신에게 연결된 개인 가디언을 해제할 수 있다.
- 캡틴은 여행 내 모든 가디언 관계(개인/전체 포함)를 관리(추가/해제)할 수 있다.
- 전체 가디언의 해제는 캡틴만 가능하다.
- 멤버 탈퇴 시 해당 멤버에 연결된 개인 가디언 관계는 자동 해제된다 (전체 가디언은 유지).
- 미성년자 크루의 가디언 해제는 캡틴 승인이 필요하다 (§10 참조).
- 유료 추가 가디언 해제 시 이미 결제된 요금은 환불하지 않는다.

### 03.4 가디언 긴급 요청

프라이버시 우선 등급에서 가디언의 위치 접근이 제한되므로, 긴급 상황에 대응할 수단이 필요하다.

- **가디언은 프라이버시 등급과 무관하게 "긴급 위치 요청"을 보낼 수 있다.**
  - 긴급 위치 요청 시, 연결된 멤버에게 FCM 높은 우선순위 알림이 전송된다.
  - 멤버는 알림을 확인하고 **수동으로 위치 공유를 승인**하거나, 무시할 수 있다.
  - 승인 시 1회성으로 현재 위치가 가디언에게 전송된다 (지속 공유가 아님).
  - 이벤트 타입: `GUARDIAN_LOCATION_REQUEST`
- **안전 최우선 등급에서는 승인 없이 자동으로 위치가 전송된다** (이미 항상 공유 중이므로 추가 기능은 불필요하지만, 명시적 안부 확인 용도로 사용 가능).
- **표준 등급에서는 멤버 승인 없이 자동 응답**할 수 있다. `auto_responded=TRUE`, `auto_response_reason='standard_grade_auto'`로 기록된다.
- 요청 빈도 제한: 1시간에 최대 3회. (DB 인덱스 `idx_guardian_location_request_hourly`로 빈도 조회를 최적화한다.)

### 03.5 기업 출장 가디언 운영 모델

기업 출장에서도 팀원 개인의 안전을 위해 가디언을 지정할 수 있다. 운영 방식은 두 가지 모델을 제공한다.

#### 모델 A: 개별 설정 (팀원 자율)

- 팀원(크루)이 개인적으로 가디언을 등록한다.
- 가디언 대상: 팀원의 가족(배우자, 부모 등)
- 일반 B2C 가디언 규칙과 동일하게 적용된다.
- 프라이버시 우선 등급이 기본이므로, 업무 시간 외에는 가디언에게 위치가 공유되지 않는다.

#### 모델 B: 관리자 일괄 설정 (학교 모델 준용)

- 회사 안전담당자 또는 관리자가 B2B 관리 포털을 통해 가디언을 일괄 등록한다.
- 가디언 대상: 회사 안전담당자, HR 담당자, 또는 팀원 지정 가족
- 학교 수학여행의 "학부모 일괄 연결" 기능과 동일한 방식으로 처리한다.
- 가디언 연결 시 팀원(크루) 본인의 동의가 필수이다 (성인이므로 법정대리인 동의는 불필요).
- CSV 업로드를 통한 팀원-가디언 매핑 일괄 등록을 지원한다.

> **두 모델의 혼용**: 동일 여행에서 일부 팀원은 모델 A, 일부는 모델 B를 사용할 수 있다. 개인 가디언 상한(무료 2명 + 유료 3명)은 모델과 무관하게 동일하게 적용된다.

## §04 위치 공유 원칙

### 04.1 위치 공유 기본 구조

위치 공유 설정은 세 가지 요소로 구성된다.

```
[1] 여행 프라이버시 등급 (§02.7) → 전체 정책의 기조를 결정
    │
    └── [2] 위치 공유 모드 (강제 / 자유) → 멤버 간 공유 방식
           │
           └── [3] 공유 스케줄 (시간대 / 일정 연동 / 24시간)
                   │
                   └── [별도] 공개 범위 (전체 / 관리자만 / 지정 멤버)

[독립] 가디언 공유 정책 → 프라이버시 등급에 따라 자동 결정
```

### 04.2 위치 공유 모드

| 모드 | 설명 | 개인 스케줄 설정 | 개인 공개 범위 설정 |
|------|------|:----------------:|:-------------------:|
| **강제 공유** | 캡틴이 지정한 스케줄에 따라 전 멤버 위치 공유 | ❌ | ❌ |
| **자유 설정** | 각 멤버가 개인별 스케줄 및 공개 범위 설정 가능 | ✅ | ✅ |

- 여행 도중에도 캡틴은 모드를 변경할 수 있다. 변경 시 전 멤버에게 알림을 발송한다.

### 04.3 공유 스케줄 유형

스케줄은 **세 가지 유형** 중 하나를 선택한다.

#### 유형 A: 24시간 공유

- 여행 기간 내내 위치를 공유한다.
- 별도 설정 불필요.
- 프라이버시 등급 "안전 최우선"의 기본값.

#### 유형 B: 시간대 기반

- 하루 중 공유/비공유 시간대를 직접 지정한다.
- 시간대 스케줄은 `tb_location_schedule` 테이블에 요일별/시간대별로 저장된다.
  - `day_of_week`이 NULL이면 매일 동일 시간대 적용.
  - `share_start ~ share_end` 범위 밖에서는 자동 비공유.

```
예시:
  공유 시간: 오전 9시 ~ 오후 6시
  비공유 시간: 오후 6시 ~ 오전 9시
```

- 적용 범위 옵션:
  - **여행 전체 기간**에 동일하게 적용
  - **요일별**로 다른 시간대를 지정하여 적용
  - **특정 일자**에만 선택적으로 적용

#### 유형 C: 일정 연동

- **등록된 여행 일정(`tb_schedule`, `tb_travel_schedule`)의 시간에만 자동으로 위치가 공유된다.**
- 일정이 없는 시간대에는 자동으로 비공유 상태가 된다.
- 프라이버시 등급 "프라이버시 우선"의 기본값.

##### 일정 연동 상세 규칙

- **버퍼 시간**: 일정 시작 전 15분 ~ 일정 종료 후 15분 동안 위치 공유를 유지한다 (이동 시간 고려).
  - 버퍼 시간은 캡틴이 0분/15분/30분 중 선택 가능. 기본값: 15분.
- **일정 미등록 시**: 해당 날짜에 일정이 하나도 없으면 **종일 비공유**가 된다.
  - 일정 미등록 날짜에 대한 안내 알림을 캡틴에게 발송한다.
- **종일 일정(`all_day = true`)**: 해당 날짜 전체(00:00~23:59)를 공유 시간으로 처리한다.
- **연속 일정 간 간격이 버퍼 이내(30분 이하)인 경우**: 비공유로 전환하지 않고 연속 공유한다.
- 일정 연동 모드에서 멤버가 **수동으로 공유를 켤 수 있는 옵션**:
  - 자유 설정 모드일 때: 멤버가 일정 외 시간에도 자발적으로 공유를 켤 수 있다 (버튼 토글).
  - 강제 공유 모드일 때: 불가. 일정에 따른 자동 ON/OFF만 동작한다.

### 04.4 공개 범위 설정 (자유 설정 모드 전용)

| 옵션 | DB 값 (`visibility_type`) | 설명 |
|------|:------------------------:|------|
| **전체 공개** | `all` | 여행 내 모든 멤버에게 위치 공개 (기본값) |
| **관리자만** | `admin_only` | 캡틴과 크루장에게만 위치 공개 |
| **지정 멤버** | `specified` | 직접 선택한 멤버에게만 위치 공개 |

- 기본값은 **전체 공개**(`all`)이다.
- 위치 공유 설정은 여행 단위(`trip_id`)로 관리된다. 동일 그룹 내 여행별 설정이 분리된다.
- 공개 범위 설정은 스케줄과 독립적으로 동작한다.
- 공개 범위 설정은 **단방향**이다.
- 강제 공유 모드에서는 공개 범위 설정이 비활성화된다.
- 비공유 사유(선택적 비공개 vs 전체 비공유 vs 오프라인)는 구분하지 않는다.

### 04.5 가디언 위치 공유 — 프라이버시 등급별 정책

가디언에 대한 위치 공유는 **여행 프라이버시 등급(§02.7)**에 따라 동작이 달라진다.

#### 안전 최우선 등급 (`safety_first`)

```
스케줄 ON 시간대:   가디언 ✅ 실시간 공유
스케줄 OFF 시간대:  가디언 ✅ 실시간 공유 (항상)
가디언 일시 중지:   ❌ 불가
```

#### 표준 등급 (`standard`)

```
스케줄 ON 시간대:   가디언 ✅ 실시간 공유
스케줄 OFF 시간대:  가디언 ✅ 저빈도 공유 (30분 간격 스냅샷)
가디언 일시 중지:   ✅ 최대 12시간
```

#### 프라이버시 우선 등급 (`privacy_first`)

```
스케줄 ON 시간대:   가디언 ✅ 실시간 공유
스케줄 OFF 시간대:  가디언 ❌ 비공유
가디언 일시 중지:   ✅ 최대 24시간
```

#### 등급별 비교 요약

```
                    안전 최우선        표준              프라이버시 우선
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
ON 시간대 가디언     실시간            실시간            실시간
OFF 시간대 가디언    실시간            30분 스냅샷       ❌ 비공유
일시 중지 가능       ❌                최대 12시간       최대 24시간
SOS 시 위치 전달     ✅ 항상           ✅ 항상           ✅ 항상
긴급 위치 요청       불필요(항상 공유)  1회성 위치 전달   멤버 승인 후 전달

```

### 04.6 가디언 일시 중지

- 멤버는 가디언 연결을 해제하지 않고 **일시 중지**를 요청할 수 있다.
- 일시 중지 최대 시간은 프라이버시 등급에 따라 다르다:
  - 안전 최우선: 불가
  - 표준: 최대 12시간
  - 프라이버시 우선: 최대 24시간
- 일시 중지 시 가디언에게 "일시 중지됨" 상태가 표시된다.
- 일시 중지 사실은 캡틴에게 알림으로 전달된다.
- 만료 시 자동으로 공유가 재개된다.

### 04.7 위치 공유 우선순위 (최종 결정 로직)

#### 멤버 간 공유

```
1. 여행 모드 확인
   ├── 강제 공유 → 캡틴 스케줄 확인 → 2단계로
   └── 자유 설정 → 개인 스케줄 확인 → 2단계로

2. 스케줄 유형별 ON/OFF 판정
   ├── 24시간 → 항상 ON → 3단계로
   ├── 시간대 기반 → 현재 시간 확인 → ON이면 3단계, OFF이면 비공유
   └── 일정 연동 → 현재 일정 존재 여부 확인 (버퍼 포함) → 일정 중이면 3단계, 아니면 비공유

3. 공개 범위 확인 (자유 설정 모드일 때)
   ├── 전체 공개 → 모든 멤버에게 공유
   ├── 관리자만 → 캡틴/크루장에게만 공유
   └── 지정 멤버 → 선택된 멤버에게만 공유
```

#### 가디언 공유

```
1. 가디언 연결 여부 확인
   ├── 연결 안 됨 → 비공유
   └── 연결됨 → 2단계로

2. 일시 중지 여부 확인
   ├── 일시 중지 중 → 비공유 (상태 표시)
   └── 정상 → 3단계로

3. 프라이버시 등급 + 스케줄 ON/OFF 확인
   ├── 안전 최우선 → ✅ 항상 공유
   ├── 표준 + ON 시간대 → ✅ 실시간 공유
   ├── 표준 + OFF 시간대 → ✅ 30분 스냅샷
   ├── 프라이버시 우선 + ON 시간대 → ✅ 실시간 공유
   └── 프라이버시 우선 + OFF 시간대 → ❌ 비공유
```

### 04.8 위치 공유 관련 제약

- 위치 공유가 OFF인 멤버도 **SOS 발송은 가능**하다 (SOS는 모든 설정과 무관하게 동작).
- SOS 발송 시에는 발송 시점의 위치가 **프라이버시 등급과 무관하게** 강제로 포함된다.
- 위치 공유 OFF 상태에서도 지오펜스 진입/이탈 감지는 **기기 로컬에서만** 동작하며, 서버/타 멤버에게 전송되지 않는다.
- 위치 공유 설정 변경은 실시간으로 반영되어야 한다.

### 04.9 위치 공유 설정 변경 알림

- 프라이버시 등급 변경 시: 전체 멤버 + 가디언에게 알림 발송.
- 강제 ↔ 자유 모드 전환 시: 전체 멤버 + 가디언에게 알림 발송.
- 강제 모드에서 캡틴이 스케줄을 변경할 때: 전체 멤버에게 알림 발송.
- 자유 모드에서 개인이 스케줄/공개 범위를 변경할 때: 별도 알림 없음.
- 가디언 일시 중지 시: 캡틴에게 알림 발송.
- 일정 연동 모드에서 새 일정 등록/수정/삭제 시: 해당 날짜에 공유 시간이 변경되므로 전체 멤버에게 알림 발송.

### 04.10 모드 전환 시 개인 설정 처리

- 강제 → 자유 전환: 모든 멤버에게 **기본값(24시간 전체 공개)**이 적용된다. 이전 개인 설정은 복원하지 않는다.
- 자유 → 강제 전환: 개인 설정은 삭제되지 않고 비활성화되지만, 복원을 보장하지 않는다.

## §05 알림 및 SOS 원칙

### 05.1 SOS 발송

- 모든 여행 멤버(캡틴/크루장/크루)는 **SOS를 발송**할 수 있다.
- SOS 발송 시 수신 대상: 그룹 전체 멤버 + 발송자에 연결된 가디언 전원.
- SOS 발송 시 발송자의 현재 위치가 **프라이버시 등급과 무관하게** 강제로 포함된다.
- **SOS는 모든 프라이버시 설정을 무시하는 최상위 안전 기능이다.**

### 05.2 가디언 긴급 알림

- **가디언은 연결된 멤버에게 긴급 알림을 발송할 수 있다.**
  - 긴급 알림은 SOS와 구분되는 별도 이벤트 타입(`GUARDIAN_ALERT`)이다.
  - 수신 대상: 연결된 멤버 + 해당 멤버의 캡틴/크루장.
  - FCM 알림 강도는 SOS와 동일하게 높은 우선순위로 발송한다.
- 전체 가디언은 여행 전체 멤버에게 긴급 알림을 발송할 수 있다.

### 05.3 가디언 긴급 위치 요청

- 가디언은 연결된 멤버에게 **긴급 위치 요청**을 보낼 수 있다 (§03.4 참조).
- 프라이버시 등급별 동작:
  - 안전 최우선: 이미 항상 공유 중이므로 추가 기능은 안부 확인 용도. 자동 응답.
  - 표준: 멤버에게 알림 → 1회성 위치 자동 전달 (별도 승인 불필요).
  - 프라이버시 우선: 멤버에게 알림 → **멤버가 승인**해야 위치가 전달됨.
- 요청 빈도 제한: 1시간에 최대 3회.
- 이벤트 타입: `GUARDIAN_LOCATION_REQUEST`

### 05.4 지오펜스 알림

- 지오펜스 진입/이탈 이벤트는 그룹 관리자(캡틴/크루장)에게 전송된다.
- 가디언에 대한 지오펜스 이벤트 전달은 프라이버시 등급에 따라 다르다:
  - 안전 최우선: 항상 전달
  - 표준: 스케줄 ON 시간대에만 전달
  - 프라이버시 우선: 전달하지 않음
- 크루 간에는 서로의 지오펜스 이벤트를 수신하지 않는다.

### 05.5 출석 체크

- 출석 체크 실행: 캡틴/크루장만 가능. `tb_attendance_check` 테이블에 세션이 생성된다.
- 출석 응답: 크루가 응답. `tb_attendance_response` 테이블에 응답이 기록된다.
  - 응답 유형: `present`(출석 확인), `absent`(미출석/미응답), `unknown`(응답 전)
- 응답 마감(`deadline_at`) 경과 시, 미응답자는 자동으로 `absent` 처리된다.
- 출석 체크 완료 후 캡틴에게 요약 알림이 전송된다.
- 가디언은 연결된 멤버의 출석 체크 결과를 열람할 수 있다.
- 출석 체크는 위치 공유 설정 및 프라이버시 등급과 무관하게 동작한다.

### 05.6 알림 수신 규칙 요약

| 이벤트 | 캡틴 | 크루장 | 크루 | 가디언 |

## §06 회원가입 및 온보딩 원칙

### 06.1 신규 가입 규칙

- **신규 유저는 역할 미지정 상태로 가입한다.** 가입 시 역할이 즉시 결정되지 않는다.
- 가입 완료 후 유저는 다음 중 하나를 선택한다:
  - **여행 생성** → 해당 여행의 `captain`으로 자동 배정
  - **초대코드로 참여** → 코드에 지정된 역할(`crew_chief`/`crew`/`guardian`)로 배정
  - **앱 탐색** → 역할 없이 제한된 기능 사용 가능 (여행 가이드, 데모 모드)
- 초대코드로 가입한 유저도 이후 별도로 여행을 생성하면 해당 여행에서 캡틴이 된다.

### 06.2 온보딩 흐름 규칙

- 기존 회원이 재인증(전화번호 OTP)하면 **프로필 설정 과정 없이** 메인 화면으로 즉시 이동한다.
- 인증 완료 후 기존 참여 여행이 있으면, **현재 날짜 기준으로 가장 가까운 여행**에 자동 배정한다.
  - 우선순위: `active` 상태 여행 > `planning` 상태 여행 (시작일이 현재에 가장 가까운 순)
  - 동일 우선순위의 여행이 여러 개인 경우, 시작일이 빠른 여행을 선택한다.

### 06.3 여행 미참여 유저의 메인 화면

- 참여 중인 여행이 없는 유저의 메인 화면은 **탐색 모드**로 동작한다:
  - 상단에 **여행 생성 / 초대코드 입력** CTA 버튼을 고정 표시한다.
  - **여행 가이드 탭**: 활성 — 국가별 안전 정보를 자유롭게 탐색할 수 있다.
  - **지도 탭**: 활성 — 지도 탐색 가능 (멤버 마커, 지오펜스 없음)
  - **멤버/채팅 탭**: 비활성 — 여행 참여 후 사용 가능하다는 안내 표시.
- 데모 모드: 앱 최초 설치 후 1회, 샘플 여행 데이터로 주요 기능을 체험할 수 있는 인터랙티브 투어를 제공한다.

### 06.4 앱 재실행 시 흐름

- 앱 재실행 시 저장된 인증 토큰이 유효하면 온보딩 과정을 생략하고 메인 화면으로 직행한다.
- 토큰이 만료되었거나 없으면 전화번호 인증 화면으로 이동한다.
- 재실행 시에도 가장 가까운 여행 자동 배정 규칙을 동일하게 적용한다.

### 06.5 약관 동의 체계

- 온보딩 약관 동의 화면에서 다음 항목을 필수/선택으로 구분하여 수집한다:

| 동의 항목 | 필수/선택 | 근거 |
|----------|:--------:|------|
| 서비스 이용약관 | 필수 | 전자상거래법 |
| 개인정보처리방침 | 필수 | 개인정보보호법 제30조 |
| 위치기반서비스 이용약관 | 필수 | 위치정보법 제15조 |
| 마케팅 정보 수신 | 선택 | 정보통신망법 |

- 필수 동의 미체크 시 가입 진행이 차단된다.
- 동의 이력은 `tb_user_consent`에 항목별 동의 시간·버전과 함께 기록한다.
- 약관 버전 변경 시 재동의 절차가 트리거된다 (설정 메뉴 원칙 참조).

## §07 권한 위임 원칙

### 07.1 리더(캡틴) 권한 위임

- 캡틴은 **여행 내 다른 멤버(크루장 또는 크루)**에게 리더 권한을 위임할 수 있다.
- 위임 시 우선 추천: 크루장 → 크루 순으로 UI에서 추천하되, 캡틴이 자유롭게 선택 가능.
- 위임 후:
  - 기존 캡틴 → 크루장으로 역할 변경
  - 위임 받은 멤버 → 캡틴으로 승격
- 위임 이력은 `TB_LEADER_TRANSFER_LOG`에 기록한다.

### 07.2 캡틴 탈퇴 시 위임 강제

- 여행이 `active` 또는 `planning` 상태일 때 캡틴이 탈퇴하려면:
  - 다른 멤버가 있는 경우: **반드시 리더 권한을 위임해야 한다.**
  - **캡틴만 남은 경우**: 여행을 종료(`completed`)하거나 삭제 후 탈퇴한다.
- 여행이 `completed` 상태이면 위임 없이 탈퇴 가능하다.

## §08 데이터 정합성 원칙

### 08.1 그룹 멤버 레코드

- `tb_group_member.trip_id`는 **절대 NULL이 되어서는 안 된다.**

### 08.2 역할 유일성 제약

- 하나의 그룹 내에서 `captain` 역할은 **정확히 1명**만 존재해야 한다.
  - DB 레벨 강제: `idx_group_member_captain` 부분 유니크 인덱스로 `member_role='captain' AND status='active'` 조건에서 `group_id` 기준 유일성을 보장한다.
- `UNIQUE(group_id, user_id)` 제약 조건을 유지한다.

### 08.3 일정 충돌 검증

- 캡틴/크루장/크루의 여행 참여 시 일정 겹침을 서버 측에서 반드시 검증한다.
- 기존 여행의 일정 변경 시에도 충돌 검증을 수행한다.
- `active`, `planning` 상태만 검사한다.

### 08.4 가디언 상한 및 중복 검증

- **무료 개인 가디언**: 멤버당 2명 제한을 서버에서 검증한다.
- **유료 추가 가디언**: 결제 완료 상태에서만 3명째~5명째 개인 가디언 연결을 허용한다.
- **전체 가디언**: 여행당 2명 제한을 서버에서 검증한다.
- **가디언 연결 중복 방지**: `tb_guardian_link`의 UNIQUE 제약은 부분 인덱스 2개로 구현된다.
  - 가입 완료 가디언(`guardian_id IS NOT NULL`): `(trip_id, member_id, guardian_id)` 조합 유니크
  - 미가입 초대(`guardian_id IS NULL`): `(trip_id, member_id, guardian_phone)` 조합 유니크
  - 이 설계는 `guardian_id`가 NULL일 때 `NULL ≠ NULL` 문제를 회피한다.

### 08.5 위치 공개 범위 정합성

- 멤버 탈퇴 시 해당 멤버를 공개 범위에 지정한 다른 멤버의 설정에서 자동 제거한다.
- 새 멤버 합류 시:
  - "전체 공개" → 자동 공유
  - "관리자만" → 관리자 역할일 때만 공유
  - "지정 멤버" → 수동 추가 필요

### 08.6 가디언 일시 중지 정합성

- 일시 중지 최대 시간은 프라이버시 등급에 따라 서버에서 검증한다.
- 만료 시 서버에서 자동으로 상태를 해제한다.
- 일시 중지 중 앱을 삭제/재설치해도 서버 기준으로 상태가 유지된다.

### 08.7 일정 연동 공유 정합성

- 일정 등록/수정/삭제 시, 해당 일정에 연결된 위치 공유 ON/OFF 스케줄이 자동으로 재계산되어야 한다.
- 일정이 삭제되어 특정 날짜에 일정이 0건이 되면, 해당 날짜는 종일 비공유가 된다.
  - 이 경우 캡틴에게 경고 알림을 발송한다.
- 일정 연동 모드에서 일정 시간이 겹치는 경우, 가장 이른 시작 ~ 가장 늦은 종료 시간이 공유 시간으로 설정된다 (합집합).
- 프라이버시 등급 변경 시 기본 스케줄 유형이 변경될 수 있으므로, 변경 전에 캡틴에게 확인 다이얼로그를 표시한다.

### 08.8 프라이버시 등급 정합성

- `tb_trip`에 `privacy_level` 컬럼을 추가한다 (`safety_first` / `standard` / `privacy_first`, 기본값: `standard`).
- 등급 변경 시 관련 가디언 공유 정책, 일시 중지 상한, 지오펜스 이벤트 전달 규칙이 즉시 변경된다.
- 등급 변경으로 인해 진행 중인 가디언 일시 중지가 새 상한을 초과하면:
  - 안전 최우선으로 변경 → 일시 중지 즉시 해제
  - 표준으로 변경 → 12시간 초과분 자동 단축
  - 프라이버시 우선으로 변경 → 기존 일시 중지 유지

### 08.9 약관 동의 정합성

- `tb_user_consent`에 약관별 동의 이력을 기록한다.
- 약관 버전 갱신 시 미동의 유저에게 앱 진입 시 재동의 팝업을 표시한다.
- 필수 약관 철회 시 서비스 이용이 제한된다 (설정 메뉴 원칙 참조).

### 08.10 여행 기간 정합성

- 여행 기간은 최대 15일을 초과할 수 없다. 서버에서 검증한다.
- 여행 기간 변경 시에도 15일 초과 여부를 재검증한다.

## §09 과금 모델 원칙

### 09.1 과금 철학

SafeTrip은 **트랜잭션(여행) 기반 과금**을 원칙으로 한다. 월 구독이 아닌 여행별 요금을 부과하여, 여행하지 않는 기간에 비용이 발생하지 않는 공정한 구조를 제공한다.

### 09.2 B2C 기본 과금

| 인원 규모 | 기본 요금 (여행당) | 여행 기간 | 포함 사항 |
|----------|:------------------:|----------|----------|
| 1~5명 | 무료 | 15일 이내 자유 설정 | 핵심 안전 기능 전체 |
| 6~15명 | 9,900원 | 15일 이내 자유 설정 | 기본 + 출석 체크 |
| 16~30명 | 14,900원 | 15일 이내 자유 설정 | 기본 + 출석 체크 + 그룹 관리 강화 |
| 31명+ | 19,900원 | 15일 이내 자유 설정 | 전체 기능 |

- **여행 기간**: 1회 여행당 최대 15일 이내에서 자유롭게 설정한다. 15일 초과 시 여행을 분할해야 하며, 분할된 각 여행은 별도 과금한다 (§02.3 참조).
- **인원 산정 기준**: 여행 멤버(캡틴/크루장/크루)만 인원으로 산정한다. 가디언은 인원 산정에 포함하지 않는다.

### 09.3 가디언 추가 과금

| 항목 | 무료 범위 | 추가 과금 |
|------|----------|----------|
| 개인 가디언 | 멤버당 2명까지 무료 | 3명째부터 1명당 1,900원/여행 |
| 전체 가디언 | 여행당 2명까지 무료 | — (추가 불가) |

- 추가 가디언 결제는 여행 단위로 이루어진다.
- 무료 가디언 2명 한도를 초과하여 연결을 시도하면 과금 안내 모달이 표시된다.
- 추가 가디언 해제 시 이미 결제된 요금은 환불하지 않는다.

### 09.4 부가 상품

| 상품명 | 가격 | 설명 |
|-------|:----:|------|
| 움직임 세션 보기 | 2,900원/여행 | 멤버별 이동기록 과거 일자 열람 |
| AI Plus | 4,900원/월 또는 2,900원/여행 | Convenience + Intelligence AI |
| AI Pro | 9,900원/월 또는 5,900원/여행 | AI Plus + 프리미엄 브리핑, 일정 최적화 |

### 09.5 무료 구간 보장

- **안전 기능은 과금과 무관하게 항상 사용 가능하다**: SOS 발송/수신, 가디언 긴급 알림, 안전가이드, 기본 위치 공유.
- 무료(1~5명) 여행에서도 핵심 안전 기능의 품질(SLA)은 유료와 동일하게 보장한다.
- 무료 가디언 2명은 과금 상태와 무관하게 항상 연결 가능하다.

### 09.6 결제 유형 체계

DB 상 결제/구독/명세 항목의 유형은 다음과 같이 통일된다:

| 유형 (`payment_type`) | 설명 |
|:---------------------:|------|
| `trip_base` | 여행 기본 이용료 (6인 이상 유료) |
| `addon_movement` | 움직임 세션 애드온 (2,900원/세션) |
| `addon_ai_plus` | AI Plus 애드온 (4,900원/월 또는 2,900원/여행) |
| `addon_ai_pro` | AI Pro 애드온 (9,900원/월 또는 5,900원/여행) |
| `addon_guardian` | 추가 가디언 슬롯 (1,900원/여행, 3명째 이상) |
| `b2b_contract` | B2B 계약 일괄 과금 |

- `tb_subscription.plan_type`, `tb_payment.payment_type`, `tb_billing_item.item_type`은 위 체계와 일관되게 사용한다.

### 09.7 결제/환불 규칙

- 결제 시점: 여행 생성 시 인원 확정 후 (planning 상태에서 결제 가능).
- 환불 규칙:
  - `planning` 상태: 전액 환불
  - `active` 상태 + 여행 시작 24시간 이내: 50% 환불
  - `active` 상태 + 여행 시작 24시간 이후: 환불 불가
  - `completed` 상태: 환불 불가

## §10 미성년자 보호 기반 원칙

> 상세 규칙은 미성년자 보호 원칙(별도 Tier 0 문서)에서 정의한다. 본 절은 비즈니스 원칙에서 보장해야 하는 기반 규칙을 명시한다.

### 10.1 연령별 동의 체계

| 연령 구분 | 동의 주체 | 추가 요건 |
|----------|----------|----------|
| 14세 미만 | 법정대리인(부모/보호자) | 법정대리인 동의 확인 절차 필수 |
| 14세~17세 | 본인 + 보호자 이중 동의 | 보호자 확인 SMS/인앱 승인 |
| 18세 이상 | 본인 | 일반 성인 동의 절차 |

### 10.2 미성년자 여행 제약

- **미성년자 멤버가 1명이라도 포함된 여행은 `safety_first` 등급만 허용된다.**
- 미성년자 크루의 가디언 해제는 **캡틴의 승인이 필요**하다.
- 미성년자 크루는 가디언 일시 중지 기능을 사용할 수 없다.
- 미성년자의 위치 데이터 보유 기간은 성인보다 단축된다 (여행 종료 후 30일, §13 참조).

### 10.3 미성년자 AI 기능 제한

- Safety AI (안전 관련): 제한 없이 사용 가능
- Convenience AI (편의 기능): 제한 없이 사용 가능
- Intelligence AI (분석/인사이트): 미성년자 개인 데이터 기반 분석 제한 (그룹 단위만 허용)

### 12.2 B2B 계약 구조

- B2B 고객은 **관리자 계정(Admin Portal)**을 통해 여행을 일괄 생성하고 초대코드를 대량 발행할 수 있다.

- B2B 계약별 여행 수 제한: `tb_b2b_contract.max_trips` 컬럼으로 계약당 최대 여행 수를 관리한다 (NULL이면 `max_groups`로 대체).
- B2B SLA는 앱 성능·안정성 원칙에서 별도로 정의한다.

### 12.3 B2B 전용 기능

| 기능 | 설명 | 대상 고객 |
|------|------|----------|

| 보호자/가디언 일괄 연결 | 멤버-가디언 매핑 일괄 등록 | 학교, 기업 |
| 브랜딩 커스터마이징 | 앱 내 조직 로고 표시 | 여행사, 학교, 기업 |
| 관리 대시보드 | 실시간 그룹 현황, 출석 통계, 안전 리포트 | 전체 |
| 데이터 리포트 | 여행 종료 후 안전 지표 리포트 제공 | 학교, 보험사 |

### 12.4 기업 출장 B2B 가디언 모델

기업 출장 B2B 계약에서는 학교 수학여행과 유사한 방식으로 가디언을 일괄 관리할 수 있다.

- **회사 관리자(안전담당자/HR)가 B2B 관리 포털에서 가디언을 일괄 등록**한다.
- 가디언 대상은 팀원 지정 가족 또는 회사 안전담당자이다.
- CSV 업로드를 통한 팀원-가디언 매핑 일괄 등록을 지원한다.
- 팀원(크루) 본인의 동의가 필수이며, 동의하지 않은 가디언 연결은 보류 상태로 처리한다.
- B2B 계약에 포함된 가디언 수는 별도 협의하며, B2C 무료/유료 가디언 한도와 별개로 적용할 수 있다.

## §14 삭제 및 비활성화 원칙

### 14.1 소프트 삭제

- 모든 주요 테이블은 **소프트 삭제(`deleted_at`)** 방식을 사용한다.

### 14.2 여행 삭제 시 연쇄 처리

1. 관련 그룹 비활성화
2. 그룹 멤버 `status` → `left`
3. 모든 가디언 연결 관계 해제
4. 활성 지오펜스 비활성화
5. 위치 공유 즉시 중지
6. 위치 공유 설정(스케줄/공개 범위) 비활성화
7. 관련 초대코드 비활성화
8. 삭제 이벤트 로그 기록
9. 전체 멤버 + 가디언에게 여행 삭제 알림 발송

### 14.3 멤버 탈퇴 시 연쇄 처리

1. `tb_group_member.status` → `left`, `left_at` 기록
2. 해당 멤버에 연결된 개인 가디언 관계 해제
3. 해당 멤버의 위치 공유 즉시 중지
4. 해당 멤버를 공개 범위에 지정한 다른 멤버의 설정에서 제거
5. 해당 멤버가 다른 멤버의 공개 범위에 지정한 설정 제거
6. 탈퇴 이벤트 로그 기록
7. 캡틴/크루장 + 해당 멤버의 가디언에게 탈퇴 알림 발송

### 14.4 계정 삭제(탈퇴) 시 데이터 처리

- 계정 삭제 요청 시 **7일 유예 기간**을 부여한다 (유예 기간 내 철회 가능).
  - DB 컬럼: `tb_user.deletion_requested_at`에 삭제 요청 시각을 기록하며, 이 시점이 7일 유예의 기산점이 된다.
  - 유예 기간 내 철회: `deletion_requested_at`을 NULL로 초기화한다.
- 유예 기간 만료 후 처리:
  - 즉시 삭제: 프로필 정보(이름, 이미지), 긴급 연락처, 기기 토큰
  - 익명화 후 보관: 위치 데이터, 이벤트 로그 (법적 보관 의무 기간까지)
  - 영구 보관: SOS 발송/수신 기록 (§13 데이터 생명주기 참조)
- 삭제 처리 진행 중에도 참여 중인 `active` 여행이 있으면, 여행 탈퇴가 먼저 실행된다 (§14.3).

| 모든 멤버가 동시 탈퇴 | 마지막 탈퇴자가 캡틴이면 여행 자동 종료(completed) |
| 초대코드로 이미 참여 중인 여행에 재참여 시도 | "이미 참여 중인 여행입니다" 안내 후 차단 |
| B2B 일괄 초대 중 일부 사용자 가입 실패 | 성공/실패 리포트 생성, 실패 건 재초대 가능 |

### 15.2 위치 공유 엣지케이스

| 상황 | 처리 규칙 |
|------|----------|
| 기기 위치 권한 거부 | 위치 관련 기능 비활성화 + 권한 요청 배너 상시 표시 |
| GPS 신호 없음 (터널, 실내) | 마지막 알려진 위치 유지 + "마지막 갱신: N분 전" 표시 |
| 일정 연동 모드에서 일정 0건 여행일 | 종일 비공유 + 캡틴 경고 배너 |
| 프라이버시 등급 변경 중 네트워크 단절 | 로컬 캐시에 변경 저장 → 복구 시 재시도 |

### 15.3 과금 엣지케이스

| 상황 | 처리 규칙 |
|------|----------|
| 무료(5명) 여행에서 6번째 멤버 합류 시도 | "유료 전환이 필요합니다" 안내 → 캡틴에게 결제 유도 |
| 결제 완료 후 멤버가 5명 이하로 감소 | 환불 없음, 이미 결제된 요금 유지 |
| 결제 실패 (카드 오류 등) | 여행 생성/멤버 추가 차단 → 재결제 유도 |
| B2B 계약 기간 만료 | 계약 갱신까지 신규 여행 생성 차단 (기존 active 여행은 유지) |
| 무료 가디언 2명 초과 연결 시도 | 추가 가디언 과금 안내 모달 표시 → 결제 후 연결 |
| 15일 초과 여행 생성 시도 | "여행은 최대 15일까지 설정 가능합니다. 분할 생성해 주세요" 안내 |

### 15.4 가디언 과금 엣지케이스

| 상황 | 처리 규칙 |
|------|----------|
| 유료 가디언 결제 후 여행 삭제 | 환불 규칙은 기본 과금과 동일 (§09.6 적용) |
| 유료 가디언 연결 후 해제 | 결제 취소 불가, 재연결 시 추가 과금 없음 (동일 여행 내) |
| B2B 계약에서 가디언 수 초과 | B2B 계약 범위 내에서 처리, 초과 시 추가 계약 필요 |

## 부록 B: 여행 유형별 시나리오 예시

### 시나리오 1: 수학여행 (안전 최우선)

> 고등학교 수학여행. 캡틴 = 인솔 교사, 크루 = 학생 30명, 전체 가디언 = 학년 부장, 개인 가디언 = 각 학부모.

- **프라이버시 등급**: 안전 최우선 (`safety_first`)
- **위치 공유 모드**: 강제 공유
- **스케줄**: 24시간
- B2B 학교 계약, 학생-학부모 일괄 가디언 연결, 미성년자 보호 정책 적용

**동작**:
- 낮 시간: 모든 학생 위치 → 교사 + 학부모 실시간 공유
- 밤 시간: 모든 학생 위치 → 교사 + 학부모 **계속 실시간 공유** ✅
- 학생이 가디언 일시 중지 시도 → **차단됨** (안전 최우선 + 미성년자)
- 학부모는 새벽에도 자녀 위치 확인 가능
- 학생(미성년자) 위치 데이터 → 여행 종료 후 30일 보관 (성인 90일보다 단축)

---

### 시나리오 2: 업무 출장 (프라이버시 우선 + 가디언 활용)

> 해외 출장. 캡틴 = 팀장, 크루 = 팀원 5명. 가디언 = 팀원 개별 지정(가족) 또는 회사 안전담당자.

- **프라이버시 등급**: 프라이버시 우선 (`privacy_first`)
- **위치 공유 모드**: 강제 공유
- **스케줄**: 시간대 기반 (오전 9시 ~ 오후 6시)
- **가디언 운영**: 모델 A(팀원 자율) + 모델 B(회사 안전담당자 일괄 등록) 혼용

**동작**:
- 오전 9시~오후 6시: 팀원 위치 → 팀장 + 팀원 간 공유, 가디언에게도 공유
- 오후 6시~오전 9시: **누구에게도 위치 비공유** (퇴근 후 완전 프라이버시)
- 팀장은 퇴근 후 팀원 위치를 볼 수 없음 → 부적절한 감시 방지
- 가디언(가족)도 퇴근 후에는 위치 확인 불가 (프라이버시 우선 등급)
- 가디언이 긴급 상황 우려 시 → 긴급 위치 요청 발송 → 팀원 승인 후 1회 위치 공유

---

### 시나리오 3: 성인 패키지 투어 (프라이버시 우선 + 일정 연동)

> 유럽 7박 8일 패키지 투어. 캡틴 = 여행사 가이드, 크루 = 관광객 20명. 개인 가디언 = 일부 관광객의 가족.

- **프라이버시 등급**: 프라이버시 우선 (`privacy_first`)
- **위치 공유 모드**: 강제 공유
- **스케줄**: 일정 연동 (버퍼 15분)

**동작 (3일차 예시)**:
```
08:45  일정 "에펠탑 관광" 시작 15분 전 → 위치 공유 자동 ON
09:00  에펠탑 관광 시작
12:00  에펠탑 관광 종료
12:15  버퍼 종료
12:30  다음 일정 "세느강 크루즈" 시작 15분 전 → 위치 공유 자동 ON
       (12:15~12:15 간격 = 0분, 연속 공유 유지)
12:30  세느강 크루즈 시작
17:00  크루즈 종료
17:15  버퍼 종료 → 위치 공유 자동 OFF
17:15~ 자유시간: 가이드(캡틴)는 관광객 위치를 볼 수 없음 ✅
```

---

### 시나리오 4: 성인 가족 여행 (표준)

> 부모 + 성인 자녀 가족 여행. 캡틴 = 부모, 크루 = 성인 자녀 2명. 가디언 = 한국에 남은 조부모.

- **프라이버시 등급**: 표준 (`standard`)
- **위치 공유 모드**: 자유 설정
- **스케줄**: 각자 설정 (자녀 A: 밤 11시~오전 8시 비공유)

**동작**:
- 오전 8시~밤 11시: 자녀 A 위치 → 가족 전원 + 조부모(가디언) 실시간 공유
- 밤 11시~오전 8시: 자녀 A 위치 → 가족 간 비공유, 조부모에게는 **30분 스냅샷** 제공
  - 조부모 화면: "마지막 확인: 23분 전 · 호텔 근처" 표시

---

### 시나리오 5: 가디언 긴급 위치 요청 (프라이버시 우선)

> 시나리오 3의 패키지 투어에서 자유시간(위치 비공유) 중, 관광객 B의 가디언(배우자)이 여행지에 지진 뉴스를 접함.

1. 가디언(배우자): 긴급 위치 요청 발송
2. 관광객 B: 높은 우선순위 FCM 알림 수신 — "가디언이 현재 위치를 요청합니다"
3. 관광객 B: **"위치 공유" 버튼 탭** (승인)
4. 가디언: 1회성으로 관광객 B의 현재 위치 수신
5. 가이드(캡틴)/크루장: 동시에 알림 수신 (관리 목적)
6. 이벤트 로그: `GUARDIAN_LOCATION_REQUEST` + `APPROVED` 기록

- 관광객 B가 무응답(10분 경과) → 가디언에게 "응답 없음" 표시
- 이 경우 가디언은 긴급 알림(`GUARDIAN_ALERT`)을 추가로 발송하여 주의를 환기할 수 있음
