## 1. 문서 목적 및 적용 범위

### 1.1 목적

본 문서는 SafeTrip 메인 화면 바텀시트의 **일정 탭(Schedule Tab)**이 제공하는 기능 원칙을 정의한다. 여행 스케줄 CRUD, 위치 공유 일정 연동 시각화, 역할별 접근 제어, 일정 카드 디자인 시스템, 알림 규칙, 지도 연동 규칙을 포함한다.

### 1.2 적용 범위

- **적용 대상**: SafeTrip Flutter 앱의 바텀시트 일정 탭 (ScheduleTab Widget)
- **적용 여행 상태**: `planning` (일정 등록·수정), `active` (일정 표시·실행), `completed` (일정 열람 전용)
- **기준 문서**: SafeTrip_비즈니스_원칙_v5.1
- **제외 범위**: 지도 화면에서의 일정 마커 표시 규칙 (→ #17 지도 기본화면 고유 원칙 참조), 출석 체크 상세 규칙 (→ #19 멤버탭 원칙 참조)

## 2. 일정탭 설계 철학 및 이중 역할

### 2.1 설계 철학 5원칙

| 원칙 | 설명 |
|------|------|
| **시간 중심** | 모든 일정은 시간순으로 정렬·표시하며, 현재 진행 중인 일정을 최우선으로 강조한다 |
| **위치 공유 일체화** | 일정은 단순 스케줄 관리가 아닌 위치 공유 스케줄과 연동된 핵심 기능이다 |
| **역할 기반** | 일정 수정·삭제 권한은 역할에 따라 엄격하게 제한된다 |
| **지도 연결** | 모든 일정 카드에서 [지도▶] 버튼을 통해 지도 화면으로 즉시 이동한다 |
| **오프라인 우선** | 오프라인 상태에서도 일정 목록을 열람하고 수정 내용을 로컬에 임시 저장한다 |

### 2.2 일정탭의 이중 역할

일정탭은 두 가지 독립된 역할을 동시에 수행한다.

```
역할 A: 여행 스케줄 CRUD
   - 일정 생성 / 수정 / 삭제 / 조회
   - 시간순 정렬, 날짜 타임라인, 일정 카드 표시
   - 이동(이동)/숙박/식사/관광/쇼핑/모임/기타 유형 관리

역할 B: 위치 공유 스케줄 시각화 (schedule_linked 모드)
   - 프라이버시 우선 등급에서 공유 타임라인 바 시각화
   - 일정 시간대 = 녹색 (공유), 비일정 시간대 = 회색 (비공유)
   - 버퍼 구간(시작 전 15분 ~ 종료 후 30분) 연한 녹색으로 표시
   - 연속 일정 간 30분 이내 간격 → 자동 연결 공유
```

> 두 역할은 서로 연동된다. 일정을 등록하면 역할 B의 공유 시간대가 자동 갱신된다.

## 3. 화면 구성 5개 영역

일정탭은 위에서 아래로 5개 영역(A~E)으로 구성된다.

### 영역 A: 위치 연동 배너

```
[위치 연동 배너]
"일정 연동 공유 모드 활성 — 일정 시간대에만 위치가 공유됩니다"
```

- **표시 조건**: 프라이버시 우선 등급(`privacy_first`)에서만 표시
- 안전 최우선 / 표준 등급에서는 이 배너가 표시되지 않는다.
- 배너 탭 시 위치 공유 설정 화면으로 이동.

### 영역 B: 날짜 타임라인 바

```
[ 3/1 토 ] [ 3/2 일 ] [ 3/3 월 ] ... [ 3/15 일 ]
              ▲ (현재 날짜 강조 — 파란 테두리)
```

- **최대 15일까지만 표시 (§02.3 여행 기간 15일 제한)**.
- 현재 날짜: 파란 테두리 + 진한 텍스트로 강조.
- 가로 스크롤 가능.
- 날짜 탭 시 해당 날짜의 일정 카드로 스크롤 이동.
- 일정이 있는 날짜: 날짜 아래 파란 점 표시.

### 영역 C: 공유 타임라인 바

```
[00:00──────09:00 공유 ████████████████ 18:00──────23:59]
             연한 녹색(버퍼)  진한 녹색(공유 ON)  회색(비공유)
```

- **표시 조건**: 프라이버시 우선 등급(`privacy_first`)에서만 표시
- 24시간 타임라인에서 공유 시간대를 색상으로 구분.
  - 진한 녹색: 일정 등록된 공유 시간 (ON)
  - 연한 녹색: 버퍼 구간 (일정 시작 전 15분 ~ 종료 후 30분)
  - 회색: 비공유 시간 (일정 없음)
- 타임라인 바 탭 시 해당 시간대의 일정 카드로 스크롤 이동.

### 영역 D: 일정 카드 목록

- 선택된 날짜의 일정을 시간순으로 정렬하여 표시.
- 현재 시각에 해당하는 진행 중 일정: 파란 테두리 + 상단 고정 (active 상태에서만).
- 일정 없는 날짜: "이 날짜에 등록된 일정이 없습니다" 안내 + [+ 일정 추가] 버튼 (캡틴/크루장만).

#### 일정 카드 구조

```
┌────────────────────────────────────────────────┐
│  09:00  │ ✈️ 인천공항 출발 │ [진행 중]  │ [지도▶] │
│         │  인천국제공항 1터미널                   │
│         │  → 09:00 ~ 11:30                      │
└────────────────────────────────────────────────┘
```

#### 일정 카드 유형별 아이콘 7종

| 유형 | `schedule_type` | 아이콘 | 설명 |
|------|:---------------:|:------:|------|
| 이동 | `move` | ✈️ | 교통편, 이동 수단 |
| 숙박 | `stay` | 🏨 | 호텔, 숙소 체크인/아웃 |
| 식사 | `meal` | 🍽️ | 식당, 카페 |
| 관광 | `sightseeing` | 📍 | 관광지, 명소 방문 |
| 쇼핑 | `shopping` | 🛍️ | 쇼핑몰, 시장 |
| 모임 | `meeting` | 👥 | 집합, 미팅 포인트 |
| 기타 | `other` | 📌 | 분류 외 일정 |

#### 진행 중 강조 처리

- 현재 시각이 `start_datetime ~ end_datetime` 범위에 있는 일정: 파란 테두리 (2px) + 상단 고정.
- 현재 시각이 버퍼 구간(시작 전 15분)에 있는 일정: 노란색 테두리 + "곧 시작" 배지.
- 종료된 일정: 회색 처리 (텍스트, 아이콘 모두 회색).

### 영역 E: + 일정 추가 버튼

- **활성 조건**: 캡틴, 크루장만 활성 (크루, 가디언은 비활성/숨김).
- 완료된 여행(`completed` 상태): 전체 역할에서 비활성화.
- 탭 시 일정 추가 바텀시트 표시.

## 4. 일정 CRUD 규칙

### 4.1 일정 생성

| 입력 방식 | 설명 | Phase |
|----------|------|:-----:|
| 직접 입력 | 일정 이름, 유형, 시작·종료 시간, 장소명 직접 입력 | Phase 1 |
| 지도 핀 탭 | 지도에서 장소를 핀으로 선택 후 일정 정보 입력 | Phase 1 |
| AI 일정 제안 | 목적지 기반 AI 추천 일정 자동 생성 | Phase 2 |

#### 일정 생성 시 여행 기간 제한 (§02.3)

- **날짜 선택 범위**: 여행의 `start_date` ~ `end_date` (최대 15일 이내) 날짜만 선택 가능.
- 여행 기간 외 날짜 입력 시도: 날짜 선택기에서 해당 날짜 비활성화 + "여행 기간 내의 날짜를 선택해 주세요" 안내 토스트.
- 기존 여행 기간 변경 시 15일 초과 경고: "여행 기간은 최대 15일입니다. 분할 여행을 고려해 주세요" 안내.

#### 종일 일정 처리

- `all_day = TRUE` 설정 시: 해당 날짜 전체(00:00~23:59)를 공유 시간으로 처리 (§04.3).
- 공유 타임라인 바에서 해당 날짜 전체가 진한 녹색으로 표시.

### 4.2 일정 수정

- **수정 권한**: 캡틴, 크루장만 가능.
- 수정 이력은 `tb_schedule_history` 테이블에 기록 (수정자, 수정 시각, 변경 전/후 값).
- 수정 완료 시 전체 멤버 + 가디언에게 "일정이 변경되었습니다" 푸시 알림 발송.
- 진행 중(`active` 상태) 일정 수정: 경고 다이얼로그 표시 후 수정 진행.

### 4.3 일정 삭제

- **삭제 권한**: 캡틴, 크루장만 가능.
- 삭제 전 확인 다이얼로그 필수: "이 일정을 삭제하시겠습니까? 삭제된 일정은 복구할 수 없습니다."
- 논리 삭제: `deleted_at` 타임스탬프 기록 (물리 삭제 아님).
- 삭제 완료 시 전체 멤버 + 가디언에게 "일정이 삭제되었습니다" 푸시 알림 발송.

### 4.4 시간 충돌 처리

- 동일 날짜에 시간이 겹치는 일정 등록 시도: 경고 배너 표시 ("시간이 겹치는 일정이 있습니다").
- **차단하지 않는다** (경고만 표시, 사용자 판단에 맡김).
- 겹치는 일정이 있는 날짜: 날짜 타임라인 바에 주황색 경고 아이콘 표시.

### 4.5 일정 연동 공유 시스템 (프라이버시 우선 등급)

프라이버시 우선 등급(`privacy_first`)에서 일정과 위치 공유가 연동되는 세부 규칙이다.

| 구간 | 색상 | 공유 상태 |
|------|:----:|:--------:|
| 일정 등록된 시간 | 진한 녹색 | ON (실시간 공유) |
| 버퍼 구간 (일정 전 15분 ~ 후 30분) | 연한 녹색 | ON (버퍼 공유) |
| 일정 없는 시간 | 회색 | OFF (비공유) |

#### 연속 일정 자동 연결

- 연속된 두 일정의 간격이 30분 이내인 경우: 중간 비공유 구간 없이 자동 연결하여 연속 공유.
- 간격 30분 초과: 중간 구간 비공유 (회색) 처리.

#### 일정 미등록 날짜 처리

- 해당 날짜에 등록된 일정이 없으면 **종일 비공유** (§04.3).
- 캡틴에게 "일정이 없는 날짜에는 위치가 공유되지 않습니다" 안내 알림 발송.

## 6. 프라이버시 등급별 동작 차이

비즈니스 원칙 §02.7 (여행 프라이버시 등급) 및 §04.3 (공유 스케줄 유형)에 따른 일정탭 동작 차이를 정의한다.

### 6.1 안전 최우선 등급 (`safety_first`)

- 영역 A (위치 연동 배너): **미표시**
- 영역 C (공유 타임라인 바): **미표시**
- 일정 CRUD: 정상 동작
- 위치 공유: 24시간 항상 공유 (일정 등록과 무관)
- 일정 추가 시 공유 변화: 없음 (항상 공유이므로 일정과 무관)
- 미성년자 포함 여행에 강제 적용 (§10)

```
[일정탭 표시]

B(날짜 타임라인): ✅ (최대 15일)
C(공유 타임라인): ❌
D(일정 카드 목록): ✅
E(추가 버튼): ✅ (캡틴/크루장)
```

- 위치 공유: 기본 24시간 공유, 일정 등록과 무관

- 영역 A (위치 연동 배너): **표시** ("일정 연동 공유 모드 활성")

## 7. 에러 및 엣지케이스 처리

### 7.1 여행 기간 제한 관련

| 상황 | 처리 방법 |
|------|----------|
| 15일 초과 날짜 선택 시도 | 날짜 선택기에서 범위 외 날짜 비활성화 + "여행 기간 내의 날짜를 선택해 주세요" 토스트 |
| 기존 여행 종료일을 15일 초과로 변경 시도 | "여행 기간은 최대 15일입니다" 오류 + 변경 차단 |
| 일정 날짜가 여행 기간 밖에 이미 존재하는 경우 (데이터 정합 오류) | 해당 일정에 경고 아이콘 표시 + "여행 기간과 일정 날짜를 확인해 주세요" 안내 |
| DB CHECK 제약 위반 (`end_date - start_date > 15`) | 서버 400 에러 반환, "최대 15일 이내 여행만 생성할 수 있습니다" 안내 |

### 7.2 CRUD 권한 오류

| 상황 | 처리 방법 |
|------|----------|
| 크루가 일정 수정/삭제 시도 | 버튼 비활성화 (UI 레벨 차단), API 403 반환 시 "권한이 없습니다" 토스트 |
| 완료된 여행(`completed`)에서 일정 추가 시도 | "종료된 여행의 일정은 수정할 수 없습니다" 안내 |
| 캡틴 권한 이임 중 일정 수정 시도 | "권한 이임 중에는 일정을 수정할 수 없습니다" 안내 |

### 7.3 시간 충돌 및 데이터 오류

| 상황 | 처리 방법 |
|------|----------|
| 시간이 겹치는 일정 등록 | 경고 배너 표시 (차단하지 않음), 날짜 타임라인에 경고 아이콘 |
| `end_datetime < start_datetime` 입력 | 저장 차단, "종료 시간이 시작 시간보다 빠릅니다" 오류 |
| 일정 로딩 실패 (API 오류) | "일정을 불러올 수 없습니다. 다시 시도해 주세요" 오류 + 재시도 버튼 |
| 일정 0개 (날짜 선택 후) | "이 날짜에 등록된 일정이 없습니다" 안내 + [+ 일정 추가] 버튼 표시 |

### 7.4 위치 연동 오류 (프라이버시 우선 등급)

| 상황 | 처리 방법 |
|------|----------|
| 일정 연동 공유 계산 실패 | 공유 타임라인 바 "계산 중..." 표시 후 재시도 |
| 일정 없는 날짜에 공유 ON 요청 (자유 설정 모드 한정) | 멤버가 수동으로 공유를 켤 수 있는 버튼 표시 (§04.3 자유 설정 모드) |
| 일정 수정 시 공유 시간대 변경 알림 실패 | 로컬 재시도 3회, 실패 시 "알림 발송에 실패했습니다" 안내 |

### 7.5 가디언 관련

| 상황 | 처리 방법 |
|------|----------|
| 무료 가디언이 일정탭 직접 접근 시도 | 403 처리, 상태 탭으로 리다이렉트 |
| 유료 가디언 일정 요약 발송 실패 | 재시도 3회 (1시간 간격), 실패 시 다음 일 요약에 포함 |
| 연결 멤버의 일정이 없는 경우 (유료 가디언 요약) | "오늘은 등록된 일정이 없습니다" 요약 메시지 발송 |

## 8. DB 스키마

본 섹션은 일정탭에서 직접 참조하는 테이블 구조를 명세한다. 전체 스키마는 `07_T2_DB_설계_및_관계_v3_4.md`를 참조한다.

### 8.1 `tb_schedule` — 여행 일정

```sql
-- 여행 일정 (일정탭의 핵심 데이터)
CREATE TABLE tb_schedule (
  id              UUID          PRIMARY KEY DEFAULT gen_random_uuid(),
  trip_id         UUID          NOT NULL REFERENCES tb_trip(id) ON DELETE CASCADE,
  title           VARCHAR(100)  NOT NULL,
  schedule_type   VARCHAR(20)   NOT NULL DEFAULT 'other',
                  -- 'move' | 'stay' | 'meal' | 'sightseeing' | 'shopping' | 'meeting' | 'other'
  start_datetime  TIMESTAMPTZ   NOT NULL,
  end_datetime    TIMESTAMPTZ,
  all_day         BOOLEAN       NOT NULL DEFAULT FALSE,  -- 종일 일정 여부 (§04.3)
  location_name   VARCHAR(200),
  location_lat    DECIMAL(10,8),
  location_lng    DECIMAL(11,8),
  created_by      UUID          NOT NULL REFERENCES tb_user(id),
  created_at      TIMESTAMPTZ   NOT NULL DEFAULT NOW(),
  updated_at      TIMESTAMPTZ   NOT NULL DEFAULT NOW(),
  deleted_at      TIMESTAMPTZ   -- 논리 삭제

  CONSTRAINT chk_schedule_datetime CHECK (end_datetime IS NULL OR end_datetime > start_datetime)
);

-- 날짜별 일정 조회 최적화
CREATE INDEX idx_schedule_trip_datetime ON tb_schedule(trip_id, start_datetime)
  WHERE deleted_at IS NULL;
```

## 9. 오프라인 대응

비즈니스 원칙 §11 (오프라인 기본 정책) 및 오프라인 동작 통합 원칙 (#16)에 따른 일정탭 오프라인 대응을 정의한다.

### 9.1 오프라인 상태의 일정 기능

| 기능 | 오프라인 상태 | 대응 |
|------|:-----------:|------|
| 일정 목록 열람 | ✅ | 로컬 SQLite `local_schedule_draft` 캐시에서 표시 |
| 일정 카드 상세 열람 | ✅ | 로컬 캐시 데이터 표시 |
| 일정 생성 (오프라인) | ⚠️ | 로컬에 임시 저장 → 온라인 복구 시 서버 동기화 |
| 일정 수정 (오프라인) | ⚠️ | 로컬에 변경 사항 저장 → 온라인 복구 시 동기화 |
| 일정 삭제 (오프라인) | ⚠️ | 로컬에서 논리 삭제 표시 → 온라인 복구 시 서버 반영 |
| 공유 타임라인 바 표시 | ✅ | 마지막 캐시된 일정 기반으로 계산 |
| 타 멤버 일정 실시간 업데이트 | ❌ | 네트워크 필수 |
| 알림 발송 | ❌ | 네트워크 복구 시 자동 재시도 |

### 9.2 오프라인 수정 동기화 규칙

- **로컬 저장 테이블**: `local_schedule_draft` (SQLite)
  - `sync_status`: `pending` / `synced` / `conflict`
  - `local_updated_at`: 로컬 수정 시각
- **충돌 해결**: last-write-wins (서버 타임스탬프 기준)
  - 서버의 `updated_at`이 로컬 `local_updated_at`보다 늦으면 서버 데이터 우선.
  - 충돌 시 사용자에게 "서버와 로컬 데이터가 다릅니다. 최신 서버 데이터로 덮어쓰겠습니까?" 안내.
- **동기화 순서**: 온라인 복구 시 SOS 이벤트 → 위치 데이터 → 일정 변경 순서로 동기화 (§11.2).

### 9.3 오프라인 UI 표시

- 화면 상단에 오렌지색 배너: "오프라인 — 변경 사항은 연결 후 저장됩니다"
- 로컬 미동기화 일정 카드: 우상단에 동기화 대기 아이콘 (점선 테두리)
- 오프라인 중 [지도▶] 버튼: 오프라인 지도 모드로 이동 (지도탭 #17 §9 참조)
