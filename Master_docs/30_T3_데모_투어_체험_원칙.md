## §1. 목적 및 적용 범위

### 1.1 목적

데모 투어 모드(Demo Tour Mode)는 **앱 다운로드 없이, 회원가입 없이** SafeTrip의 핵심 기능을 체험할 수 있도록 제공하는 비로그인 온보딩 보조 기능이다.

본 문서는 데모 투어 모드의 기능 범위, 시나리오 구성, 역할 전환, 가디언 과금 체험, 여행 기간 시뮬레이션, 전환 유도 전략 등 데모 모드 설계 및 구현에 필요한 모든 원칙을 정의한다.

### 1.2 적용 범위

- Flutter 앱 비로그인 상태에서 진입하는 데모 투어 화면 전체
- 웰컴화면(#29)의 "먼저 둘러보기" CTA를 통해 진입하는 모든 플로우
- 데모 모드 내 시나리오 선택, 역할 전환, 등급 비교, 가디언 과금 체험 패널
- 데모 종료 후 실제 앱 전환 유도 모달

### 1.3 진입 경로

```
웰컴화면(#29) → "먼저 둘러보기" CTA 버튼
    │
    └── 시나리오 선택 화면 (S1 / S2 / S3)
            │
            └── 데모 메인 화면 (지도·일정·멤버·채팅·안전가이드 탭)
```

## §2. 데모 모드 기본 원칙 (D1~D5)

데모 모드는 다음 5개 기본 원칙(D1~D5)을 모든 상황에서 준수해야 한다.

| 원칙 ID | 원칙명 | 설명 |
|---------|--------|------|
| **D1** | 서버 미기록 | 모든 데모 데이터는 로컬 시뮬레이션으로 처리하며, 서버 API를 호출하지 않는다. Analytics 이벤트만 예외적으로 서버에 전송한다. |
| **D2** | 실제 UI 동일 | 실제 앱과 동일한 화면 구성, 컴포넌트, 인터랙션을 사용한다. 데모 전용 간소화 UI를 별도로 제작하지 않는다. |
| **D3** | 데모 배지 | 모든 화면 상단에 "데모 모드" 배지를 항상 표시한다. 사용자가 실제 앱으로 오인하지 않도록 명확히 구분한다. |
| **D4** | 안전 격리 | 데모 중 실제 계정·데이터·결제 정보와 완전히 격리된다. 데모 모드에서 생성된 어떤 데이터도 실제 서비스에 영향을 주지 않는다. |
| **D5** | 탈출구 항상 제공 | 언제든지 "실제 앱으로 전환" 버튼을 화면 내에서 접근 가능하게 표시한다. |

## §3. 본문

### §3.1 3개 프리셋 시나리오

데모 모드는 SafeTrip의 대표적인 사용 사례를 체험할 수 있도록 3개의 프리셋 시나리오를 제공한다.

#### 시나리오 개요

| 시나리오 | 제목 | 프라이버시 등급 | 멤버 구성 | 여행 기간 | 특징 |
|---------|------|:--------------:|---------|---------|------|
| **S1** | 학생 단체 여행 (제주도) | 안전 최우선 (`safety_first`) | 캡틴(Captain, 교사) 1 + 크루장(Crew Chief) 2 + 크루(Crew) 20 + 가디언(Guardian, 학부모) 10 | 3일 | SOS 훈련, 실시간 위치 전체 공개, 가디언 항상 공유 |
| **S2** | 친구들과 해외여행 (도쿄) | 표준 (`standard`) | 캡틴(Captain) 1 + 크루장(Crew Chief) 1 + 크루(Crew) 4 | 7일 | 채팅, 일정 공유, 안전가이드, 자유 위치 설정 |
| **S3** | 해외 출장 / 패키지 투어 | 프라이버시 우선 (`privacy_first`) | 캡틴(Captain, 가이드) 1 + 크루(Crew) 15 + 가디언(Guardian, 회사) 2 | 5일 | 위치 공개 최소화, 체크포인트만, 일정 연동 스케줄 |

#### 시나리오별 시뮬레이션 데이터 구성

각 시나리오는 다음 데이터를 로컬 JSON 파일(`assets/demo/scenario_*.json`)에 포함하여 완전한 체험을 제공한다.

| 데이터 항목 | 내용 |
|------------|------|
| 멤버 목록 | 가상 이름, 프로필 이미지(저해상도), 역할, 연결 가디언 정보 |
| 일정 | D+1~D+N 날짜별 여행 일정 (최대 15일 범위 내) |
| 가상 위치 이벤트 | 맵 핀, 이동 경로 폴리라인, 실시간 이동 시뮬레이션 |
| 시뮬레이션 이벤트 | SOS 발생, 멤버 이탈, 일정 변경, 지오펜스 이벤트 |

#### S1 — 학생 단체 여행 (제주도) 시뮬레이션 이벤트 예시

```
[D+1 09:00] 출발 — 공항 집결 체크포인트
[D+1 14:00] 성산일출봉 도착 — 지오펜스 IN 이벤트
[D+1 16:30] 크루(학생) 1명 지오펜스 이탈 이벤트 → 가디언 알림 시뮬
[D+2 10:00] SOS 훈련 시뮬레이션 → 전체 멤버 위치 공유 + 가디언 긴급 알림
[D+3 17:00] 귀국 — 여행 종료 안내
```

#### S2 — 친구들과 해외여행 (도쿄) 시뮬레이션 이벤트 예시

```
[D+1] 도착, 채팅으로 일정 공유
[D+3] 개별 자유 일정 — 위치 공개 범위 "관리자만" 전환 체험
[D+5] 일정 변경 — 크루장이 일정 수정 알림
[D+7] 귀국 — 여행 완료 화면
```

#### S3 — 해외 출장 / 패키지 투어 시뮬레이션 이벤트 예시

```
[D+1] 현지 도착, 일정 연동 체크포인트만 위치 공유
[D+2] 업무 회의 시간 — 위치 비공유 구간
[D+4] 가디언(회사 안전담당자) 긴급 위치 요청 — 멤버 승인 플로우 시뮬
[D+5] 귀국
```

### §3.2 ★ 여행 기간 15일 시뮬레이션 (§02.3)

비즈니스 원칙 §02.3에 따라 1회 여행의 최대 기간은 15일이다. 데모 모드는 이 제한을 시각적으로 체험할 수 있도록 구현한다.

#### 15일 제한 시뮬레이션 규칙

| 규칙 | 내용 |
|------|------|
| 시뮬레이션 최대 기간 | 15일 이내 (비즈니스 원칙 §02.3 기준) |
| 15일 초과 시도 | 자동 종료 안내 체험 화면 표시: "여행은 최대 15일까지 설정 가능합니다. 분할 생성해 주세요" |
| 타임라인 UI | D-1 ~ D+15 범위 표시, 초과 구간은 회색 처리 및 잠금 아이콘 표시 |

#### 타임 슬라이더 구성

데모 모드 내 타임 슬라이더를 통해 여행 전후 시점을 자유롭게 이동하여 체험할 수 있다.

```
[여행 전 구간]  여행 중 구간 (D+1 ~ D+15 범위)  [여행 후 구간]
     D-7  ◄──────────────────────────────────►  D+18
           ↑                              ↑
        출발일(D+1)                 귀국일(D+N, max D+15)

회색 처리 구간: D+16 이상 (15일 제한 초과)
```

| 구간 | 표시 방식 | 데모 체험 내용 |
|------|----------|--------------|
| 여행 전 (D-7 ~ D-1) | 활성 (파란색 테두리) | planning 상태, 멤버 초대 화면, 일정 등록 |
| 여행 중 (D+1 ~ D+N) | 활성 (강조 색상) | active 상태, 실시간 위치, 채팅, 이벤트 |
| 여행 후 (D+N+1 ~ D+N+3) | 활성 (회색조) | completed 상태, 이동기록 조회 |
| 15일 초과 구간 | 비활성 (진한 회색) | 잠금 아이콘 + "최대 15일 제한" 툴팁 |

### §3.3 ★ 가디언 과금 체험 (§09.3)

비즈니스 원칙 §09.3에 따라 개인 가디언은 멤버당 2명까지 무료이며, 3명째부터 1명당 1,900원/여행의 추가 과금이 발생한다. 데모 모드는 이 차이를 직접 체험할 수 있는 비교 뷰를 제공한다.

#### 무료 / 유료 가디언 시뮬레이션 사양

| 항목 | 무료 가디언 시뮬 | 유료 가디언 시뮬 |
|------|:---------------:|:---------------:|
| 연결 가능 인원 | 기본 최대 2명 | 추가 최대 3명 (3~5번째) |
| 위치 이력 조회 | 당일만 | 전체 이동기록 |
| 이동기록 보존 | 24시간 제한 | 여행 기간 전체 |
| 알림 | 기본 알림 (SOS, 지오펜스) | 확장 알림 (이탈, 체크포인트, 주기 리포트) |
| 히스토리 조회 | 불가 | 가능 (날짜별 이동 경로) |
| 과금 | 무료 | 1,900원/여행 (시뮬레이션, 실제 결제 없음) |

#### 비교 뷰 UI 구성

```
┌─────────────────────────────────────────────┐
│  [가디언 과금 체험]                           │
├──────────────────┬──────────────────────────┤
│   무료 가디언    │     유료 가디언                      │
│  (최대 2명)      │  (3~5번째 가디언, 1,900원/여행)      │
│                  │                           │
│  • 당일 위치만   │  • 전체 이동기록            │
│  • 기본 알림     │  • 확장 알림               │
│  • 24시간 이력   │  • 히스토리 조회            │
│                  │                           │
│   [현재 체험 중] │  [업그레이드 체험하기 →]    │
└──────────────────┴──────────────────────────┘
```

- 비교 뷰는 좌(무료) / 우(유료)를 나란히 배치하여 차이를 즉각 인지할 수 있게 한다.
- "업그레이드 체험하기" 버튼 클릭 시 유료 가디언 시뮬레이션으로 전환된다.
- 유료 가디언 시뮬레이션 전환 시 안내 모달을 표시한다:

```
[안내 모달]
"실제 앱에서는 가디언 추가 시 1,900원/여행의 결제가 필요합니다.
 데모에서는 결제 없이 유료 기능을 체험해 보실 수 있습니다."

[계속 체험하기]  [실제 앱 시작하기]
```

### §3.4 역할 전환 패널

데모 모드에서는 하나의 시나리오를 4가지 역할(캡틴/크루장/크루/가디언) 시점으로 전환하여 각 역할의 기능 차이를 체험할 수 있다.

#### 역할 전환 패널 구성

```
[역할 전환]
  [캡틴(Captain)]  [크루장(Crew Chief)]  [크루(Crew)]  [가디언(Guardian)]
         ↑
    현재 선택된 역할 (강조 표시)
```

#### 역할 전환 동작 규칙

| 규칙 | 내용 |
|------|------|
| 전환 방식 | 동일 시나리오를 선택한 역할 시점으로 즉시 재렌더링 |
| 데이터 일관성 | 역할 전환 시 동일 시뮬레이션 데이터 유지, 뷰만 변경 |
| 잠긴 기능 표시 | 해당 역할에서 접근 불가한 기능은 자물쇠 아이콘(🔒) + 그레이아웃 처리 |
| 전환 애니메이션 | 페이드 인/아웃 (200ms) |

#### 역할별 접근 가능 기능 요약 (데모 기준)

| 기능 | 캡틴 | 크루장 | 크루 | 가디언 |
|------|:----:|:------:|:----:|:------:|
| 여행 설정 변경 | ✅ | 🔒 | 🔒 | 🔒 |
| 멤버 초대 | ✅ | ✅ | 🔒 | 🔒 |
| 일정 편집 | ✅ | ✅ | 🔒 | 🔒 |
| 전체 멤버 위치 조회 | ✅ | ✅ | ✅ | 연결 멤버만 |
| SOS 발송 | ✅ | ✅ | ✅ | 🔒 |
| 채팅 | ✅ | ✅ | ✅ | 🔒 |
| 가디언 과금 체험 | ✅ | 🔒 | 🔒 | 🔒 |

### §3.5 등급 비교 체험

데모 모드에서는 동일한 상황을 3개 프라이버시 등급(안전 최우선/표준/프라이버시 우선)으로 전환하여 차이를 직접 체험할 수 있다.

#### 등급 비교 체험 UI

```
┌─────────────────────────────────────────────────┐
│  [프라이버시 등급 비교 체험]                      │
│                                                  │
│  [안전 최우선]    [표준]    [프라이버시 우선]      │
│       ↑                                          │
│   현재 선택                                      │
│                                                  │
│  [등급별 차이 시각화]                             │
│  • 위치 공유 범위: 24시간 ← 현재 → 일정 연동만   │
│  • 가디언 공유:  항상 ← 현재 → 스케줄 OFF 비공유 │
│  • 마커 표시:   실시간 ← 현재 → 30분 스냅샷      │
└─────────────────────────────────────────────────┘
```

#### 등급별 데모 차이 상세

| 차이 항목 | 안전 최우선 | 표준 | 프라이버시 우선 |
|----------|:----------:|:----:|:--------------:|
| 위치 공유 범위 | 24시간 실시간 | 24시간 (OFF시 저빈도) | 일정 연동 시간대만 |
| 가디언 공유 | 항상 공유 | 항상 (OFF시 30분 스냅샷) | 스케줄 OFF 비공유 |
| 지도 멤버 마커 | 실시간 갱신 | 실시간 갱신 | 체크포인트만 핀 표시 |
| 가디언 일시 중지 | 불가 | 최대 12시간 가능 | 최대 24시간 가능 |
| 지오펜스 이벤트 → 가디언 전달 | 항상 | 스케줄 ON 시만 | 전달 안 함 |

### §3.6 화면별 데모 플로우

데모 모드의 전체 화면 진행 순서는 다음과 같다.

#### 전체 플로우

```
[1단계] 진입
  웰컴화면(#29) → "먼저 둘러보기" 버튼 탭
        │
        ▼
[2단계] 시나리오 선택
  3개 카드 (S1/S2/S3) 중 하나 선택
  각 카드: 시나리오명 + 대표 이미지 + 멤버 구성 요약 + 여행 기간
        │
        ▼
[3단계] 메인 탭 자유 체험
  지도 탭 / 일정 탭 / 멤버 탭 / 채팅 탭 / 안전가이드 탭
  코치마크 가이드 표시 (첫 방문 기능)
        │
        ▼
[4단계] 이벤트 시뮬레이션
  시나리오별 주요 이벤트 자동 실행 (SOS, 지오펜스, 일정 변경 등)
  코치마크로 이벤트 흐름 안내
        │
        ▼
[5단계] 역할·등급 전환 체험
  역할 전환 패널 + 등급 비교 체험 패널
  가디언 과금 체험 비교 뷰
  15일 타임 슬라이더 체험 (메인 탭 지도에서도 접근 가능)
        │
        ▼
[6단계] 데모 종료
  "데모 체험을 완료했습니다" 화면
  전환 유도 모달: "여행 만들기" / "초대코드로 참여"
```

## §5. 프라이버시 등급별 동작 차이

3개 프리셋 시나리오는 각각 3개 프라이버시 등급에 대응하며, 등급 비교 체험 패널에서 직접 전환하여 차이를 체험할 수 있다.

| 동작 항목 | 안전 최우선 (S1) | 표준 (S2) | 프라이버시 우선 (S3) |
|----------|:--------------:|:---------:|:-----------------:|
| 대표 시나리오 | 학생 단체 여행 | 친구들과 해외여행 | 해외 출장 / 패키지 투어 |
| 지도 멤버 마커 | 실시간 표시 | 실시간 표시 | 체크포인트만 핀 표시 |
| 가디언 위치 공유 | 항상 (24시간) | 스케줄 ON 시 실시간 / OFF 시 30분 스냅샷 | 스케줄 OFF 시 비공유 |
| 가디언 일시 중지 시뮬 | 버튼 비활성 (불가) | 최대 12시간 | 최대 24시간 |
| 지오펜스 이탈 → 가디언 알림 | 항상 전달 | 스케줄 ON 시만 | 전달 안 함 |
| SOS 시 가디언 위치 수신 | 항상 | 항상 | 항상 |
| 위치 비공유 구간 | 없음 | 일부 있음 | 일정 외 전체 비공유 |

## §6. 에러 및 엣지케이스

| 상황 | 처리 규칙 |
|------|---------|
| 데모 중 실제 푸시 알림 수신 | 데모 배지 유지, 실제 알림은 시스템 알림 영역에만 표시. 데모 화면 내에서는 별도 처리 없음 |
| 데모 중 네트워크 오프라인 전환 | 로컬 JSON 데이터 기반이므로 데모 계속 진행. 오프라인 상태 표시는 시스템 알림바 기준 |
| 기기 가로 모드 전환 | 지도 탭, 비교 뷰는 가로 모드 레이아웃 지원. 탭 구성은 하단에서 우측 사이드로 전환 |
| 앱 백그라운드 후 복귀 | 데모 상태 유지 (최대 30분). 30분 초과 후 복귀 시 시나리오 선택 화면으로 초기화 |
| 시뮬레이션 JSON 로딩 실패 | 기본 S1 시나리오로 대체 실행. 에러 토스트: "데이터 로딩에 실패했습니다. 기본 시나리오로 시작합니다." |
| 역할 전환 중 이벤트 재생 중 | 현재 재생 이벤트 일시 정지 → 역할 전환 완료 후 재개 |
| 타임 슬라이더 15일 초과 드래그 | 15일 경계에서 슬라이더 고정, 진동 햅틱 + "여행은 최대 15일까지 설정 가능합니다. 분할 생성해 주세요" 툴팁 표시 |
| 가디언 역할에서 채팅 탭 접근 시도 | 잠금 화면 표시: "가디언은 채팅 기능을 이용할 수 없습니다. 역할을 전환해 보세요." |

| `demo_completed` | Firebase Analytics / 서버 분석 DB |
| `demo_converted` | Firebase Analytics / 서버 분석 DB |

> 이 Analytics 이벤트에는 개인 식별 정보를 포함하지 않는다. 데모 세션 ID(임시 UUID)만 사용한다.

## §8. 오프라인 대응
