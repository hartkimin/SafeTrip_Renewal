## §1. 글로벌 레이아웃 시스템

### 1.1 전체 구조

```
┌──────────────────────────────────────────────────────────────────────┐
│  [A] 글로벌 헤더 (56px)                                               │
│  ┌──────────┬───────────────────────────────────────┬──────────────┐ │
│  │  Logo    │  🔍 글로벌 검색                        │ 🔔 알림  👤  │ │
│  └──────────┴───────────────────────────────────────┴──────────────┘ │
├──────┬───────────────────────────────────────────────────────────────┤
│  [B] │  [C] 메인 콘텐츠 영역                                         │
│ 사이 │  ┌─────────────────────────────────────────────────────────┐  │
│ 드바 │  │  [C1] 페이지 헤더 (타이틀 + 액션 버튼)                     │  │
│      │  ├─────────────────────────────────────────────────────────┤  │
│  240 │  │  [C2] 페이지 콘텐츠                                      │  │
│  px  │  │                                                         │  │
│      │  │  (각 모듈별 콘텐츠 영역 — 스크롤 가능)                     │  │
│ 접힘 │  │                                                         │  │
│ 시   │  │                                                         │  │
│ 72px │  └─────────────────────────────────────────────────────────┘  │
│      │                                                               │
└──────┴───────────────────────────────────────────────────────────────┘
```

### 1.2 영역별 사양

| 영역 | 고정/스크롤 | 높이/너비 | 설명 |
|------|:---------:|----------|------|
| [A] 글로벌 헤더 | 고정 (sticky) | H: 56px | 로고, 검색, 알림, 사용자 메뉴 |
| [B] 사이드바 | 고정 | W: 240px (접힘 시 72px) | 네비게이션 메뉴 |
| [C] 메인 콘텐츠 | 스크롤 | 나머지 영역 | 각 페이지 콘텐츠 |
| [C1] 페이지 헤더 | 고정 (sticky) | H: 64px | 페이지 타이틀, 브레드크럼, 액션 |
| [C2] 페이지 콘텐츠 | 스크롤 | 가변 | 실제 데이터 영역 |

### 1.3 반응형 브레이크포인트

| 브레이크포인트 | 너비 | 사이드바 | 레이아웃 |
|:------------:|:----:|:-------:|---------|
| Desktop XL | ≥ 1440px | 펼침 (240px) | 4컬럼 그리드 |
| Desktop | ≥ 1024px | 펼침 (240px) | 3컬럼 그리드 |
| Tablet | ≥ 768px | 접힘 (72px) | 2컬럼 그리드 |
| Mobile | < 768px | 숨김 (오버레이) | 1컬럼 스택 |

## §2. 글로벌 헤더 [A]

```
┌───────────────────────────────────────────────────────────────────────┐
│  🛫 SafeTrip Admin  │  🔍 사용자, 여행, SOS 검색...       │ 🔔(3) 👤 Admin ▾ │
└───────────────────────────────────────────────────────────────────────┘
```

| 요소 | 위치 | 설명 |
|------|------|------|
| **로고** | 좌측 | SafeTrip Admin 로고, 클릭 시 대시보드 이동 |
| **글로벌 검색** | 중앙 | 사용자/여행/SOS 통합 검색, Cmd+K 단축키 지원 |
| **알림 벨** | 우측 | 미확인 알림 수 뱃지, 클릭 시 알림 드롭다운 |
| **사용자 메뉴** | 우측 | 관리자 이름, 역할 표시, 프로필/로그아웃 드롭다운 |


### 3.2 사이드바 상태

| 상태 | 너비 | 표시 | 전환 트리거 |
|------|:----:|------|-----------|
| 펼침 | 240px | 아이콘 + 텍스트 | 기본 (Desktop) |
| 접힘 | 72px | 아이콘만 (호버 시 툴팁) | Tablet 또는 수동 토글 |
| 숨김 | 0px | 오버레이 (햄버거 메뉴) | Mobile |

### 3.3 활성 SOS 뱃지

- SOS 관제 센터 메뉴 항목에 활성 SOS 건수 뱃지 상시 표시
- 빨간색 원 + 숫자, 깜빡임 애니메이션 (0건이면 숨김)
- 사이드바 접힘 상태에서도 아이콘 옆에 미니 뱃지 표시

## §8. 인터랙션 패턴

### 8.1 목록 → 상세 네비게이션

```
목록 페이지  ──[행 클릭]──→  상세 페이지 (새 라우트)
                              또는
목록 페이지  ──[행 클릭]──→  슬라이드 패널 (SOS 상세)
```

- SOS, 알림: 슬라이드 패널 (빠른 확인 및 대응)
- 사용자, 여행, 결제, B2B: 별도 상세 페이지 (풍부한 정보)

### 8.2 데이터 요청 패턴

| 패턴 | 사용처 | 설명 |
|------|--------|------|
| **SWR/TanStack Query** | 모든 서버 데이터 | 자동 캐시, 재검증, 백그라운드 갱신 |
| **WebSocket** | SOS, 알림 | 실시간 스트리밍 |
| **Polling (30s)** | 대시보드 KPI | 자동 새로고침 |
| **Pagination** | 목록 페이지 | 서버 사이드 페이지네이션 (`page`, `limit`) |
| **Optimistic Update** | Ban, 상태 변경 | UI 즉시 반영 후 서버 확인 |

### 8.3 에러 상태 처리

| 상태 | UI 처리 |
|------|---------|
| 로딩 | 스켈레톤 UI (컴포넌트별 독립) |
| 빈 데이터 | Empty State 일러스트 + 안내 메시지 |
| API 에러 | Error Banner + 재시도 버튼 |
| 인증 만료 | 로그인 페이지 자동 리다이렉트 |
