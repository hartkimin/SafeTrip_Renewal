## 1. 문서 목적 및 적용 범위

### 1.1 목적

본 문서는 SafeTrip 앱 서비스(이하 "서비스")의 개인정보처리방침에 관한 설계 원칙을 정의한다. 「개인정보 보호법」 제30조 및 동법 시행령 제31조에 따라 정보주체에게 개인정보 처리에 관한 절차 및 기준을 안내하고, 관련 고충을 신속하게 처리하기 위한 기반 문서이다.

본 문서는 두 가지 역할을 수행한다.

1. **내부 설계 원칙**: SafeTrip 개발·운영팀이 개인정보 처리 로직을 구현할 때 참조하는 기술적 기준
2. **외부 공개 방침의 원천**: 앱 내 및 웹사이트에 공개되는 개인정보처리방침의 원문 소스

### 1.2 적용 범위

- SafeTrip 모바일 앱(iOS, Android)
- SafeTrip 웹 서비스(관리 콘솔 포함)
- SafeTrip API를 통해 처리되는 모든 개인정보
- B2B 고객(학교, 여행사, 보험사)이 SafeTrip을 통해 처리하는 개인정보

### 1.3 법적 근거

| 법령 | 관련 조항 | 적용 사항 |
|------|----------|----------|
| 개인정보 보호법 | 제30조 | 개인정보처리방침 수립·공개 의무 |

| 정보통신망법 | 제27조의2 | 개인정보 처리 위탁 고지 |

## 3. 개인정보 수집 항목 및 목적

### 3.1 서비스 이용 계약 체결·이행을 위해 처리하는 개인정보 (동의 불요)

2024년 9월 개정된 개인정보 보호법에 따라, 서비스 이용 계약의 체결·이행에 필요한 개인정보는 정보주체의 별도 동의 없이 처리할 수 있다.

| 수집 시점 | 수집 항목 | 처리 목적 | 보유 기간 |
|----------|----------|----------|----------|
| 회원가입 | 휴대전화번호, 국가번호 | 본인 인증, 계정 식별, 긴급 연락 | 회원 탈퇴 시까지 (탈퇴 후 7일 유예) |
| 회원가입 | 표시 이름 | 그룹 내 멤버 식별 | 회원 탈퇴 시까지 |
| 서비스 이용 | FCM 토큰, 디바이스 정보(OS, 버전, 모델) | 푸시 알림 발송, SOS 알림 전달, 앱 호환성 | 회원 탈퇴 시까지 |
| 여행 생성 | 여행명, 목적지 국가/도시, 여행 기간 | 여행 관리, 안전 정보 제공 | 여행 종료 후 90일 |
| 유료 서비스 | 결제 정보(결제 수단, 거래 ID) | 가디언 추가 과금, 유료 상품 결제·환불 | 전자상거래법에 따라 5년 |

> **DB 참조**: `TB_USER` (phone_number, phone_country_code, display_name, fcm_token, device_info), `TB_TRIP` (trip_name, destination_country_code, start_date, end_date)

### 3.2 정보주체의 동의를 받아 처리하는 개인정보

민감정보, 고유식별정보, 제3자 제공에 해당하는 항목은 반드시 명시적 동의를 받아 처리한다.

#### 3.2.1 위치정보 (별도 동의 필수)

| 수집 항목 | 처리 목적 | 수집 빈도 | 보유 기간 |
|----------|----------|----------|----------|
| 실시간 GPS 좌표 (위도, 경도) | 그룹 내 멤버 위치 공유, 안전 모니터링 | 이동 시 30초, 정지 시 120초 | 여행 종료 후 90일 |
| 실시간 GPS 좌표 (SOS 발동 시) | SOS 긴급 알림에 위치 포함, 구조 요청 지원 | SOS 발동 시 즉시 | 여행 종료 후 3년 (법적 분쟁 대비) |
| 체류 지점 데이터 | 이동기록 타임라인, 일정 매칭 | 체류 감지 알고리즘 (5분 주기) | 여행 종료 후 30일 |
| 지오펜스 출입 기록 | 안전 구역 이탈·진입 알림 | 이벤트 발생 시 | 여행 종료 후 90일 |

> **위치정보법 준수**: 위치정보 수집·이용 동의는 서비스 이용약관 동의와 별도로 받으며, 언제든 동의를 철회할 수 있다. 위치정보 동의 철회 시에도 SOS 발동 상황에서는 프라이버시 설정과 무관하게 위치가 포함된다 (SOS 원칙 참조).

#### 3.2.2 선택 동의 항목

| 수집 항목 | 처리 목적 | 동의 거부 시 영향 | 보유 기간 |
|----------|----------|----------------|----------|
| 프로필 이미지 | 멤버 식별, 프로필 표시 | 이니셜 아바타 자동 적용 | 회원 탈퇴 시까지 |
| 이메일 주소 | 긴급 시 추가 연락 수단, 계정 복구 | 이메일 기반 기능 이용 불가 | 회원 탈퇴 시까지 |
| 긴급 연락처 (최대 2명) | SOS 발동 시 긴급 연락처에 알림 전달 | SOS 알림이 그룹 멤버에게만 전달 | 회원 탈퇴 시까지 |
| 생년월일 | 미성년자 여부 판별, 연령별 동의 체계 적용 | 미성년자 보호 기능 미적용 | 회원 탈퇴 시까지 |

> **DB 참조**: `TB_USER` (profile_image_url, email, date_of_birth), 긴급 연락처는 별도 관리

### 3.3 자동 수집 항목

| 수집 항목 | 처리 목적 | 수집 방법 | 보유 기간 |
|----------|----------|----------|----------|
| 앱 사용 로그 | 서비스 개선, 오류 분석 | 자동 기록 | 수집일로부터 1년 |

### 4.2 계정 삭제 시 데이터 처리 흐름

```
계정 삭제 요청
│
├── 즉시 처리 (소프트 삭제)
│   · TB_USER.deleted_at 설정
│   · 활성 여행에서 자동 탈퇴 (캡틴인 경우 위임 강제)
│   · 가디언 연결 관계 전체 해제
│   · 로그인 차단
│
├── 7일 유예 기간 내 재로그인
│   · 전체 데이터 복원
│   · "복원 되었습니다" 안내
│
└── 7일 유예 기간 경과 (하드 삭제)
    · 프로필 이미지: CDN에서 삭제
    · 표시 이름 → "알 수 없는 사용자"로 대체
    · 전화번호, 이메일: 영구 삭제
    · 긴급 연락처: 영구 삭제
    · 위치 데이터: 즉시 삭제 (법적 보관 의무 항목 제외)
    · SOS 기록: 법적 보관 기간(3년)까지 익명화 보관 후 삭제
```

> **구현 참조**: 프로필화면 원칙 v1.0 §11.3, 설정 메뉴 원칙 v1.0 §3.2.3

## 6. 개인정보의 제3자 제공

### 6.1 그룹 내 멤버 간 정보 공유

SafeTrip의 핵심 기능인 그룹 안전 모니터링을 위해, 같은 여행 그룹 멤버 간에 §5에 정의된 범위 내에서 개인정보가 공유된다. 이는 서비스 이용 계약 체결 시 고지되며, 여행 그룹 참여 시 동의한 것으로 본다.

### 6.2 외부 제3자 제공

| 제공받는 자 | 제공 항목 | 제공 목적 | 보유 기간 | 동의 |
|-----------|----------|----------|----------|:----:|
| 외교부 (MOFA API) | 여행 목적지 국가 코드 | 여행 안전 정보·경보 조회 | 조회 즉시 파기 (API 호출용) | 동의 불요 (공공 API) |
| LLM 서비스 제공사 | 여행 국가, 기간, 일정 (비식별) | AI 안전 브리핑, 일정 추천, 번역 | 제공사 약관에 따름 | 별도 동의 |
| 결제 대행사 (PG사) | 결제 정보 | 유료 서비스 결제 처리 | PG사 약관에 따름 | 결제 시 동의 |

> **AI 데이터 비식별화**: AI 기능 사용 시 LLM 제공사에 전달되는 데이터에는 개인 식별 정보(이름, 전화번호, 이메일)가 포함되지 않는다. 여행 정보(국가, 기간, 일정)만 비식별 형태로 전달된다. (AI 기능 원칙 v1.0 §8 참조)

### 6.3 SOS 발동 시 긴급 정보 제공

SOS 발동 시 아래 정보가 그룹 멤버 및 가디언에게 즉시 공유된다. 이는 생명·신체 안전을 위한 긴급 조치로서 별도 동의 없이 처리된다 (개인정보 보호법 제15조 제1항 제3호).

| 제공 정보 | 제공 대상 | 근거 |
|----------|----------|------|
| SOS 발송자 위치 (GPS 좌표) | 전체 그룹 멤버, 가디언 | 급박한 위험 방지 |
| SOS 발송자 이름, 배터리 잔량 | 전체 그룹 멤버, 가디언 | 구조 지원 |
| SOS 이벤트 기록 | 긴급 구조기관 (요청 시) | 수사 협조 |

| 삭제 요청 중 진행 중인 여행 존재 | "참여 중인 여행({trip_name})이 있습니다. 여행 탈퇴 후 삭제를 진행해주세요." |

### 16.3 오프라인 동의 관련

| 상황 | 처리 |
|------|------|

## 17. DB 스키마

### 17.1 TB_USER_CONSENT (동의 이력 관리)

```sql
CREATE TABLE tb_user_consent (
    consent_id          BIGINT PRIMARY KEY GENERATED ALWAYS AS IDENTITY,
    user_id             VARCHAR(128) NOT NULL REFERENCES TB_USER(user_id),
    consent_type        VARCHAR(50) NOT NULL,
        -- 'terms_of_service'        : 서비스 이용약관 (필수)
        -- 'privacy_policy'          : 개인정보처리방침 (필수)
        -- 'location_collection'     : 위치정보 수집·이용 (필수)
        -- 'lbs_terms'               : 위치기반서비스 이용약관 (필수)
        -- 'international_transfer'  : 국외 이전 동의 (필수)
        -- 'ai_data_usage'           : AI 데이터 활용 (선택)
        -- 'marketing'               : 마케팅 정보 수신 (선택)
        -- 'minor_guardian'           : 미성년자 법정대리인 동의
    consent_version     VARCHAR(20) NOT NULL,       -- 동의한 약관 버전 (e.g., 'v1.0')
    is_agreed           BOOLEAN NOT NULL,
    agreed_at           TIMESTAMPTZ,
    withdrawn_at        TIMESTAMPTZ,                -- 동의 철회 시각
    guardian_user_id    VARCHAR(128),                -- 법정대리인 user_id (14세 미만)
    ip_address          VARCHAR(45),                 -- 동의 시 IP (감사 추적용)
    device_info         VARCHAR(200),                -- 동의 시 디바이스
    created_at          TIMESTAMPTZ DEFAULT NOW(),
    updated_at          TIMESTAMPTZ
);

CREATE INDEX idx_user_consent_user ON tb_user_consent(user_id, consent_type);
CREATE INDEX idx_user_consent_type ON tb_user_consent(consent_type, is_agreed);
```

### 17.2 TB_DATA_DELETION_LOG (데이터 삭제 이력)

```sql
CREATE TABLE tb_data_deletion_log (
    deletion_id         BIGINT PRIMARY KEY GENERATED ALWAYS AS IDENTITY,
    user_id             VARCHAR(128) NOT NULL,       -- FK 아님 (삭제된 유저 참조)
    deletion_type       VARCHAR(30) NOT NULL,
        -- 'account_soft_delete'    : 계정 소프트 삭제
        -- 'account_hard_delete'    : 계정 하드 삭제
        -- 'location_batch_delete'  : 위치 데이터 배치 삭제
        -- 'trip_data_delete'       : 여행 데이터 삭제
        -- 'consent_withdrawal'     : 동의 철회에 따른 삭제
    affected_tables     TEXT[],                      -- 삭제 영향 테이블 목록
    record_count        INTEGER,                     -- 삭제 레코드 수
    requested_by        VARCHAR(20) NOT NULL,
        -- 'user'    : 정보주체 요청
        -- 'system'  : 배치 자동 삭제
        -- 'admin'   : 관리자 처리
    executed_at         TIMESTAMPTZ DEFAULT NOW(),
    notes               TEXT
);

CREATE INDEX idx_deletion_log_user ON tb_data_deletion_log(user_id);
CREATE INDEX idx_deletion_log_date ON tb_data_deletion_log(executed_at DESC);
```
