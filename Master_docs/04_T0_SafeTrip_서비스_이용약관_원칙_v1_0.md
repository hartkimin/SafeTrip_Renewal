## 1. 문서 목적 및 적용 범위

### 1.1 목적

본 문서는 SafeTrip 앱 서비스(이하 "서비스")의 서비스 이용약관에 관한 설계 원칙을 정의한다. 「전자상거래 등에서의 소비자보호에 관한 법률」(이하 "전자상거래법"), 「정보통신망 이용촉진 및 정보보호 등에 관한 법률」(이하 "정보통신망법") 등 관련 법령에 따라 서비스 이용 조건, 유료 서비스 결제·환불, SOS 기능의 면책 범위 등을 명시하여 법적 분쟁을 예방하기 위한 기반 문서이다.

본 문서는 두 가지 역할을 수행한다.

1. **내부 설계 원칙**: SafeTrip 개발·운영팀이 서비스 정책을 구현할 때 참조하는 기술적·운영적 기준
2. **외부 공개 약관의 원천**: 앱 내 및 웹사이트에 공개되는 서비스 이용약관의 원문 소스

### 1.2 적용 범위

- SafeTrip 모바일 앱(iOS, Android)에서 제공하는 모든 서비스
- SafeTrip 웹 서비스(관리 콘솔 포함)
- 무료 서비스 및 유료 서비스(트랜잭션 과금, AI 구독, 가디언 추가 과금)
- B2B 고객(학교, 여행사, 보험사)이 SafeTrip을 통해 이용하는 서비스
- 데모 모드에서의 체험 기능(실제 서비스 기능 제외)

## 3. 용어 정의

본 약관에서 사용하는 주요 용어의 정의는 다음과 같다.

| 용어 | 정의 |
|------|------|
| **서비스** | 회사가 SafeTrip 앱 및 웹을 통해 제공하는 여행 안전 관련 일체의 서비스 |
| **회원** | 본 약관에 동의하고 회원가입을 완료하여 서비스를 이용하는 자 |
| **여행** | 회원이 서비스 내에서 생성하거나 참여하는 하나의 여행 단위 (Trip) |
| **캡틴** | 여행을 생성하고 관리하는 최상위 역할. 프라이버시 등급 설정, 멤버 관리, 여행 삭제 권한 보유 |
| **크루장** | 캡틴으로부터 일부 관리 권한을 위임받은 역할. 일정 관리, 멤버 초대 권한 보유 |
| **크루** | 여행에 참여하는 일반 멤버 역할 |
| **가디언** | 여행에 직접 참여하지 않으면서 멤버의 위치·안전 정보를 모니터링하는 보호자 역할 |
| **SOS** | 긴급 상황 시 그룹 전체 멤버 및 가디언에게 위치 정보를 포함한 긴급 알림을 발송하는 기능 |
| **Heartbeat** | 멤버의 기기와 서버 간 주기적인 통신 신호. 통신 두절 감지에 사용 |
| **프라이버시 등급** | 여행 내 위치 공유 수준을 결정하는 3단계 등급 (안전 최우선, 표준, 프라이버시 우선) |
| **트랜잭션 과금** | 월정액이 아닌, 여행 참여(건) 단위로 부과되는 과금 방식 |
| **유료 서비스** | 가디언 추가 과금, AI Plus/Pro 구독 등 별도 결제가 필요한 서비스 |
| **데모 모드** | 회원가입 없이 서비스의 주요 기능을 시뮬레이션 데이터로 체험할 수 있는 모드 |
| **초대코드** | 여행 참여를 위해 캡틴 또는 크루장이 생성하는 고유 코드 (QR코드, 딥링크 포함) |

## 4. 약관의 효력 및 변경

### 4.1 약관의 효력

본 약관은 서비스 화면에 게시하거나 기타의 방법으로 회원에게 공지함으로써 효력이 발생한다. 회사는 본 약관의 내용을 회원이 쉽게 알 수 있도록 서비스 초기 화면 또는 연결화면에 게시한다.

### 4.2 약관의 변경

| 변경 유형 | 사전 고지 기간 | 고지 방법 |
|----------|:------------:|----------|
| 이용자에게 불리한 변경 | 시행일 30일 전 | 앱 푸시 + 인앱 팝업 + 홈페이지 공지 |
| 단순 표현 수정, 법률 반영 등 | 시행일 7일 전 | 앱 내 공지 + 홈페이지 공지 |

회사가 약관을 변경하는 경우, 변경 내용과 시행일을 명시하여 현행 약관과 함께 서비스 내에 고지한다. 이용자에게 불리하게 약관을 변경하는 경우에는 시행일 30일 전에 공지하며, 이메일(등록된 경우), 앱 내 팝업 등 전자적 수단으로 개별 통지한다.

### 4.3 변경 동의 처리

회원이 변경된 약관에 동의하지 않는 경우 서비스 이용을 중단하고 이용 계약을 해지할 수 있다. 변경된 약관의 시행일 이후에도 서비스를 계속 이용하는 경우 변경된 약관에 동의한 것으로 간주한다.

### 4.4 약관 외 준칙

본 약관에 명시되지 않은 사항은 전자상거래법, 정보통신망법, 개인정보 보호법, 위치정보법, 약관규제법 등 관련 법령 및 회사가 정한 서비스 세부이용지침에 따른다.

## 5. 이용 계약의 체결

### 5.1 회원가입

이용 계약은 회원이 되고자 하는 자(이하 "가입 신청자")가 본 약관의 내용에 동의한 후 회원가입 신청을 하고, 회사가 이를 승낙함으로써 체결된다.

**회원가입 절차:**

```
1. 휴대전화번호 입력 및 인증번호 확인
2. 서비스 이용약관 동의 (필수)
3. 개인정보처리방침 동의 (필수)
4. 위치기반서비스 이용약관 동의 (필수)
5. 마케팅 정보 수신 동의 (선택)
6. 표시 이름 설정 (2자 이상)
7. 회원가입 완료
```

> **온보딩 참조**: SafeTrip_온보딩_UX_시나리오_v3.0 A-2 약관 동의 화면

### 5.2 가입 승낙 거부 사유

회사는 다음의 경우 가입을 거부하거나 사후에 이용 계약을 해지할 수 있다.

| 거부 사유 | 설명 |
|----------|------|
| 타인의 명의 도용 | 본인 소유가 아닌 휴대전화번호로 가입한 경우 |
| 허위 정보 기재 | 가입 정보에 허위 사항을 기재한 경우 |
| 이전 이용 제한 이력 | 이전에 본 약관 위반으로 이용이 제한된 자가 재가입을 시도하는 경우 |
| 기술적 불가 | 서비스 관련 설비의 여유가 없거나 기술적 장애가 있는 경우 |
| 법적 제한 | 관련 법령에 따라 가입이 제한되는 경우 |

### 5.3 미성년자 이용

비즈니스 원칙 v5.1 §10 및 미성년자 보호 원칙에 따라 연령별 동의 체계를 적용한다.

| 연령 구간 | 가입 동의 주체 | 동의 방식 |
|----------|--------------|----------|
| 8세 이하 | 보호의무자 단독 | 보호의무자 본인 인증 + 관계 증빙 |
| 9~13세 (14세 미만) | 본인 + 법정대리인 | 본인 동의 + 법정대리인 휴대전화 인증 |
| 14~17세 | 본인 (법정대리인 동의 권장) | 본인 동의, 법정대리인에게 고지 |
| 18세 이상 | 본인 | 본인 동의 |

> **법적 근거**: 개인정보 보호법 제22조 (14세 미만 아동), 위치정보법 제25조·제26조

## 6. 서비스의 내용

### 6.1 서비스 구성

SafeTrip은 여행 안전을 목적으로 다음의 서비스를 제공한다.

| 구분 | 서비스 | 설명 |
|------|--------|------|
| 핵심 안전 | 실시간 위치 공유 | 여행 그룹 내 멤버 간 실시간 위치 확인 |
| 핵심 안전 | SOS 긴급 알림 | 긴급 상황 시 그룹 전원에게 위치 포함 알림 발송 |
| 핵심 안전 | Heartbeat 모니터링 | 기기 통신 두절 감지 및 자동 알림 |
| 핵심 안전 | 지오펜스 알림 | 안전 구역 이탈·진입 시 알림 |
| 여행 관리 | 여행 생성·관리 | 여행 일정 생성, 멤버 초대, 역할 배정 |
| 여행 관리 | 일정 관리 | 공유 일정 작성·편집, 위치 기반 스케줄 |
| 여행 관리 | 그룹 채팅 | 여행 그룹 내 실시간 메시지·공지·투표 |
| 정보 제공 | 안전가이드 | 외교부 API 기반 여행 안전 정보 |
| 정보 제공 | 이동기록 | 개인별 이동 경로 타임라인 |
| 부가 서비스 | AI 기능 | AI 일정 생성, 번역, 안전 어시스턴트 등 |
| 부가 서비스 | 가디언 연결 | 여행 외부 보호자의 안전 모니터링 |

### 6.2 여행 기간 제한

비즈니스 원칙 v5.1 §02.3에 따라, 하나의 여행은 최대 15일(15박 16일)까지 설정할 수 있다. 15일을 초과하는 장기 여행의 경우 여행을 분할하여 생성해야 한다.

| 항목 | 내용 |
|------|------|
| 최대 여행 기간 | 15일 (15박 16일) |
| 초과 시 처리 | 여행 생성 불가, 분할 생성 안내 |
| DB 제약 | `TB_TRIP` CHECK 제약 조건 적용 |

### 6.3 역할 체계

서비스 내 역할 및 주요 권한은 다음과 같다.

| 역할 | 여행 참여 | 위치 공유 | 설정 변경 | 멤버 관리 | SOS 발송 | SOS 수신 |
|------|:--------:|:--------:|:--------:|:--------:|:--------:|:--------:|
| 캡틴 | ✅ | ✅ | ✅ (전체) | ✅ | ✅ | ✅ |
| 크루장 | ✅ | ✅ | 일부 | 일부 | ✅ | ✅ |
| 크루 | ✅ | ✅ | 개인만 | ❌ | ✅ | ✅ |
| 가디언 | ❌ | 열람만 | ❌ | ❌ | ❌ | ✅ |

### 6.4 프라이버시 등급

캡틴은 여행 생성 시 3단계 프라이버시 등급 중 하나를 선택하며, 이에 따라 위치 공유 범위가 결정된다.

| 등급 | 위치 수집 | 공유 모드 | 가디언 실시간 공유 | SOS 시 위치 제공 |
|------|:--------:|:--------:|:----------------:|:---------------:|
| 🛡️ 안전 최우선 | 24시간 상시 | 강제 공유 | 항상 | ✅ (등급 무관) |
| 📍 표준 | 24시간 상시 | 자유 설정 | ON 시간대만 | ✅ (등급 무관) |
| 🔒 프라이버시 우선 | 일정 연동 | 자유 설정 | ON 시간대만 | ✅ (등급 무관) |

> **SOS 최상위 원칙**: SOS 기능은 모든 프라이버시 등급, 위치 공유 설정, 가디언 일시 중지를 무시하고 동작한다 (SOS 원칙 v1.0 §1.1).

### 6.5 데모 모드

## 8. SOS 기능 및 면책

### 8.1 SOS 기능의 법적 위치

**SafeTrip의 SOS 기능은 여행 안전을 위한 보조 수단이며, 공식 긴급 구조 서비스(119, 112, 해외 긴급 전화 등)를 대체하지 않는다.**

이용자는 긴급 상황 발생 시 반드시 해당 국가의 공식 긴급 구조 서비스에 먼저 연락해야 하며, SafeTrip의 SOS 기능은 그룹 내 알림 및 위치 공유를 통한 추가적인 안전 수단으로 활용되어야 한다.

### 8.2 SOS 서비스 범위

| SOS 기능 | 서비스 내용 | 한계 |
|----------|-----------|------|
| 수동 SOS 발송 | 이용자가 SOS 버튼을 눌러 그룹 전원에게 위치 포함 긴급 알림 발송 | 수신자의 기기 상태·네트워크 환경에 따라 전달 지연 가능 |
| 자동 SOS (Heartbeat 타임아웃) | 서버와의 통신이 일정 시간 두절되면 자동 알림 발송 | 기기 전원 차단, 배터리 고갈 시 발동 불가 |
| 배터리 부족 경고 | 배터리 임계치 이하 시 그룹에 알림 | 급격한 배터리 고갈 시 알림 전 전원 차단 가능 |
| 긴급 전화 연결 안내 | 해당 국가 긴급 전화번호 안내 및 다이얼 연결 지원 | 실제 통화 연결은 이동통신 환경에 의존 |

### 8.3 면책 조항

회사는 다음의 경우 SOS 기능의 미작동 또는 지연에 대하여 책임을 지지 아니한다.

| 면책 사유 | 구체적 상황 |
|----------|-----------|
| 이용자 기기 문제 | 기기 전원 꺼짐, 배터리 고갈, GPS 비활성, 앱 강제 종료 |
| 네트워크 환경 | 인터넷 연결 불가, 이동통신 음영 지역, 로밍 미설정 |
| OS 제한 | iOS/Android의 백그라운드 프로세스 제한으로 인한 Heartbeat 간격 연장 |
| 수신자 환경 | 수신자의 기기 꺼짐, 앱 미설치, 푸시 알림 비허용 |
| 불가항력 | 자연재해, 전쟁, 폭동, 해당 국가의 통신 차단 등 |
| 제3자 서비스 장애 | FCM(Firebase Cloud Messaging) 장애, 이동통신사 장애 |

### 8.4 SOS 투명성 고지

회사는 SOS 기능의 제약 사항을 다음의 시점에 이용자에게 명확히 고지한다.

| 고지 시점 | 고지 내용 | 고지 방법 |
|----------|----------|----------|
| 회원가입 시 | SOS는 보조 수단이며 공식 구조 서비스를 대체하지 않음 | 약관 동의 화면 |
| 최초 여행 생성 시 | SOS 기능의 구체적 제약 사항 (네트워크, 배터리 등) | 인앱 안내 카드 |
| SOS 발송 확인 시 | "공식 긴급 전화도 함께 이용해주세요" | SOS 확인 다이얼로그 |
| SOS 설정 화면 | 기능별 한계 상세 안내 | 설정 > SOS 화면 |

> **SOS 원칙 참조**: SafeTrip_SOS_원칙_v1.0 §15 제약 사항 및 투명성 고지

## 11. 이용 제한 및 계약 해지

### 11.1 이용 제한

회사는 회원이 본 약관 제9조의 금지 행위를 하거나 관련 법령을 위반한 경우, 다음의 단계적 조치를 취할 수 있다.

| 단계 | 조치 | 기간 | 적용 사유 |
|:----:|------|------|----------|
| 1 | 서면 경고 | — | 경미한 약관 위반 (첫 위반) |
| 2 | 서비스 일시 정지 | 7일~30일 | 경고 후 재위반, 허위 SOS 반복 |
| 3 | 서비스 이용 제한 | 30일~90일 | 위치 조작, 타인 사칭 |
| 4 | 이용 계약 해지 | 영구 | 서비스 방해, 불법 행위, 3단계 이후 재위반 |

### 11.2 이의 신청

이용 제한을 받은 회원은 조치 통보일로부터 15일 이내에 이의를 신청할 수 있으며, 회사는 접수일로부터 15일 이내에 처리 결과를 통보한다.

### 11.3 회원 탈퇴

회원은 언제든 앱 내 설정 > 계정 관리 또는 고객센터를 통해 탈퇴를 요청할 수 있다.

**탈퇴 처리 절차:**

```
탈퇴 요청
│
├── 진행 중인 여행 확인
│   ├── active 여행 있음 → "진행 중인 여행이 있습니다" 안내
│   │   ├── 캡틴인 경우 → 캡틴 위임 또는 여행 삭제 후 탈퇴 가능
│   │   └── 크루/크루장인 경우 → 여행에서 자동 탈퇴 후 계정 탈퇴
│   └── active 여행 없음 → 탈퇴 진행
│
├── 2단계 확인
│   ├── 1차: 탈퇴 안내 (데이터 삭제 범위, 복구 불가 고지)
│   └── 2차: "탈퇴합니다" 텍스트 입력 확인
│
├── 소프트 삭제 (즉시)
│   ├── TB_USER.deleted_at 설정
│   ├── 로그인 차단
│   └── 타 멤버에게 "알 수 없는 사용자"로 표시
│
├── 7일 유예 기간
│   ├── 재로그인 시 계정 복원 가능
│   └── 유예 기간 내 미복원 시 하드 삭제 진행
│
└── 하드 삭제 (7일 경과 후)
    ├── 개인 식별 정보 영구 삭제
    ├── 프로필 이미지 CDN 삭제
    ├── SOS 관련 위치 기록: 익명화 후 법적 보관 기간까지 보존
    └── 결제 기록: 전자상거래법에 따라 5년 보관
```

> **개인정보처리방침 참조**: SafeTrip_개인정보처리방침_원칙_v1.0 §4.2 계정 삭제 흐름

서비스에 포함된 모든 콘텐츠(텍스트, 그래픽, 로고, 아이콘, 이미지, 소프트웨어, UI 디자인 등)에 대한 저작권 및 지적재산권은 회사에 귀속된다.

### 12.2 이용자 콘텐츠

회원이 서비스 내에서 생성한 콘텐츠(채팅 메시지, 일정 내용 등)의 저작권은 해당 회원에게 귀속된다. 다만, 회사는 서비스 운영 및 개선 목적으로 해당 콘텐츠를 사용할 수 있으며, 이 경우 개인을 식별할 수 없는 형태로 가공하여 사용한다.

### 12.3 외부 데이터

서비스 내에서 제공되는 외교부 여행 안전 정보, 지도 데이터 등 외부 출처의 콘텐츠에 대한 저작권은 해당 출처에 귀속되며, 회사는 적법한 라이선스에 따라 이를 제공한다.

---

## 13. 손해배상

### 13.1 회사의 배상 책임

회사는 회사의 고의 또는 중대한 과실로 인하여 회원에게 손해가 발생한 경우 그 손해를 배상한다. 다만, 회사의 고의 또는 과실이 없음을 입증한 경우에는 그러하지 아니하다.

### 13.2 배상 범위

회사의 손해배상 범위는 통상 손해에 한하며, 특별한 사정으로 인한 손해는 회사가 그 사정을 알았거나 알 수 있었을 때에 한하여 배상한다.

### 13.3 무료 서비스의 책임 제한

회사가 무료로 제공하는 서비스(데모 모드, 무료 AI 기능 등)의 이용으로 발생한 손해에 대하여는 회사의 고의 또는 중대한 과실이 있는 경우를 제외하고 책임을 지지 아니한다.

### 13.4 회원의 배상 책임

회원이 본 약관에 위반하는 행위 또는 불법 행위로 인하여 회사에 손해를 발생시킨 경우, 해당 회원은 회사에 대하여 그 손해를 배상해야 한다.

### 14.1 분쟁 조정

서비스 이용과 관련하여 분쟁이 발생한 경우, 회사와 회원은 분쟁의 해결을 위해 성실히 협의한다. 협의로 해결되지 않는 경우 다음의 기관에 분쟁 조정을 신청할 수 있다.

- 한국소비자원 소비자분쟁조정위원회

- 방송미디어통신위원회 (위치정보 관련)

### 14.2 관할 법원

본 약관에 관한 소송은 민사소송법에 따른 관할 법원에 제기한다.

### 14.3 준거법

## 17. 에러 및 엣지케이스

### 17.1 결제 관련

| 상황 | 처리 |
|------|------|
| 결제 중 네트워크 단절 | 결제 상태 불명 → 서버 측 결제 검증 후 결과 반영. "결제 확인 중입니다" 안내 + 수동 확인 버튼 |
| 앱 스토어 인앱 결제 오류 | 앱 스토어 결제 영수증 검증 실패 → "결제가 완료되지 않았습니다. 다시 시도해주세요" 안내 |
| 이중 결제 발생 | 결제 ID 기준 중복 감지 → 자동 환불 처리 + 이메일(등록된 경우) 안내 |
| 결제 후 서비스 미반영 | 결제 완료 확인 후 수동 서비스 적용 → 고객지원 접수 유도 |
| 환불 후 서비스 재이용 시도 | 환불 완료 서비스는 즉시 이용 제한 → 재결제 안내 |

### 17.2 계정 관련

| 상황 | 처리 |
|------|------|
| 탈퇴 유예 기간(7일) 중 재가입 시도 | 기존 계정 복원 안내 → 재로그인으로 복원 |
| 캡틴인 상태에서 탈퇴 시도 | "진행 중인 여행의 캡틴입니다. 캡틴 위임 또는 여행 삭제 후 탈퇴해주세요" |
| 탈퇴 후 동일 번호로 재가입 | 7일 유예 기간 내 → 복원 안내. 유예 기간 경과 → 신규 가입 처리 |
| 이용 정지 중 유료 서비스 결제일 도래 | 구독 자동 갱신 일시 중지 → 정지 해제 후 수동 갱신 안내 |
| 이용 제한 해제 후 데이터 상태 | 제한 기간 중 생성된 여행 데이터는 보존. 실시간 위치 수집은 제한 기간 중 중단 |

### 17.3 역할 충돌

| 상황 | 처리 |
|------|------|
| 캡틴 위임 중 여행 삭제 시도 | "캡틴 권한 변경 중입니다. 완료 후 다시 시도해주세요" |
| 크루장 → 캡틴 위임 직후 기존 캡틴의 설정 변경 시도 | 즉시 권한 변경 반영 → "권한이 변경되었습니다" 안내 + 설정 화면 새로고침 |
| 동시에 2명이 캡틴 위임 수락 | 서버 측 먼저 도착한 요청 우선 처리 (선착순) → 후순위 요청 거부 + 안내 |
| 가디언 해제 후 해당 여행 접근 시도 | 즉시 접근 차단 → "가디언 권한이 해제되었습니다" 안내 |
