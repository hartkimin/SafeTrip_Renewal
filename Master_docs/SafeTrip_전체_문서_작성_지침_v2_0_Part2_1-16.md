# SafeTrip 전체 원칙 문서 작성 지침서 v2.0

> **문서 목적**: SafeTrip 프로젝트의 35개 원칙 문서 각각을 최신화(v5.1 기준)하기 위한 작성 프롬프트, 참조 관계, 필수 포함 항목, 정합성 검증 방법을 정의한다. 본 지침서를 프롬프트로 사용하여 각 문서를 순차적으로 업데이트한다.
>
> **기준 문서**: SafeTrip_마스터_원칙_거버넌스_v2.0, SafeTrip_비즈니스_원칙_v5.1
>
> **v1.0 → v2.0 변경 사유**: 비즈니스 원칙 v5.0→v5.1 반영, 기존 문서 현황 기반 갭 분석 반영, 정합성 검증 강화
>
> **작성일**: 2026-03-01

## 공통 작성 규칙

### 모든 문서에 반드시 포함해야 하는 구조

```
1. 문서 헤더 (메타데이터 테이블)
   - 문서 ID, 계층, 등급, 버전, 작성일
   - 기준 문서(상위 문서) — 반드시 v5.1 참조 명시
   - 관련 법령 (해당 시)
   - 후행 문서 목록
   - 검토 주기, 담당 부서
2. 문서 목적 및 적용 범위
3. 본문 (문서별 고유 내용 — 아래 개별 지침 참조)
4. 역할별 접근 권한 매트릭스 (캡틴/크루장/크루/가디언)
5. 프라이버시 등급별 동작 차이 (안전 최우선/표준/프라이버시 우선)
6. 에러 및 엣지케이스 처리
7. DB 스키마 (해당 시) — v5.1 부록 C 스키마 반영 필수
8. 오프라인 대응 (해당 시)
9. 구현 우선순위 (P0~P3) 및 Phase 배치
10. 검증 체크리스트 (법적 + 기능 + 거버넌스 12항목)
11. 기존 문서 대비 변경·확장 사항 (변경 이력)
12. 부록 (관련 문서 참조 테이블)
```

### 문서 헤더 표준 템플릿

모든 문서는 다음 형식의 메타데이터 테이블로 시작해야 한다.

```markdown

|------|------|
| **문서 ID** | `DOC-{Tier}-{약어}-{번호}` |
| **문서 계층** | Tier {N} — {계층명} |
| **문서 등급** | {S/A/B/C} — {등급 설명} |
| **버전** | v{X.Y} |
| **작성일** | {YYYY-MM-DD} |
| **기준 문서** | SafeTrip_비즈니스_원칙_v5.1 (Tier 1) [+ 추가 상위 문서] |
| **관련 법령** | {해당 법령 목록} |
| **후행 문서** | {영향 받는 하위 문서 목록} |
```

### 거버넌스 품질 체크리스트 (12항목)

모든 문서는 작성 완료 시 아래 12개 항목을 **전수 충족**해야 한다.

| # | 체크 항목 |
|---|----------|
| 1 | 문서 목적과 적용 범위가 명시되어 있다 |
| 2 | 기준 문서(상위 문서)가 명시되어 있다 — 비즈니스 원칙은 **v5.1**을 참조 |
| 3 | 역할별(캡틴/크루장/크루/가디언) 접근 권한이 정의되어 있다 |
| 4 | 프라이버시 등급별(안전 최우선/표준/프라이버시 우선) 동작 차이가 정의되어 있다 |
| 5 | 에러 및 엣지케이스 처리가 포함되어 있다 |
| 6 | 검증 체크리스트가 포함되어 있다 |
| 7 | 기존 문서 대비 변경/확장 사항이 명시되어 있다 |
| 8 | DB 스키마가 필요한 경우 테이블 구조가 포함되어 있다 — v5.1 스키마 변경 반영 여부 확인 |
| 9 | 구현 우선순위(P0~P3)와 Phase 배치가 포함되어 있다 |
| 10 | 오프라인 동작이 해당되는 경우 대응 방안이 포함되어 있다 |
| 11 | 가디언 과금(무료/유료)이 해당되는 경우 과금 분기가 포함되어 있다 |
| 12 | 여행 기간 제한(15일)이 해당되는 경우 제한 처리가 포함되어 있다 |

## 부록 C: 프롬프트 사용법

### 각 문서 업데이트 시 프롬프트 사용 순서

```
1. 해당 문서의 "작성 프롬프트" 섹션을 복사한다.
2. 기존 문서 원본을 함께 첨부한다.
3. 다음 형식으로 AI에게 요청한다:

   "다음 기준에 따라 [문서명]을 v{X.Y} → v{X.Y+1}로 갱신하라.

   [작성 프롬프트 붙여넣기]

   [기존 문서 원본 첨부]

   갱신 시 주의사항:
   - 기준 문서를 v5.1로 갱신
   - 공통 갱신 항목(여행 기간, 가디언 과금, DB 스키마) 확인
   - 거버넌스 12항목 체크리스트 충족 확인
   - 정합성 검증 방법에 따라 자가 검증 수행
   - 변경 이력을 부록에 기록"

4. 갱신 완료 후 "정합성 검증 방법" 테이블로 교차 검증한다.
5. 교차 참조 문서가 있는 경우 해당 문서도 함께 확인한다.
```

### 미작성 문서 신규 작성 시

```
1. 해당 문서의 "작성 프롬프트"를 사용한다.

3. "공통 작성 규칙"의 11개 구조를 모두 포함한다.
4. 상위 문서 원본을 참조로 첨부한다.
5. 작성 완료 후 거버넌스 12항목 체크리스트를 자가 검증한다.
```
