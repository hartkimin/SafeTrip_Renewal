## §2. MOD-01: 대시보드 (Dashboard)

> **목적**: 운영 현황을 한눈에 파악하는 Command Center

### 2.1 KPI 카드 (실시간 업데이트)

| KPI | 데이터 소스 | 갱신 주기 | 설명 |
|-----|-----------|:---------:|------|
| Active SOS | `/emergencies/stats` | ⚡ 5초 | 현재 활성 SOS 건수, 심각도별 색상 |
| Active Trips | `/trips/admin/stats` | 30초 | 진행 중인 여행 그룹 수 |
| DAU (일일 활성 사용자) | `/users/admin/stats` | 5분 | 오늘 활성 사용자 수 |
| B2B 파트너 수 | `/b2b/organizations` | 5분 | 활성 B2B 조직 수 |
| 오늘 매출 | `/payments/admin/stats` | 5분 | 금일 결제 완료 합산 |
| 신규 가입자 | `/users/admin/stats` | 5분 | 오늘 신규 가입 수 |

### 2.2 SOS 실시간 피드

- 최근 20건의 SOS 이벤트를 실시간 스트림으로 표시
- 각 이벤트: ID, 사용자명, 여행명, 위치, 시간, 배터리, 상태(ACTIVE/RESOLVED)
- 신규 SOS 발생 시 **사운드 알림 + 화면 깜박임** 제공
- 클릭 시 SOS 상세 페이지로 이동

### 2.3 시계열 차트

| 차트 | 데이터 | 기간 |
|------|--------|------|
| 사용자 추이 | DAU/WAU/MAU | 7일/30일/90일 |
| 여행 생성 추이 | 일별 생성 수 | 7일/30일 |
| 매출 추이 | 일별 매출 | 7일/30일/90일 |
| SOS 발생 추이 | 일별 발생/해소 수 | 7일/30일 |

### 2.4 운영 상태 패널

- 시스템 헬스체크 (API 응답 시간, DB 상태, Firebase 상태)
- 최근 시스템 알림 5건
- 미처리 고객 문의 건수

## §3. MOD-02: SOS 관제 센터 (SOS Command Center)

> **목적**: 긴급 SOS 상황의 실시간 모니터링, 분류, 대응, 종결 관리

### 3.1 기능 목록

| 기능 ID | 기능명 | 우선순위 | 설명 |
|---------|--------|:--------:|------|
| SOS-01 | SOS 실시간 목록 | P0 | 전체 SOS 이벤트의 필터링·정렬 가능한 목록 |
| SOS-02 | SOS 상세 보기 | P0 | 개별 SOS의 상세 정보 (위치, 이벤트 타임라인, 관련 여행/사용자) |
| SOS-03 | SOS 지도 뷰 | P0 | 모든 활성 SOS를 지도 위에 핀으로 표시 |
| SOS-04 | SOS 상태 관리 | P0 | ACTIVE → ACKNOWLEDGED → IN_PROGRESS → RESOLVED/FALSE_ALARM |
| SOS-05 | SOS 대응 기록 | P1 | 관리자가 대응 내용, 조치 사항을 기록 |
| SOS-06 | 구조기관 연락 | P1 | 해당 국가의 긴급 전화번호 표시 및 직접 연결 |
| SOS-07 | SOS 통계 대시보드 | P1 | 기간별 발생 건수, 평균 대응 시간, 유형별 분포 |
| SOS-08 | SOS 이력 검색 | P1 | 과거 SOS 이벤트 검색 및 분석 |

### 3.2 SOS 상태 전이

```
ACTIVE ──┬──→ ACKNOWLEDGED ──→ IN_PROGRESS ──→ RESOLVED
         │                                        │
         └────────────────────→ FALSE_ALARM ──────┘ (통계)
```

### 3.3 SOS 상세 정보 구성

- **발송자 정보**: 사용자 ID, 이름, 전화번호, 프로필
- **위치 정보**: 최종 위치 좌표, 주소 (역지오코딩), 이동 경로 (최근 30분)
- **여행 정보**: 소속 여행명, 출발/도착 국가, 여행 기간
- **기기 정보**: 배터리 잔량, 마지막 Heartbeat 시간, 네트워크 상태
- **타임라인**: SOS 발생 → 각 관리자 확인 → 대응 조치 → 해소 전 과정

## §4. MOD-03: 사용자 관리 (User Management)

| 기능 ID | 기능명 | 우선순위 | 설명 |
|---------|--------|:--------:|------|

| USR-02 | 사용자 상세 프로필 | P0 | 기본 정보, 여행 이력, 결제 이력, 활동 로그 |
| USR-03 | 사용자 검색 | P0 | 이름, 전화번호, 이메일, UID로 검색 |
| USR-04 | 사용자 차단(Ban) | P0 | 차단/차단 해제, 사유 기록, 이력 관리 |
| USR-05 | 사용자 통계 | P1 | 가입 추이, 활성 사용자, 이탈률 |
| USR-06 | 약관 동의 현황 | P1 | 각 약관별 동의 상태 및 버전 확인 |
| USR-07 | 미성년자 관리 | P1 | 미성년자 목록, 법정대리인 동의 상태 확인 |
| USR-08 | 사용자 활동 타임라인 | P2 | 시간순 활동 로그 (가입, 여행 생성, SOS 등) |
| USR-09 | 사용자 내보내기 | P2 | CSV/Excel 형태로 사용자 목록 내보내기 |

### 4.2 사용자 상세 프로필 탭 구성

| 탭 | 내용 |
|----|------|
| **기본 정보** | UID, 이름, 전화번호, 가입일, 최근 활동일, 상태 |
| **여행 이력** | 참여한 여행 목록 (역할, 기간, 상태) |
| **결제 이력** | 결제 건 목록 (금액, 유형, 상태) |
| **가디언 관계** | 가디언으로 연결된 멤버 / 가디언에 연결된 관계 |
| **활동 로그** | 이벤트 로그 (로그인, SOS, 설정 변경 등) |
| **제재 이력** | 차단 기록 (시간, 사유, 해제 여부) |

## §5. MOD-04: 여행 관리 (Trip Management)

> **목적**: 전체 여행의 현황 파악, 상세 조회, 운영 개입

### 5.1 기능 목록

| 기능 ID | 기능명 | 우선순위 | 설명 |
|---------|--------|:--------:|------|
| TRP-01 | 여행 목록 | P0 | 페이지네이션 + 검색 + 상태별 필터 (planning/active/completed) |
| TRP-02 | 여행 상세 | P0 | 여행 기본 정보, 멤버 목록, 일정, 지오펜스, 채팅 요약 |
| TRP-03 | 여행 검색 | P0 | 여행명, 목적지, 캡틴 이름으로 검색 |
| TRP-04 | 여행 상태 관리 | P1 | 관리자에 의한 강제 상태 변경 (비정상 상황 대응) |
| TRP-05 | 여행 통계 | P1 | 생성 추이, 국가별 인기도, 평균 인원수, 평균 기간 |

| TRP-09 | 여행 지도 뷰 | P2 | 여행의 멤버 위치/경로를 지도 위에 표시 |

### 5.2 여행 상세 탭 구성

| 탭 | 내용 |
|----|------|
| **개요** | 여행명, 기간, 목적지, 프라이버시 등급, 상태, B2B 연결 여부 |

| **일정** | 등록된 일정 목록 (날짜, 시간, 장소) |
| **SOS 이력** | 해당 여행에서 발생한 SOS 이벤트 |
| **이벤트 로그** | 여행 관련 이벤트 (멤버 합류/탈퇴, 설정 변경 등) |
