### 시나리오별 설계 의도

- **S1**: SafeTrip의 핵심 — **대규모 단체의 안전 관리**. 33명의 학생단체에서 이탈 감지, 전원 도착 확인, SOS 훈련 등 안전최우선 모드의 모든 기능을 시연
- **S2**: **자유여행의 유연함**. 6명 친구 그룹이 자유시간에 흩어졌다 모이는 패턴, 일정 변경, 채팅 활용
- **S3**: **비즈니스 환경의 프라이버시**. 업무시간 vs 자유시간의 위치공유 전환, 프라이버시우선 모드
- **S4**: **가족 + 미성년자 보호**. 부모-자녀 구성에서 미성년자 위치 항시 추적, 원격 조부모 가디언의 실시간 알림
- **S5**: **2인 최소 구성의 간결한 UX**. 커플여행에서 위치공유 on/off 토글, 프라이버시우선의 최소 기능 시연

## §7. DB 시드 데이터 매핑

### 7.1 에뮬레이터 시드 대상 테이블

각 시나리오를 실제 DB에 시드할 때 필요한 테이블:

| 테이블 | 용도 | 시나리오별 레코드 수 |
|--------|------|---------------------|
| tb_user | 유저 생성 | S1:33, S2:6, S3:18, S4:6, S5:3 |
| tb_trip | 여행 생성 | 각 1건 |
| tb_group | 그룹 생성 | 각 1건 |
| tb_group_member | 멤버 등록 | = 여행멤버 수 (가디언 제외) |
| tb_guardian_link | 가디언 링크 | S1:10, S2:0, S3:2, S4:2, S5:1 |
| tb_schedule | 일정 등록 | S1:14, S2:23, S3:22, S4:18, S5:17 |
| tb_schedule_place | 장소 좌표 | = 일정 수 |
| tb_chat_message | 채팅 메시지 | S1:2, S2:7, S3:2, S4:3, S5:5 |
| tb_event_log | 이벤트 로그 | = 시뮬레이션 이벤트 수 |

### 7.2 시드 순서

```
1. tb_user (전체 유저)
2. tb_trip (여행 5건)
3. tb_group (그룹 5건)
4. tb_group_member (멤버 매핑)
5. tb_guardian_link (가디언 링크)
6. tb_schedule + tb_schedule_place (일정·장소)
7. tb_chat_message (채팅 메시지)
8. tb_event_log (이벤트 로그)
```

### 7.3 시드 파일 위치 (예정)

```
emulator-data/
  seed/
    demo-scenario-s1.sql
    demo-scenario-s2.sql
    demo-scenario-s3.sql

    demo-scenario-s5.sql
```
