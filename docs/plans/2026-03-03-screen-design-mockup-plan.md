# SafeTrip 전체 화면 디자인 목업 계획

> **작성일:** 2026-03-03
> **버전:** v1.0
> **목적:** Flutter 구현 가이드 / 디자인 시스템 확립 / UX 흐름 검증
> **분류 체계:** 하이브리드 (User Journey 기반 + 도메인별 그룹)
> **대상 앱:** 하나의 앱 내 크루/가디언 모드 전환
> **범위:** Phase 1 + 2 + 3 전체 (104개 화면)

---

## 전체 화면 구조 (대분류)

```
SafeTrip 전체 화면 체계 (104개)
│
├─ A. 온보딩 & 인증 ─────────── 7개  (P0)
├─ B. 여행 생성 & 참여 ─────── 8개  (P0)
├─ C. 메인 맵 & 공통 UI ────── 12개 (P0-P1)
├─ D. 여행 관리 ──────────────── 16개 (P0-P1)
├─ E. 위치공유 & 프라이버시 ─── 8개  (P0-P1)
├─ F. 가디언 시스템 ──────────── 10개 (P0-P1)
├─ G. SOS & 긴급 기능 ─────── 5개  (P0-P1)
├─ H. 출석 체크 ───────────────── 5개 (P1)
├─ I. 채팅 & 커뮤니케이션 ──── 5개  (P0-P2)
├─ J. 안전 가이드 ─────────────── 7개 (P0-P1)
├─ K. 설정 & 프로필 ──────────── 8개 (P0-P1)
├─ L. 결제 & 구독 ─────────── 10개 (P2)
├─ M. 미성년자 보호 ──────────── 4개 (P2)
├─ N. B2B 관리자 포털 ──────── 8개  (P3)
└─ O. AI 기능 ────────────────── 5개 (P2-P3)

### Phase별 분포

| Phase | 화면 수 | 비율 | 내용 |
|:-----:|:-------:|:----:|------|
| P0 | 52개 | 50% | 핵심 앱 경험 (온보딩~SOS~가디언~안전가이드) |
| P1 | 20개 | 19% | 확장 기능 (출석, 지오펜스, 세부 설정) |
| P2 | 19개 | 18% | 수익화 (결제, 미성년자 보호, AI 기초) |
| P3 | 13개 | 13% | 엔터프라이즈 (B2B 포털, 고급 AI) |

| A-02 | `onboarding-intro` | 웰컴 온보딩 | 3-4 페이지 스와이프. SafeTrip 핵심 가치 소개 (위치공유, SOS, 가디언) | P0 | 신규 | ✅ |

| A-07 | `consent-agree` | 약관 동의 | 이용약관, 개인정보처리방침, 위치정보 이용약관 (필수) + 마케팅 수신 (선택) | P0 | 신규 | ❌ |

### User Journey Flow

```
A-01 Splash
 ├─ [토큰 유효] → C-01 메인맵

      → A-03 역할 선택
        ├─ [크루] → A-07 약관 → A-04 전화인증 → A-06 프로필
        └─ [가디언] → A-07 약관 → A-04 전화인증 → A-06 프로필
```

### User Journey Flow

```

    ├─ "여행 만들기" → B-02 → B-03 국가 → B-04 프라이버시 → B-05 공유모드 → B-06 확인 → C-01 메인맵
