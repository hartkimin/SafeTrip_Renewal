### 1.1 레이아웃 계층 (3-Layer)

메인 화면은 3개의 레이어로 구성된다. 하위 레이어부터 상위 레이어 순으로 렌더링된다.

```
┌─────────────────────────────────────┐
│      [Layer 3] 상단 오버레이         │  ← z-index 최상위
│  여행정보카드 | 설정⚙️ | 알림🔔       │
│                      [SOS 버튼]     │
├─────────────────────────────────────┤
│                                     │
│        [Layer 1] 지도 뷰             │  ← 기본 레이어
│     (Google Maps / Kakao Map)       │
│   멤버 위치 마커 | 지오펜스 오버레이   │
│                                     │
├─────────────────────────────────────┤
│     [Layer 2] 바텀시트 (탭 포함)      │  ← 지도 위에 슬라이딩
│  [일정] [멤버] [채팅] [안전가이드]     │
│             탭 컨텐츠                │
└─────────────────────────────────────┘
```

| **비고** | 로딩 상태 표시. 버튼 내 로딩 시 Circular 사용 |

#### Divider

| 항목 | 값 |
|------|-----|
| **Flutter 위젯** | `Divider` |
| **크기** | 높이 1px, 전체 너비 |
| **색상 토큰** | `outline` (`#EDEDED`) 또는 `outlineVariant` (`#F5F5F5`) |
| **상태** | default |
| **비고** | 섹션 구분선. 얇은 구분선은 `outlineVariant` 사용 |

## 4. 여행 상태별 UI 매트릭스

> 출처: 화면구성원칙 &sect;5

4가지 여행 상태(`none` / `planning` / `active` / `completed`)에 따라 화면 구성이 변경된다.

### 4.1 상태 전이 다이어그램

```
none ──────────┐
               │
planning ──────┤──→ active ──→ completed
               │       │
               │       └──→ active (재활성화, 24시간 이내 1회)
               └── (여행 생성)
```

### 4.2 상태별 UI 요소

| UI 요소 | none | planning | active | completed |
|---------|:----:|:--------:|:------:|:---------:|
| 지도 (멤버 위치 마커) | ❌ | ❌ | ✅ | ❌ (마지막 스냅샷) |
| SOS 버튼 | ❌ | ❌ | ✅ | ❌ |
| D+N 경과일 표시 | ❌ | ❌ | ✅ | ❌ |
| D-N 남은일 표시 | ❌ | ✅ | ❌ | ❌ |
| 일정탭 | ❌ | ✅ 편집 가능 | ✅ 편집 가능 | ✅ 열람 전용 |
| 멤버탭 | ❌ | ✅ | ✅ | ✅ 열람 |
| 채팅탭 | ❌ | ✅ | ✅ | ✅ 열람 |
| 안전가이드탭 | ❌ | ✅ | ✅ | ✅ |
| 여행 시작 버튼 | ❌ | ✅ (캡틴만) | ❌ | ❌ |
| 여행 종료 버튼 | ❌ | ❌ | ✅ (캡틴만) | ❌ |
| 재활성화 버튼 | ❌ | ❌ | ❌ | ✅ (24h 이내, 캡틴) |
| "종료된 여행" 뱃지 | ❌ | ❌ | ❌ | ✅ |
| 탭 상태 | 비활성화 (회색) | 전체 활성 | 전체 활성 | 전체 (읽기전용) |

### 4.3 상태별 특수 UI

#### `none` 상태 (여행 없음)

- 홈 화면: 지도 배경 위에 "+ 새 여행 만들기" 버튼 중앙 표시
- 탭바 비활성화 (4개 탭 모두 회색 처리, 전환 불가)
- 여행정보카드 대신 온보딩 안내 배너 표시

#### `completed` 상태 (여행 종료)

- 지도는 여행 종료 시점의 마지막 스냅샷 표시 (실시간 위치 아님)
- 모든 탭 열람 전용 (편집/전송 비활성화)
- 상단바에 "종료된 여행" 뱃지 표시
- 재활성화 버튼: 종료 시점으로부터 24시간 이내에만 캡틴에게 표시

## 6. 바텀시트 5단계 스냅 포인트

> 출처: DESIGN.md &sect;5.2

### 6.1 스냅 포인트 정의

| Step | 이름 | 화면 비율 | 지도 노출 | 용도 |
|:----:|------|:--------:|:--------:|------|
| 1 | `peek` | 6% | 94% | 탭 바만 살짝 노출. 지도 최대 노출 시 |
| 2 | `collapsed` | 18% | 82% | 탭 바 + 헤더만 노출. SOS 발동 시 강제 전환 대상 |
| 3 | `half` | 35% | 65% | 기본 컨텐츠 열람. 일정탭/채팅탭 기본 높이 |
| 4 | `tall` | 55% | 45% | 확장 컨텐츠. 목록 탐색 시 |
| 5 | `expanded` | 75% | 25% | 최대 확장. 지도 일부만 노출 |

### 6.2 바텀시트 공통 스타일

| 속성 | 값 |
|------|-----|
| 배경 | `surface` (`#FFFFFF`) |
| 상단 모서리 | `radius20` (20px) |
| 핸들 바 | 가운데 정렬, 4px x 44px, `line04` 색상 |
| 그림자 | 상단 방향, black 4% opacity |

### 6.3 탭별 기본 높이

| 탭 | planning | active | completed |
|----|:--------:|:------:|:---------:|
| 일정 | half (35%) | half (35%) | half (35%) |
| 멤버 | peek (6%) | peek (6%) | peek (6%) |
| 채팅 | half (35%) | half (35%) | half (35%) |
| 안전가이드 | peek (6%) | peek (6%) | peek (6%) |

### 6.4 동작 규칙

- 탭 전환 시 이전 탭의 스크롤 위치와 바텀시트 높이는 **유지**
- 탭 전환 시 바텀시트가 `expanded` 상태이면 새 탭도 `expanded` 유지
- SOS 발동 시 모든 탭이 `collapsed` **강제 전환**
- 뒤로가기: `expanded` 상태 → `half` 축소, `half` 이하 → 앱 종료 확인
