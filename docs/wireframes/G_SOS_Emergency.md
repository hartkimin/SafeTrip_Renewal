## 1. 개요

- **화면 수:** 3개 (G-01, G-02, G-03)
- **Phase:** P0 3개 (전 화면 런칭 필수)
- **핵심 역할:** 캡틴/크루장/크루 (SOS 발신), 캡틴/크루장/크루/가디언 (SOS 수신)
- **여행 상태:** `active` 상태에서만 SOS 발송 가능
- **법적 근거:** 위치정보법 제29조 (긴급구조), 개인정보보호법 제15조 제1항 제3호 (급박한 위험 방지)

### SOS 최상위 원칙

> **SOS는 모든 프라이버시 설정을 무시하는 최상위 안전 기능이다.** (비즈니스 원칙 §05.1)
> SafeTrip SOS는 보조 수단이며, 119/112 등 공식 구조기관 직접 연락이 최우선이다.

### User Journey Flow

```
[여행 active 상태]
     │
     ├── [캡틴/크루장/크루] ──→ G-01 SOS 버튼 (메인맵 우하단 상시 표시)
     │                         │
     │                    [3초 롱프레스]
     │                         ↓
     │                    G-02 SOS 활성 오버레이 (발신자 기기)
     │                         │
     │                    ├── [SOS 해제] → 확인 다이얼로그 → 해제 완료 → C-01 복귀
     │                    └── [긴급 구조기관 연결] → 현지 긴급전화 발신
     │                         │
     │                    [FCM PRIORITY_HIGH 알림 발송]
     │                         │
     │               ┌─────────┴──────────┐
     │               ↓                    ↓
     │         [멤버 수신]          [가디언 수신]
     │         G-03 SOS 수신        G-03 SOS 수신
     │         알림                  알림 (연결 멤버)
     │               │
     │          ├── [확인] → 알림 닫힘, 상단 배너 유지
     │          └── [위치 보기] → C-01 맵 (발신자 위치 센터링)
     │
     └── [가디언] ──→ SOS 버튼 미표시 (수신 전용)
```

### SOS 디자인 토큰

|--------|-----|------|
| `sosDanger` | `#D32F2F` | SOS 버튼 배경, SOS 오버레이 배경, SOS 관련 전체 |

**SOS 발동 시 자동 처리 시퀀스:**

```
SOS 발동 (3초 롱프레스 완료)
    ↓
1. 서버 SOS 이벤트 생성 (tb_sos_event, status: 'active')
    ↓
2. FCM PRIORITY_HIGH 알림 발송
    ├── 그룹 전체 멤버 (캡틴 + 크루장 + 크루)
    └── 발송자에 연결된 가디언 전원 (무료/유료 모두)
    ↓
3. 채팅탭 SOS 시스템 메시지 자동 삽입
    ↓
4. G-02 오버레이 표시 (발신자 기기)
    ↓
5. G-03 SOS 수신 알림 표시 (수신자 기기)
    ↓
```

## 3. 에러 & 엣지케이스

| # | 케이스 | 처리 방법 |
|---|--------|----------|
| 1 | **SOS 발송 실패 (네트워크)** | 오프라인 시나리오 자동 전환: SMS 발송 (긴급 연락처 + 가디언 전화번호) + 로컬 알람 (기기 최대 볼륨) + SQLite 큐잉 → 네트워크 복귀 시 Priority 1 재전송 |
| 2 | **FCM 알림 미전달** | 서버 재시도 (30초 간격 3회), 이후 SMS 폴백 |
| 3 | **SOS 중 앱 강제 종료** | 서버 SOS 이벤트 유지, 앱 재진입 시 G-02 오버레이 자동 표시 |
| 4 | **오발송 (3초 미만 해제)** | 카운트다운 중 손 떼면 자동 취소, SOS 발송 안 됨 |
| 5 | **동시 다중 SOS** | 각각 별도 이벤트로 처리, 화면에 발신자별 배너 스택 |
| 6 | **SOS 발동 중 여행 종료** | SOS 이벤트 유지, 여행 종료 후에도 SOS 해제 전까지 알림 지속 |
| 7 | **가디언 연결 없는 멤버 SOS** | 그룹 멤버에게만 알림, "가디언 없음" 표시 |
| 8 | **GPS 신호 없음 (SOS 발동)** | 마지막 알려진 위치 + "GPS 신호 없음" 메시지 포함하여 발송 |
| 9 | **SOS 해제 실패 (네트워크)** | 로컬에서 해제 처리 → 네트워크 복구 시 서버에 해제 전송 |
| 10 | **SOS 중 배터리 부족 (<20%)** | Heartbeat 주기 10초 → 60초 전환 (배터리 보존), 배터리 잔량 알림에 포함 |
| 11 | **위치 권한 없음** | SOS 발송은 가능 (위치 없이), "GPS를 켜주세요" 안내 + 설정 이동 버튼 |
| 12 | **알림 권한 없음 (수신자)** | SOS 수신 제한 경고 배너 상시 표시 (설정 이동 링크 포함) |
