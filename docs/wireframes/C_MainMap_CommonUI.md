## 1. 개요

### 1.1 문서 목적

본 문서는 SafeTrip 앱의 **메인 맵 화면과 공통 UI**에 해당하는 7개 화면을 정의한다. 메인 맵은 앱 전체의 허브 화면으로, 3-Layer 아키텍처를 통해 지도, 바텀시트 탭, 상단 오버레이를 통합적으로 제공한다. 각 화면은 메타데이터, 레이아웃(ASCII 390x844px 기준), 역할별 분기, 여행상태별 분기, 프라이버시등급별 분기, 인터랙션을 포함한다.

### 1.2 화면 구성

| 화면 ID | 화면명 | Phase | 비고 |
|---------|--------|:-----:|------|

| C-02 | 가디언 맵 (Guardian Map Mode) | P0 | 가디언 전용 3탭 지도 화면 |

#### Layer 2: 바텀시트 4탭 컴포넌트

**바텀시트 5단계 스냅 포인트** (출처: 화면구성원칙 v1.1 SS2.3)

| Step | 이름 | 화면 비율 | 지도 노출 | 용도 |
|:----:|------|:--------:|:--------:|------|
| 1 | `collapsed` | 10% | 90% | 탭 바 + 헤더만 노출 (약 72dp). SOS 발동 시 강제 전환 |
| 2 | `peek` | 25% | 75% | 탭 컨텐츠 일부 미리보기 |
| 3 | `half` | 50% | 50% | 기본 컨텐츠 열람 |
| 4 | `expanded` | 75% | 25% | 상세 컨텐츠 열람 |
| 5 | `full` | 100% | 0% | 전체 화면. 지도 완전히 숨겨짐 |

**바텀시트 동작 규칙**

- 탭 전환 시 이전 탭의 스크롤 위치와 바텀시트 높이는 **유지**된다.
- 탭 전환 시 바텀시트가 `full` 상태이면 새 탭도 `full` 유지.
- SOS 발동 시 → **모든 탭이 `collapsed`** 강제 전환.
- 뒤로가기: `full` → `half` 축소, `half` 이하 → 앱 종료 확인 다이얼로그.

**바텀시트 컴포넌트 상세**

| 컴포넌트 | Flutter 위젯 | 속성 |

## 3. 에러 & 엣지케이스

### 3.1 네트워크 에러

| 상황 | UI 처리 |
|------|---------|
| 위치 데이터 로딩 실패 | 마지막 캐시 위치 표시 + "위치 정보를 불러오는 중..." 오버레이 |
| 탭 데이터 로딩 실패 | 해당 탭 내 재시도 버튼 표시 |
| SOS 발송 실패 | 에러 다이얼로그 + 재시도 버튼 (자동 3회 재시도 후 표시) |
| 전체 오프라인 | 오렌지 배너 표시, 캐시 데이터로 열람만 허용 |
| 알림 목록 로딩 실패 | 재시도 버튼 표시, 로컬 캐시 알림은 표시 |

### 3.2 권한 에러

| 상황 | UI 처리 |
|------|---------|
| 위치 권한 없음 | 권한 요청 다이얼로그 → 거부 시 지도 비활성화 + 설정 이동 안내 |
| 알림 권한 없음 | SOS 수신 제한 경고 배너 (설정 이동 링크 포함) |
| 위치 서비스 OFF | "GPS를 켜주세요" 시스템 설정 이동 버튼 |

### 3.3 엣지케이스

| 상황 | 처리 규칙 |
|------|----------|
| 여행 기간 15일 초과 생성 시도 | 생성 화면에서 실시간 오류 표시, 완료 버튼 비활성화 |
| 멤버 0명 여행 | 멤버탭에 "아직 멤버가 없습니다 -- 초대하기" CTA 표시 |
| 동시에 2개 여행 active | 서버 차단 + "현재 진행 중인 여행이 있습니다" 에러 메시지 |
| 가디언이 연결 멤버 0명 | "연결된 멤버가 없습니다. 멤버가 가디언으로 초대해야 합니다" 안내 |
| 재활성화 24시간 초과 | 재활성화 버튼 비표시, "재활성화 기간이 만료되었습니다" 안내 |
| 멤버 전원이 같은 좌표 | 클러스터 숫자 배지 표시, 탭 시 멤버 목록 팝업 |
| 클러스터 내 SOS 발신자 포함 | 클러스터를 빨간색으로 강조 + SOS 배지 표시 |
| SOS 발동 중 앱 재시작 | 서버에서 SOS 상태 복원, Layer 6 즉시 재활성화 |
| SOS 발동 중 오프라인 | Layer 6 유지, SOS 로컬 큐잉, 네트워크 복구 시 자동 전송 |
| 동시 다수 SOS | 가장 최근 SOS 발신자 위치로 카메라 이동, SOS 목록 Layer 6에 표시 |
| Google Maps 로딩 실패 | 오프라인 캐시 타일로 대체, "오프라인 지도" 안내 |
| 오프라인 캐시 없음 | 회색 배경 + 멤버 마커만 좌표 기반 점 표시 (타일 없이) |

### 3.4 알림 엣지케이스

| 상황 | 처리 규칙 |

- 상단에 오렌지색 배너: "오프라인 상태 -- 실시간 위치가 업데이트되지 않습니다"

- SOS 버튼 및 기능은 오프라인에서도 항상 활성 상태 유지 (G5 원칙)
